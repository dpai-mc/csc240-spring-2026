💡 From Correctness to Communication

In the previous modules, you learned how to construct valid proofs. Now, you'll learn how to communicate those proofs effectively. A proof that is correct but poorly written fails to serve its purpose—convincing the reader of a mathematical truth.

Think of this module as learning the "style guide" for mathematical writing.

1. The Four Principles of Good Proof Writing

Every well-written mathematical proof adheres to four fundamental principles:

Principle Description Questions to Ask
Clarity Every step is understandable to the intended reader Can a classmate follow each step? Are terms defined before use?
Completeness Sufficient detail to justify each step; no logical gaps Is every claim supported? Are all cases covered? Are closure properties mentioned?
Correctness Every inference is logically valid; no errors Does each step follow from the previous? Are definitions applied correctly?
Conciseness No unnecessary repetition or verbosity Can anything be removed without losing meaning? Is there redundancy?

📖 The "4 C's" of Proof Writing

Remember the 4 C's: Clarity, Completeness, Correctness, Conciseness.

These principles sometimes tension with each other—more detail (completeness) can reduce conciseness. Good mathematical writing finds the right balance for the intended audience.

Balancing the Principles

⚖️ Finding the Right Balance

  • Too much detail: The proof becomes tedious and hard to follow. The main ideas get lost in routine calculations.
  • Too little detail: The reader can't verify the proof. Logical gaps make the argument unconvincing.
  • The goal: Include enough detail that a competent reader can fill in any remaining gaps, but not so much that the proof becomes overwhelming.

Rule of thumb: For this course, err on the side of more detail. As you gain experience, you'll develop intuition for what can be omitted.

2. Proof Writing Conventions

Mathematical writing follows specific conventions that have developed over centuries. Following these conventions makes your proofs easier to read and demonstrates professionalism.

Standard Phrases and Their Meanings

Phrase When to Use Example
"Let..." Introducing a variable or assumption "Let n be an even integer."
"Assume..." Stating the hypothesis in a direct proof "Assume that n is divisible by 6."
"Suppose..." Similar to "Assume"; often used in proof by contradiction "Suppose, for contradiction, that √2 is rational."
"By definition..." Applying a formal definition "By definition of even, n = 2k for some integer k."
"Then..." / "So..." Indicating a consequence of previous statements "Then n² = 4k²."
"Therefore..." / "Thus..." / "Hence..." Drawing a conclusion (often the final one) "Therefore, n² is even."
"Since..." Providing justification for a claim "Since k is an integer, 2k² is also an integer."
"Note that..." / "Observe that..." Highlighting an important observation "Note that n + 1 is odd when n is even."
"Without loss of generality (WLOG)..." When cases are symmetric "WLOG, assume a ≤ b."
"We consider two cases..." Beginning a proof by cases "We consider two cases: n is even, or n is odd."
"We prove the contrapositive..." Announcing a contrapositive proof "We prove the contrapositive: If n is odd, then n² is odd."

Mathematical Notation Conventions

Convention Correct Usage Incorrect Usage
Variables Use italics: n, x, k Don't use regular text: n, x, k
Equations in sentences Integrate smoothly: "Since n = 2k, we have..." Don't start sentences with symbols: "= 2k, so..."
Definitions Use "if and only if" or "iff" for definitions Don't use "if" alone for definitions
Quantifiers in prose "For all integers n" or "For every integer n" Don't write "∀n" in flowing prose
End of proof Use ∎, □, QED, or "This completes the proof." Don't leave the end unmarked

Writing Mathematics in Sentences

⚠️ Common Formatting Mistakes

  • Don't start a sentence with a symbol:
    • ❌ "n = 2k. So n² = 4k²."
    • ✓ "We have n = 2k, so n² = 4k²."
  • Don't use symbols as shorthand in prose:
    • ❌ "∀ integers n, n² ≥ 0"
    • ✓ "For all integers n, n² ≥ 0."
  • Don't chain equations without explanation:
    • ❌ "n² = (2k)² = 4k² = 2(2k²)"
    • ✓ "Then n² = (2k)² = 4k² = 2(2k²), which has the form 2m."

💡 The "Newspaper Test"

A good mathematical proof should read like a well-written article. Try reading your proof aloud. Does it flow naturally? Does each sentence connect logically to the next?

If you find yourself stumbling or confused, your reader will too.

3. Common Proof Writing Errors

Learning to recognize and avoid common errors will significantly improve your proof writing. Here are the most frequent mistakes students make:

Error 1: Assuming What You're Trying to Prove

❌ Incorrect
Claim: If n is even, then n² is even.

Proof: Assume n² is even. Then n² = 2k for some integer k. Since n² is even, n must be even. ∎

Problem: The proof assumes the conclusion (n² is even) instead of the hypothesis (n is even)!

✓ Correct
Claim: If n is even, then n² is even.

Proof: Assume n is even. Then n = 2k for some integer k. So n² = (2k)² = 4k² = 2(2k²). Since 2k² is an integer, n² is even. ∎

Why it's correct: Assumes the hypothesis (n is even) and derives the conclusion (n² is even).

Error 2: Using the Same Variable for Different Quantities

❌ Incorrect
Claim: The sum of two even integers is even.

Proof: Let m and n be even. Then m = 2k and n = 2k. So m + n = 4k = 2(2k), which is even. ∎

Problem: Using the same k for both m and n implies m = n, which isn't given!

✓ Correct
Claim: The sum of two even integers is even.

Proof: Let m and n be even. Then m = 2a and n = 2b for some integers a and b. So m + n = 2a + 2b = 2(a + b). Since a + b is an integer, m + n is even. ∎

Why it's correct: Uses different variables (a and b) for the different integers.

Error 3: Proving by Example

❌ Incorrect
Claim: For all integers n, n² + n is even.

Proof: Let n = 3. Then n² + n = 9 + 3 = 12, which is even. Let n = 4. Then n² + n = 16 + 4 = 20, which is even. Therefore, n² + n is always even. ∎

Problem: Checking specific examples doesn't prove a universal statement!

✓ Correct
Claim: For all integers n, n² + n is even.

Proof: Note that n² + n = n(n + 1). This is the product of two consecutive integers. Among any two consecutive integers, one is even. Therefore, n(n + 1) is even. ∎

Why it's correct: Uses a general argument that works for all n.

Error 4: Missing Justifications

❌ Incomplete
Claim: If n is even, then n² is even.

Proof: Let n be even. Then n = 2k. So n² = 4k² = 2(2k²), which is even. ∎

Problem: Why is 2k² an integer? This step needs justification.

✓ Complete
Claim: If n is even, then n² is even.

Proof: Let n be even. By definition, n = 2k for some integer k. Then n² = (2k)² = 4k² = 2(2k²). Let m = 2k². Since k is an integer, m is an integer (integers are closed under multiplication). Thus n² = 2m where m is an integer, so n² is even by definition. ∎

Why it's correct: Every step is justified, including the closure property.

Error 5: Confusing Conditional Directions

❌ Incorrect
Claim: If n² is even, then n is even.

Proof: Assume n is even. Then n = 2k, so n² = 4k² = 2(2k²), which is even. ∎

Problem: This proves the converse (if n is even, then n² is even), not the original statement!

✓ Correct (using contrapositive)
Claim: If n² is even, then n is even.

Proof: We prove the contrapositive: if n is odd, then n² is odd. Assume n is odd. Then n = 2k + 1 for some integer k. So n² = (2k + 1)² = 4k² + 4k + 1 = 2(2k² + 2k) + 1. Since 2k² + 2k is an integer, n² is odd. ∎

Why it's correct: Explicitly states we're proving the contrapositive, then proves it correctly.

Error 6: Incomplete Case Analysis

❌ Incomplete
Claim: For all real x, |x| ≥ 0.

Proof: Case 1: x > 0. Then |x| = x > 0 ≥ 0. ✓
Case 2: x < 0. Then |x| = -x > 0 ≥ 0. ✓
Therefore, |x| ≥ 0. ∎

Problem: What about x = 0? The cases aren't exhaustive!

✓ Complete
Claim: For all real x, |x| ≥ 0.

Proof: Case 1: x > 0. Then |x| = x > 0 ≥ 0. ✓
Case 2: x = 0. Then |x| = 0 ≥ 0. ✓
Case 3: x < 0. Then |x| = -x > 0 ≥ 0. ✓
Since these cases cover all real numbers, |x| ≥ 0. ∎

Why it's correct: All cases (positive, zero, negative) are covered.

✅ Proof Writing Error Checklist

☐ Did I assume the hypothesis (not the conclusion)?
☐ Did I use different variables for different quantities?
☐ Did I prove the statement generally (not just with examples)?
☐ Did I justify each step (including closure properties)?
☐ Did I prove the correct direction of the conditional?
☐ Did I cover all cases in a proof by cases?
☐ Did I verify that denominators are nonzero (for rational proofs)?

4. The Revision Process

Good mathematical writing rarely emerges fully formed. Like any writing, proofs benefit from revision. Here's a systematic process:

📝 The Three-Pass Revision Process

Pass 1: Check Correctness
  • Does each step follow logically from previous steps?
  • Are all definitions applied correctly?
  • Are there any logical gaps?
  • Is the conclusion what was claimed?
Pass 2: Check Completeness
  • Is every claim justified?
  • Are closure properties mentioned where needed?
  • Are all cases covered (for proof by cases)?
  • Is it clear which proof technique is being used?
Pass 3: Check Clarity and Style
  • Does the proof flow smoothly?
  • Are standard phrases used appropriately?
  • Is there unnecessary repetition?
  • Would a classmate understand each step?

📝 Example: Revising a Proof

Original (rough draft):

Prove: If a | b and b | c then a | c.

a | b so b = ak. b | c so c = bm. c = akm. a | c.

After Pass 1 (Correctness): The logic is correct, but let's make it clearer.

After Pass 2 (Completeness): Need to justify that km is an integer.

After Pass 3 (Clarity): Add structure and flow.

Final (polished):

Theorem: If a | b and b | c, then a | c.

Proof:

Assume a | b and b | c.

By definition of divisibility, b = ak for some integer k.

By definition of divisibility, c = bm for some integer m.

Substituting the first equation into the second:

c = bm = (ak)m = a(km).

Let j = km. Since k and m are integers, j is an integer.

Thus c = aj where j is an integer.

By definition of divisibility, a | c.

∎

5. Connecting Proofs to Algorithm Correctness

The proof techniques you've learned this week directly apply to computer science, particularly in verifying that algorithms are correct.

Proof Techniques in CS Contexts

Proof Technique CS Application Example
Direct Proof Proving loop invariants, algorithm properties "If the array is sorted before iteration i, it remains sorted after."
Proof by Cases Analyzing conditional branches "We consider two cases: the element is found, or it is not found."
Contrapositive Proving properties by their logical equivalents "If the output is incorrect, then the input violated the precondition."
Counterexample Finding bugs, disproving claimed properties "This sorting algorithm fails for input [3, 1, 2]."

📝 Example: Proving a Simple Algorithm Property

Consider this algorithm to compute the absolute value:

function absoluteValue(x):
    if x >= 0:
        return x
    else:
        return -x

Claim: For all real numbers x, absoluteValue(x) ≥ 0.

Proof (by cases):

Let x be any real number. We consider two cases based on the conditional in the algorithm.

Case 1: x ≥ 0.

The algorithm returns x. Since x ≥ 0, the return value is ≥ 0. ✓

Case 2: x < 0.

The algorithm returns -x. Since x < 0, we have -x > 0 ≥ 0. ✓

In both cases, absoluteValue(x) ≥ 0.

∎

Notice how the proof structure mirrors the algorithm's conditional structure!

💡 Connection to CSLO 4: Induction and Recursion

In Week 6, you'll learn mathematical induction, which is the primary technique for proving properties of:

  • Recursive algorithms: Proving that a recursive function computes the correct result
  • Loop invariants: Proving that a property holds through all iterations of a loop
  • Recurrence relations: Proving closed-form solutions for recursive definitions

The clear, structured proof writing you've learned this week will be essential for induction proofs!

7. Practice Problems

For each problem, write a polished proof following all the conventions discussed in this module.

📝 Problem 1 Easy

Prove: If n is odd, then n + 1 is even.

Theorem: If n is odd, then n + 1 is even.

Proof:

Assume n is an odd integer.

By definition of odd, n = 2k + 1 for some integer k.

Then n + 1 = (2k + 1) + 1 = 2k + 2 = 2(k + 1).

Let m = k + 1. Since k is an integer, m is an integer.

Thus n + 1 = 2m where m is an integer.

By definition of even, n + 1 is even.

∎

📝 Problem 2 Medium

Prove: For any integer n, n(n + 1) is even.

Theorem: For any integer n, n(n + 1) is even.

Proof (by cases):

Let n be any integer. We consider two cases based on the parity of n.

Case 1: n is even.

By definition, n = 2k for some integer k.

Then n(n + 1) = 2k(n + 1) = 2[k(n + 1)].

Since k(n + 1) is an integer, n(n + 1) is even.

Case 2: n is odd.

Then n + 1 is even (since odd + 1 = even).

By definition, n + 1 = 2j for some integer j.

Then n(n + 1) = n(2j) = 2(nj).

Since nj is an integer, n(n + 1) is even.

In both cases, n(n + 1) is even.

∎

📝 Problem 3 Medium

Prove by contrapositive: If n² is even, then n is even.

Theorem: If n² is even, then n is even.

Proof (by contrapositive):

We prove the contrapositive: If n is odd, then n² is odd.

Assume n is an odd integer.

By definition of odd, n = 2k + 1 for some integer k.

Then n² = (2k + 1)² = 4k² + 4k + 1 = 2(2k² + 2k) + 1.

Let m = 2k² + 2k. Since k is an integer, m is an integer.

Thus n² = 2m + 1 where m is an integer.

By definition of odd, n² is odd.

This proves the contrapositive, so the original statement holds.

∎

📝 Problem 4 Hard

Edit the following rough proof into a polished version:

"prove sum of rationals is rational. r=a/b s=c/d. r+s = (ad+bc)/bd. ad+bc and bd are integers and bd≠0 so r+s rational."

Theorem: The sum of two rational numbers is rational.

Proof:

Let r and s be rational numbers.

By definition of rational, r = a/b for some integers a, b with b ≠ 0.

By definition of rational, s = c/d for some integers c, d with d ≠ 0.

Then r + s = a/b + c/d = (ad + bc)/(bd).

Let p = ad + bc and q = bd.

Since a, b, c, d are integers, p is an integer (closure under multiplication and addition).

Since b, d are integers, q is an integer (closure under multiplication).

Since b ≠ 0 and d ≠ 0, we have q = bd ≠ 0 (product of nonzero integers is nonzero).

Thus r + s = p/q where p and q are integers with q ≠ 0.

By definition of rational, r + s is rational.

∎
