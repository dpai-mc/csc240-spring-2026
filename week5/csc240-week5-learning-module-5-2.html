1. What is a Direct Proof?

📖 Definition: Direct Proof

A direct proof is a method of proving a conditional statement P → Q by:

  1. Assuming the hypothesis P is true
  2. Using definitions, axioms, and previously proven theorems
  3. Logically deriving that the conclusion Q must also be true

The logical structure is: Assume P → [logical steps] → Therefore Q

Why Direct Proof Works: The Logical Foundation

The direct proof technique is grounded in the truth table for implication:

P Q P → Q Analysis
T T T ✓ This is what we want to show
T F F ✗ This would make the implication false
F T T Not our concern (P is false)
F F T Not our concern (P is false)

💡 Key Insight

The implication P → Q is only false when P is true and Q is false (row 2).

To prove P → Q is always true, we need to show that whenever P is true, Q must also be true.

That's exactly what a direct proof does: we assume P is true, then show Q must follow—eliminating the possibility of row 2.

The Direct Proof Template

📋 Direct Proof Template for P → Q

  1. State the theorem: "Theorem: If P, then Q."
  2. Begin the proof: "Proof:"
  3. Assume the hypothesis: "Assume P." or "Let [variable] satisfy P."
  4. Apply definitions: "By definition of [term], [variable] = [expression]."
  5. Perform logical/algebraic steps: "Then... Therefore... It follows that..."
  6. Apply definitions to establish conclusion: "By definition of [term], Q is satisfied."
  7. State the conclusion: "Therefore, Q." or "Thus, Q holds."
  8. End the proof: ∎

The Role of Definitions: The Key to Direct Proofs

Definitions are the engine of direct proofs. They serve two critical purposes:

🔑 Two Uses of Definitions in Direct Proofs

1. Unpacking the Hypothesis When you assume P, the definition of P tells you what mathematical facts you can use.
Example: "Assume n is even" → By definition, n = 2k for some integer k.
2. Establishing the Conclusion To prove Q, you must show your result satisfies the definition of Q.
Example: To prove "m is even" → Show m = 2j for some integer j.

💡 Think of Definitions as Translators

Definitions translate between:

  • English descriptions ("n is even") and
  • Mathematical expressions (n = 2k)

A direct proof typically follows this pattern:

English (hypothesis) → Definition → Math → Algebra → Math → Definition → English (conclusion)

2. Your First Direct Proof: A Complete Walkthrough

Let's work through a complete direct proof step-by-step, explaining every decision along the way.

📝 Example 1: The Sum of Two Even Numbers

Theorem: If m and n are even integers, then m + n is even.

Step-by-Step Construction

Identify the structure

This is a conditional statement: "If P, then Q"

  • P (hypothesis): "m and n are even integers"
  • Q (conclusion): "m + n is even"

For a direct proof, we'll assume P and derive Q.

Assume the hypothesis

We begin by assuming P is true:

"Assume m and n are even integers."

This is our starting point—everything that follows must build from this assumption.

Apply the definition to unpack the hypothesis

What does it mean for m to be even? What does it mean for n to be even?

Definition of even: An integer x is even if x = 2k for some integer k.

Applying this definition:

  • Since m is even, m = 2a for some integer a
  • Since n is even, n = 2b for some integer b

"By definition of even, m = 2a for some integer a, and n = 2b for some integer b."

Perform algebraic manipulation

Now we have mathematical expressions to work with. Let's compute m + n:

m + n = 2a + 2b = 2(a + b)

"Then m + n = 2a + 2b = 2(a + b)."

Identify the form needed for the conclusion

To prove m + n is even, we need to show m + n = 2k for some integer k.

We have m + n = 2(a + b). Is (a + b) an integer?

Yes! Since a and b are integers, and integers are closed under addition, (a + b) is an integer.

"Let c = a + b. Since a and b are integers, c is an integer."

Apply the definition to establish the conclusion

We now have m + n = 2c where c is an integer.

This is exactly the definition of even!

"Thus m + n = 2c where c is an integer. By definition of even, m + n is even."

Conclude and end the proof

"Therefore, if m and n are even integers, then m + n is even. ∎"

The Complete Proof

Theorem: If m and n are even integers, then m + n is even.

Proof:

Assume m and n are even integers. (Assumption)

By definition of even, m = 2a for some integer a, and n = 2b for some integer b. (Definition of even)

Then m + n = 2a + 2b = 2(a + b). (Algebra: substitution and factoring)

Let c = a + b. Since a and b are integers, c is an integer. (Closure of integers under addition)

Thus m + n = 2c where c is an integer. (Substitution)

By definition of even, m + n is even. (Definition of even)

∎

⚠️ Critical Mistake to Avoid: Using the Same Variable

Notice that we used different variables (a and b) when applying the definition to m and n:

  • m = 2a
  • n = 2b

Why? If we wrote "m = 2k and n = 2k," we would be claiming m = n, which isn't necessarily true!

Wrong: "Since m is even, m = 2k. Since n is even, n = 2k." (This implies m = n)

Correct: "Since m is even, m = 2a. Since n is even, n = 2b." (Different variables for different values)

3. Proofs Involving Parity (Even and Odd)

Parity refers to whether an integer is even or odd. These are among the most common types of direct proofs you'll encounter.

Quick Reference: Parity Definitions

Property Definition Examples
Even n = 2k for some integer k ..., -4, -2, 0, 2, 4, 6, ...
Odd n = 2k + 1 for some integer k ..., -3, -1, 1, 3, 5, 7, ...

📝 Example 2: Square of an Even Number

Theorem: If n is even, then n² is even.

Strategy:

  • Hypothesis: n is even
  • Conclusion: n² is even
  • Plan: Use the definition of even to write n = 2k, compute n², and show the result has the form 2m.

Theorem: If n is even, then n² is even.

Proof:

Assume n is an even integer. (Assumption)

By definition of even, n = 2k for some integer k. (Definition of even)

Then n² = (2k)² = 4k² = 2(2k²). (Algebra)

Let m = 2k². Since k is an integer, 2k² is an integer, so m is an integer. (Closure of integers)

Thus n² = 2m where m is an integer. (Substitution)

By definition of even, n² is even. (Definition of even)

∎

📝 Example 3: Square of an Odd Number

Theorem: If n is odd, then n² is odd.

Strategy:

  • Hypothesis: n is odd
  • Conclusion: n² is odd
  • Plan: Use the definition of odd to write n = 2k + 1, compute n², and show the result has the form 2m + 1.

Theorem: If n is odd, then n² is odd.

Proof:

Assume n is an odd integer. (Assumption)

By definition of odd, n = 2k + 1 for some integer k. (Definition of odd)

Then n² = (2k + 1)² (Substitution)

= 4k² + 4k + 1 (Expanding the square)

= 2(2k² + 2k) + 1. (Factoring out 2 from the first two terms)

Let m = 2k² + 2k. Since k is an integer, m is an integer. (Closure of integers)

Thus n² = 2m + 1 where m is an integer. (Substitution)

By definition of odd, n² is odd. (Definition of odd)

∎

📝 Example 4: Sum of Even and Odd

Theorem: The sum of an even integer and an odd integer is odd.

Theorem: If m is even and n is odd, then m + n is odd.

Proof:

Assume m is an even integer and n is an odd integer. (Assumption)

By definition, m = 2a for some integer a, and n = 2b + 1 for some integer b. (Definitions)

Then m + n = 2a + (2b + 1) = 2a + 2b + 1 = 2(a + b) + 1. (Algebra)

Let c = a + b. Since a and b are integers, c is an integer. (Closure)

Thus m + n = 2c + 1 where c is an integer. (Substitution)

By definition of odd, m + n is odd. (Definition of odd)

∎

📝 Example 5: Product of Two Odd Numbers

Theorem: The product of two odd integers is odd.

Theorem: If m and n are odd integers, then mn is odd.

Proof:

Assume m and n are odd integers. (Assumption)

By definition of odd, m = 2a + 1 for some integer a, and n = 2b + 1 for some integer b. (Definition of odd)

Then mn = (2a + 1)(2b + 1) (Substitution)

= 4ab + 2a + 2b + 1 (FOIL expansion)

= 2(2ab + a + b) + 1. (Factoring out 2)

Let c = 2ab + a + b. Since a and b are integers, c is an integer. (Closure)

Thus mn = 2c + 1 where c is an integer. (Substitution)

By definition of odd, mn is odd. (Definition of odd)

∎

Summary: Parity Proof Patterns

To Prove Target Form Strategy
Result is even 2k for some integer k Factor out 2 from your expression
Result is odd 2k + 1 for some integer k Factor out 2 from all but one term, leaving +1

4. Proofs Involving Divisibility

Divisibility proofs are fundamental in number theory and have important applications in computer science, particularly in cryptography and algorithm analysis.

Quick Reference: Divisibility Definition

📖 Definition: Divisibility (Reminder)

An integer a divides an integer b (written a ∣ b) if and only if there exists an integer k such that b = ak.

Equivalent statements:

  • "a divides b"
  • "a is a divisor of b"
  • "a is a factor of b"
  • "b is a multiple of a"
  • "b is divisible by a"

Notation: a ∣ b means "a divides b" — a ∤ b means "a does not divide b"

⚠️ Important: a ∣ b is NOT a Fraction!

Don't confuse the divisibility notation a ∣ b with the fraction a/b.

  • 3 ∣ 12 is a statement (true or false) meaning "3 divides 12" — TRUE
  • 3/12 is a number equal to 0.25

📝 Example 6: Transitivity of Divisibility

Theorem: If a ∣ b and b ∣ c, then a ∣ c.

(In words: If a divides b and b divides c, then a divides c.)

Strategy:

  • Hypothesis: a ∣ b and b ∣ c
  • Conclusion: a ∣ c
  • Plan: Apply the definition of divisibility to both hypotheses, then combine to show c = a(something).

Theorem: If a ∣ b and b ∣ c, then a ∣ c.

Proof:

Assume a ∣ b and b ∣ c. (Assumption)

By definition of divisibility, b = am for some integer m. (Definition applied to a ∣ b)

By definition of divisibility, c = bn for some integer n. (Definition applied to b ∣ c)

Substituting the first equation into the second: (Substitution)

c = bn = (am)n = a(mn). (Algebra: associativity of multiplication)

Let k = mn. Since m and n are integers, k is an integer. (Closure of integers under multiplication)

Thus c = ak where k is an integer. (Substitution)

By definition of divisibility, a ∣ c. (Definition of divisibility)

∎

📝 Example 7: Divisibility of Sum

Theorem: If a ∣ b and a ∣ c, then a ∣ (b + c).

Theorem: If a ∣ b and a ∣ c, then a ∣ (b + c).

Proof:

Assume a ∣ b and a ∣ c. (Assumption)

By definition, b = am for some integer m, and c = an for some integer n. (Definition of divisibility)

Then b + c = am + an = a(m + n). (Algebra: factoring out a)

Let k = m + n. Since m and n are integers, k is an integer. (Closure)

Thus b + c = ak where k is an integer. (Substitution)

By definition of divisibility, a ∣ (b + c). (Definition)

∎

📝 Example 8: Linear Combinations

Theorem: If a ∣ b and a ∣ c, then a ∣ (bx + cy) for any integers x and y.

(This is a generalization of Example 7—the sum b + c is the special case where x = y = 1.)

Theorem: If a ∣ b and a ∣ c, then a ∣ (bx + cy) for any integers x and y.

Proof:

Assume a ∣ b and a ∣ c, and let x and y be any integers. (Assumption)

By definition, b = am for some integer m, and c = an for some integer n. (Definition)

Then bx + cy = (am)x + (an)y (Substitution)

= a(mx) + a(ny) (Associativity and commutativity)

= a(mx + ny). (Factoring out a)

Let k = mx + ny. Since m, n, x, y are integers, k is an integer. (Closure)

Thus bx + cy = ak where k is an integer. (Substitution)

By definition of divisibility, a ∣ (bx + cy). (Definition)

∎

📝 Example 9: Divisibility Chain

Theorem: If 6 ∣ n, then 3 ∣ n.

(More generally: If ab ∣ n, then a ∣ n.)

Theorem: If 6 ∣ n, then 3 ∣ n.

Proof:

Assume 6 ∣ n. (Assumption)

By definition of divisibility, n = 6k for some integer k. (Definition)

Then n = 6k = (3 · 2)k = 3(2k). (Algebra: 6 = 3 · 2)

Let m = 2k. Since k is an integer, m is an integer. (Closure)

Thus n = 3m where m is an integer. (Substitution)

By definition of divisibility, 3 ∣ n. (Definition)

∎

💡 Computer Science Connection: Divisibility in Programming

Divisibility is fundamental to many CS applications:

  • Hash functions: h(k) = k mod m — understanding divisibility helps analyze hash distribution
  • Cryptography: RSA encryption relies heavily on divisibility properties of prime numbers
  • Algorithm analysis: Proving that loop invariants involve divisibility conditions
  • Data structures: Circular buffers use modular arithmetic based on divisibility

5. Proofs Involving Rational Numbers

Rational number proofs require careful attention to the condition that the denominator cannot be zero.

Quick Reference: Rational Number Definition

📖 Definition: Rational Number (Reminder)

A real number r is rational if and only if there exist integers a and b with b ≠ 0 such that r = a/b.

Key requirements:

  • a must be an integer
  • b must be an integer
  • b must be nonzero

Notation: The set of rational numbers is denoted ℚ.

📝 Example 10: Sum of Rational Numbers

Theorem: The sum of two rational numbers is rational.

Strategy:

  • Hypothesis: r and s are rational
  • Conclusion: r + s is rational
  • Plan: Write r = a/b and s = c/d, compute r + s using common denominators, verify the result is a ratio of integers with nonzero denominator.

Theorem: If r and s are rational numbers, then r + s is rational.

Proof:

Assume r and s are rational numbers. (Assumption)

By definition of rational, r = a/b for integers a, b with b ≠ 0. (Definition)

By definition of rational, s = c/d for integers c, d with d ≠ 0. (Definition)

Then r + s = a/b + c/d = (ad + bc)/(bd). (Algebra: common denominator)

Let p = ad + bc and q = bd. (Substitution)

Since a, b, c, d are integers, p = ad + bc is an integer. (Closure under multiplication and addition)

Since b, d are integers, q = bd is an integer. (Closure under multiplication)

Since b ≠ 0 and d ≠ 0, we have q = bd ≠ 0. (Product of nonzero integers is nonzero)

Thus r + s = p/q where p and q are integers with q ≠ 0. (Substitution)

By definition of rational, r + s is rational. (Definition)

∎

📝 Example 11: Product of Rational Numbers

Theorem: The product of two rational numbers is rational.

Theorem: If r and s are rational numbers, then rs is rational.

Proof:

Assume r and s are rational numbers. (Assumption)

By definition, r = a/b and s = c/d for integers a, b, c, d with b, d ≠ 0. (Definition)

Then rs = (a/b) · (c/d) = ac/(bd). (Algebra)

Let p = ac and q = bd. (Substitution)

Since a, c are integers, p = ac is an integer. (Closure)

Since b, d are integers, q = bd is an integer. (Closure)

Since b ≠ 0 and d ≠ 0, we have q = bd ≠ 0. (Nonzero product)

Thus rs = p/q where p, q are integers with q ≠ 0. (Substitution)

By definition of rational, rs is rational. (Definition)

∎

📝 Example 12: Reciprocal of a Nonzero Rational

Theorem: If r is a nonzero rational number, then 1/r is rational.

Theorem: If r is a nonzero rational number, then 1/r is rational.

Proof:

Assume r is a nonzero rational number. (Assumption)

By definition, r = a/b for integers a, b with b ≠ 0. (Definition)

Since r ≠ 0, we have a/b ≠ 0, which means a ≠ 0. (Nonzero fraction implies nonzero numerator)

Then 1/r = 1/(a/b) = b/a. (Algebra: reciprocal of a fraction)

Now b is an integer, a is an integer, and a ≠ 0. (From above)

Thus 1/r = b/a where b and a are integers with a ≠ 0. (Substitution)

By definition of rational, 1/r is rational. (Definition)

∎

⚠️ Critical: Don't Forget the Nonzero Condition!

In rational number proofs, you must always verify that denominators are nonzero:

  • When applying the definition: "...with b ≠ 0"
  • When forming new fractions: "Since b ≠ 0 and d ≠ 0, we have bd ≠ 0"
  • When taking reciprocals: "Since r ≠ 0, we have a ≠ 0"

Forgetting this step is a common error that makes proofs incomplete!

6. Common Mistakes and How to Avoid Them

Learning to write proofs involves learning what not to do. Here are the most common mistakes students make in direct proofs:

Mistake 1: Reusing Variables

❌ Incorrect

"Assume m and n are even."

"Then m = 2k and n = 2k for some integer k."

Problem: This claims m = n!

✓ Correct

"Assume m and n are even."

"Then m = 2a and n = 2b for some integers a and b."

Why: Different variables allow m ≠ n.

Mistake 2: Assuming What You're Trying to Prove

❌ Incorrect

"Prove: If n is even, then n² is even."

"Assume n² is even..."

Problem: You assumed the conclusion!

✓ Correct

"Prove: If n is even, then n² is even."

"Assume n is even..."

Why: Assume the hypothesis, prove the conclusion.

Mistake 3: Using Examples Instead of General Proof

❌ Incorrect

"Prove: The sum of two even integers is even."

"Let m = 4 and n = 6. Then m + n = 10, which is even. ∎"

Problem: This only shows one case!

✓ Correct

"Prove: The sum of two even integers is even."

"Let m and n be any even integers. Then m = 2a and n = 2b..."

Why: Works for ALL even integers.

Mistake 4: Skipping the Closure Justification

❌ Incomplete

"...so m + n = 2(a + b)."

"Therefore m + n is even. ∎"

Problem: Why is (a + b) an integer?

✓ Complete

"...so m + n = 2(a + b)."

"Let c = a + b. Since a and b are integers, c is an integer."

"Thus m + n = 2c where c is an integer, so m + n is even. ∎"

Mistake 5: Working Backwards Without Stating It

❌ Confusing

"We need to show n² is even."

"n² = 2k for some integer k."

"So n² = (2m)² = 4m² = 2(2m²)."

Problem: Mixes goal with derivation.

✓ Clear

"Assume n is even, so n = 2m for some integer m."

"Then n² = (2m)² = 4m² = 2(2m²)."

"Let k = 2m². Then n² = 2k, so n² is even."

Why: Clear forward progression.

Mistake 6: Forgetting to Introduce Variables

❌ Unclear

"Assume n is even."

"Then n = 2k."

Problem: Where did k come from?

✓ Clear

"Assume n is even."

"By definition, n = 2k for some integer k."

Why: Explicitly introduces k.

✅ Proof Writing Checklist

Before submitting a direct proof, verify:

  • ☐ Did I assume the hypothesis (not the conclusion)?
  • ☐ Did I use different variables for different quantities?
  • ☐ Did I explicitly apply definitions?
  • ☐ Did I justify that new expressions are integers (closure)?
  • ☐ Did I verify denominators are nonzero (for rational proofs)?
  • ☐ Did I explicitly state the conclusion?
  • ☐ Does every step follow logically from previous steps?

7. The "Work Backwards" Strategy

When you're stuck on a proof, it often helps to work backwards from what you need to show. This is a planning strategy, not the final proof structure.

🎯 The Work Backwards Strategy

  1. Identify the goal: What does the conclusion look like when you apply its definition?
  2. Identify the start: What does the hypothesis give you when you apply its definition?
  3. Find the bridge: What algebraic or logical steps connect start to goal?
  4. Write the proof forward: Present the proof from hypothesis to conclusion.

📝 Example 13: Applying the Strategy

Theorem: If n is even, then 3n + 2 is even.

Planning Phase (Work Backwards)

1
Goal: Show 3n + 2 is even
Need to show: 3n + 2 = 2m for some integer m
2
Start: n is even
This means: n = 2k for some integer k
3
Bridge: Substitute and simplify
3n + 2 = 3(2k) + 2 = 6k + 2 = 2(3k + 1)
4
Check: Is (3k + 1) an integer?
Yes! Since k is an integer, 3k + 1 is an integer.

Final Proof (Written Forward)

Theorem: If n is even, then 3n + 2 is even.

Proof:

Assume n is an even integer. (Assumption)

By definition of even, n = 2k for some integer k. (Definition)

Then 3n + 2 = 3(2k) + 2 = 6k + 2 = 2(3k + 1). (Algebra)

Let m = 3k + 1. Since k is an integer, m is an integer. (Closure)

Thus 3n + 2 = 2m where m is an integer. (Substitution)

By definition of even, 3n + 2 is even. (Definition)

∎

📝 Example 14: A Trickier Case

Theorem: If n is odd, then 5n + 3 is even.

Planning Phase

  • Goal: Show 5n + 3 = 2m for some integer m
  • Start: n = 2k + 1 for some integer k
  • Bridge: 5n + 3 = 5(2k + 1) + 3 = 10k + 5 + 3 = 10k + 8 = 2(5k + 4)
  • Check: 5k + 4 is an integer ✓

Theorem: If n is odd, then 5n + 3 is even.

Proof:

Assume n is an odd integer. (Assumption)

By definition of odd, n = 2k + 1 for some integer k. (Definition)

Then 5n + 3 = 5(2k + 1) + 3 = 10k + 5 + 3 = 10k + 8 = 2(5k + 4). (Algebra)

Let m = 5k + 4. Since k is an integer, m is an integer. (Closure)

Thus 5n + 3 = 2m where m is an integer. (Substitution)

By definition of even, 5n + 3 is even. (Definition)

∎

Theorem: If n is even, then n² + 2n is even.

Prove: If m and n are both odd, then m + n is even.

Let m = 2a + 1 and n = 2b + 1. Then m + n = 2a + 1 + 2b + 1 = 2(a + b + 1).

Divisibility Proofs

📝 Problem 5 Easy

Prove: If 6 | n, then 2 | n.

If n = 6k, then n = 2(3k). Since 3k is an integer, 2 | n.

📝 Problem 6 Medium

Prove: If a | b, then a | (-b).

If b = ak, then -b = a(-k). Is -k an integer? Yes!

📝 Problem 7 Medium

Prove: If a | b and a | c, then a | (2b + 3c).

This is a special case of the linear combination theorem (Example 8). Use b = am and c = an.

📝 Problem 8 Hard

Prove: For all integers n, 3 | n(n+1)(n+2).

n(n+1)(n+2) is the product of three consecutive integers. Among any three consecutive integers, at least one must be divisible by 3. (This actually requires proof by cases—we'll cover this in Module 5.3!)

Rational Number Proofs

📝 Problem 9 Medium

Prove: If r is rational and r ≠ 0, then r² is rational.

If r = a/b, then r² = a²/b². Check that a² and b² are integers with b² ≠ 0.

📝 Problem 10 Medium

Prove: If r and s are rational with s ≠ 0, then r/s is rational.

r/s = r · (1/s). Use the facts that the product of rationals is rational and the reciprocal of a nonzero rational is rational.

11. Summary and Key Takeaways

📋 Key Takeaways from Module 5.2

  1. Direct proof proves P → Q by assuming P and deriving Q through logical steps.
  2. Definitions are essential: Use them to unpack the hypothesis and establish the conclusion.
  3. Use different variables when applying definitions to different quantities.
  4. Justify closure: Explain why new expressions are integers (or rationals, etc.).
  5. The "work backwards" strategy helps plan proofs by identifying what form the conclusion needs.
  6. Common patterns:
    • To prove even: Show result = 2k for integer k
    • To prove odd: Show result = 2k + 1 for integer k
    • To prove a | b: Show b = ak for integer k
    • To prove rational: Show result = p/q for integers p, q with q ≠ 0
