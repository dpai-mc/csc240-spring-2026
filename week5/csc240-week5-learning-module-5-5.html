💡 A Different Kind of "Proof"

In Modules 5.2-5.4, you learned how to prove that statements are true. But what if a statement is false? You can't prove a false statement true—instead, you need to disprove it!

This module teaches you how to disprove false statements using counterexamples.

1. What is a Counterexample?

📖 Definition: Counterexample

A counterexample is a specific example that demonstrates a universal statement is false.

To disprove the statement ∀x ∈ D, P(x) ("For all x in domain D, P(x) is true"), you need to find just ONE value c ∈ D such that P(c) is false.

The Logical Foundation

The power of counterexamples comes from a fundamental logical equivalence:

¬(∀x, P(x)) ≡ ∃x, ¬P(x)

In plain English:

  • Original: "For all x, P(x) is true"
  • Negation: "There exists some x where P(x) is false"

📐 Key Principle: One Counterexample Suffices

To prove that a universal statement is FALSE, you only need to find ONE example where it fails.

This single example—the counterexample—proves that "for all" is too strong a claim.

Proving vs. Disproving: A Critical Distinction

✓ To PROVE ∀x, P(x)
  • Must show P(x) holds for ALL x
  • Cannot use specific examples as proof (only for intuition)
  • Need a general argument (direct proof, contrapositive, etc.)
  • One example proves nothing
✗ To DISPROVE ∀x, P(x)
  • Must find ONE x where P(x) fails
  • A specific example IS the proof
  • Show the counterexample and verify it fails
  • One example is enough!

⚠️ Critical Warning: Examples Don't Prove Universal Statements!

Finding examples where P(x) is TRUE does NOT prove ∀x, P(x)!

Example: The statement "All primes are odd" seems supported by 3, 5, 7, 11, 13... but it's FALSE because 2 is prime and even.

No matter how many supporting examples you find, you haven't proven a universal statement. But ONE counterexample disproves it!

💡 Computer Science Connection: Testing vs. Verification

This asymmetry between proving and disproving directly relates to software testing:

  • Testing can find bugs (counterexamples to "the program works correctly")
  • Testing cannot prove a program is correct (no matter how many tests pass)
  • As Edsger Dijkstra famously said: "Testing shows the presence, not the absence of bugs."

A counterexample is like a failing test case—one failure proves the code has a bug!

2. The Structure of a Disproof by Counterexample

A disproof by counterexample has a specific structure. Here's the template:

📋 Disproof by Counterexample Template

  1. State what you're disproving: "We disprove the claim: [Universal statement]."
  2. Present the counterexample: "Consider x = c." (where c is your specific value)
  3. Verify it's in the domain: "Note that c is in the domain [if not obvious]."
  4. Show the property fails: "For this value, P(c) is false because [show calculation/reasoning]."
  5. Conclude: "Therefore, the claim '[Universal statement]' is false."

📝 Example 1: Complete Disproof Structure

Claim: For all integers n, n² > n.

Disproof:

We disprove the claim by counterexample.

Counterexample: Let n = 1.

n is an integer, so it's in the domain.

Then n² = 1² = 1 and n = 1.

We need n² > n, but 1 > 1 is false (since 1 = 1).

Therefore, the claim "For all integers n, n² > n" is false.

Counterexample Found!

n = 1
Verification: n² = 1, n = 1, so n² = n, not n² > n.
CLAIM IS FALSE

Another Counterexample

What about n = 0? If n = 0, then 0² = 0, so 0 > 0 is also false. Thus, n = 0 is also a valid counterexample!

⚠️ Common Mistake: Incomplete Verification

Always verify that your counterexample actually fails! Don't just state a value—show the calculation or reasoning that proves it doesn't satisfy the property.

Incomplete: "n = 0 is a counterexample." ✗

Complete: "Let n = 0. Then n² = 0 and n = 0, so n² = n, not n² > n, thus the claim fails." ✓

3. Classic Counterexamples

Let's work through several classic counterexamples that illustrate common patterns.

📝 Example 2: All Primes Are Odd

Claim: All prime numbers are odd.

Disproof:

We disprove the claim by counterexample.

Counterexample: Consider the number 2.

2 is a prime number (its only positive divisors are 1 and 2).

However, 2 is an even number (since 2 = 2 × 1), not odd.

Therefore, 2 is a prime number that is not odd, which contradicts the claim.

The claim "All prime numbers are odd" is false.

Counterexample Found!

2
Why it works: 2 is prime, but 2 is even, not odd.
CLAIM IS FALSE

📝 Example 3: Square Root of Sum

Claim: For all positive real numbers x and y, √(x + y) = √x + √y.

Disproof:

We disprove the claim by counterexample.

Counterexample: Let x = 1 and y = 1.

Both are positive real numbers, so they are in the domain.

Left side: √(x + y) = √(1 + 1) = √2.

Right side: √x + √y = √1 + √1 = 1 + 1 = 2.

Since √2 ≈ 1.414 and 2, we see that √2 ≠ 2.

Therefore, the claim "∀x, y ∈ ℝ⁺, √(x + y) = √x + √y" is false.

Counterexample Found!

x = 1, y = 1
Calculation:
√(1+1) = √2 ≈ 1.414
√1 + √1 = 1 + 1 = 2
√2 ≠ 2
CLAIM IS FALSE

📝 Example 4: Divisibility Doesn't Multiply

Claim: For all integers a, b, c, if a ∣ c and b ∣ c, then ab ∣ c.

Disproof:

We disprove the claim by counterexample.

Counterexample: Let a = 2, b = 2, and c = 2.

All are integers, so they are in the domain.

Check hypothesis a ∣ c: Does 2 ∣ 2? Yes, since 2 = 2 × 1. ✓

Check hypothesis b ∣ c: Does 2 ∣ 2? Yes, since 2 = 2 × 1. ✓

Check conclusion ab ∣ c: Does (2 × 2) ∣ 2? Does 4 ∣ 2? No, because there is no integer k such that 2 = 4k. ✗

Since the conclusion fails for these values, the claim is false.

Counterexample Found!

a = 2, b = 2, c = 2
Verification:
2 ∣ 2 ✓
2 ∣ 2 ✓
But (2 × 2) ∣ 2 (i.e., 4 ∣ 2) ✗
CLAIM IS FALSE

Note: Another Counterexample

Consider a = 6, b = 4, c = 12. Both 6 ∣ 12 and 4 ∣ 12, but ab = 24, and 24 ∤ 12.

📝 Example 5: Square Divisibility

Claim: For all integers n, if n² is divisible by 4, then n is divisible by 4.

Disproof:

We disprove the claim by counterexample.

Counterexample: Let n = 2.

n is an integer, so it's in the domain.

First, check the hypothesis: Is n² divisible by 4?

n² = 2² = 4. Yes, 4 ∣ 4 since 4 = 4 × 1. ✓ (Hypothesis satisfied)

Next, check the conclusion: Is n divisible by 4?

Is 4 ∣ 2? No, because there is no integer k such that 2 = 4k. ✗ (Conclusion fails)

Since the hypothesis is true but the conclusion is false for n = 2, the claim is false.

Counterexample Found!

n = 2
Verification:
n² = 4, and 4 ∣ 4 ✓
But 4 ∤ 2 ✗
CLAIM IS FALSE

Contrast with a TRUE statement

Compare this with the true statement: "If n² is even, then n is even." (We proved this by contrapositive in Module 5.4). This illustrates that specific divisors matter!

While 4 ∣ n² implies 2 ∣ n, it does NOT imply 4 ∣ n.

📝 Example 6: Fermat Numbers

Claim: For all integers n ≥ 0, the number Fₙ = 2^(2ⁿ) + 1 is prime.

(These are called Fermat numbers. F₀ = 3, F₁ = 5, F₂ = 17, F₃ = 257, F₄ = 65537 are all prime, leading Fermat to conjecture they all are.)

Disproof:

We disprove the claim by counterexample.

Counterexample: Let n = 5.

Then F₅ = 2^(2⁵) + 1 = 2³² + 1.

Calculating 2³² + 1 gives 4,294,967,297.

Euler showed that F₅ is divisible by 641:

4,294,967,297 = 641 × 6,700,417

Since F₅ has a divisor other than 1 and itself (namely 641), F₅ is composite, not prime.

Therefore, the claim "For all integers n ≥ 0, Fₙ = 2^(2ⁿ) + 1 is prime" is false.

Counterexample Found!

n = 5
Calculation:
F₅ = 2³² + 1 = 4,294,967,297
4,294,967,297 = 641 × 6,700,417
Since F₅ is not prime, the claim fails.
CLAIM IS FALSE

Historical Note

This was a famous conjecture for over 100 years until Euler found the counterexample in 1732. It shows that even brilliant mathematicians can be misled by early examples!

4. Strategies for Finding Counterexamples

Finding a counterexample can sometimes feel like searching for a needle in a haystack. However, there are systematic strategies that can significantly increase your chances of success:

🔍 Strategies for Counterexample Hunting

Strategy Description Examples to Try (for integers)
Boundary Values Test values at the edges of the domain or near critical points. 0, 1, -1, smallest/largest possible value, values that make parts of expression zero.
Small Values / Systematic Search Start with the simplest values and work your way up (or down). For integers: 0, 1, 2, 3, -1, -2, ...
For reals: 0, 0.5, 1, -0.5, -1, ...
Special Cases Consider values with unique properties relevant to the statement. Primes, composites, perfect squares, powers of 2, rationals, irrationals.
"Breaking" Conditions Think about what would make the statement fail. If it's P → Q, try to make P true and Q false. If "n is even", try an even n. If "x > 0", try a positive x.
Negative Numbers Many algebraic properties behave differently with negative numbers than with positive ones. -1, -2, -0.5. Often overlooked!
Fractions / Irrational Numbers If the domain is real numbers, test non-integers. 1/2, 0.3, √2, π.

📝 Example 7: Applying Search Strategies

Claim: For all integers n, if n² is odd, then n is prime.

Let's try to find a counterexample using our strategies:

  • Goal: Find an integer n such that n² is odd, but n is NOT prime.
  • Small values:
    • n = 1: n² = 1 (odd). Is 1 prime? No. Counterexample!
    • n = 3: n² = 9 (odd). Is 3 prime? Yes. (Not a counterexample)
    • n = 5: n² = 25 (odd). Is 5 prime? Yes. (Not a counterexample)
  • "Breaking" Conditions: We need n to be NOT prime.
    • Try a composite number: n = 9. n² = 81 (odd). Is 9 prime? No. Counterexample!
    • Try n = -3. n² = 9 (odd). Is -3 prime? No (primes are > 1). Counterexample!

Many counterexamples exist! The simplest is usually preferred.

⚠️ The "No Counterexample Found" Fallacy

Just because you can't find a counterexample doesn't mean one doesn't exist, nor does it prove the statement is true. It simply means you haven't found one yet!

Remember the Fermat numbers Fₙ = 2^(2ⁿ) + 1. It took over 100 years to find the counterexample for n = 5!

6. Practice Problems

Disprove each false claim with a counterexample. For each problem, clearly state the counterexample and show why it disproves the claim.

📝 Problem 1 Easy

For all integers n, n² ≥ n.

Consider non-positive integers. What happens when n = 0 or n = 1?

📝 Problem 2 Medium

For all integers n > 1, 2ⁿ + 1 is prime.

Start checking small integer values for n: 2, 3, 4, 5... You might find a number that's easily factorable.

📝 Problem 3 Medium

For all real numbers x, y, √(x² + y²) = x + y.

Try simple positive values like x = 1, y = 1. Or consider cases where x + y might be negative.

📝 Problem 4 Medium

For all integers a, b, c, if a ∣ c and b ∣ c, then ab ∣ c.

Think about common multiples. What if a and b share common factors?

Try a = 2, b = 4, c = 4.

📝 Problem 5 Medium

For all integers n, if n² is divisible by 4, then n is divisible by 4.

Consider numbers that are even but not multiples of 4. What happens when you square them?

📝 Problem 6 Hard

Every odd integer can be written as the sum of two primes.

Start with small odd integers greater than 2. Can 3 be written as the sum of two primes? What about 5? 7? 9? Remember, 1 is not prime.

📝 Problem 7 Hard

For all positive real numbers x, x + 1/x ≥ 2. (Careful! This one might be true...)

The problem states "Careful! This one might be true...". If you can't find a counterexample after trying various values (small, large, fractions), consider if it's actually a true statement that needs proof. If it is true, you would state that no counterexample exists and it requires a proof.

📝 Problem 8 Hard

For all integers a, b, if a² ∣ b², then a ∣ b.

This looks like it might be true, but the wording "for all integers" includes negatives. What if a and b have different signs?

Try a = 2, b = −2. Does a² ∣ b²? Does a ∣ b?

8. Summary and Key Takeaways

📋 Key Takeaways from Module 5.5

  1. Counterexamples are used to disprove universal statements (∀x, P(x)).
  2. Only one counterexample is needed to disprove a universal statement.
  3. A counterexample must be a specific value in the domain for which the statement's property fails.
  4. Always verify that your chosen counterexample actually makes the statement false.
  5. Strategies for finding counterexamples include testing boundary values, small values, special cases, negative numbers, and fractions/irrationals.
  6. Failing to find a counterexample does NOT prove a statement true.
  7. The concept of counterexamples is analogous to failing test cases or bug reports in software engineering.
