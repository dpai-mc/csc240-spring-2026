💡 Building on Direct Proofs

In Module 5.2, you learned to prove statements by assuming the hypothesis and deriving the conclusion. Proof by cases extends this technique by handling situations where the hypothesis naturally splits into multiple scenarios. You'll essentially perform several "mini direct proofs"—one for each case.

1. What is Proof by Cases?

📖 Definition: Proof by Cases (Proof by Exhaustion)

A proof by cases is a proof technique where we:

  1. Divide the domain (or hypothesis) into several mutually exclusive cases that together cover all possibilities
  2. Prove that the conclusion holds in each case separately
  3. Conclude that since the cases are exhaustive, the conclusion holds in general

Also known as: Proof by exhaustion, case analysis, or case-by-case proof.

The Logical Foundation

Proof by cases is based on a fundamental logical equivalence. If we know that either $$P_1$$ or $$P_2$$ (or ... or $$P_n$$) must be true, and we can show that each one implies Q, then Q must be true:

$$[(P_1 \vee P_2 \vee \cdots \vee P_n) \wedge (P_1 \rightarrow Q) \wedge (P_2 \rightarrow Q) \wedge \cdots \wedge (P_n \rightarrow Q)] \rightarrow Q$$

In simpler terms:

📐 Logical Principle Behind Proof by Cases

If we can partition the domain into cases $$C_1, C_2, \ldots, C_n$$ such that:

  1. Exhaustive: Every element belongs to at least one case (no gaps)
  2. The conclusion holds in each case: $$C_i \rightarrow Q$$ for all i

Then Q is true for all elements in the domain.

When to Use Proof by Cases

Proof by cases is appropriate when:

Situation Common Cases Example Statement
Parity of integers n is even, n is odd "For all integers n, n² + n is even"
Sign of numbers x > 0, x = 0, x < 0 "For all real x, |x| ≥ 0"
Absolute value x ≥ 0, x < 0 "For all real x, |x| = |-x|"
Divisibility classes n ≡ 0, 1, 2 (mod 3) "For all integers n, n³ ≡ n (mod 3)"
Comparison a ≤ b, a > b "max(a,b) + min(a,b) = a + b"
Disjunctive hypothesis Given "P or Q" "If x = 2 or x = 3, then x² - 5x + 6 = 0"

💡 Computer Science Connection

Proof by cases directly corresponds to conditional statements in programming:

// Proof by cases ↔ if-else structure
if (n % 2 == 0) {
    // Case 1: n is even
    // ... prove property holds for even n
} else {
    // Case 2: n is odd
    // ... prove property holds for odd n
}
// Since both branches establish the property,
// it holds for ALL integers n

When you verify that a program handles all possible inputs correctly by checking each branch, you're essentially doing proof by cases!

2. The Proof by Cases Template

Every proof by cases follows a standard structure. Here's the template:

📋 Proof by Cases Template

  1. State the theorem: "Theorem: [Statement to prove]"
  2. Begin the proof: "Proof:"
  3. Introduce the cases: "We consider [n] cases." or "We proceed by cases."
  4. State and verify exhaustiveness: "Note that every [element] is either [Case 1] or [Case 2] (or ... [Case n])." (Optional but helpful)
  5. For each case:
    • "Case k: [Condition for case k]"
    • Prove the conclusion holds using direct proof
  6. Conclude: "In all cases, [conclusion] holds. Therefore, [theorem statement]."
  7. End the proof: ∎

Key Requirements for Valid Cases

✅ Checklist for Valid Case Division

Exhaustive The cases must cover ALL possibilities. No element can fall outside all cases.
Example (Good) For integers: "n is even" and "n is odd" — every integer is one or the other
Example (Bad) For integers: "n > 0" and "n < 0" — misses n = 0!

⚠️ Common Mistake: Non-Exhaustive Cases

If your cases don't cover all possibilities, your proof is incomplete!

Wrong: "We consider two cases: x > 0 and x < 0."

Problem: What about x = 0?

Correct: "We consider three cases: x > 0, x = 0, and x < 0."

Do Cases Need to Be Mutually Exclusive?

Technically, cases don't need to be mutually exclusive (non-overlapping), but it's strongly recommended:

  • Mutually exclusive cases are cleaner and easier to follow
  • Overlapping cases are valid but may cause confusion or redundant work

📝 Example: Overlapping vs. Non-Overlapping Cases

Overlapping (Valid but not ideal):

  • Case 1: n ≤ 5
  • Case 2: n ≥ 3

These overlap when 3 ≤ n ≤ 5. Valid, but you might prove the same thing twice.

Non-overlapping (Better):

  • Case 1: n < 3
  • Case 2: 3 ≤ n ≤ 5
  • Case 3: n > 5

Each integer falls into exactly one case.

3. Parity-Based Case Proofs

The most common application of proof by cases involves the parity (evenness or oddness) of integers. Since every integer is either even or odd, we can always split into these two cases.

Case 1: n is even
n = 2k
Use this form in the proof
Case 2: n is odd
n = 2k + 1
Use this form in the proof

📝 Example 1: Product of Consecutive Integers

Theorem: For any integer n, the product n(n + 1) is even.

Why use cases? The expression n(n + 1) involves n, and the parity of n affects the calculation. If n is even, then n itself contributes a factor of 2. If n is odd, then n + 1 is even and contributes a factor of 2.

Cases:

  • Case 1: n is even → n = 2k → n(n+1) = 2k(n+1) = 2[k(n+1)]
  • Case 2: n is odd → n+1 is even → n+1 = 2m → n(n+1) = n(2m) = 2[nm]

Theorem: For any integer n, the product n(n + 1) is even.

Proof:

Let n be any integer. We consider two cases based on the parity of n.

Case 1: n is even.

By definition of even, n = 2k for some integer k. (Definition)

Then n(n + 1) = 2k(n + 1) = 2[k(n + 1)]. (Algebra)

Let m = k(n + 1). Since k and n + 1 are integers, m is an integer. (Closure)

Thus n(n + 1) = 2m where m is an integer, so n(n + 1) is even. (Definition of even)

Case 2: n is odd.

Since n is odd, n + 1 is even. (Odd + 1 = Even, proven in Module 5.2)

By definition of even, n + 1 = 2j for some integer j. (Definition)

Then n(n + 1) = n(2j) = 2(nj). (Algebra)

Let m = nj. Since n and j are integers, m is an integer. (Closure)

Thus n(n + 1) = 2m where m is an integer, so n(n + 1) is even. (Definition of even)

In both cases, n(n + 1) is even. Since every integer is either even or odd, we conclude that for any integer n, n(n + 1) is even.

∎

📝 Example 2: n² + n is Always Even

Theorem: For any integer n, n² + n is even.

Theorem: For any integer n, n² + n is even.

Proof:

Let n be any integer. We proceed by cases on the parity of n.

Case 1: n is even.

Then n = 2k for some integer k. (Definition of even)

So n² + n = (2k)² + 2k = 4k² + 2k = 2(2k² + k). (Algebra)

Since 2k² + k is an integer, n² + n is even. (Definition of even)

Case 2: n is odd.

Then n = 2k + 1 for some integer k. (Definition of odd)

So n² + n = (2k + 1)² + (2k + 1) (Substitution)

= 4k² + 4k + 1 + 2k + 1 (Expanding)

= 4k² + 6k + 2 (Simplifying)

= 2(2k² + 3k + 1). (Factoring out 2)

Since 2k² + 3k + 1 is an integer, n² + n is even. (Definition of even)

In both cases, n² + n is even.

∎

Alternative Approach

Notice that n² + n = n(n + 1). This is the product of consecutive integers, which we already proved is even in Example 1! Sometimes recognizing a factorization lets you reuse previous results.

📝 Example 3: Square Modulo 4

Theorem: For any integer n, n² leaves a remainder of 0 or 1 when divided by 4.

(Equivalently: n² ≡ 0 (mod 4) or n² ≡ 1 (mod 4))

Theorem: For any integer n, n² ≡ 0 or 1 (mod 4).

Proof:

Let n be any integer. We consider two cases.

Case 1: n is even.

Then n = 2k for some integer k. (Definition of even)

So n² = (2k)² = 4k². (Algebra)

Since n² = 4k² = 4k² + 0, we have n² ≡ 0 (mod 4). (Definition of mod)

Case 2: n is odd.

Then n = 2k + 1 for some integer k. (Definition of odd)

So n² = (2k + 1)² = 4k² + 4k + 1 = 4(k² + k) + 1. (Algebra)

Since n² = 4(k² + k) + 1, we have n² ≡ 1 (mod 4). (Definition of mod)

In both cases, n² ≡ 0 or 1 (mod 4).

∎

Why This Matters

This result is useful for proving that certain numbers are NOT perfect squares. For example, any number that is ≡ 2 or 3 (mod 4) cannot be a perfect square!

4. Absolute Value Proofs

Proofs involving absolute values naturally require case analysis because the absolute value function is defined differently for non-negative and negative numbers.

📖 Definition: Absolute Value

For any real number x, the absolute value of x, denoted |x|, is defined as:

$$|x| = \begin{cases} x & \text{if } x \geq 0 \\ -x & \text{if } x < 0 \end{cases}$$

The absolute value represents the distance from x to 0 on the number line.

Case 1: x ≥ 0
|x| = x
The number is already non-negative
Case 2: x < 0
|x| = -x
Negate to make it positive

📝 Example 4: Absolute Value is Non-Negative

Theorem: For any real number x, |x| ≥ 0.

Theorem: For any real number x, |x| ≥ 0.

Proof:

Let x be any real number. We consider two cases.

Case 1: x ≥ 0.

By definition of absolute value, |x| = x. (Definition, since x ≥ 0)

Since x ≥ 0, we have |x| = x ≥ 0. (Substitution)

Case 2: x < 0.

By definition of absolute value, |x| = -x. (Definition, since x < 0)

Since x < 0, we have -x > 0, so |x| = -x > 0 ≥ 0. (Properties of inequalities)

In both cases, |x| ≥ 0.

∎

📝 Example 5: Absolute Value of Negation

Theorem: For any real number x, |-x| = |x|.

Theorem: For any real number x, |-x| = |x|.

Proof:

Let x be any real number. We consider two cases.

Case 1: x ≥ 0.

Then -x ≤ 0. (Negating both sides of x ≥ 0)

By definition, |x| = x (since x ≥ 0). (Definition)

By definition, |-x| = -(-x) = x (since -x ≤ 0). (Definition)

Therefore, |-x| = x = |x|. (Transitivity of equality)

Case 2: x < 0.

Then -x > 0. (Negating both sides of x < 0)

By definition, |x| = -x (since x < 0). (Definition)

By definition, |-x| = -x (since -x > 0, so |-x| = -x). (Definition)

Therefore, |-x| = -x = |x|. (Transitivity of equality)

In both cases, |-x| = |x|.

∎

📝 Example 6: Triangle Inequality (Simple Case)

Theorem: For any real number x, -|x| ≤ x ≤ |x|.

Theorem: For any real number x, -|x| ≤ x ≤ |x|.

Proof:

Let x be any real number. We consider two cases.

Case 1: x ≥ 0.

Then |x| = x. (Definition)

We need to show: -|x| ≤ x ≤ |x|, i.e., -x ≤ x ≤ x.

The right inequality x ≤ x is trivially true. (Reflexivity)

For the left inequality: Since x ≥ 0, we have -x ≤ 0 ≤ x, so -x ≤ x. ✓

Case 2: x < 0.

Then |x| = -x. (Definition)

We need to show: -|x| ≤ x ≤ |x|, i.e., -(-x) ≤ x ≤ -x, i.e., x ≤ x ≤ -x.

The left inequality x ≤ x is trivially true. (Reflexivity)

For the right inequality: Since x < 0, we have x < 0 < -x, so x ≤ -x. ✓

In both cases, -|x| ≤ x ≤ |x|.

∎

5. Without Loss of Generality (WLOG)

Sometimes, cases are symmetric—the proof for one case is essentially identical to another, just with variables swapped. In such situations, we can use the phrase "without loss of generality" to handle only one case.

📖 Definition: Without Loss of Generality (WLOG)

"Without loss of generality" (abbreviated WLOG or w.l.o.g.) is a phrase used when:

  • Multiple cases are symmetric (interchangeable by relabeling)
  • The proof for one case works identically for the others
  • We choose to present only one case to avoid redundancy

When you write "WLOG, assume [condition]," you're claiming that proving this case is sufficient because the other case(s) follow by symmetry.

📝 Example 7: Sum of Even and Odd (Using WLOG)

Theorem: If one of two integers is even and the other is odd, then their sum is odd.

Without WLOG, we would need two cases:

  • Case 1: m is even and n is odd
  • Case 2: m is odd and n is even

But these cases are symmetric! The proof for Case 2 is identical to Case 1, just swapping m and n.

Theorem: If one of m and n is even and the other is odd, then m + n is odd.

Proof:

Without loss of generality, assume m is even and n is odd.

(The case where m is odd and n is even follows by symmetry, since addition is commutative.)

By definition, m = 2a for some integer a, and n = 2b + 1 for some integer b. (Definitions)

Then m + n = 2a + (2b + 1) = 2(a + b) + 1. (Algebra)

Let c = a + b. Since a and b are integers, c is an integer. (Closure)

Thus m + n = 2c + 1 where c is an integer, so m + n is odd. (Definition of odd)

∎

📝 Example 8: Maximum Function

Theorem: For any real numbers a and b, max(a, b) + min(a, b) = a + b.

Theorem: For any real numbers a and b, max(a, b) + min(a, b) = a + b.

Proof:

Let a and b be any real numbers.

Without loss of generality, assume a ≤ b.

(The case a > b follows by symmetry, swapping a and b.)

Since a ≤ b, by definition of max and min: (Definitions)

max(a, b) = b and min(a, b) = a.

Therefore, max(a, b) + min(a, b) = b + a = a + b. (Commutativity of addition)

∎

⚠️ When NOT to Use WLOG

Only use WLOG when cases are truly symmetric. If the cases have different properties or require different reasoning, you must handle them separately.

Symmetric (WLOG OK):

  • "m is even and n is odd" vs. "m is odd and n is even" (for addition/multiplication)
  • "a ≤ b" vs. "a > b" (when the roles of a and b are interchangeable)

NOT Symmetric (WLOG NOT OK):

  • "x > 0" vs. "x < 0" — positive and negative numbers have different properties
  • "n is prime" vs. "n is composite" — these have fundamentally different structures
  • "a divides b" vs. "b divides a" — divisibility is not symmetric

6. Proofs with More Than Two Cases

Sometimes we need more than two cases. Common examples include:

  • Three cases: positive, zero, negative
  • Three cases: divisibility by 3 (remainder 0, 1, or 2)
  • Four or more cases: divisibility by larger numbers

📝 Example 9: n³ - n is Divisible by 3

Theorem: For any integer n, 3 | (n³ - n).

Why three cases? We're proving divisibility by 3, so it makes sense to consider the three possible remainders when n is divided by 3:

  • Case 1: n ≡ 0 (mod 3), i.e., n = 3k
  • Case 2: n ≡ 1 (mod 3), i.e., n = 3k + 1
  • Case 3: n ≡ 2 (mod 3), i.e., n = 3k + 2

Every integer falls into exactly one of these cases.

Theorem: For any integer n, 3 | (n³ - n).

Proof:

Let n be any integer. Every integer has a remainder of 0, 1, or 2 when divided by 3. We consider these three cases.

Case 1: n ≡ 0 (mod 3).

Then n = 3k for some integer k. (Definition of congruence)

So n³ - n = (3k)³ - 3k = 27k³ - 3k = 3(9k³ - k). (Algebra)

Since 9k³ - k is an integer, 3 | (n³ - n). (Definition of divisibility)

Case 2: n ≡ 1 (mod 3).

Then n = 3k + 1 for some integer k. (Definition)

So n³ - n = (3k + 1)³ - (3k + 1). (Substitution)

= (27k³ + 27k² + 9k + 1) - (3k + 1) (Expanding)

= 27k³ + 27k² + 9k + 1 - 3k - 1 (Distributing)

= 27k³ + 27k² + 6k (Simplifying)

= 3(9k³ + 9k² + 2k). (Factoring out 3)

Since 9k³ + 9k² + 2k is an integer, 3 | (n³ - n). (Definition)

Case 3: n ≡ 2 (mod 3).

Then n = 3k + 2 for some integer k. (Definition)

So n³ - n = (3k + 2)³ - (3k + 2). (Substitution)

= (27k³ + 54k² + 36k + 8) - (3k + 2) (Expanding)

= 27k³ + 54k² + 36k + 8 - 3k - 2 (Distributing)

= 27k³ + 54k² + 33k + 6 (Simplifying)

= 3(9k³ + 18k² + 11k + 2). (Factoring out 3)

Since 9k³ + 18k² + 11k + 2 is an integer, 3 | (n³ - n). (Definition)

In all three cases, 3 | (n³ - n).

∎

Alternative Approach

Notice that n³ - n = n(n² - 1) = n(n-1)(n+1) = (n-1)n(n+1).

This is the product of three consecutive integers! Among any three consecutive integers, exactly one is divisible by 3. Therefore, the product is divisible by 3.

This factoring approach is more elegant but still requires case analysis to prove that one of three consecutive integers is divisible by 3.

📝 Example 10: Three Consecutive Integers

Theorem: Among any three consecutive integers, exactly one is divisible by 3.

Theorem: Among any three consecutive integers n, n+1, n+2, exactly one is divisible by 3.

Proof:

Let n be any integer. Consider the three consecutive integers n, n+1, and n+2.

Every integer has remainder 0, 1, or 2 when divided by 3. We consider three cases based on n.

Case 1: n ≡ 0 (mod 3).

Then 3 | n. (Definition)

Also, n + 1 ≡ 1 (mod 3) and n + 2 ≡ 2 (mod 3), so 3 ∤ (n+1) and 3 ∤ (n+2).

Exactly one (namely n) is divisible by 3. ✓

Case 2: n ≡ 1 (mod 3).

Then n + 2 ≡ 1 + 2 ≡ 0 (mod 3), so 3 | (n + 2). (Arithmetic mod 3)

Also, n ≡ 1 and n + 1 ≡ 2 (mod 3), so 3 ∤ n and 3 ∤ (n+1).

Exactly one (namely n+2) is divisible by 3. ✓

Case 3: n ≡ 2 (mod 3).

Then n + 1 ≡ 2 + 1 ≡ 0 (mod 3), so 3 | (n + 1). (Arithmetic mod 3)

Also, n ≡ 2 and n + 2 ≡ 1 (mod 3), so 3 ∤ n and 3 ∤ (n+2).

Exactly one (namely n+1) is divisible by 3. ✓

In all cases, exactly one of n, n+1, n+2 is divisible by 3.

∎

10. Summary and Key Takeaways

📋 Key Takeaways from Module 5.3

  1. Proof by cases divides the domain into exhaustive scenarios and proves the conclusion in each.
  2. Cases must be exhaustive—every element must fall into at least one case.
  3. Common case divisions:
    • Parity: n is even, n is odd
    • Sign: x > 0, x = 0, x < 0
    • Absolute value: x ≥ 0, x < 0
    • Modular: n ≡ 0, 1, ..., k-1 (mod k)
  4. WLOG can be used when cases are symmetric, allowing you to prove just one case.
  5. Each case is essentially a mini direct proof—assume the case condition and derive the conclusion.
  6. Proof by cases corresponds to if-else structures in programming.

Preparing for Module 5.4: Proof by Contrapositive

In the next module, you'll learn Proof by Contrapositive—a technique that proves $$P \rightarrow Q$$ by instead proving $$\neg Q \rightarrow \neg P$$. Make sure you:

  • ✓ Can construct proofs by cases with proper structure

Module 5.4: Proof by Contrapositive will teach you an alternative to direct proof that's especially useful when:

  • The conclusion is easier to negate than to prove directly
  • Assuming ¬Q gives you more useful information than assuming P
  • The statement has the form "If [something], then [something] is not..."

The classic example: proving "If n² is even, then n is even" is much easier using the contrapositive!
