1. What is a Mathematical Proof?

📖 Definition: Mathematical Proof

A mathematical proof is a rigorous, logical argument that establishes the truth of a mathematical statement beyond any doubt. It consists of a sequence of statements, where each statement follows logically from:

  • Previous statements in the proof
  • Axioms (statements accepted as true without proof)
  • Definitions
  • Previously proven theorems

Why Proofs Matter in Computer Science

You might wonder why a computer science course emphasizes mathematical proofs. The answer is that proofs are essential tools for building reliable software and understanding algorithms. Here's why:

Application Description Real-World Example
Algorithm Correctness Proving that an algorithm produces the correct output for ALL valid inputs Proving that a sorting algorithm always produces a sorted list, regardless of input
Program Verification Formally verifying that software meets its specifications Proving that a banking system never allows an account balance to go negative
Security Analysis Proving that cryptographic protocols are secure against attacks Proving that an encryption scheme cannot be broken without the key
Complexity Analysis Proving bounds on algorithm running time and space usage Proving that quicksort has O(n log n) average-case time complexity
Data Structure Properties Proving that data structures maintain required invariants Proving that a binary search tree always maintains the BST property after insertion

Proofs vs. Testing: A Critical Distinction

As programmers, you're familiar with testing code. But there's a fundamental difference between testing and proving:

⚠️ The Limitation of Testing

Testing can show that something works for specific cases, but it can NEVER prove that something works for ALL cases.

Proofs establish truth for ALL cases simultaneously—including infinitely many cases that could never be tested.

As computer scientist Edsger Dijkstra famously said: "Testing shows the presence, not the absence of bugs."

📝 Example: The Danger of Relying on Testing

Consider the claim: "For all positive integers n, the expression $$n^2 + n + 41$$ is prime."

Testing approach: Let's check several values:

  • n = 1: $$1 + 1 + 41 = 43$$ ✓ (prime)
  • n = 2: $$4 + 2 + 41 = 47$$ ✓ (prime)
  • n = 3: $$9 + 3 + 41 = 53$$ ✓ (prime)
  • n = 4: $$16 + 4 + 41 = 61$$ ✓ (prime)
  • n = 5: $$25 + 5 + 41 = 71$$ ✓ (prime)
  • ...
  • n = 39: $$1521 + 39 + 41 = 1601$$ ✓ (prime)

After 39 successful tests, you might be confident the claim is true. But wait!

When n = 40:

$$40^2 + 40 + 41 = 1600 + 40 + 41 = 1681 = 41 \times 41$$

1681 is NOT prime—it's 41². The claim is FALSE!

Lesson: Even 39 successful tests couldn't reveal this failure. A single counterexample disproves a universal statement, but no amount of testing can prove one true.

💡 Connection to Your Programming Experience

Think about debugging: when you trace through your code step-by-step to verify it works correctly, you're performing an informal proof. Mathematical proofs formalize this reasoning process, making it:

  • Rigorous: Every step must be justified
  • Universal: Applies to all cases, not just tested ones
  • Verifiable: Others can check your reasoning
  • Permanent: Once proven, always true

2. Types of Mathematical Statements

In mathematics, we use different terms to describe statements based on their importance, how they're established, and their role in the mathematical hierarchy:

Term Definition Example
Axiom (or Postulate) A fundamental statement accepted as true without proof; the starting point for a mathematical system "For all real numbers a and b, a + b = b + a." (Commutative property)
Definition A precise description that gives meaning to a mathematical term "An integer n is even if n = 2k for some integer k."
Theorem An important mathematical statement that has been proven true The Pythagorean Theorem: In a right triangle, $$a^2 + b^2 = c^2$$
Proposition A proven statement of lesser importance than a theorem "The sum of two even numbers is even."
Lemma A "helper" theorem—a proven statement used as a stepping stone to prove a larger result "If gcd(a, b) = 1 and a | bc, then a | c."
Corollary A result that follows easily and directly from a theorem "If n² is even, then n is even." (follows from properties of even numbers)
Conjecture A statement believed to be true but not yet proven Goldbach's Conjecture: "Every even integer greater than 2 is the sum of two primes."

💡 Computer Science Analogy

These mathematical concepts map naturally to programming constructs:

  • Axioms → Language specifications, primitive operations
  • Definitions → Type definitions, interfaces, contracts
  • Theorems → Proven properties of algorithms or data structures
  • Lemmas → Helper functions or methods
  • Corollaries → Simple consequences of main results
  • Conjectures → Unverified assumptions in code (potential bugs!)

The Hierarchy of Mathematical Truth

Mathematical knowledge is built in layers:

🏗️ Building Mathematical Knowledge

  1. Start with axioms (accepted without proof)
  2. Establish definitions (give meaning to terms)
  3. Prove lemmas (small helper results)
  4. Prove theorems (major results, using lemmas)
  5. Derive corollaries (easy consequences of theorems)

Each level builds on the previous levels. You can never use an unproven statement (conjecture) as justification in a proof!

3. Components of a Well-Structured Proof

Every well-written proof contains several essential components. Understanding these components will help you both read and write proofs effectively.

The Anatomy of a Proof

Component Purpose Example Phrases
Statement The theorem or proposition to be proven; clearly states what we're proving "Theorem: If n is even, then n² is even."
Proof Declaration Signals the beginning of the proof "Proof:" or "Proof."
Assumptions States what we assume to be true (the hypothesis) "Assume n is even." / "Let n be an even integer." / "Suppose that..."
Definitions Applied Expands terms using their formal definitions "By definition of even, n = 2k for some integer k."
Logical Steps The chain of reasoning from assumptions to conclusion "Then...", "Therefore...", "It follows that...", "Thus..."
Justifications Explains why each step is valid "by algebra", "by definition of...", "by Theorem X", "since..."
Conclusion The final statement that was to be proven "Therefore, n² is even."
End Marker Signals the proof is complete "∎" or "□" or "QED" or "■"

📖 Definition: QED and ∎

QED stands for the Latin phrase "quod erat demonstrandum," meaning "which was to be demonstrated" or "what was to be shown."

The symbol ∎ (called a "tombstone," "Halmos symbol," or "end-of-proof symbol") serves the same purpose—it indicates that the proof is complete. Named after mathematician Paul Halmos who popularized its use.

A Complete Example: Annotated Proof

Let's examine a complete proof with each component labeled:

Theorem: If n is an even integer, then n² is an even integer.

Proof:

Let n be an even integer. (Assumption—we assume the hypothesis is true)

By definition of even, n = 2k for some integer k. (Applying the definition of "even")

Then n² = (2k)² (Substitution)

= 4k² (Algebra—expanding the square)

= 2(2k²). (Algebra—factoring out 2)

Let m = 2k². (Introducing a new variable for clarity)

Since k is an integer, 2k² is an integer, so m is an integer. (Justification—integers are closed under multiplication)

Thus n² = 2m where m is an integer. (Substitution)

By definition of even, n² is even. (Applying the definition of "even" to conclude)

∎

💡 Key Observations

  • The proof starts by assuming the hypothesis (n is even)
  • The definition of "even" is used twice: once to unpack the assumption, once to establish the conclusion
  • Every algebraic step is justified
  • The proof ends by explicitly stating what was proven
  • The structure mirrors the logical form: P → Q (assume P, derive Q)

4. Common Theorem Forms and Logical Structures

Mathematical theorems come in several common forms. Recognizing these forms is crucial because the form of a theorem often determines which proof technique to use.

Common Theorem Forms

Form Logical Structure Example Typical Proof Method
Conditional $$P \rightarrow Q$$ "If n is divisible by 6, then n is divisible by 2." Direct proof or contrapositive
Biconditional $$P \leftrightarrow Q$$ "n is even if and only if n² is even." Prove both directions: P→Q and Q→P
Universal $$\forall x, P(x)$$ "For all integers n, n + 0 = n." Direct proof with arbitrary element
Existential $$\exists x, P(x)$$ "There exists a prime number greater than 100." Construct a specific example
Uniqueness $$\exists! x, P(x)$$ "There exists a unique real number x such that x + 3 = 5." Show existence, then show uniqueness
Negation of Universal $$\neg(\forall x, P(x)) \equiv \exists x, \neg P(x)$$ "Not all primes are odd." Provide a counterexample

Understanding Conditional Statements: The Most Common Form

The conditional statement "If P, then Q" (written $$P \rightarrow Q$$) is by far the most common theorem form. Let's understand its components thoroughly:

📖 Terminology for $$P \rightarrow Q$$

P Called the hypothesis, antecedent, premise, or assumption
Q Called the conclusion or consequent

Alternative phrasings:

  • "P implies Q"
  • "P is sufficient for Q" (if we have P, that's enough to guarantee Q)
  • "Q is necessary for P" (P can't happen without Q)
  • "P only if Q" (P happens only when Q also happens)
  • "Q whenever P" (every time P occurs, Q occurs)

📝 Example: Identifying P and Q

Statement: "If a number is divisible by 6, then it is divisible by 2."

  • P (hypothesis): "a number is divisible by 6"
  • Q (conclusion): "it is divisible by 2"

Statement: "Whenever it rains, the ground gets wet."

  • P (hypothesis): "it rains"
  • Q (conclusion): "the ground gets wet"

5. Related Conditional Statements

Given any conditional statement $$P \rightarrow Q$$, we can form three related statements. Understanding these relationships is essential for proof writing.

The Four Related Statements

Name Symbolic Form Example (Original: "If it rains, then the ground is wet.")
Original (Conditional) $$P \rightarrow Q$$ "If it rains, then the ground is wet."
Converse $$Q \rightarrow P$$ "If the ground is wet, then it rains."
Inverse $$\neg P \rightarrow \neg Q$$ "If it doesn't rain, then the ground is not wet."
Contrapositive $$\neg Q \rightarrow \neg P$$ "If the ground is not wet, then it doesn't rain."

📐 Critical Theorem: Contrapositive Equivalence

A conditional statement is logically equivalent to its contrapositive:

$$(P \rightarrow Q) \equiv (\neg Q \rightarrow \neg P)$$

This means: proving the contrapositive is exactly the same as proving the original statement!

⚠️ Critical Warning: Converse ≠ Original

A conditional statement is NOT logically equivalent to its converse or inverse!

  • $$(P \rightarrow Q) \not\equiv (Q \rightarrow P)$$ — The converse may be true or false independently
  • $$(P \rightarrow Q) \not\equiv (\neg P \rightarrow \neg Q)$$ — The inverse may be true or false independently

However: The converse and inverse are equivalent to each other (they're contrapositives of each other).

Logical Equivalence Diagram

🔗 Equivalence Relationships

Original:       P → Q    ←——— EQUIVALENT ———→    ¬Q → ¬P   :Contrapositive
                 ↑                                    ↑
                 |         NOT equivalent             |
                 ↓                                    ↓
Converse:       Q → P    ←——— EQUIVALENT ———→    ¬P → ¬Q   :Inverse

Summary:

  • Original ≡ Contrapositive (always have the same truth value)
  • Converse ≡ Inverse (always have the same truth value)
  • Original and Converse are independent (one can be true while the other is false)

📝 Example: Why Converse ≠ Original

Original (TRUE): "If n is divisible by 4, then n is divisible by 2."

Converse (FALSE): "If n is divisible by 2, then n is divisible by 4."

Counterexample: n = 6

  • 6 is divisible by 2 ✓ (6 = 2 × 3)
  • 6 is NOT divisible by 4 ✗ (6 ÷ 4 = 1.5)

The converse is false even though the original is true!

📝 Example: Forming All Four Statements

Original: "If a function is differentiable, then it is continuous."

Let P = "a function is differentiable" and Q = "it is continuous"

Original ($$P \rightarrow Q$$): "If a function is differentiable, then it is continuous." (TRUE)
Converse ($$Q \rightarrow P$$): "If a function is continuous, then it is differentiable." (FALSE—|x| is continuous but not differentiable at 0)
Inverse ($$\neg P \rightarrow \neg Q$$): "If a function is not differentiable, then it is not continuous." (FALSE—same counterexample)
Contrapositive ($$\neg Q \rightarrow \neg P$$): "If a function is not continuous, then it is not differentiable." (TRUE—equivalent to original)

Why This Matters for Proofs

The contrapositive equivalence gives us a powerful proof technique:

💡 Proof Strategy: Contrapositive

To prove $$P \rightarrow Q$$, you can instead prove $$\neg Q \rightarrow \neg P$$.

When is this useful?

  • When starting from ¬Q gives you more useful information than starting from P
  • When the negation of Q is easier to work with
  • Classic example: To prove "If n² is even, then n is even," prove "If n is odd, then n² is odd" (much easier!)

We'll explore this technique in depth in Module 5.4.

6. Essential Definitions for Proofs

To write proofs effectively, you must know and use precise mathematical definitions. Here are the essential definitions we'll use throughout this course:

Number Theory Definitions

📖 Definition: Even Integer

An integer n is even if and only if there exists an integer k such that n = 2k.

Symbolically: $$n \text{ is even} \Leftrightarrow \exists k \in \mathbb{Z}, n = 2k$$

Examples:

  • 6 is even because 6 = 2(3), where k = 3
  • 0 is even because 0 = 2(0), where k = 0
  • -8 is even because -8 = 2(-4), where k = -4

📖 Definition: Odd Integer

An integer n is odd if and only if there exists an integer k such that n = 2k + 1.

Symbolically: $$n \text{ is odd} \Leftrightarrow \exists k \in \mathbb{Z}, n = 2k + 1$$

Examples:

  • 7 is odd because 7 = 2(3) + 1, where k = 3
  • 1 is odd because 1 = 2(0) + 1, where k = 0
  • -5 is odd because -5 = 2(-3) + 1, where k = -3

📖 Definition: Divisibility

An integer a divides an integer b (written $$a \mid b$$) if and only if there exists an integer k such that b = ak.

Symbolically: $$a \mid b \Leftrightarrow \exists k \in \mathbb{Z}, b = ak$$

Terminology:

  • "a divides b"
  • "a is a divisor of b"
  • "a is a factor of b"
  • "b is a multiple of a"
  • "b is divisible by a"

Examples:

  • $$3 \mid 12$$ because 12 = 3(4)
  • $$5 \mid 0$$ because 0 = 5(0)
  • $$7 \nmid 15$$ because there's no integer k with 15 = 7k

📖 Definition: Prime Number

An integer p > 1 is prime if and only if its only positive divisors are 1 and p itself.

Equivalently: p > 1 is prime if whenever p = ab for positive integers a and b, then a = 1 or b = 1.

Examples: 2, 3, 5, 7, 11, 13, 17, 19, 23, ...

Non-examples: 1 (not greater than 1), 4 = 2×2, 6 = 2×3

📖 Definition: Composite Number

An integer n > 1 is composite if and only if it is not prime—that is, if it has a positive divisor other than 1 and itself.

Examples: 4, 6, 8, 9, 10, 12, ...

📖 Definition: Rational Number

A real number r is rational if and only if there exist integers a and b with b ≠ 0 such that r = a/b.

Symbolically: $$r \in \mathbb{Q} \Leftrightarrow \exists a, b \in \mathbb{Z}, b \neq 0, r = \frac{a}{b}$$

Examples:

  • 3/4 is rational (a = 3, b = 4)
  • -2/7 is rational (a = -2, b = 7)
  • 5 is rational (5 = 5/1, so a = 5, b = 1)
  • 0.75 is rational (0.75 = 3/4)

📖 Definition: Irrational Number

A real number is irrational if and only if it is not rational—that is, it cannot be expressed as a ratio of two integers.

Examples: $$\sqrt{2}$$, $$\pi$$, $$e$$, $$\sqrt{3}$$

⚠️ Common Mistake: Definition vs. Test

Don't confuse the definition of a property with a test for that property:

Definition of even: n = 2k for some integer k
Test for even (in code): n % 2 == 0

In proofs, always use the formal definition, not the computational test!

Correct: "Since n is even, n = 2k for some integer k."

Incorrect: "Since n is even, n % 2 = 0."

Why Definitions Are Critical

In proofs, definitions serve two essential purposes:

🎯 Using Definitions in Proofs

  1. Unpacking assumptions: When you assume something is true, the definition tells you what that means mathematically.
    • "Assume n is even" → "Then n = 2k for some integer k"
  2. Establishing conclusions: To prove something has a property, show it satisfies the definition.
    • To prove n is even → Show n = 2m for some integer m

7. Overview of Proof Techniques

There are several standard proof techniques, each suited to different types of statements. This week (Week 5), we'll focus on the first four; mathematical induction will be covered in Week 6.

Technique When to Use Basic Approach Module
Direct Proof Proving $$P \rightarrow Q$$ Assume P is true, derive Q through logical steps 5.2
Proof by Cases When the domain naturally splits into cases Prove the result holds in each case separately 5.3
Proof by Contrapositive When direct proof is difficult Prove $$\neg Q \rightarrow \neg P$$ instead 5.4
Proof by Counterexample Disproving universal statements Find one example where the statement fails 5.5
Proof by Contradiction Proving something is impossible Assume the opposite, derive a contradiction Week 6
Mathematical Induction Proving $$\forall n \geq n_0, P(n)$$ Prove base case, prove inductive step Week 6

Preview of This Week's Techniques

To prove $$P \rightarrow Q$$ directly:

  1. Assume P is true
  2. Use definitions, axioms, and previously proven theorems
  3. Derive Q through a chain of logical implications

Example: To prove "If n is even, then n² is even":

  • Assume n is even
  • Apply definition: n = 2k
  • Compute: n² = 4k² = 2(2k²)
  • Conclude: n² is even

When the domain naturally divides into cases:

  1. Identify exhaustive, mutually exclusive cases
  2. Prove the result holds in each case
  3. Conclude the result holds in general

Example: To prove "For any integer n, n(n+1) is even":

  • Case 1: n is even → n(n+1) = (even)(anything) = even
  • Case 2: n is odd → n+1 is even → n(n+1) = (anything)(even) = even

Since $$(P \rightarrow Q) \equiv (\neg Q \rightarrow \neg P)$$:

  1. Assume ¬Q is true
  2. Derive ¬P through logical steps
  3. Conclude that $$P \rightarrow Q$$ is true

Example: To prove "If n² is even, then n is even":

  • Prove the contrapositive: "If n is odd, then n² is odd"
  • Assume n is odd: n = 2k + 1
  • Compute: n² = 4k² + 4k + 1 = 2(2k² + 2k) + 1
  • Conclude: n² is odd

To disprove a universal statement $$\forall x, P(x)$$:

  1. Find a specific value c in the domain
  2. Show that P(c) is false
  3. Conclude that $$\forall x, P(x)$$ is false

Example: Disprove "All prime numbers are odd":

  • Counterexample: 2
  • 2 is prime (only divisors are 1 and 2)
  • 2 is even (not odd)
  • Therefore, the statement is false

10. Summary and Key Takeaways

📋 Key Takeaways from Module 5.1

  1. Mathematical proofs establish truth for ALL cases, unlike testing which only checks specific cases.
  2. A well-structured proof contains: statement, assumptions, definitions applied, logical steps, justifications, and conclusion.
  3. The most common theorem form is the conditional: $$P \rightarrow Q$$.
  4. A conditional is logically equivalent to its contrapositive: $$(P \rightarrow Q) \equiv (\neg Q \rightarrow \neg P)$$
  5. A conditional is NOT equivalent to its converse or inverse.
  6. Formal definitions are essential for proofs—always use them precisely.
  7. Different proof techniques are suited for different types of statements.
