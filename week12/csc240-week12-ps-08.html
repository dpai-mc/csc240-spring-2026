Overview & Objectives

PS-08 assesses your understanding of the foundational graph theory concepts introduced in Week 12. You will work with graph diagrams, formal notation, adjacency representations, the Handshaking Lemma, special graph families, and graph isomorphism.

🎯 Learning Objectives Assessed
  • Define fundamental graph terminology and use it precisely in written descriptions.
  • Construct adjacency matrix and adjacency list representations and convert between them.
  • Analyze space/time tradeoffs of graph representations using Big-O notation.
  • Apply the Handshaking Lemma to verify or reject degree sequences.
  • Recognize special graph types and their properties.
  • Determine whether two graphs are isomorphic.

Problem 1 — Formal Representation and Degrees (2 pts)

2 points

Using Graph G1 above, complete the following:

  1. Write the formal representation G₁ = (V, E). List V in roster notation and E as a set of 2-element subsets.

Count the edges touching each vertex in G₁ to find its degree — trace every line connected to a given vertex dot. To write E formally, each edge is an unordered pair like {a, b}; count them carefully in the diagram.

Problem 2 — Paths, Cycles, and Connectivity (2 pts)

2 points

Using Graph G2 above, answer the following:

  1. Write the formal representation G₂ = (V, E).
  2. Compute deg(v) for every vertex.
  3. Identify a path of length ≥ 3 (list the vertex sequence). Verify it is a valid path (no repeated vertices).
  4. Identify a cycle (list the vertex sequence, starting and ending at the same vertex). State its length.
  5. Is G2 connected? Justify your answer in one or two sentences.
💡 Hint for Problem 2

A path is a sequence of vertices where each consecutive pair is connected by an edge and no vertex is repeated. Trace a route through the diagram without lifting your pencil and without revisiting a vertex.

A cycle starts and ends at the same vertex with no other repetitions — the shortest possible cycle in a simple graph is a triangle (length 3). Look for three mutually connected vertices.

For connectivity, ask whether you can reach every vertex from every other vertex by following edges. One disconnected vertex is enough to make the graph not connected.

Problem 3: Real-World Graph Modeling

Problem 3 — Modeling a Scenario as a Graph (1 pt)

1 point

Read the scenario below and answer the questions that follow.

📚 Scenario: University Course Prerequisites

A university catalog lists the following prerequisite relationships among five computer science courses:

  • CS 101 is a prerequisite for CS 201 and CS 210.
  • CS 201 is a prerequisite for CS 301.
  • CS 210 is a prerequisite for CS 301.
  • CS 301 is a prerequisite for CS 400.

There are no other prerequisite relationships.

  1. What type of graph best models this scenario? Choose from: directed or undirected, and weighted or unweighted. State both choices.
  2. Justify each choice in 2–3 sentences. Explain why the alternative would be inappropriate or less informative.
  3. Is this graph a DAG (directed acyclic graph)? Why or why not?
💡 Hint for Problem 3

Ask yourself: does the relationship have a direction? Is "CS 101 is a prerequisite for CS 201" the same as "CS 201 is a prerequisite for CS 101"? If not, an undirected edge loses important information.

For weighted vs. unweighted: do the edges in this scenario have meaningful numerical magnitudes (distance, cost, capacity)? Or is the only information whether a prerequisite relationship exists at all?

For the DAG question, try to follow prerequisite chains in sequence. A directed cycle would mean some course is indirectly a prerequisite for itself — check whether that can happen here.

Problem 4 — Diagram → Adjacency Matrix and Adjacency List (2 pts)

2 points

Using Graph G3 above:

  1. Construct the adjacency matrix. Use the vertex ordering 1, 2, 3, 4. Present it as a clearly labeled 4 × 4 matrix.
  2. Construct the adjacency list. For each vertex, list its neighbors in ascending order.
💡 Hint for Problem 4

For the matrix, entry A[i][j] = 1 if and only if there is an edge between vertex i and vertex j; otherwise it is 0. Since the graph is undirected, the matrix must be symmetric — A[i][j] = A[j][i] for all i, j. Check each edge in the diagram and set both symmetric positions.

For the adjacency list, scan each row of your completed matrix and record every column position that holds a 1 — those are the neighbors of that vertex.

Problem 6: Tradeoff Analysis

Problem 6 — Space Complexity Comparison (2 pts)

2 points

A social network is modeled as an undirected, unweighted graph with 10,000 users (vertices) and 40,000 friendships (edges).

  1. Calculate the space complexity (in Big-O notation) of storing this graph as an adjacency matrix. Show your substitution.
  2. Calculate the space complexity of storing it as an adjacency list. Show your substitution.
  3. Which representation is more space-efficient for this graph? Justify your choice by comparing the two results.
  4. Under what condition (in terms of |E| relative to |V|) would the other representation become preferable? Answer in one or two sentences using Big-O reasoning.
💡 Hint for Problem 6

Adjacency matrix space is always O(|V|²) — it stores one entry for every possible pair of vertices regardless of how many edges actually exist. Substitute |V| = 10,000 to get a concrete number.

Adjacency list space is O(|V| + |E|) — one slot per vertex plus one entry per edge endpoint. Substitute both given values.

For part (d), think about when the two expressions become roughly comparable: what relationship between |E| and |V| would make O(|V| + |E|) ≈ O(|V|²)?

Problems 7–8: Special Graphs and the Handshaking Lemma

Problem 7 — Special Graph Identification (2 pts)

2 points

For each description below, identify the named special graph (using standard notation such as Kₙ, Kₘ,ₙ, Cₙ, Pₙ, or Wₙ) and state its edge count and degree sequence.

  1. A graph on 6 vertices in which every vertex is adjacent to every other vertex.
  2. A graph on 7 vertices arranged in a single cycle (each vertex has exactly two neighbors).
  3. A graph whose vertex set can be partitioned into two groups of sizes 3 and 4, where every vertex in one group is adjacent to every vertex in the other group, and there are no edges within either group.
  4. A graph consisting of a cycle on 5 vertices plus one additional "hub" vertex that is adjacent to all 5 cycle vertices.
💡 Hint for Problem 7

Review the named families and their edge-count formulas:

Kₙ (complete graph on n vertices): every pair connected; edge count = n(n−1)/2; every vertex has degree n−1.

Cₙ (cycle on n vertices): n edges; every vertex has degree 2.

Kₘ,ₙ (complete bipartite, groups of size m and n): m×n edges; vertices in the size-m group each have degree n, vertices in the size-n group each have degree m.

Wₙ (wheel — hub + Cₙ): the hub has degree n; each cycle vertex has degree 3 (two cycle neighbors + hub); total edges = 2n.

Problem 8 — Handshaking Lemma Application (1 pt)

1 point

For each proposed degree sequence, determine whether a simple undirected graph with that degree sequence can exist. Use the Handshaking Lemma to justify your answer. If the sequence passes the Handshaking Lemma test, state how many edges the graph would have.

  1. Degree sequence: (4, 3, 3, 2, 2)
  2. Degree sequence: (3, 3, 3, 2, 1)
  3. Degree sequence: (5, 3, 3, 3, 2, 2)

Reminder: The Handshaking Lemma states that ∑(v ∈ V) deg(v) = 2|E|. A necessary condition for a degree sequence to be realizable is that the sum of degrees is even.

💡 Hint for Problem 8

Add up all the values in the degree sequence. The Handshaking Lemma requires this sum to equal 2|E|, so the sum must be even.

If the sum is odd, the sequence cannot be realized as a simple graph — you are done, no further work needed.

If the sum is even, the potential number of edges is sum ÷ 2. Note: passing this parity test is necessary but not sufficient for realizability — a sequence can have an even sum and still be impossible for other reasons, but for this problem the parity check is the required test.

Problem 9: Graph Isomorphism

Graphs H1 (vertices {p, q, r, s}) and H2 (vertices {w, x, y, z}).

Problem 9 — Isomorphism Determination (1 pt)

1 point

Examine graphs H1 and H2 shown above.

  1. List the edge set for each graph.
  2. Compare the following invariants: number of vertices, number of edges, and degree sequence.
  3. Based on your invariant comparison, are the graphs potentially isomorphic, or can you already rule it out?
  4. If the invariants match, provide an explicit bijection f: V(H₁) → V(H₂) and verify that it preserves all edges. If the invariants do not match, clearly state which invariant fails and conclude that the graphs are not isomorphic.
💡 Hint for Problem 9

Work through the invariants in order — each one is quick to check and any mismatch immediately proves non-isomorphism:

Step 1: Count vertices in each graph. Step 2: Count edges in each graph. Step 3: Compute and sort the degree sequence of each graph (list all degrees in non-increasing order).

If all three invariants match, the graphs are possibly isomorphic. Try to construct a bijection by pairing vertices of equal degree. Then verify every edge in H₁ maps to a valid edge in H₂ under your mapping — if all edges check out, you have proven isomorphism.
