1 · From Graphs to Graph Types

Where We Are
In Module 12.1 you learned the formal definition of a graph $$G = (V, E)$$ and the core vocabulary—vertices, edges, degree, paths, cycles, and connectivity. You know what a graph is; now let's look at the different flavors.

Not every graph is created equal. The edges in a social network work very differently from the edges in a GPS routing system. In this module you will learn three independent classification axes that, combined, determine the type of graph you need:

  1. Direction — Are the edges one-way or two-way?
  2. Weight — Do edges carry a numerical value?
  3. Multiplicity — Can there be parallel edges or self-loops?

After pinning down these distinctions, we will walk through four real-world systems and see how the choice of graph type follows naturally from the problem being modeled.

2 · Directed vs. Undirected Graphs

Definition — Undirected Graph
An undirected graph has edges that are unordered pairs $$\{u, v\}$$. The relationship is symmetric: if $$u$$ is connected to $$v$$, then $$v$$ is connected to $$u$$.
Definition — Directed Graph (Digraph)
A directed graph (or digraph) has edges that are ordered pairs $$(u, v)$$. An edge from $$u$$ to $$v$$ does not imply an edge from $$v$$ to $$u$$. Each directed edge is called an arc.

In-Degree and Out-Degree

In a directed graph every vertex $$v$$ has two degree measures:

  • In-degree $$\deg^{-}(v)$$ — number of arcs coming in to $$v$$.
  • Out-degree $$\deg^{+}(v)$$ — number of arcs going out of $$v$$.

A fundamental identity for directed graphs is:

$$\sum_{v \in V} \deg^{+}(v) \;=\; \sum_{v \in V} \deg^{-}(v) \;=\; |E|$$

Every arc contributes exactly 1 to the out-degree of its tail and 1 to the in-degree of its head.

Side-by-Side Diagram — Same Structure, Two Interpretations

The diagrams below show the same four vertices $$\{a, b, c, d\}$$ and the same connections, first as an undirected graph and then as a directed graph.

Undirected Graph
$$E = \bigl\{\{a,b\},\{b,c\},\{c,d\},\{a,d\},\{a,c\}\bigr\}$$
$$\sum \deg(v) = 3+2+3+2 = 10 = 2|E|$$
Directed Graph (Digraph)
$$E = \{(a,b),(b,c),(c,d),(d,a),(a,c)\}$$
$$\sum \deg^{+}(v) = \sum \deg^{-}(v) = 5 = |E|$$
Key Observation — When to Choose Directed

Choose a directed graph whenever the relationship between entities is not necessarily symmetric. Examples:

  • Web hyperlinks — page A can link to page B without B linking back.
  • Prerequisite chains — CSC 221 is a prerequisite for CSC 240, but not vice-versa.
  • One-way streets in a city map.

Choose an undirected graph when the relationship is inherently mutual: Facebook friendships, Ethernet links, chemical bonds.

3 · Weighted vs. Unweighted Graphs

Definition — Weight Function
A weighted graph augments $$G = (V, E)$$ with a weight function $$w : E \to \mathbb{R}$$ that assigns a real number to every edge. When no such function is present, the graph is unweighted (equivalently, every edge has implicit weight 1).

Weights can represent many real-world quantities:

DomainWhat the Weight Represents
Road / GPS networkDistance (km) or travel time (min)
Airline routesTicket price ($) or flight duration
Computer networkBandwidth (Mbps) or latency (ms)
Pipeline systemCapacity (liters/sec) or construction cost
Weighted Undirected Graph
$$w(\{a,b\})=5,\; w(\{b,c\})=3,\; w(\{c,d\})=7,\; w(\{a,d\})=2,\; w(\{a,c\})=4$$
Why Does the Weight Function Matter?

Many of the algorithms you will study in Weeks 14–17 depend on edge weights:

  • Dijkstra's algorithm (Week 14) finds the shortest weighted path between two vertices.
  • Kruskal's and Prim's algorithms (Week 14) find the minimum spanning tree—the cheapest way to connect every vertex.

Without weights, "shortest" just means "fewest edges" (BFS handles that in Week 13). With weights, the problem becomes fundamentally different.

4 · Simple Graphs, Multigraphs, and Pseudographs

Definition — Simple Graph
A simple graph has no self-loops (an edge from a vertex to itself) and no parallel edges (two or more edges between the same pair of vertices). Unless stated otherwise, "graph" in this course means simple graph.
Definition — Multigraph
A multigraph allows parallel edges (also called multiple edges) between the same pair of vertices but does not allow self-loops.
Definition — Pseudograph
A pseudograph allows both parallel edges and self-loops. It is the most general undirected graph model.

When Do Multigraphs and Pseudographs Appear in Practice?
  • Multigraph — multi-lane highway map: Two cities may be connected by both a freeway and a surface road. Each road is a distinct edge with potentially different weights (travel time).
  • Multigraph — redundant network links: Data centers often run multiple physical cables between the same pair of switches for fault tolerance.
  • Pseudograph — finite-state automaton: A state in a state machine can have a transition back to itself (a self-loop), such as a state that stays put when reading a particular input character.

In this course, unless a problem explicitly says "multigraph" or "pseudograph," assume a simple graph.

5 · Real-World Modeling Gallery

For each mini-case below, ask yourself: What are the vertices? What are the edges? Is the graph directed or undirected? Weighted or unweighted?

Case 1 — Social Network

Undirected Unweighted

Vertices = people. Edges = mutual friendships. A friendship on a platform like Facebook is symmetric—if Alice is friends with Bob, Bob is friends with Alice—so the graph is undirected. There is no natural numerical value on a friendship edge, so the graph is unweighted. This is a simple, undirected, unweighted graph.

Case 2 — Road / GPS Network

Directed * Weighted

Vertices = intersections. Edges = road segments. Weights represent distance or travel time. Most streets are two-way (undirected), but one-way streets make the graph directed in general. GPS routing uses directed, weighted graphs so it can respect one-way constraints and minimize travel cost.

* If all streets are two-way, the graph is undirected. In the general case (one-way streets present), use a digraph.

Case 3 — Web Pages & Hyperlinks

Directed Unweighted

Vertices = web pages. Edges = hyperlinks. A link from page A to page B does not imply a link from B to A, so the graph is directed. Links typically carry no numerical value, making the graph unweighted. Search-engine algorithms like PageRank operate on this directed, unweighted graph.

Case 4 — Task Dependency (DAG)

Directed Acyclic (DAG) Unweighted

Vertices = tasks. Edges = "must complete before" relationships. An edge $$(A, B)$$ means task A must finish before task B can start. Because circular dependencies are forbidden, the graph must be acyclic—a Directed Acyclic Graph (DAG). In Week 13 you will learn topological sorting, which finds a valid execution order for the tasks.

6 · Connection to Relations (Week 2 Callback)

Week 2 Callback — Graphs as Relations

In Week 2 you studied binary relations on a set. A graph is just a relation you can draw!

  • A directed graph on vertex set $$V$$ is exactly a binary relation $$R \subseteq V \times V$$. The edge $$(u,v)$$ exists if and only if $$u\,R\,v$$.
  • An undirected simple graph corresponds to a symmetric, irreflexive relation: symmetric because $$\{u,v\} \in E$$ means both $$u\,R\,v$$ and $$v\,R\,u$$; irreflexive because a simple graph has no self-loops.
Relation PropertyGraph Interpretation
Reflexive — every element is related to itselfEvery vertex has a self-loop (pseudograph)
Irreflexive — no element is related to itselfNo self-loops (simple graph)
Symmetric — $$u\,R\,v \Rightarrow v\,R\,u$$Undirected edges (or every directed arc has a reverse arc)
Antisymmetric — $$u\,R\,v$$ and $$v\,R\,u \Rightarrow u = v$$No pair of vertices has arcs in both directions (except self-loops)
Transitive — $$u\,R\,v$$ and $$v\,R\,w \Rightarrow u\,R\,w$$If there is a directed path of length 2 from $$u$$ to $$w$$, there is also a direct arc

This connection will reappear in Week 13 when you study traversal algorithms—BFS and DFS effectively compute the transitive closure of the adjacency relation, revealing all reachable vertices.
