1   Why Graphs?

Think about your social-media feed, a GPS navigation app, or the dependency chain that determines the order your code compiles. Each of these systems is, at its core, a collection of things and connections between things. In mathematics and computer science, the structure that captures this idea is called a graph.

🔗 Connection to Prior Weeks

In Weeks 1–2 you studied sets, ordered pairs, and binary relations. A graph is built from exactly those ingredients: a set of vertices and a relation (the edges) defined on that set. If you are comfortable writing a set in roster notation and identifying properties of a relation, you already have the prerequisites for this module.

Figure 1 — A symmetric, irreflexive relation on {a, b, c, d} drawn as an undirected graph.

Graphs appear everywhere in computing: network routing, social-network analysis, compiler dependency resolution, web crawling, database query planning, and more. This module gives you the precise vocabulary you need to describe and reason about these structures. Every remaining week of CSC 240 builds on what you learn here.

2   Formal Definition of a Graph

📘 Definition — Graph

A graph is an ordered pair G = (V, E) where:

  • V is a finite, non-empty set of vertices (also called nodes).
  • E is a set of edges. In an undirected graph, each edge is a 2-element subset of V, written {u, v}. In a directed graph, each edge is an ordered pair (u, v) ∈ V × V.

Notice how the definition relies on concepts from Week 1 (sets, subsets) and Week 2 (ordered pairs, Cartesian product). Edges are literally elements of the power set of V restricted to size 2, or elements of V × V.

Worked Example

Let V = {a, b, c, d} and E = { {a,b}, {b,c}, {c,d}, {a,d} }.

Figure 2 — The graph G = (V, E) with |V| = 4 and |E| = 4. This is the cycle C₄.

Adjacencies from the diagram:

  • a is adjacent to b and d.
  • b is adjacent to a and c.
  • c is adjacent to b and d.
  • d is adjacent to c and a.

Every vertex has exactly 2 neighbors, so this graph is a cycle — a concept we will define precisely in the next section.

3   Core Vocabulary

The terms below form the language you will use for the rest of the course. Each definition includes a brief visual cue tied to the worked example above or a small supplementary diagram.

📘 Definition

Two vertices u and v are adjacent (or neighbors) if there is an edge {u,v} ∈ E.

In Figure 2: a and b are adjacent because {a,b} ∈ E. Vertices a and c are not adjacent — there is no edge {a,c}.

📘 Definition

An edge e = {u,v} is incident to vertex u (and to vertex v). We also say u and v are the endpoints of e.

In Figure 2: The edge {a,b} is incident to both a and b.

📘 Definition

The degree of a vertex v, written deg(v), is the number of edges incident to v.

In Figure 2:

  • deg(a) = 2  (edges {a,b} and {a,d})
  • deg(b) = 2, deg(c) = 2, deg(d) = 2

Quick check: ∑ deg(v) = 2 + 2 + 2 + 2 = 8 = 2 × |E| = 2 × 4. This is the Handshaking Lemma — you will study it formally in Module 12.4.

📘 Definitions
  • A walk is a sequence of vertices v₀, v₁, …, vₖ where each consecutive pair {vᵢ, vᵢ₊₁} ∈ E. Vertices and edges may repeat.
  • A trail is a walk in which no edge is repeated (vertices may still repeat).
  • A path is a walk in which no vertex (and therefore no edge) is repeated.

Key takeaway: Every path is a trail, and every trail is a walk — but not the other way around.

📘 Definition

A cycle is a path of length ≥ 3 that starts and ends at the same vertex (the only repeated vertex is the start/end vertex).

In Figure 2: a → b → c → d → a is a cycle of length 4. The entire graph is a cycle.

📘 Definition

An undirected graph G is connected if there exists a path between every pair of vertices in V.

In Figure 2: You can reach any vertex from any other vertex, so the graph is connected.

📘 Definition

A connected component of a graph G is a maximal connected subgraph — meaning you cannot add any more vertices or edges from G and keep it connected.

Figure 3 — A disconnected graph with two connected components.

This graph has |V| = 5, |E| = 4, and 2 connected components. There is no path from vertex 1 to vertex 4.

📘 Definition

A graph H = (V′, E′) is a subgraph of G = (V, E) if V′ ⊆ V and E′ ⊆ E, and every edge in E′ has both endpoints in V′.

Example from Figure 2: Take V′ = {a, b, c} and E′ = { {a,b}, {b,c} }. This is a valid subgraph (a path P₃).

4   Directed Graphs: In-Degree and Out-Degree

In a directed graph (or digraph), each edge is an ordered pair (u, v), meaning the edge goes from u to v. Direction matters: (a, b) and (b, a) are different edges.

📘 Definitions — In-Degree and Out-Degree
  • Out-degree deg⁺(v): the number of edges leaving v.
  • In-degree deg⁻(v): the number of edges entering v.

For any directed graph: ∑ deg⁺(v) = ∑ deg⁻(v) = |E|

Figure 4 — A directed graph. Edge (a,b) goes from a to b but not from b to a.
Vertex deg⁺(v) deg⁻(v)
a20
b11
c12
d01
Total44

Both columns sum to |E| = 4, confirming the directed-graph handshaking property.

⚠️ Undirected vs. Directed — Don't Mix Them Up

In an undirected graph, every vertex has a single degree value deg(v). In a directed graph, degree splits into deg⁺(v) and deg⁻(v). When a problem says "degree" without qualification, assume the graph is undirected unless stated otherwise.

5   Mini-Case: A Social Network as a Graph

Six friends — Ali, Bo, Cal, Dee, Eve, Fay — are connected by the following friendships (friendship is mutual, so this is an undirected graph):

Figure 5 — Social network: V = {Ali, Bo, Cal, Dee, Eve, Fay}, |E| = 8.

Formal notation:

E = { {Ali,Bo}, {Ali,Fay}, {Ali,Dee}, {Bo,Cal}, {Bo,Fay}, {Cal,Dee}, {Eve,Dee}, {Fay,Eve} }

Person Friends deg(v)
AliBo, Fay, Dee3
BoAli, Cal, Fay3
CalBo, Dee2
DeeAli, Cal, Eve3
EveDee, Fay2
FayAli, Bo, Eve3
Total16

Handshaking check: ∑ deg(v) = 16 = 2 × 8 = 2|E|. ✓

Yes. From any person, you can reach every other person by following friendship edges. For example, to get from Cal to Eve: Cal → Dee → Eve. The graph has 1 connected component.

There are several paths. Two short ones:

  • Bo → Fay → Eve  (length 2)
  • Bo → Ali → Dee → Eve  (length 3)

The shortest path has length 2: Bo → Fay → Eve.

One cycle: Ali → Bo → Fay → Ali (length 3). Another: Ali → Bo → Cal → Dee → Ali (length 4). Cycles indicate that there are multiple routes between people — the network has redundancy.

Check 12.1a — Edge Set Notation

Consider the graph below with 5 vertices. List the edge set E using set notation.

Which of the following correctly represents E?

✔ Correct. Reading the edges from the diagram: v₁–v₂ (top), v₁–v₃ (diagonal left), v₂–v₃ (diagonal right), v₂–v₄ (right), and v₄–v₅ (bottom). Option A incorrectly lists {v₃,v₄} instead of {v₂,v₄}. Options C and D also have edge mismatches.
✘ Not quite. Trace each line in the diagram carefully. The edge between the right-side vertices connects v₂ to v₄, not v₃ to v₄. The correct answer is B: { {v₁,v₂}, {v₁,v₃}, {v₂,v₃}, {v₂,v₄}, {v₄,v₅} }.

Check 12.1b — Degree and the Handshaking Lemma

Using the same graph from Check 12.1a, compute deg(v) for every vertex and verify the Handshaking Lemma. Which row below is correct?

✔ Correct. v₁ touches edges {v₁,v₂} and {v₁,v₃} → degree 2. v₂ touches {v₁,v₂}, {v₂,v₃}, {v₂,v₄} → degree 3. v₃ touches {v₁,v₃}, {v₂,v₃} → degree 2. v₄ touches {v₂,v₄}, {v₄,v₅} → degree 2. v₅ touches {v₄,v₅} → degree 1. Sum = 10 = 2 × 5 = 2|E|. ✓ The Handshaking Lemma holds.
✘ Not quite. Count the edges touching each vertex carefully. v₂ is the most connected vertex with 3 edges. The sum of all degrees must equal 2|E| = 2 × 5 = 10. If your sum is odd (like 11 in option C), the Handshaking Lemma tells you something is wrong. The correct answer is B.

Check 12.1c — Walk vs. Path (True / False)

Statement: "Every walk is a path."

✔ Correct — the statement is false. A walk may repeat vertices and edges. A path requires all vertices to be distinct. For example, in the social network from Section 5, the walk Ali → Bo → Fay → Ali revisits Ali, so it is a walk (and actually a cycle) but not a path. The correct hierarchy is: every path is a trail, and every trail is a walk — but not every walk is a path.
✘ Not quite. A walk allows repeated vertices and edges, while a path requires all vertices to be distinct. Since walks are less restrictive, not every walk qualifies as a path. The statement is false.

Check 12.1d — Connected Components

A graph has vertices {1, 2, 3, 4, 5, 6, 7} and edges { {1,2}, {2,3}, {1,3}, {4,5}, {6,7} }. How many connected components does this graph have?

✔ Correct — 3 components. Component 1: {1, 2, 3} (connected by edges {1,2}, {2,3}, {1,3}). Component 2: {4, 5} (connected by {4,5}). Component 3: {6, 7} (connected by {6,7}). There is no edge linking any vertex in one component to a vertex in another, so the graph is disconnected with exactly 3 maximal connected subgraphs.
✘ Not quite. Group the vertices by reachability. Vertices 1, 2, 3 are all connected to each other. Vertices 4 and 5 are connected only to each other. Vertices 6 and 7 are connected only to each other. No edges cross between these three groups, giving 3 connected components (answer C).
