1  From BFS to DFS — Explore Deep, Then Backtrack

In Module 13.1 you learned BFS, which fans out level by level like ripples on a pond. Depth-First Search (DFS) takes the opposite strategy: it picks one neighbor and immediately dives deeper, continuing along a single path until it reaches a dead end. Only then does it backtrack and try a different branch.

🏰 The Maze Analogy

Imagine you are exploring a large hedge maze. You carry a ball of string so you can retrace your steps. At every fork, you always take the leftmost unexplored passage and walk as far as you can. When you hit a dead end, you follow the string back to the most recent fork that still has an unexplored passage, then continue from there.

That is exactly how DFS works:

  • Go forward (explore an unvisited neighbor).
  • Dead end? Backtrack to the most recent vertex that still has unvisited neighbors.
  • Repeat until every reachable vertex has been visited.

While BFS uses a queue (first-in, first-out), DFS relies on a stack (last-in, first-out). The recursive version uses the call stack implicitly; the iterative version manages an explicit stack data structure. Both produce the same deep-before-wide exploration pattern.

2  Recursive DFS Algorithm

DFS has a natural recursive structure: visiting a vertex means visiting each of its unvisited neighbors, and visiting a neighbor is the same problem on a smaller portion of the graph. This mirrors the recursive thinking you practiced in Week 11.

2.1  Pseudocode

The standard recursive DFS has two parts:

Algorithm DFS(G):
    Input: Graph G = (V, E) represented as an adjacency list
    Output: All vertices marked visited; d[] and f[] timestamps set

    time ← 0                          // global clock
    for each vertex v ∈ V:
        color[v] ← WHITE              // unvisited
        parent[v] ← NIL

    for each vertex v ∈ V:            // outer loop handles disconnected graphs
        if color[v] = WHITE:
            DFS-Visit(G, v)
Algorithm DFS-Visit(G, u):
    time ← time + 1
    d[u] ← time                       // discovery time
    color[u] ← GRAY                   // u is being explored

    for each neighbor w of u:          // iterate adjacency list of u
        if color[w] = WHITE:
            parent[w] ← u
            DFS-Visit(G, w)            // recursive call — go deeper!

    color[u] ← BLACK                   // u is fully explored
    time ← time + 1
    f[u] ← time                        // finish time
📖 Vertex Colors
  • WHITE — not yet discovered.
  • GRAY — discovered but still has descendants being explored (on the call stack).
  • BLACK — fully explored; all descendants finished.

2.2  The Call Stack as an Implicit Stack

Every time DFS-Visit calls itself, the current vertex is pushed onto the call stack. When the recursive call returns (because a dead end was reached), the program pops back to the previous vertex — exactly the backtracking behavior of the maze analogy.

🔗 Connection to Week 11

In Week 11 you traced recursion trees for functions like fibonacci(n). The DFS call tree works the same way: each call to DFS-Visit(G, u) spawns child calls for u's unvisited neighbors. The recursion tree of DFS is the DFS tree of the graph.

3  Iterative DFS Algorithm

Just as BFS uses an explicit queue, we can implement DFS with an explicit stack. This avoids deep recursion that could overflow the call stack on very large graphs.

3.1  Pseudocode

Algorithm DFS-Iterative(G, s):
    Input: Graph G = (V, E), source vertex s
    Output: All vertices reachable from s marked visited

    let S be an empty stack
    for each vertex v ∈ V:
        visited[v] ← false

    S.push(s)

    while S is not empty:
        u ← S.pop()
        if visited[u] = false:
            visited[u] ← true
            process(u)                  // e.g., print u
            for each neighbor w of u:   // push all unvisited neighbors
                if visited[w] = false:
                    S.push(w)
⚠️ Subtle Difference: Iterative vs. Recursive DFS

The iterative version may visit vertices in a different order than the recursive version, even on the same graph with the same adjacency-list ordering. Here is why:

  • The recursive version processes the first unvisited neighbor immediately (depth-first) before looking at the second neighbor.
  • The iterative version pushes all unvisited neighbors onto the stack at once. Because a stack is LIFO, the last neighbor pushed is the first popped. This effectively reverses the adjacency-list order unless you push neighbors in reverse order.

To make the iterative version produce the same order as the recursive version, push neighbors in reverse adjacency-list order.

3.2  BFS ↔ Iterative DFS: One Data Structure Apart

AspectBFSIterative DFS
Data structureQueue (FIFO)Stack (LIFO)
Core loopu ← Q.dequeue()u ← S.pop()
Neighbor handlingEnqueue unvisited neighborsPush unvisited neighbors
Traversal shapeLevel by levelDeep path by path

Swapping the queue for a stack is all it takes to change BFS into DFS. This elegant symmetry is one of the most important observations in graph algorithms.

4  Discovery and Finish Timestamps

The recursive DFS algorithm maintains a global clock. Each vertex $$v$$ gets two timestamps:

  • $$d[v]$$ — discovery time: when $$v$$ is first encountered (colored GRAY).
  • $$f[v]$$ — finish time: when $$v$$'s subtree is fully explored (colored BLACK).

For a graph with $$|V|$$ vertices, the timestamps use the integers $$1$$ through $$2|V|$$ (each vertex accounts for exactly two ticks of the clock).

4.1  The Parenthesis Theorem

📖 Parenthesis Theorem

For any two vertices $$u$$ and $$v$$ in a DFS, exactly one of the following is true:

  1. The intervals $$[d[u], f[u]]$$ and $$[d[v], f[v]]$$ are entirely disjoint — neither is an ancestor of the other in the DFS tree.
  2. One interval entirely contains the other — the vertex with the containing interval is an ancestor of the other in the DFS tree.

No two intervals partially overlap. If you replace each discovery with "(" and each finish with ")", you get a string of properly nested parentheses — hence the name.

5  DFS Tree and Edge Classification

As DFS explores a graph, every edge is classified into one of four types based on the timestamps and colors of the endpoints. This classification is one of the most powerful tools in graph algorithm design.

5.1  Four Edge Types (Directed Graphs)

DFS Edge Classification
Edge TypeDescriptionTimestamp Condition
Tree edge Edge $$(u, v)$$ where $$v$$ is WHITE when explored from $$u$$. These edges form the DFS tree. $$d[u] < d[v] < f[v] < f[u]$$
Back edge Edge $$(u, v)$$ where $$v$$ is GRAY — $$v$$ is an ancestor of $$u$$ in the DFS tree. Back edges indicate cycles. $$d[v] < d[u] < f[u] < f[v]$$
Forward edge Edge $$(u, v)$$ where $$v$$ is BLACK and $$v$$ is a descendant of $$u$$ (not via a tree edge). $$d[u] < d[v] < f[v] < f[u]$$
Cross edge Edge $$(u, v)$$ where $$v$$ is BLACK and $$v$$ is neither an ancestor nor a descendant of $$u$$. $$d[v] < f[v] < d[u] < f[u]$$

5.2  Undirected Graphs: Simpler Classification

⚠️ Undirected Graph Simplification

In an undirected graph, DFS produces only tree edges and back edges. There are no forward or cross edges. This is because every edge is traversed in both directions, so any non-tree edge must connect a vertex to one of its ancestors (making it a back edge).

5.3  Why Edge Classification Matters

  • Cycle detection: A graph has a cycle if and only if DFS finds a back edge. (Module 13.4 explores this in depth.)
  • Topological sorting: A DAG has no back edges. Sorting by reverse finish time gives a valid topological order. (Module 13.4.)
  • Strongly connected components: Edge classification drives Tarjan's and Kosaraju's algorithms (beyond this course, but good to know).

6  Time Complexity: $$O(|V| + |E|)$$

The complexity analysis for DFS mirrors BFS almost exactly.

6.1  On an Adjacency List

  1. Initialization: The outer loop in DFS(G) visits every vertex once to check its color → $$O(|V|)$$.
  2. DFS-Visit calls: Each vertex is the argument of DFS-Visit at most once (it must be WHITE to enter). Inside DFS-Visit(G, u), we scan the adjacency list of $$u$$, which has $$\deg(u)$$ entries.
  3. Total adjacency-list work: Summing over all vertices: $$\sum_{u \in V} \deg(u) = 2|E|$$ for undirected graphs (by the Handshaking Lemma from Week 12) or $$|E|$$ for directed graphs.

Therefore:

$$T_{\text{DFS}} = O(|V| + |E|)$$

This is the same asymptotic complexity as BFS. Both algorithms are linear in the size of the graph.

6.2  On an Adjacency Matrix

If the graph is stored as an adjacency matrix, checking all neighbors of a vertex takes $$O(|V|)$$ instead of $$O(\deg(u))$$. Summing over all vertices:

$$T_{\text{DFS-matrix}} = O(|V|^2)$$

This is why adjacency lists are preferred for traversal algorithms on sparse graphs — the same conclusion you reached for BFS in Module 13.1.

8  Java Code Walkthrough

Below are Java implementations of both recursive and iterative DFS. Review these after you are comfortable with the pseudocode above.

☕ Java: Recursive DFS with Timestamps 
import java.util.*;

public class DFSRecursive {
    static int time = 0;  // global clock

    static final int WHITE = 0, GRAY = 1, BLACK = 2;

    public static void dfs(List<List<Integer>> adj, int n) {
        int[] color  = new int[n];   // all WHITE (0) by default
        int[] d      = new int[n];   // discovery times
        int[] f      = new int[n];   // finish times
        int[] parent = new int[n];
        Arrays.fill(parent, -1);
        time = 0;

        for (int v = 0; v < n; v++) {
            if (color[v] == WHITE) {
                dfsVisit(adj, v, color, d, f, parent);
            }
        }

        // Print results
        System.out.println("Vertex | d[v] | f[v]");
        for (int v = 0; v < n; v++) {
            System.out.printf("  %d    |  %2d  |  %2d%n", v, d[v], f[v]);
        }
    }

    static void dfsVisit(List<List<Integer>> adj, int u,
                         int[] color, int[] d, int[] f, int[] parent) {
        time++;
        d[u] = time;
        color[u] = GRAY;
        System.out.println("Discover vertex " + u + " at time " + d[u]);

        for (int w : adj.get(u)) {
            if (color[w] == WHITE) {
                parent[w] = u;
                dfsVisit(adj, w, color, d, f, parent);  // recurse
            }
            // color[w] == GRAY  → back edge
            // color[w] == BLACK → forward or cross edge
        }

        color[u] = BLACK;
        time++;
        f[u] = time;
        System.out.println("Finish vertex " + u + " at time " + f[u]);
    }

    public static void main(String[] args) {
        // Build the 7-vertex example: A=0, B=1, C=2, D=3, E=4, F=5, G=6
        int n = 7;
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < n; i++) adj.add(new ArrayList<>());

        // Undirected edges (add both directions)
        int[][] edges = {{0,1},{0,2},{1,3},{1,4},{2,4},{2,5},{4,6}};
        for (int[] e : edges) {
            adj.get(e[0]).add(e[1]);
            adj.get(e[1]).add(e[0]);
        }
        // Sort each list for consistent alphabetical order
        for (List<Integer> list : adj) Collections.sort(list);

        dfs(adj, n);
    }
}
💡 Key Observation

The dfsVisit method directly mirrors the pseudocode from Section 2. The color array tracks WHITE/GRAY/BLACK, and the global time variable stamps discovery and finish events.

☕ Java: Iterative DFS with Explicit Stack 
import java.util.*;

public class DFSIterative {

    public static void dfsIterative(List<List<Integer>> adj, int n, int source) {
        boolean[] visited = new boolean[n];
        Deque<Integer> stack = new ArrayDeque<>();
        List<Integer> order = new ArrayList<>();

        stack.push(source);

        while (!stack.isEmpty()) {
            int u = stack.pop();
            if (visited[u]) continue;   // may have been pushed multiple times
            visited[u] = true;
            order.add(u);

            // Push neighbors in REVERSE order so the first neighbor
            // in the adjacency list is popped first (matches recursive order)
            List<Integer> neighbors = adj.get(u);
            for (int i = neighbors.size() - 1; i >= 0; i--) {
                int w = neighbors.get(i);
                if (!visited[w]) {
                    stack.push(w);
                }
            }
        }

        System.out.println("Iterative DFS order: " + order);
    }

    public static void main(String[] args) {
        int n = 7;
        List<List<Integer>> adj = new ArrayList<>();
        for (int i = 0; i < n; i++) adj.add(new ArrayList<>());

        int[][] edges = {{0,1},{0,2},{1,3},{1,4},{2,4},{2,5},{4,6}};
        for (int[] e : edges) {
            adj.get(e[0]).add(e[1]);
            adj.get(e[1]).add(e[0]);
        }
        for (List<Integer> list : adj) Collections.sort(list);

        dfsIterative(adj, n, 0);  // source = A (vertex 0)
    }
}
⚠️ Why Reverse Order?

We push neighbors from last to first so that the first neighbor in the sorted adjacency list ends up on top of the stack and is popped next. Without this reversal, the iterative version would explore neighbors in the opposite order from the recursive version.

9  DFS Correctness and Mathematical Induction

One of the course learning outcomes (CSLO 4) asks you to connect recursion to mathematical induction. DFS is a perfect example: its recursive structure lets us prove correctness using the same induction framework you learned in Weeks 6 and 11.

🔗 The Connection

Recursion and induction are two sides of the same coin. Recursion solves a problem by reducing it to smaller instances. Induction proves that the solution is correct by showing the base case works and that correctness carries from smaller instances to larger ones.

9.1  What We Want to Prove

Claim: Starting from any source vertex $$s$$, DFS-Visit(G, s) correctly visits every vertex reachable from $$s$$.

9.2  Proof by Strong Induction on $$k$$, the Number of Reachable Vertices

Base Case ($$k = 1$$)

If only one vertex is reachable from $$s$$ (namely $$s$$ itself, with no unvisited neighbors), then DFS-Visit(G, s) discovers $$s$$, finds no WHITE neighbors, and finishes. The single reachable vertex is correctly visited. ✓

Inductive Hypothesis

Assume that for every graph and every source vertex, if the number of reachable unvisited vertices is at most $$k$$ (for some $$k \geq 1$$), then DFS-Visit correctly visits all of them.

Inductive Step (show it works for $$k + 1$$)

Suppose $$s$$ can reach $$k + 1$$ unvisited vertices (including itself). When DFS-Visit(G, s) runs:

  1. It discovers $$s$$ and marks it GRAY.
  2. For each WHITE neighbor $$w$$ of $$s$$, it calls DFS-Visit(G, w). At the time of this call, the number of unvisited vertices reachable from $$w$$ is at most $$k$$ (because $$s$$ itself is already GRAY and won't be counted again, and some vertices may have been visited by earlier recursive calls).
  3. By the inductive hypothesis, each such call correctly visits all vertices reachable from $$w$$.
  4. After all neighbors are processed, every vertex reachable from $$s$$ has been visited.

By the principle of mathematical induction, DFS-Visit correctly visits all reachable vertices for any graph of any size. ∎

💡 Why This Matters

This proof pattern — base case, inductive hypothesis, inductive step — is the same structure you used in Week 6 for proofs about sums and divisibility, and in Week 11 for proving properties of recursive functions. The only difference is that here the "smaller instance" is a subgraph with fewer unvisited vertices instead of a smaller integer. Recognizing this pattern across different domains is a core skill in discrete mathematics.
