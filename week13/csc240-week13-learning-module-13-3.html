BFS vs. DFS: Comparative Analysis

A structured reference for choosing the right traversal strategy.

⏱️ ~20–30 minutes 📖 Reference & Comparison 🎯 Ungraded Knowledge Checks

1 · Introduction

In Modules 13.1 and 13.2 you learned Breadth-First Search (BFS) and Depth-First Search (DFS) as individual algorithms. Both systematically visit every reachable vertex in a graph, yet they do so in fundamentally different orders—and that difference determines which problems each algorithm solves well.

This short module puts the two algorithms side by side so you can quickly compare their properties and make informed choices in problem-solving contexts.

🎯 Module Objectives
  • Compare BFS and DFS across key structural and complexity dimensions.
  • Apply decision rules to select the appropriate algorithm for a given problem.
  • Analyze space behavior on extreme graph topologies (star graph, path graph).
  • Trace both traversals on the same graph and explain the different visit orders.
💡 How to Use This Module

This module is designed as a scannable reference. Bookmark the comparison table and decision rules—they will be useful when you work on this week's problem set and in future courses.

2 · Comparison Table

The table below summarizes the key properties of BFS and DFS. Where both algorithms share a property, the cell is marked Same. Where they differ, the distinguishing behavior is highlighted.

BFS vs. DFS — Property Comparison
Property BFS DFS
Data structure Queue (FIFO) Stack (LIFO) / recursion
Traversal pattern Level-by-level (breadth-first) Branch-by-branch (depth-first)
Time — adjacency list O(V + E)
Time — adjacency matrix O(V²)
Space (worst case) O(V) — queue may hold an entire level O(V) — stack may reach depth V
Shortest path (unweighted) ✔ Yes — guarantees minimum hops ✘ No — may find a longer path first
Timestamps (discovery / finish) Not standard (discovery time only) ✔ Yes — d[v] and f[v]
Edge classification Limited (tree vs. non-tree) ✔ Full — tree, back, forward, cross
Cycle detection Possible (undirected) ✔ Preferred — back edge ⟹ cycle (directed & undirected)
Topological sort Possible via Kahn's algorithm (in-degree) ✔ Natural — reverse finish-time order
Connected-component detection Both work — run from each unvisited vertex
🔍 Key Takeaway

BFS and DFS share the same asymptotic time complexity, but they differ in what structural information they reveal. BFS discovers shortest-path distances; DFS discovers timestamps and edge types. Choosing the right algorithm means matching these capabilities to the problem at hand.

3 · When to Use BFS vs. DFS

Use the following decision cards as a quick reference. Each card lists the problem types best suited to the algorithm and a brief justification.

🔵 Choose BFS when …

  • Shortest path (unweighted): BFS guarantees the minimum number of edges from source to every reachable vertex.
  • Level-order exploration: You need to process all vertices at distance k before distance k + 1 (e.g., social-network "friends of friends").
  • Minimum hops: Routing problems where each edge has equal cost—fewest hops = shortest path.
  • Bipartiteness testing: Level-based coloring naturally checks two-colorability.

🟣 Choose DFS when …

  • Topological sort: Reverse finish-time ordering gives a valid topological order for DAGs.
  • Cycle detection (directed): A back edge in the DFS tree signals a cycle—clean and efficient.
  • Exhaustive search / backtracking: Maze solving, puzzle solving, and constraint satisfaction naturally follow DFS's "go deep, then backtrack" pattern.
  • Strongly connected components: Algorithms like Kosaraju's and Tarjan's are built on DFS.
💡 Quick Heuristic

Need the closest answer? → BFS (think "level by level").
Need to classify structure or explore every branch? → DFS (think "deep dive").

4 · Space Behavior on Extreme Graphs

While both algorithms have O(V) worst-case space, the shape of the graph determines which algorithm actually hits that worst case. Two extreme topologies make this vivid.

⭐ Star Graph K1,n — BFS worst case

BFS from A: After dequeuing A, all n leaves are enqueued simultaneously → queue size = n.

DFS from A: Visits one leaf at a time, immediately backtracks → max stack depth = 2 (A + one leaf).

⚠️ BFS pain point

Wide, shallow graphs inflate the BFS queue to hold an entire level.

📏 Path Graph Pn — DFS worst case

DFS from A: Descends the entire chain before backtracking → max stack depth = n.

BFS from A: Each level has exactly one vertex → max queue size = 1.

⚠️ DFS pain point

Deep, narrow graphs inflate the DFS stack (or recursion depth) to n.

🔍 Pattern

BFS space ∝ maximum level width. DFS space ∝ maximum path depth. In practice, most real-world graphs fall between these extremes, and both algorithms use moderate space.

5 · Same Graph, Two Traversals

Below is the 7-vertex running example used throughout Week 13 (the same graph from Modules 13.1 and 13.2). We traverse it from source vertex A using both BFS and DFS. Ties are broken alphabetically. Compare the visit orders.

🔵 BFS from A

Visit order: A B D C E F G

Levels: {A} → {B, D} → {C, E, F} → {G}. All level-k vertices visited before level-k+1.

🟣 DFS from A

Visit order: A B C E F G D

DFS dives A → B → C → E → F → G, then backtracks to visit D last.

🔑 Notice the Difference

Both algorithms visit all 7 vertices, but in completely different orders. BFS spreads outward level by level; DFS plunges down one branch before exploring alternatives. The same graph, the same source—different strategies, different orders.

✅ Knowledge Check 1 of 5

A ride-sharing app needs to find the route with the fewest stops (edges) between two subway stations in an unweighted graph. Which algorithm should it use?

✔️ Correct! BFS explores vertices level by level, so the first time it reaches any vertex, it has found the minimum number of hops. DFS might find a longer path first and has no mechanism to guarantee shortest distance in an unweighted graph.
✘ Not quite. The correct answer is B. BFS guarantees the shortest path (minimum hops) in unweighted graphs because it visits all vertices at distance k before any vertex at distance k + 1. DFS does not provide this guarantee.
✅ Knowledge Check 2 of 5

A build system must check whether its dependency graph contains a circular dependency (cycle in a directed graph). Which algorithm is preferred?

✔️ Correct! DFS is the standard choice for directed cycle detection. During DFS, encountering a back edge (an edge to a vertex still on the recursion stack) definitively proves a cycle exists. BFS can detect cycles in undirected graphs but is not the standard approach for directed graphs.
✘ Not quite. The correct answer is B. DFS naturally classifies edges into tree, back, forward, and cross edges. A back edge (pointing to an ancestor in the DFS tree) is the definitive signal of a cycle in a directed graph. This is the standard technique for directed cycle detection.
✅ Knowledge Check 3 of 5

Consider a path graph P10,000 (10,000 vertices in a single chain). Which statement about space usage is correct when starting from one end?

✔️ Correct! In a path graph, each "level" has exactly one vertex, so the BFS queue never holds more than 1 or 2 vertices. DFS, however, descends the entire chain before backtracking, pushing all 10,000 vertices onto the stack. This is DFS's worst-case space scenario.
✘ Not quite. The correct answer is B. A path graph has depth n but width 1. DFS dives to the end (stack depth ≈ 10,000), while BFS processes one vertex per level (queue size ≈ 1). This illustrates how graph shape determines which algorithm hits its worst-case space.
✅ Knowledge Check 4 of 5

Which of the following is a capability of DFS that BFS does not naturally provide?

✔️ Correct! DFS's timestamp structure (discovery and finish times) enables full edge classification into tree, back, forward, and cross edges. BFS can only distinguish tree edges from non-tree edges—it does not support the finer classification that DFS provides.
✘ Not quite. The correct answer is C. Options A, B, and D are shared by both algorithms. Only DFS provides full edge classification (tree, back, forward, cross) thanks to its discovery and finish timestamps. BFS only distinguishes tree edges from non-tree edges.
✅ Knowledge Check 5 of 5

A course prerequisite system needs to produce a valid ordering of courses such that every prerequisite is listed before the course that requires it. This is a:

✔️ Correct! Ordering items so that every dependency comes first is exactly the topological sort problem. DFS solves it naturally: list vertices in reverse finish-time order. (Note: Kahn's BFS-based algorithm also works, but the DFS approach is the classic method covered in this course.)
✘ Not quite. The correct answer is C. Producing a valid prerequisite ordering is a topological sort on a directed acyclic graph (DAG). DFS naturally provides this: after a full DFS, listing vertices in reverse order of their finish times yields a valid topological order.

Module Summary

🗂️ Key Takeaways

  • Same time complexity, different information: Both BFS and DFS run in $$O(V + E)$$ time, but BFS reveals shortest-path distances while DFS reveals timestamps and edge classifications.
  • Data structure drives the pattern: A queue (FIFO) produces level-order traversal; a stack (LIFO) produces depth-first exploration.
  • Choose BFS for shortest paths, minimum hops, and level-order processing.
  • Choose DFS for topological sort, cycle detection, edge classification, and exhaustive backtracking.
  • Space depends on graph shape: BFS struggles on wide graphs (star); DFS struggles on deep graphs (path).
  • Both detect connected components equally well.
