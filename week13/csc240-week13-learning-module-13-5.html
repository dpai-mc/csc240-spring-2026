Section A — Real-World Applications Gallery

Each mini-case below maps a real computing scenario to one or more algorithms from Modules 13.1–13.4. Notice how the same core traversal ideas appear across very different domains.

BFS – Shortest Path

🔲 Maze Solving

A robot navigates a grid-based maze where each cell is either open or blocked. The grid maps naturally to an unweighted graph: each open cell is a vertex and adjacent open cells share an edge. BFS from the start cell discovers the shortest path (fewest moves) to the exit because it explores all cells at distance $$d$$ before any cell at distance $$d+1$$. This is exactly the level-order exploration you practiced in Module 13.1.

DFS – Deep Exploration

🌐 Web Crawling

A search-engine crawler starts at a seed URL and follows hyperlinks to discover new pages. Each page is a vertex; each hyperlink is a directed edge. DFS is a natural fit because the crawler follows a chain of links as deeply as possible before backtracking, which keeps memory usage manageable compared to BFS on the enormous web graph. Cycle detection (Module 13.4) prevents the crawler from revisiting pages it has already indexed.

Topological Sort

📦 Package Manager Dependency Resolution

When you run an install command, the package manager must determine a valid installation order: library A before framework B if B depends on A. The dependency graph is a DAG (directed acyclic graph). A topological sort (Module 13.3) produces a linear ordering that respects every dependency. If a cycle is detected, the manager reports a circular-dependency error—exactly the cycle-detection application from Module 13.4.

Connected Components

👥 Social-Network Community Detection

In a social network, users are vertices and friendships are undirected edges. Identifying connected components (Module 13.4) reveals isolated communities—groups of users who are all reachable from one another but disconnected from other groups. Repeated BFS or DFS on unvisited vertices partitions the entire network into these communities, a first step toward recommendation engines and content targeting.

🔗 Pattern to Remember

BFS excels when you need the shortest or nearest result in an unweighted graph. DFS excels for deep exploration, backtracking, and structural analysis (cycles, components, topological order). Many real systems combine both.

Section B — Preview of Week 14

🔭 What's Ahead

BFS and DFS are the foundation for more powerful graph algorithms you will study next week. Kruskal's and Prim's algorithms find a Minimum Spanning Tree (MST)—the cheapest set of edges connecting all vertices—by extending ideas from traversal and greedy selection. Dijkstra's algorithm generalizes BFS to find shortest paths in weighted graphs using a priority queue instead of a plain queue.

Two key deliverables build directly on Weeks 12–13 material: PE-09 (a practice exercise on weighted-graph algorithms) and the Final Project, which asks you to apply multiple graph techniques to a larger problem. Mastering this week's traversals will make Week 14 significantly smoother.

Section D — Study Tips

  • Rework knowledge checks from Modules 13.1–13.4 without looking at the answers first. If you can answer correctly from memory, the concept is solid.
  • Hand-trace BFS and DFS on new graphs. Draw a small graph (6–8 vertices), pick a start vertex, and trace each algorithm. Write out the queue/stack at every step.
  • Time yourself: aim to complete a full BFS or DFS trace on a 6-vertex graph in under 5 minutes. Speed indicates fluency.
  • Review PS-08 and PE-08. Revisit any problems you found difficult and redo them on a blank sheet of paper.
  • Teach it to someone. Explain topological sort or cycle detection to a classmate (or a rubber duck). If you stumble, that is your study target.
  • Build a one-page cheat sheet (for studying, not for the quiz). Summarize each algorithm's data structure, time complexity, and primary use case.

📝 Knowledge Check 2 — Mini Practice Quiz (5 Questions)

Answer each question, then click Check Answers for immediate feedback.

Q1 (Week 11 – Recurrence). What is the asymptotic solution to $$T(n) = 2\,T(n/2) + n$$ using the Master Theorem?

✅ Correct! With $$a=2, b=2, f(n)=n$$, we have $$n^{\log_b a} = n^1 = n$$. Since $$f(n) = \Theta(n^{\log_b a})$$, Case 2 gives $$T(n) = \Theta(n \log n)$$.
❌ Not quite. Apply the Master Theorem: $$a=2, b=2$$, so $$n^{\log_2 2} = n$$. Since $$f(n) = n = \Theta(n)$$, this is Case 2, giving $$\Theta(n \log n)$$.

Q2 (Week 12 – Representations). A simple undirected graph has 5 vertices and 4 edges. How many 1s appear in its adjacency matrix?

✅ Correct! Each undirected edge is stored twice in the adjacency matrix (once in each direction), so 4 edges × 2 = 8 ones.
❌ Remember: in an undirected graph's adjacency matrix, each edge $$\{u,v\}$$ produces a 1 at position $$(u,v)$$ and $$(v,u)$$. So 4 × 2 = 8.

Q3 (Week 13 – BFS). Consider the graph with edges: A–B, A–C, B–D, C–D, C–E. Starting BFS at vertex A (processing neighbors in alphabetical order), what is the discovery order?

✅ Correct! BFS from A enqueues B, C (level 1, alphabetical). Dequeue B → enqueue D. Dequeue C → D already visited, enqueue E. Order: A, B, C, D, E.
❌ Trace carefully: Start with A. Enqueue neighbors B, C (alphabetical). Dequeue B → neighbor D enqueued. Dequeue C → D visited, E enqueued. Dequeue D, then E. Discovery order: A, B, C, D, E.

Q4 (Week 13 – Topological Sort). Which of the following is true about topological sort?

✅ Correct! Topological sort requires a DAG. If the graph has a cycle, no valid topological ordering exists.
❌ Topological sort is defined only for DAGs. It does not require weights, it can be done with DFS or Kahn's (BFS-based) algorithm, and a DAG may have multiple valid orderings.

Q5 (Week 13 – Algorithm Selection). You need to determine whether a directed graph contains a cycle. Which approach is most appropriate?

✅ Correct! DFS with back-edge detection is the standard method for cycle detection in directed graphs. A back edge (an edge to a vertex currently on the recursion stack) indicates a cycle.
❌ The standard approach is DFS with back-edge detection. During DFS, if you encounter an edge to a vertex still on the recursion stack, a cycle exists.
