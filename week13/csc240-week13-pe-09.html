Scenario

You are a junior systems engineer at NetTrace Solutions, a company that manages large-scale build pipelines for software deployments. Your team has flagged two issues in the current infrastructure:

  1. The internal server network may have isolated clusters — groups of machines that cannot communicate with each other. You need to identify every connected cluster.
  2. The build pipeline (a directed task graph) may contain a circular dependency that prevents tasks from completing. You need to determine whether a valid build order exists and, if not, pinpoint the cycle.

You will use BFS and DFS to investigate both graphs systematically.

Practice Objectives

By completing this lab you will practice the following Week 13 objectives (not graded):

  • Objective 3: Trace BFS and DFS by hand on small graphs (≤ 10 vertices), recording queue/stack state at each step and the resulting traversal order.
  • Objective 5: Detect connected components in an undirected graph using BFS or DFS and explain why the algorithm correctly identifies all components.
  • Objective 6: Perform topological sorting on a DAG using DFS-based finish-time ordering, and explain why the algorithm fails on graphs with cycles.
  • Objective 7: Detect cycles in both directed and undirected graphs using DFS edge classification (back edges), and distinguish between the two cases.
  • Objective 4: Compare BFS and DFS in terms of time complexity, space complexity, and suitability for different problem types.

Phase 5: Cycle Detection on Graph D-Cyc

Task: Now switch to Graph D-Cyc (which adds the edge T6 → T3). Perform DFS starting from T1 using the same neighbor processing order. Determine whether the graph contains a cycle by looking for back edges — edges that point to a vertex currently on the recursion stack (gray vertex).

Directed Cycle Detection Rule

⚠ Back Edge Rule (Directed Graphs)

In a directed graph, a back edge is an edge $$ (u, v) $$ where $$ v $$ is an ancestor of $$ u $$ in the DFS tree — equivalently, $$ v $$ is currently gray (discovered but not yet finished). A back edge exists if and only if the directed graph contains a cycle.

Bonus: Undirected Cycle Detection Comparison

ⓘ Undirected vs. Directed — The Parent Check

In an undirected graph, every edge $$ \{u, v\} $$ appears in both adjacency lists. When DFS visits $$ u $$ and sees neighbor $$ v $$, if $$ v $$ is gray and $$ v $$ is not the parent of $$ u $$ in the DFS tree, then $$ (u, v) $$ is a back edge indicating a cycle. You must exclude the parent to avoid falsely detecting the edge you just traversed as a cycle.

In a directed graph, edges are one-way, so the parent check is not needed — any edge to a gray vertex is a genuine back edge.

Hint — Phase 5

The key moment occurs when DFS reaches T6 in D-Cyc. In the DAG version, T6 only has neighbor T8. In D-Cyc, T6 has neighbors [T8, T3]. After processing T8, DFS checks T3. At this point, T3 was discovered earlier (when T1 explored it) and has NOT yet finished — T3 is still gray because we are inside the call chain T1 → T3 → ... → T6. Therefore (T6, T3) is a back edge. The cycle is T3 → T4 → T6 → T3 (following the DFS tree edges from T3 down to T6, then the back edge T6 → T3 closes the loop).

Note: The trace depends on neighbor processing order. If T1 visits T2 first, then T3 second, the back edge may be detected at a different point in the trace — but the back edge (T6, T3) will still be found.
