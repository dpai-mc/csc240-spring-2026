Week 16: AVL Trees & Hashing

Advanced Trees – AVL Trees and Hashing  |  CSC 240 – Discrete Structures  |  Spring 2026

Welcome to Week 16! This week you'll complete the tree data structures arc by studying AVL trees—the elegant, self-balancing extension of the BSTs you built in Week 15—and then pivot to hashing, an entirely different strategy for storing and retrieving data in near-constant time. Together these topics give you the full toolkit for understanding how modern software chooses between tree-based and hash-based structures.

This is a "Quiz only" week—no lab, no problem set, no discussion is due. Take advantage of the lighter assignment load to go deep on the module content, prepare thoroughly for Quiz 5, and make meaningful progress on your Final Project.

Week 16 Learning Goals

By the end of this week you will be able to…
  1. Explain the AVL balance property and identify when a BST violates it after an insertion or deletion.
  2. Perform all four AVL rotation types (LL, RR, LR, RL) correctly on paper, identifying the pivot node and restructured subtrees.
  3. Trace AVL insertion and deletion operations on small trees, showing the tree state before and after each rotation.
  4. Analyze the time complexity of AVL tree operations and explain why guaranteed O(log n) height is preferable to unbalanced BSTs for worst-case scenarios.
  5. Design a hash function for a given key type and evaluate its quality based on distribution uniformity and collision minimization.
  6. Implement and compare collision resolution strategies—separate chaining and open addressing (linear probing, quadratic probing)—and trace insertions and lookups.
  7. Analyze hash table performance in terms of load factor and explain its relationship to collision frequency and resizing.
  8. Compare BST, AVL tree, and hash table data structures across time complexity, space usage, ordering support, and application suitability.

Key Concepts & Themes

AVL Trees (Modules 16.1–16.3)

  • Balance factor: BF(v) = height(v.left) − height(v.right)
  • Four rotation types: LL (single right), RR (single left), LR (double left-right), RL (double right-left)
  • AVL insertion algorithm (BST insert → update heights → rotate if needed)
  • AVL deletion and cascading rotations
  • Guaranteed O(log n) worst-case for search, insert, and delete

Hashing (Modules 16.4–16.5)

  • Hash functions: division method, polynomial hashing, quality criteria
  • Collision resolution: separate chaining vs. open addressing (linear probing, quadratic probing, double hashing)
  • Load factor α = n/m and its impact on performance
  • Resizing / rehashing

Synthesis

  • BST vs. AVL vs. Hash Table tradeoff comparison (time, space, ordering, implementation complexity)
