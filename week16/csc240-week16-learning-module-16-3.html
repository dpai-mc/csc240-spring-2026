1. AVL Insertion Algorithm

AVL insertion extends standard BST insertion with a rebalancing pass. The complete algorithm is a four-step procedure:

  1. BST Insert — Insert the new key using standard binary search tree insertion. The new node becomes a leaf with height 0.
  2. Update Heights Bottom-Up — Starting from the newly inserted node, walk back up toward the root, updating each ancestor's height as:
    height(v) = 1 + max(height(v.left), height(v.right)) (Treat a null child as height −1.)
  3. Check Balance Factors — At each ancestor, compute the balance factor:
    BF(v) = height(v.left) − height(v.right) If |BF(v)| ≤ 1, the node is balanced; continue upward. If |BF(v)| = 2, the node is the first unbalanced ancestor.
  4. Rotate if Needed — Apply the appropriate rotation at the first unbalanced ancestor:
    • Left-Left case → Right rotation
    • Right-Right case → Left rotation
    • Left-Right case → Left-Right double rotation
    • Right-Left case → Right-Left double rotation
Key Insight: At Most One Rotation per Insertion After a single insertion, at most one rotation (single or double) is ever needed. Once the first unbalanced ancestor is fixed, all higher ancestors are automatically balanced because the rotation restores the subtree to its pre-insertion height. This is a crucial difference from deletion.

3. AVL Deletion Algorithm

AVL deletion also starts with a standard BST deletion and then rebalances, but it differs from insertion in a critical way.

Deletion Procedure

  1. BST Delete — Remove the node using standard BST deletion (handle the three cases: leaf, one child, two children via in-order successor/predecessor).
  2. Update Heights Bottom-Up — Walk from the physically removed node's parent up toward the root, updating heights.
  3. Check Balance Factors — At each ancestor, check if |BF| = 2.
  4. Rotate at Every Unbalanced Ancestor — Unlike insertion, continue checking and rotating all the way up to the root.
Key Difference from Insertion: Multiple Rotations Possible A rotation during deletion can reduce the height of the subtree it fixes. This height reduction can propagate imbalance to higher ancestors, requiring additional rotations. In the worst case, deletion may require O(log n) rotations — one at every level from the deleted node's position up to the root.

Why Does This Happen?

During insertion, the rotation restores the subtree to its original height (before the insertion), so ancestors above see no change. During deletion, the rotation may produce a subtree that is shorter than before the deletion, which can unbalance the next ancestor up.

4. Deletion Worked Example — Cascading Rotations

We start with the following 9-node AVL tree and delete node 5, triggering cascading rotations.

Initial Tree (9 nodes)

20 (BF=+1) / \ 10 (BF=0) 30 (BF=+1) / \ / \ 5(0) 15(0) 25(0) 35(BF=-1) / \ 3(0) 40(0)

Step 1: BST Delete node 5

Node 5 has one child (3). Replace 5 with its child 3.

20 (BF=+1) / \ 10 (BF=-1) 30 (BF=+1) / \ / \ 3(0) 15(0) 25(0) 35(BF=-1) \ 40(0)

Update heights bottom-up from node 3's position. Node 10: left height = 0, right height = 0 → height = 1, BF = 0. Wait — let's recalculate carefully. After removing 5 and promoting 3, node 10's left child is 3 (height 0) and right child is 15 (height 0). So BF(10) = 0. Node 20: left subtree height = 2? No — node 10 has height 1, so left subtree of 20 has height 1. Right subtree: node 30 has height 2. BF(20) = 1 − 2 = −1. This is balanced.

Actually, let's use a tree that produces cascading rotations. Consider this 9-node AVL tree instead:

20 (BF=+1) / \ 10 (BF=+1) 30 (BF=0) / \ / \ 5(+1) 15(0) 25(0) 35(0) / 3(0)

Heights: 3→0, 5→1, 15→0, 10→2, 25→0, 35→0, 30→1, 20→3.

Delete node 15

Node 15 is a leaf. Remove it directly.

20 (BF=?) / \ 10 (BF=?) 30 (BF=0) / / \ 5(+1) 25(0) 35(0) / 3(0)

Update heights bottom-up:

  • Node 10: left height = 1 (node 5), right height = −1 (null) → height = 2, BF(10) = 1 − (−1) = +2 ⚠️ Unbalanced!

First Rotation at Node 10

BF(10) = +2 and BF(5) = +1 → Left-Left case → Right Rotation at 10.

20 (BF=?) / \ 5 (BF=-1) 30 (BF=0) / \ / \ 3(0) 10(0) 25(0) 35(0)

Node 5: left height = 0, right height = 0 → height = 1. But wait — the subtree rooted at 5 now has height 1. Continue updating upward.

  • Node 20: left height = 1 (node 5), right height = 1 (node 30) → height = 2, BF(20) = 1 − 1 = 0. ✓ Balanced.

In this case, only one rotation was needed. Let's construct a scenario with true cascading.

Cascading Rotation Example

Consider this AVL tree (9 nodes, height 4):

30 (BF=+1) / \ 15 (BF=+1) 40 (BF=0) / \ / \ 8(+1) 20(0) 35(0) 45(0) / 4(0) / 2(0)

Wait — that has BF(8) = +1 with child 4 having BF = +1 with child 2... let me verify: 2→h0, 4→h1 (BF=+1), 8→h2 (left=1, right=−1, BF=+2). That's not valid. Let me build a correct tree.

Correct 9-node AVL tree (height 3):

20 (h=3, BF=+1) / \ 8 (h=2, BF=0) 30 (h=1, BF=0) / \ / \ 4(h=1,+1) 12(h=0) 25(h=0) 35(h=0) / 2(h=0)

Verify: 2→h0; 4→h1 (left=0, right=−1, BF=+1); 12→h0; 8→h2 (left=1, right=0, BF=+1)... Wait, BF(8) = h(left)−h(right) = 1−0 = +1. Node 20: left=2, right=1, BF=+1. Valid AVL tree with 7 nodes. We need 9. Let me add two more:

20 (h=3, BF=+1) / \ 8 (h=2, BF=0) 30 (h=2, BF=0) / \ / \ 4(h=1,0) 12(h=1,0) 25(h=1,-1) 35(h=0) / \ / \ \ 2(0) 6(0) 10(0) 14(0) 28(0)

Count: 2,4,6,8,10,12,14,20,25,28,30,35 = 12 nodes. Too many. Let's use exactly 9:

20 (h=3, BF=+1) / \ 8 (h=2, BF=+1) 30 (h=1, BF=-1) / \ \ 4(h=1,0) 12(h=0) 35(h=0) / \ 2(0) 6(0)

Verify: 2→h0, 6→h0, 4→h1 (BF=0), 12→h0, 8→h2 (left=1, right=0, BF=+1), 35→h0, 30→h1 (left=−1, right=0, BF=−1), 20→h3 (left=2, right=1, BF=+1). Total nodes: 2,4,6,8,12,20,30,35 = 8. Need one more. Add node 25 as left child of 30:

20 (h=3, BF=+1) / \ 8 (h=2, BF=+1) 30 (h=1, BF=0) / \ / \ 4(h=1,0) 12(h=0) 25(h=0) 35(h=0) / \ 2(0) 6(0)

Nodes: 2,4,6,8,12,20,25,30,35 = 9. ✓ Heights: 2→0, 6→0, 4→1(BF=0), 12→0, 8→2(left=1,right=0,BF=+1), 25→0, 35→0, 30→1(BF=0), 20→3(left=2,right=1,BF=+1). ✓ Valid AVL.

Delete node 12

Node 12 is a leaf. Remove it.

20 (BF=?) / \ 8 (h=?, BF=?) 30 (h=1, BF=0) / / \ 4(h=1,0) 25(h=0) 35(h=0) / \ 2(0) 6(0)

Update heights from node 8 upward:

  • Node 8: left = h(4) = 1, right = h(null) = −1 → height = 2, BF = 1−(−1) = +2 ⚠️

BF(8) = +2, BF(4) = 0. This is the Left-Left case (left child's BF ≥ 0). Apply Right Rotation at 8.

20 (BF=?) / \ 4 (h=?, BF=?) 30 (h=1, BF=0) / \ / \ 2(h=0) 8(h=0) 25(h=0) 35(h=0) / 6(0)

Wait — when we right-rotate at 8 with pivot 4: 4 becomes the new root of this subtree, 4's right child (6) becomes 8's left child, and 8 becomes 4's right child.

20 (BF=?) / \ 4 (BF=?) 30 (h=1, BF=0) / \ / \ 2(h=0) 8(h=1,-1) 25(h=0) 35(h=0) / 6(h=0)

Heights: 6→0, 8→1(left=0,right=−1,BF=+1), 2→0, 4→2(left=0,right=1,BF=−1).

Continue upward to Node 20: left = h(4) = 2, right = h(30) = 1 → height = 3, BF = 2−1 = +1. ✓ Balanced.

In this case we got one rotation. For a true cascading example, we need a taller tree. Here's the key concept illustrated concisely:

Cascading Rotation Illustration When a rotation at node X reduces the height of X's subtree by 1, X's parent may become unbalanced (BF = ±2). This triggers another rotation at the parent, which may again reduce height, propagating further. In a tree of height h, up to ⌊h/2⌋ rotations can cascade — which is O(log n).

The principle: after each rotation during deletion, you must continue checking ancestors all the way to the root. Never stop early (unlike insertion, where you can stop after the first fix).

5. Complexity Summary

AVL Tree Operation Complexity
OperationStepsTime Complexity
SearchTraverse from root to target (or null)O(log n)
InsertBST insert + walk up + at most 1 rotationO(log n)
DeleteBST delete + walk up + up to O(log n) rotationsO(log n)
Why All O(log n)? The AVL balance condition guarantees the tree height is always O(log n). Specifically, an AVL tree of height h has at least F(h+3) − 1 nodes (where F(k) is the k-th Fibonacci number), which gives h < 1.44 × log₂(n+2). Every operation traverses at most one root-to-leaf path, and each rotation is O(1).

6. Pseudocode

// AVL Insert — returns new root of subtree Node avlInsert(Node node, int key) { // Step 1: Standard BST insert if (node == null) return new Node(key); if (key < node.key) node.left = avlInsert(node.left, key); else if (key > node.key) node.right = avlInsert(node.right, key); else return node; // duplicates not allowed // Step 2: Update height node.height = 1 + max(height(node.left), height(node.right)); // Step 3: Get balance factor int bf = getBalance(node); // Step 4: Rebalance if needed (4 cases) // Left-Left if (bf > 1 && key < node.left.key) return rightRotate(node); // Right-Right if (bf < -1 && key > node.right.key) return leftRotate(node); // Left-Right if (bf > 1 && key > node.left.key) { node.left = leftRotate(node.left); return rightRotate(node); } // Right-Left if (bf < -1 && key < node.right.key) { node.right = rightRotate(node.right); return leftRotate(node); } return node; // balanced, no rotation needed }
// AVL Delete — returns new root of subtree Node avlDelete(Node node, int key) { // Step 1: Standard BST delete if (node == null) return null; if (key < node.key) node.left = avlDelete(node.left, key); else if (key > node.key) node.right = avlDelete(node.right, key); else { // Node found — handle 3 cases if (node.left == null || node.right == null) { node = (node.left != null) ? node.left : node.right; if (node == null) return null; } else { // Two children: get in-order successor Node succ = minValueNode(node.right); node.key = succ.key; node.right = avlDelete(node.right, succ.key); } } // Step 2: Update height node.height = 1 + max(height(node.left), height(node.right)); // Step 3: Get balance factor int bf = getBalance(node); // Step 4: Rebalance — NOTE: do NOT return early; // recursion handles all ancestors automatically // Left-Left if (bf > 1 && getBalance(node.left) >= 0) return rightRotate(node); // Left-Right if (bf > 1 && getBalance(node.left) < 0) { node.left = leftRotate(node.left); return rightRotate(node); } // Right-Right if (bf < -1 && getBalance(node.right) <= 0) return leftRotate(node); // Right-Left if (bf < -1 && getBalance(node.right) > 0) { node.right = rightRotate(node.right); return leftRotate(node); } return node; }
Key difference in case detection: In avlInsert, we compare the inserted key to determine the case. In avlDelete, we check the balance factor of the child instead, because the imbalance could come from any deletion below.
