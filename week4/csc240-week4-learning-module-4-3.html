ü§î Why Do We Need Predicate Logic?

The Limitations of Propositional Logic

Propositional logic treats entire statements as atomic units. Let's see where this breaks down:

Example: Trying to Express "All birds can fly"

In propositional logic:

Let P = "All birds can fly"

Problems:

  • We can't talk about individual birds
  • We can't express the relationship between being a bird and being able to fly
  • We can't handle exceptions (penguins can't fly!)
  • We can't reason about specific birds vs. all birds

‚ö†Ô∏è What Propositional Logic Cannot Express

  1. Properties of objects: "x is even," "y is prime"
  2. Relationships between objects: "x is greater than y," "a is the parent of b"
  3. Universal statements: "All integers are rational"
  4. Existential statements: "There exists a prime number greater than 100"
  5. Patterns and structure: "Every number has a successor"

The Power of Predicate Logic

Predicate logic solves these problems by introducing:

Propositional Logic

  • Fixed propositions (P, Q, R)
  • Each has a single truth value
  • Can't talk about objects
  • Limited expressiveness
P: "Alice is a student" Q: "Bob is a student" R: "Carol is a student"

Need separate proposition for each person!

Predicate Logic

  • Predicates with variables
  • Truth value depends on object
  • Can talk about any object
  • Much more powerful
Student(x): "x is a student" Student(Alice) ‚Üí true Student(Bob) ‚Üí true Student(Carol) ‚Üí true

One predicate handles all people!

Real-World Scenario: Student Database

Propositional Logic Approach (inefficient):

P‚ÇÅ: "Student 1 is enrolled in CSC240" P‚ÇÇ: "Student 2 is enrolled in CSC240" P‚ÇÉ: "Student 3 is enrolled in CSC240" ... P‚ÇÅ‚ÇÄ‚ÇÄ: "Student 100 is enrolled in CSC240"

Need 100 different propositions! Can't easily add new students or courses.

Predicate Logic Approach (elegant):

Enrolled(x, c): "Student x is enrolled in course c" Enrolled(Alice, CSC240) ‚Üí true Enrolled(Bob, MTH80) ‚Üí true Enrolled(Alice, MTH80) ‚Üí true

One predicate handles all students and all courses! Easy to add more.

Key Insight

The Fundamental Difference

Propositional Logic: "This specific thing is true" (Alice is a student) Predicate Logic: "This property holds for this object" Student(Alice) = true The predicate Student(x) can be applied to ANY object x!

üìö Core Concepts

What is a Predicate?

Definition: A predicate is a function that takes one or more arguments and returns a truth value (true or false).

Notation: $$ P(x) $$, $$ Q(x, y) $$, $$ R(x, y, z) $$

Think of it as: A property or relationship that can be true or false depending on the values you plug in

Anatomy of a Predicate

Components of Predicate Logic

1. Variables - Placeholders for objects ‚ñº

Variables represent objects from the domain. Common conventions:

  • Lowercase letters: x, y, z, a, b, c (for objects)
  • Can represent: numbers, people, courses, strings, any object

Examples:

x = 5 (x is a number) y = "Alice" (y is a person) c = CSC240 (c is a course)

Free vs. Bound Variables

Free variable: Not associated with a quantifier (we'll cover this in Module 4.4)

P(x) ‚Üê x is free (value not specified)

Bound variable: Associated with a quantifier ‚àÄ or ‚àÉ

‚àÄx P(x) ‚Üê x is bound (ranges over all values)

For now, we'll work with free variables. Quantifiers come in Module 4.4!

2. Predicates - Properties and relationships ‚ñº

Predicates express properties or relationships. Convention:

  • Capitalized names: Student, Prime, GreaterThan
  • Number of arguments: Can have 1, 2, 3, or more variables

Unary Predicates (1 argument)

Express a property of a single object:

Predicate Meaning Example
$$ \text{Even}(x) $$ x is an even number Even(4) ‚Üí true, Even(5) ‚Üí false
$$ \text{Prime}(x) $$ x is a prime number Prime(7) ‚Üí true, Prime(8) ‚Üí false
$$ \text{Student}(x) $$ x is a student Student(Alice) ‚Üí true

Binary Predicates (2 arguments)

Express a relationship between two objects:

Predicate Meaning Example
$$ \text{GreaterThan}(x, y) $$ x is greater than y GreaterThan(5, 3) ‚Üí true
$$ \text{Enrolled}(x, c) $$ Student x is enrolled in course c Enrolled(Alice, CSC240) ‚Üí true
$$ \text{ParentOf}(x, y) $$ x is a parent of y ParentOf(John, Mary) ‚Üí true

N-ary Predicates (3+ arguments)

Express relationships among multiple objects:

Between(x, y, z): "y is between x and z" Between(1, 5, 10) ‚Üí true Teaches(professor, course, semester): "Professor teaches course in semester" Teaches(Smith, CSC240, Spring2026) ‚Üí true
3. Domain of Discourse - The universe of objects ‚ñº

Domain of discourse (or universe) is the set of all possible values that variables can take.

Why Domains Matter

The same predicate can have different truth values depending on the domain!

Common Domains

Symbol Domain Description
‚Ñï Natural numbers {0, 1, 2, 3, ...}
‚Ñ§ Integers {..., -2, -1, 0, 1, 2, ...}
‚Ñö Rational numbers All fractions p/q
‚Ñù Real numbers All numbers on number line
Custom Any set {Students}, {Courses}, {People}

Example: Domain Changes Meaning

Predicate: Even(x) means "x is even" Domain = ‚Ñï (natural numbers): Even(4) ‚Üí true Even(3) ‚Üí false Domain = {Students}: Even(Alice) ‚Üí undefined! (Alice is not a number) Domain matters for determining what makes sense!

‚ö†Ô∏è Always Specify the Domain

When working with predicates, always be clear about what domain you're using. The statement "all x satisfy P(x)" means very different things if x ranges over integers vs. real numbers!

4. Propositional Functions - Predicates become propositions ‚ñº

A propositional function is a predicate with free variables. When you substitute specific values for the variables, it becomes a proposition (with a definite truth value).

The Process

From Predicate to Proposition

Step 1: Start with predicate P(x): "x is even" ‚Üì (x is a variable - no truth value yet) Step 2: Substitute a value P(4): "4 is even" ‚Üì (now it's a proposition) Step 3: Evaluate P(4) = true ‚úì (definite truth value)

Examples

Propositional Function Substitution Resulting Proposition Truth Value
$$ x > 5 $$ x = 7 7 > 5 True
$$ x > 5 $$ x = 3 3 > 5 False
$$ \text{Student}(x) $$ x = Alice Student(Alice) True (if Alice is a student)
$$ x^2 = y $$ x = 3, y = 9 3¬≤ = 9 True

Combining Predicates with Logical Operators

Just like propositions, predicates can be combined using logical operators:

Logical Operators with Predicates

Operator Example Meaning
AND ($$ \land $$) $$ \text{Student}(x) \land \text{CSMajor}(x) $$ x is a student AND x is a CS major
OR ($$ \lor $$) $$ \text{Even}(x) \lor \text{Prime}(x) $$ x is even OR x is prime
NOT ($$ \neg $$) $$ \neg \text{Even}(x) $$ x is NOT even (i.e., x is odd)
IMPLIES ($$ \to $$) $$ \text{Student}(x) \to \text{HasID}(x) $$ If x is a student, then x has an ID
IFF ($$ \leftrightarrow $$) $$ \text{Even}(x) \leftrightarrow \text{Divisible}(x, 2) $$ x is even if and only if x is divisible by 2

Key Takeaways

  • Predicates are functions that return true/false
  • Variables represent objects from the domain
  • Domain determines what objects we're talking about
  • Propositional functions become propositions when variables are assigned values
  • Logical operators work the same way with predicates as with propositions

üíª Predicate Logic in Programming

Preconditions and Postconditions

Predicates are essential for formal specification of functions:

/** * Precondition: ‚àÄi (0 ‚â§ i < arr.length ‚Üí arr[i] ‚â• 0) * "All array elements are non-negative" * * Postcondition: ‚àÉi (0 ‚â§ i < arr.length ‚àß result = arr[i] ‚àß * ‚àÄj (0 ‚â§ j < arr.length ‚Üí arr[i] ‚â• arr[j])) * "Result is an element in the array and it's the maximum" */ public int findMax(int[] arr) { // Precondition check (in practice) assert arr.length > 0 : "Array must not be empty"; int max = arr[0]; for (int i = 1; i < arr.length; i++) { if (arr[i] > max) { max = arr[i]; } } return max; }

Loop Invariants

Predicates express properties that remain true throughout loop execution:

/** * Loop Invariant: ‚àÄj (0 ‚â§ j < i ‚Üí arr[j] ‚â§ arr[i]) * "All elements before index i are ‚â§ element at i" */ public void selectionSort(int[] arr) { for (int i = 0; i < arr.length - 1; i++) { // Invariant holds here: arr[0..i-1] is sorted int minIndex = i; for (int j = i + 1; j < arr.length; j++) { if (arr[j] < arr[minIndex]) { minIndex = j; } } // Swap int temp = arr[i]; arr[i] = arr[minIndex]; arr[minIndex] = temp; // Invariant still holds: arr[0..i] is now sorted } }

Conditional Logic as Predicates

Every if-statement is a predicate!

// Define predicates IsPositive(x) ‚â° (x > 0) IsEven(x) ‚â° (x % 2 == 0) InRange(x, min, max) ‚â° (x ‚â• min ‚àß x ‚â§ max) // Use in code if (x > 0) { // IsPositive(x) System.out.println("Positive"); } if (x % 2 == 0) { // IsEven(x) System.out.println("Even"); } if (x >= min && x <= max) { // InRange(x, min, max) System.out.println("In range"); }

Database Queries

SQL is built on predicate logic:
