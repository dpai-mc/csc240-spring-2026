Discrete Random Variables and Expected Value

Week 9 of 17  (March 16-22, 2026) Estimated time: 1.5 hours CSLO 2 · Objective 6

1. From Events to Numbers

In Modules 9.1 through 9.3, every question we asked had a yes-or-no character: Did event $$E$$ occur? Did the card drawn happen to be a heart? Did the test come back positive? We assigned probabilities to those yes-or-no outcomes and built a rich toolkit for reasoning about them.

Now we shift the question. Instead of asking "did something happen?" we ask: "what number did the experiment produce?"

Roll a fair six-sided die. Instead of asking "did we roll a 4?", ask: "what is the value of the roll?" The answer — 1, 2, 3, 4, 5, or 6 — is a number, not just an event.

That number-valued result is what we call a random variable. It maps every outcome in the sample space to a real number, letting us ask richer questions: What is the average value we expect over many rolls? How much does the result typically vary from that average? These questions are the subject of this module.

🔗 Connection — Modules 9.1 & 9.2
A random variable is built on top of the probability space $$(\Omega, 2^\Omega, P)$$ you defined in Module 9.1. Every probability statement about a random variable — like $$P(X = k)$$ — is still just a probability of an event in that space. Nothing from the earlier modules is discarded; we are adding a layer.

2. Discrete Random Variables

📘 Definition — Discrete Random Variable

A discrete random variable $$X$$ is a function $$X : \Omega \to \mathbb{R}$$ that assigns a real number to every outcome in the sample space $$\Omega$$, where the set of possible values $$\{X(\omega) : \omega \in \Omega\}$$ is finite or countably infinite.

We write $$X = k$$ as shorthand for the event $$\{\omega \in \Omega : X(\omega) = k\}$$ — the set of all outcomes that produce the value $$k$$.

The Die Roll as a Random Variable

Roll one fair six-sided die. The sample space is $$\Omega = \{1, 2, 3, 4, 5, 6\}$$. Define $$X(\omega) = \omega$$ — that is, the random variable simply equals the face value shown. Then:

  • $$X = 3$$ is the event $$\{3\}$$, and $$P(X = 3) = \tfrac{1}{6}$$.
  • $$X \leq 2$$ is the event $$\{1, 2\}$$, and $$P(X \leq 2) = \tfrac{2}{6} = \tfrac{1}{3}$$.

The random variable $$X$$ is not a single number — it is a function. Its value is uncertain until the die is rolled; before that, we can only describe the probabilities of each possible value.

⚠️ Notation Alert
By convention, random variables are written with capital letters ($$X$$, $$Y$$, $$Z$$) and their specific realized values with lowercase letters ($$k$$, $$x$$, $$y$$). So $$P(X = k)$$ means "the probability that the random variable $$X$$ takes the value $$k$$."

3. Probability Mass Functions

📘 Definition — Probability Mass Function (PMF)

The probability mass function of a discrete random variable $$X$$ is the function $$p : \mathbb{R} \to [0,1]$$ defined by $$p(k) = P(X = k)$$ for each possible value $$k$$. A valid PMF must satisfy two requirements:

  • Non-negativity: $$p(k) \geq 0$$ for all $$k$$.
  • Normalization: $$\displaystyle\sum_{\text{all } k} p(k) = 1$$.

4. Expected Value

📘 Definition — Expected Value

The expected value (or expectation) of a discrete random variable $$X$$ is the probability-weighted average of its possible values: $$E[X] = \sum_{\text{all } k} k \cdot P(X = k)$$

Plain-language interpretation: If you repeat the random experiment a very large number of times and average the results, that average will get closer and closer to $$E[X]$$. It is the long-run average value of the experiment.

⚠️ Expected Value Need Not Be a Possible Outcome
$$E[X]$$ is a mathematical average — it does not have to be a value that $$X$$ can actually produce. The classic example is the die roll, where $$E[X] = 3.5$$, but no face of the die shows 3.5.
▶ Worked Example 1 — Expected Value of a Fair Die Roll

Setup: Let $$X$$ be the result of rolling one fair six-sided die. Compute $$E[X]$$.

  • Write the PMF. Each value $$k \in \{1,2,3,4,5,6\}$$ has $$P(X=k) = \tfrac{1}{6}$$.
  • Apply the expected value formula. $$E[X] = \sum_{k=1}^{6} k \cdot \frac{1}{6} = \frac{1}{6}(1 + 2 + 3 + 4 + 5 + 6) = \frac{21}{6} = 3.5$$
  • Interpret. If you roll the die thousands of times and compute the running average, it will converge to $$3.5$$. No single roll ever equals $$3.5$$ — the expected value is a property of the distribution, not a property of any single outcome.

Takeaway: For a uniform distribution over $$\{1, 2, \ldots, n\}$$, the expected value is always the midpoint $$\tfrac{n+1}{2}$$. For $$n=6$$: $$\tfrac{7}{2} = 3.5$$. ✓

▶ Worked Example 2 — Expected Comparisons in a Linear Search

Setup: You have an unsorted list of $$n$$ distinct elements. You search for a target that is guaranteed to be in the list, and each position is equally likely to hold the target. Let $$X$$ be the number of comparisons made before the target is found. Compute $$E[X]$$.

  • Identify the sample space and PMF. The target is equally likely to be at any of the $$n$$ positions. If the target is at position $$k$$, exactly $$k$$ comparisons are made. So $$P(X = k) = \tfrac{1}{n}$$ for $$k = 1, 2, \ldots, n$$.
  • Apply the expected value formula. $$E[X] = \sum_{k=1}^{n} k \cdot \frac{1}{n} = \frac{1}{n} \sum_{k=1}^{n} k = \frac{1}{n} \cdot \frac{n(n+1)}{2} = \frac{n+1}{2}$$
  • Interpret for a concrete case. For $$n = 100$$ elements: $$E[X] = \tfrac{101}{2} = 50.5$$. On average, a linear search examines about half the list. This is the formal justification for the informal claim that linear search is $$O(n)$$ on average.

Takeaway: Expected value connects probability directly to algorithm analysis. The formula $$E[X] = \tfrac{n+1}{2}$$ gives the average-case cost of linear search in a single, clean expression.

🔗 Connection — Algorithm Analysis
Average-case complexity is defined as $$E[X]$$ where $$X$$ is the number of operations under a probability distribution over inputs. This exact calculation is what computer scientists mean when they say linear search has average-case complexity $$\Theta(n)$$.

5. Linearity of Expectation

📘 Definition — Linearity of Expectation

For any two discrete random variables $$X$$ and $$Y$$ defined on the same probability space, and any constants $$a, b \in \mathbb{R}$$: $$E[X + Y] = E[X] + E[Y]$$ $$E[aX + b] = a\,E[X] + b$$ This holds regardless of whether $$X$$ and $$Y$$ are independent.

Why Is This Powerful?

Many random quantities that look complicated can be written as sums of simpler random variables. Linearity of expectation lets you compute the expectation of the sum by summing the individual expectations — no need to figure out the joint distribution of the parts. This technique is called the indicator random variable method.

📘 Definition — Indicator Random Variable

An indicator random variable $$I_E$$ for an event $$E$$ is: $$I_E = \begin{cases} 1 & \text{if event } E \text{ occurs} \\ 0 & \text{otherwise} \end{cases}$$

Its expected value is simply the probability of the event: $$E[I_E] = 1 \cdot P(E) + 0 \cdot P(E^c) = P(E)$$

▶ Worked Example 3 — Expected Number of Heads in 10 Coin Flips

Setup: Flip a fair coin 10 times. Let $$X$$ be the total number of Heads. Compute $$E[X]$$ using indicator random variables.

  • Define indicator variables. For each flip $$i = 1, 2, \ldots, 10$$, define: $$I_i = \begin{cases} 1 & \text{if flip } i \text{ is Heads} \\ 0 & \text{otherwise} \end{cases}$$ Then the total number of Heads is $$X = I_1 + I_2 + \cdots + I_{10}$$.
  • Compute each indicator's expectation. Since the coin is fair, $$P(\text{flip } i = \text{Heads}) = \tfrac{1}{2}$$, so $$E[I_i] = \tfrac{1}{2}$$ for every $$i$$.
  • Apply linearity of expectation. $$E[X] = E[I_1 + I_2 + \cdots + I_{10}] = E[I_1] + E[I_2] + \cdots + E[I_{10}] = 10 \times \frac{1}{2} = 5$$
  • Interpret. On average, 5 out of 10 flips will be Heads. This matches intuition — and we obtained it without ever writing down the full binomial distribution.

Takeaway: The indicator method works even when the $$I_i$$ are not independent. For independent fair coin flips they happen to be independent, but linearity of expectation does not require it. This makes the technique applicable to much harder problems where independence fails.

🔗 Generalization — Binomial Distribution
For $$n$$ independent trials each with success probability $$p$$, the same indicator argument gives $$E[X] = np$$. For our coin flip: $$n = 10$$, $$p = \tfrac{1}{2}$$, so $$E[X] = 5$$. The binomial expected value formula is just linearity of expectation applied systematically.

6. Variance — Conceptual Introduction

📌 Scope Note — Not Assessed
Variance is introduced here for conceptual completeness only. It is NOT assessed on Quiz 3 or the Midterm Exam. You are not required to compute variance for any graded work in Week 9. Read this section to build intuition for Module 9.5 and future courses.

Expected value tells us the center of a distribution — the long-run average. But two distributions can have the same expected value while behaving very differently. Consider these two scenarios:

  • Scenario A: You always win exactly \$5. Here $$E[X] = 5$$ and the outcome never deviates from the average.
  • Scenario B: You win \$0 or \$10 with equal probability. Here $$E[X] = 5$$ as well, but the outcome swings wildly.

Variance measures this spread — how far values typically deviate from the expected value.

📘 Definition — Variance (Informational)

The variance of a discrete random variable $$X$$ is: $$\operatorname{Var}(X) = E[X^2] - (E[X])^2$$ where $$E[X^2] = \displaystyle\sum_k k^2 \cdot P(X = k)$$.

The standard deviation is $$\sigma = \sqrt{\operatorname{Var}(X)}$$, which has the same units as $$X$$ and is easier to interpret directly. A small variance means outcomes cluster tightly around $$E[X]$$; a large variance means outcomes spread far from the average.

For Scenario A above: $$\operatorname{Var}(X) = 0$$ (no spread). For Scenario B: $$\operatorname{Var}(X) = E[X^2] - 25 = 50 - 25 = 25$$, so $$\sigma = 5$$. The standard deviation of 5 reflects that outcomes deviate by \$5 from the mean on every trial.

7. The Law of Large Numbers

Plain-Language Statement

As the number of independent repetitions of a random experiment grows, the sample mean (running average of observed values) converges to the expected value $$E[X]$$. More repetitions → more reliable average.

This is why casinos are profitable despite randomness, why insurance companies can price policies, and why Monte Carlo simulations work: given enough trials, the empirical average is an accurate estimate of the theoretical expectation.

🔗 Connection — Module 9.2 Independence
The Law of Large Numbers requires that the repeated trials be independent — the result of one roll does not influence the next. The formal definition of independence from Module 9.2 ($$P(A \cap B) = P(A) \cdot P(B)$$) is the mathematical foundation that makes this convergence guarantee valid.

If you cannot compute $$E[X]$$ analytically, you can estimate it by running thousands of simulated trials and averaging the results. 9. Module Summary

📋 Module 9.4 — Key Takeaways
  • A discrete random variable $$X : \Omega \to \mathbb{R}$$ assigns a numerical value to each outcome. Its probability mass function $$P(X = k)$$ must be non-negative and sum to 1. The expected value $$E[X] = \sum_k k \cdot P(X = k)$$ is the long-run average — it need not be a possible outcome (e.g., $$E[\text{die}] = 3.5$$).
  • Linearity of expectation $$E[X + Y] = E[X] + E[Y]$$ holds regardless of independence, making the indicator random variable method a powerful shortcut: decompose a complex count into a sum of 0/1 indicators, then sum their individual expectations.
  • The Law of Large Numbers guarantees that the sample mean of independent trials converges to $$E[X]$$ as the number of trials grows — the theoretical foundation for Monte Carlo simulation in Module 9.5. Variance (not assessed) measures how spread out the distribution is around its expected value.
