Opening: From Counting to Chance

You have spent the last two weeks building a powerful counting toolkit — permutations, combinations, and the inclusion-exclusion principle. You can answer questions like "how many 5-card hands contain exactly 3 hearts?" with confidence. Now we ask a subtly different question:

"If I draw 5 cards at random, what is the probability that exactly 3 are hearts?"

The shift from counting to probability is smaller than it looks. Probability is just a ratio of counts — provided every outcome is equally likely. The heavy lifting of computing those counts is exactly what combinations were designed for.

🔗 Connection — Weeks 7 & 8
The counting tools you built in Weeks 7–8 (combinations, permutations, inclusion-exclusion) are the engine inside probability calculations. This module makes that connection explicit and formal.

Motivating Scenario: The Software Quality Engineer

Imagine you are a software quality engineer at a company that ships firmware updates. A batch of 20 devices has just arrived for final testing. Unknown to you, 8 of the 20 are defective (they passed earlier screening but have a subtle hardware fault). Your testing budget allows you to randomly select and test exactly 5 devices.

Your manager asks: "What is the probability that your sample of 5 contains exactly 2 defective devices and 3 non-defective ones?"

You already know how to count the favorable outcomes using combinations. By the end of this module, you will have the formal probability framework to turn that count into a precise probability. We will return to this scenario as our final worked example.

Why a Formal Framework?

Informal reasoning about chance leads to well-known paradoxes and errors (we will see three in the Watch Out section). A formal framework gives us three things:

  1. Precision — unambiguous definitions prevent hidden assumptions.
  2. Derivability — all probability rules follow from three axioms.
  3. Generality — the same framework covers coins, cards, software bugs, network packets, and algorithm runtimes.

The framework has three components: a sample space, events, and a probability measure. Each is a concept you have already seen in set theory — we are simply adding a measurement function on top.

Core Definitions

📘 Definition — Sample Space

The sample space $$\Omega$$ (capital omega) is the set of all possible outcomes of a random experiment. Each element $$\omega \in \Omega$$ is called an outcome (or elementary event).

Examples:
• One coin flip: $$\Omega = \{H, T\}$$
• Two coin flips: $$\Omega = \{HH,\, HT,\, TH,\, TT\}$$
• Drawing 1 card from a standard deck: $$|\Omega| = 52$$

📘 Definition — Event

An event is any subset $$E \subseteq \Omega$$. We say the event occurs when the observed outcome $$\omega \in E$$.

Special cases:
• $$\Omega$$ itself is the certain event (always occurs).
• $$\emptyset$$ is the impossible event (never occurs).
• A single outcome $$\{\omega\}$$ is called an elementary event.

📘 Definition — Probability Measure

A probability measure is a function $$P : 2^{\Omega} \to \mathbb{R}$$ (from the power set of $$\Omega$$ to the real numbers) satisfying the three Kolmogorov axioms (see next section).

The triple $$(\Omega,\, 2^{\Omega},\, P)$$ is called a finite probability space.

📘 Definition — Uniform Probability Space

When every outcome in $$\Omega$$ is equally likely, the space is called uniform and the probability of any event $$E$$ is:

$$ P(E) = \frac{|E|}{|\Omega|} $$

This is the formula you will use most often in this course. Computing $$|E|$$ and $$|\Omega|$$ is a counting problem — exactly what Weeks 7–8 prepared you for.

The Three Kolmogorov Axioms

Every valid probability measure must satisfy exactly three axioms. All other probability rules — complement, addition, conditional probability — are theorems derived from these three.

  • Axiom 1 (Non-negativity): For every event $$E \subseteq \Omega$$, $$P(E) \geq 0$$
  • Axiom 2 (Normalization): The probability of the entire sample space is 1: $$P(\Omega) = 1$$
  • Axiom 3 (Countable Additivity): If $$A$$ and $$B$$ are mutually exclusive events ($$A \cap B = \emptyset$$), then: $$P(A \cup B) = P(A) + P(B)$$ More generally, for any finite collection of pairwise mutually exclusive events $$A_1, A_2, \ldots, A_n$$: $$P(A_1 \cup A_2 \cup \cdots \cup A_n) = \sum_{i=1}^{n} P(A_i)$$
⚠️ Axiom Check
Any function that satisfies all three axioms is a valid probability measure — even if it feels unintuitive. Conversely, if a function violates any one axiom (e.g., assigns a negative value, or sums to more than 1), it is not a probability measure, no matter how reasonable it seems.

Immediate Consequences of the Axioms

Two important results follow directly:

  • Impossible event: $$P(\emptyset) = 0$$
    Proof sketch: $$\Omega$$ and $$\emptyset$$ are disjoint and $$\Omega \cup \emptyset = \Omega$$, so by Axiom 3: $$P(\Omega) = P(\Omega) + P(\emptyset)$$, giving $$P(\emptyset)=0$$.
  • Bounded: $$0 \leq P(E) \leq 1$$ for all $$E$$
    Proof sketch: $$E$$ and $$E^c$$ are disjoint and cover $$\Omega$$, so $$P(E) + P(E^c) = 1$$. Since $$P(E^c) \geq 0$$, we get $$P(E) \leq 1$$.

Core Rules — Probability as Reframed Counting

The two rules below are not new ideas. They are the counting principles from Week 7 expressed as probability statements. The table below makes this parallel explicit.

Rule Week 7 Counting Version Week 9 Probability Version
Complement $$|A^c| = |\Omega| - |A|$$ $$P(A^c) = 1 - P(A)$$
Addition (general) $$|A \cup B| = |A| + |B| - |A \cap B|$$
(Inclusion-Exclusion)		 $$P(A \cup B) = P(A) + P(B) - P(A \cap B)$$
Addition (mutually exclusive) If $$A \cap B = \emptyset$$:
$$|A \cup B| = |A| + |B|$$		 If $$A \cap B = \emptyset$$:
$$P(A \cup B) = P(A) + P(B)$$

Derivation: Complement Rule

Since $$E$$ and $$E^c$$ partition $$\Omega$$ (they are disjoint and $$E \cup E^c = \Omega$$), Axiom 3 gives:

$$ P(E) + P(E^c) = P(\Omega) = 1 \quad \Longrightarrow \quad P(E^c) = 1 - P(E) $$

Derivation: General Addition Rule

Write $$A \cup B$$ as a union of three disjoint pieces: $$A \setminus B$$, $$A \cap B$$, and $$B \setminus A$$. Applying Axiom 3 and rearranging yields:

$$ P(A \cup B) = P(A) + P(B) - P(A \cap B) $$

This is inclusion-exclusion divided through by $$|\Omega|$$. The structure is identical; only the notation changes.

🔗 Connection — Week 7 Inclusion-Exclusion
In Week 7 you proved $$|A \cup B| = |A| + |B| - |A \cap B|$$ using set-theoretic arguments. The general addition rule for probability is that same identity divided by $$|\Omega|$$. If you understand one, you understand the other.

Worked Examples

▶ Example 1 — Two Coin Flips and the Complement Rule

Setup: Flip a fair coin twice. Let $$A$$ = "at least one Head appears." Find $$P(A)$$.

  • Write the sample space. Each flip is H or T, so by the multiplication principle: $$|\Omega| = 2 \times 2 = 4$$. $$\Omega = \{HH,\; HT,\; TH,\; TT\}$$ Since the coin is fair, this is a uniform space.
  • Identify the complement. "At least one Head" is awkward to count directly. Its complement $$A^c$$ = "no Heads" = "both Tails" = $$\{TT\}$$. $$|A^c| = 1 \quad \Rightarrow \quad P(A^c) = \frac{1}{4}$$
  • Apply the complement rule. $$P(A) = 1 - P(A^c) = 1 - \frac{1}{4} = \frac{3}{4}$$
  • Verify directly. $$A = \{HH, HT, TH\}$$, so $$|A|=3$$ and $$P(A) = \tfrac{3}{4}$$. ✓

Takeaway: The complement rule is most useful when "at least one" or "not all" phrasing makes the complement smaller and easier to count than the event itself.

▶ Example 2 — Card Draw and the Addition Rule

Setup: Draw one card at random from a standard 52-card deck. Let $$A$$ = "card is a Heart" and $$B$$ = "card is a Face card (Jack, Queen, or King)." Find $$P(A \cup B)$$.

  • Identify the sample space. $$|\Omega| = 52$$. Uniform space (each card equally likely).
  • Count each event. There are 13 Hearts: $$|A| = 13$$, so $$P(A) = \tfrac{13}{52} = \tfrac{1}{4}$$. There are 12 Face cards (3 per suit × 4 suits): $$|B| = 12$$, so $$P(B) = \tfrac{12}{52} = \tfrac{3}{13}$$.
  • Count the intersection. Heart Face cards: Jack♥, Queen♥, King♥ → $$|A \cap B| = 3$$, so $$P(A \cap B) = \tfrac{3}{52}$$.
  • Apply the addition rule. $$P(A \cup B) = \frac{13}{52} + \frac{12}{52} - \frac{3}{52} = \frac{22}{52} = \frac{11}{26} \approx 0.423$$

Takeaway: Without subtracting $$P(A \cap B)$$, the three Heart Face cards would be double-counted — exactly the inclusion-exclusion correction from Week 7.

▶ Example 3 — Software Quality Engineer (Payoff Scenario)

Setup (revisited): A batch of 20 devices contains 8 defective and 12 non-defective units. You randomly select 5 devices to test. What is the probability that exactly 2 are defective and exactly 3 are non-defective?

  • Define the sample space. Choosing 5 devices from 20 without regard to order: $$|\Omega| = \binom{20}{5} = \frac{20!}{5!\,15!} = 15{,}504$$ Each combination is equally likely → uniform space.
  • Count the favorable outcomes. We need exactly 2 defective (chosen from 8) AND exactly 3 non-defective (chosen from 12). These are independent choices, so by the multiplication principle: $$|E| = \binom{8}{2} \cdot \binom{12}{3}$$ Computing each factor: $$\binom{8}{2} = \frac{8!}{2!\,6!} = 28 \qquad \binom{12}{3} = \frac{12!}{3!\,9!} = 220$$ $$|E| = 28 \times 220 = 6{,}160$$
  • Apply the uniform probability formula. $$P(E) = \frac{|E|}{|\Omega|} = \frac{6{,}160}{15{,}504} = \frac{385}{969} \approx 0.397$$
  • Interpret the result. There is approximately a 39.7% chance that your 5-device sample contains exactly 2 defective units. This type of calculation — sampling without replacement from a population with two types — is called a hypergeometric probability, and it arises constantly in software testing, quality control, and statistical auditing.

Takeaway: The probability framework added nothing computationally new — we used the same $$\binom{n}{k}$$ formula from Week 8. What it added was a precise interpretation: the ratio of favorable to total equally-likely outcomes.

Common Misconceptions

⚠️ Watch Out — Three Classic Errors

1. Forgetting that probabilities must sum to 1.

A valid probability measure over a finite sample space requires that the probabilities of all elementary events sum to exactly 1. If you assign $$P(\{H\}) = 0.6$$ and $$P(\{T\}) = 0.6$$ to a coin flip, you have violated Axiom 2. Always verify: $$\sum_{\omega \in \Omega} P(\{\omega\}) = 1$$.

2. Using the addition rule without subtracting the intersection.

A very common error: $$P(A \cup B) = P(A) + P(B)$$ is only correct when $$A$$ and $$B$$ are mutually exclusive. In general, $$P(A \cup B) = P(A) + P(B) - P(A \cap B)$$. Skipping the subtraction double-counts outcomes in $$A \cap B$$ and can produce a value greater than 1 — an immediate sign something went wrong.

3. Confusing "equally likely outcomes" with "equally likely events."

The formula $$P(E) = |E|/|\Omega|$$ requires that individual outcomes are equally likely — not events. For two coin flips, "exactly one Head" has probability $$\tfrac{2}{4}$$, not $$\tfrac{1}{3}$$, because the sample space has 4 equally likely outcomes, not 3 events. Listing the full $$\Omega$$ before computing probabilities guards against this error.

Module Summary

📋 Module 9.1 — Key Takeaways
  • A finite probability space $$(\Omega, 2^\Omega, P)$$ consists of a sample space, events as subsets, and a probability measure satisfying three axioms (non-negativity, normalization, additivity). In a uniform space, $$P(E) = |E|/|\Omega|$$ — and computing $$|E|$$ is a counting problem from Weeks 7–8.
  • The complement rule $$P(E^c) = 1 - P(E)$$ and the general addition rule $$P(A \cup B) = P(A) + P(B) - P(A \cap B)$$ are probability restatements of the set-cardinality identities you already know from inclusion-exclusion.
  • The three most common errors — probabilities not summing to 1, omitting the intersection in the addition rule, and confusing equally-likely outcomes with equally-likely events — can all be caught by returning to the formal definitions and listing $$\Omega$$ explicitly.
