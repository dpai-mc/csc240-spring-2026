🔄 The Induction-Recursion Parallel

Mathematical induction and recursive programming are two sides of the same coin. They share the same logical structure, just applied in different contexts.

The Structural Correspondence

Base Case: Prove P(n₀)
↔
Base Case: Return value for smallest input
Inductive Hypothesis: Assume P(k)
↔
Recursive Call: Trust f(k) is correct
Inductive Step: Prove P(k+1) using P(k)
↔
Recursive Case: Compute f(k+1) using f(k)
Conclusion: P(n) true for all n ≥ n₀
↔
Result: f(n) correct for all n ≥ n₀

Side-by-Side Example: Factorial

Induction Proof

Claim: n! = n × (n-1) × ... × 1 for all n ≥ 0, with 0! = 1.

Base Case (n = 0):
0! = 1 by definition. ✓

Inductive Hypothesis:
Assume k! = k × (k-1) × ... × 1 for some k ≥ 0.

Inductive Step:
(k+1)! = (k+1) × k!
= (k+1) × k × (k-1) × ... × 1 [by IH]
✓

Recursive Function

public static long factorial(int n) {
    // Base case
    if (n == 0) {
        return 1;
    }
    
    // Recursive case:
    // Trust factorial(n-1) is correct
    // Use it to compute factorial(n)
    return n * factorial(n - 1);
}

💡 The Key Insight

When you write a recursive function:

  • The base case handles the smallest input directly—no recursion needed
  • The recursive call is like the inductive hypothesis—you trust it gives the right answer for smaller inputs
  • The recursive case builds the answer for the current input using the trusted smaller result

This is exactly the structure of an induction proof!

Why Can We "Trust" the Recursive Call?

Students often wonder: "How can we trust that the recursive call works? Aren't we assuming what we're trying to prove?"

The Answer: Well-Founded Recursion

The recursion is valid because:

  1. The base case terminates: For n = 0, we return immediately without recursion
  2. Each recursive call is on a smaller input: factorial(n) calls factorial(n-1), where n-1 < n
  3. We eventually reach the base case: Starting from any n ≥ 0, repeated subtraction of 1 eventually reaches 0

This is exactly the well-ordering principle: every descending sequence of non-negative integers is finite!

✓ Proving Recursive Algorithm Correctness

Now let's formalize how to prove that a recursive function computes what it's supposed to compute.

The Correctness Proof Template

📖 Correctness Proof Structure

To prove a recursive function f(n) correctly computes some property P:

  1. State the claim: "For all n ≥ n₀, f(n) returns [specification]"
  2. Base case: Verify f(n₀) returns the correct value
  3. Inductive hypothesis: Assume f(k) is correct for some k ≥ n₀
  4. Inductive step: Show f(k+1) is correct, using the IH about f(k)

Example 1: Power Function

The Function

/**
 * Computes x^n for n >= 0.
 * @param x the base
 * @param n the exponent (non-negative)
 * @return x raised to the power n
 */
public static double power(double x, int n) {
    if (n == 0) {
        return 1;
    }
    return x * power(x, n - 1);
}

Correctness Proof

Claim: For all n ≥ 0, power(x, n) returns xⁿ.

Proof by induction on n:

Base Case (n = 0):
power(x, 0) returns 1.
By definition, x⁰ = 1 for any x.
So power(x, 0) = x⁰. ✓

Inductive Hypothesis:
Assume power(x, k) = xᵏ for some k ≥ 0.

Inductive Step:
We must show power(x, k+1) = xᵏ⁺¹.

By the code: power(x, k+1) = x * power(x, k)
= x * xᵏ    [by IH]
= xᵏ⁺¹    [exponent rule] ✓

Conclusion: By induction, power(x, n) = xⁿ for all n ≥ 0. ∎

Example 2: Fibonacci Function

The Function

/**
 * Computes the nth Fibonacci number.
 * F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2) for n >= 2
 */
public static int fib(int n) {
    if (n == 0) return 0;
    if (n == 1) return 1;
    return fib(n - 1) + fib(n - 2);
}

⚠️ Note: Two Base Cases!

The Fibonacci function has two base cases (n = 0 and n = 1) because the recursive case uses two previous values (n-1 and n-2).

This requires strong induction, which you'll learn in Week 7. For now, understand that we need enough base cases to "anchor" all the recursive calls.

Preview: Strong Induction for Fibonacci ▼

In strong induction, instead of assuming just P(k), we assume P(n₀), P(n₀+1), ..., P(k) are all true.

For Fibonacci:

  • Base cases: fib(0) = 0 = F₀ ✓ and fib(1) = 1 = F₁ ✓
  • Strong IH: Assume fib(j) = Fⱼ for all j ≤ k where k ≥ 1
  • Inductive step: fib(k+1) = fib(k) + fib(k-1) = Fₖ + Fₖ₋₁ = Fₖ₊₁ ✓

Example 3: Binary Search

The Function

/**
 * Searches for target in sorted array arr[low..high].
 * Returns index of target if found, -1 otherwise.
 */
public static int binarySearch(int[] arr, int target, 
                                  int low, int high) {
    // Base case: empty range
    if (low > high) {
        return -1;
    }
    
    int mid = low + (high - low) / 2;
    
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] > target) {
        return binarySearch(arr, target, low, mid - 1);
    } else {
        return binarySearch(arr, target, mid + 1, high);
    }
}

💡 Correctness Argument (Informal)

Binary search is correct because:

  1. Base case: If low > high, the range is empty, so target isn't there → return -1 ✓
  2. Recursive case: We check the middle element:
    • If it's the target, we found it ✓
    • If target < arr[mid], target must be in the left half (array is sorted)
    • If target > arr[mid], target must be in the right half
  3. Termination: Each recursive call reduces the range size by at least half, eventually reaching an empty range

🔁 Loop Invariants

Induction isn't just for recursive functions—it's also the foundation for proving iterative (loop-based) algorithms correct using loop invariants.

What is a Loop Invariant?

📖 Definition: Loop Invariant

A loop invariant is a property that:

  1. Initialization: Is true before the loop starts
  2. Maintenance: If true before an iteration, remains true after that iteration
  3. Termination: When the loop ends, the invariant (plus the exit condition) implies correctness

💡 Loop Invariants ARE Induction!

Induction Loop Invariant
Base case: P(n₀) is true Initialization: Invariant holds before loop
Inductive step: P(k) → P(k+1) Maintenance: Invariant preserved by each iteration
Conclusion: P(n) for all n ≥ n₀ Termination: Invariant + exit → correctness

Example: Iterative Sum

The Function

/**
 * Computes 1 + 2 + 3 + ... + n iteratively.
 */
public static int sum(int n) {
    int result = 0;
    for (int i = 1; i <= n; i++) {
        result = result + i;
    }
    return result;
}

Correctness Proof Using Loop Invariant

Loop Invariant: At the start of each iteration with loop variable i, result = 1 + 2 + ... + (i-1).

Initialization (before first iteration, i = 1):
result = 0 = (empty sum) = sum of integers from 1 to 0. ✓

Maintenance (if true before iteration i, show true after):
Before iteration: result = 1 + 2 + ... + (i-1) [by invariant]
During iteration: result = result + i = 1 + 2 + ... + (i-1) + i
After iteration: result = 1 + 2 + ... + i
Next iteration starts with i+1, and result = 1 + 2 + ... + ((i+1)-1). ✓

Termination (loop exits when i = n+1):
Invariant says: result = 1 + 2 + ... + ((n+1)-1) = 1 + 2 + ... + n. ✓

Conclusion: The function returns 1 + 2 + ... + n. ∎

Example: Finding Maximum

The Function

/**
 * Returns the maximum element in a non-empty array.
 */
public static int findMax(int[] arr) {
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    return max;
}
View Loop Invariant Proof ▼

Loop Invariant: At the start of iteration i, max = maximum of arr[0..i-1].

Initialization (i = 1):
max = arr[0] = maximum of arr[0..0]. ✓

Maintenance:
Before iteration i: max = max(arr[0..i-1])
During iteration: we set max = max(max, arr[i]) = max(arr[0..i])
After iteration: max = max(arr[0..i])
For next iteration i+1: max = max(arr[0..(i+1)-1]). ✓

Termination (i = arr.length):
max = max(arr[0..arr.length-1]) = max of entire array. ✓

📊 Complexity Analysis with Induction

Induction is also used to prove facts about algorithm complexity, particularly for recursive algorithms.

Proving Time Complexity Bounds

Example: Proving T(n) = O(n) for Linear Recursion

Consider a recursive function with recurrence: T(n) = T(n-1) + c for n > 1, T(1) = c.

Claim: T(n) = cn for all n ≥ 1.

Base case: T(1) = c = c(1). ✓

Inductive hypothesis: Assume T(k) = ck for some k ≥ 1.

Inductive step:
T(k+1) = T(k) + c = ck + c = c(k+1). ✓

Therefore, T(n) = cn = O(n). ∎

Example: Analyzing Recursive Sum

The Function

public static int sum(int n) {
    if (n == 1) return 1;       // T(1) = c₁
    return sum(n - 1) + n;       // T(n) = T(n-1) + c₂
}

Complexity Proof

Recurrence: T(n) = T(n-1) + c for n > 1, T(1) = c (where c is some constant).

Claim: T(n) ≤ cn for all n ≥ 1 (i.e., T(n) = O(n)).

Proof by induction:

Base case (n = 1): T(1) = c ≤ c(1) = c. ✓

Inductive hypothesis: Assume T(k) ≤ ck for some k ≥ 1.

Inductive step:
T(k+1) = T(k) + c ≤ ck + c = c(k+1). ✓

Conclusion: T(n) ≤ cn = O(n) for all n ≥ 1. ∎

Example: Binary Search Complexity

📐 Theorem

Binary search has time complexity O(log n).

View Complexity Analysis ▼

Recurrence: T(n) = T(n/2) + c for n > 1, T(1) = c.

Claim: T(n) ≤ c(log₂n + 1) for all n ≥ 1.

This can be proved by strong induction (Week 7), but the intuition is:

  • Each recursive call halves the problem size
  • Starting from n, we can halve at most log₂n times before reaching 1
  • Each level does O(1) work
  • Total: O(log n) levels × O(1) work = O(log n)

Common Recurrences and Their Solutions

Recurrence Solution Example Algorithm
T(n) = T(n-1) + c O(n) Linear recursion (factorial, sum)
T(n) = T(n-1) + n O(n²) Selection sort
T(n) = T(n/2) + c O(log n) Binary search
T(n) = 2T(n/2) + n O(n log n) Merge sort
T(n) = 2T(n-1) + c O(2ⁿ) Naive Fibonacci

Proof

powerOfTwo(0) returns 1 = 2⁰. ✓

Inductive hypothesis:
Assume powerOfTwo(k) = 2ᵏ for some k ≥ 0.

powerOfTwo(k+1) = 2 * powerOfTwo(k) [by code]

Before iteration i: result = (i-1)!

result = ((n+1)-1)! = n!. ✓
