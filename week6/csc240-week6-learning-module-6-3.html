📚 Introduction

Welcome to Module 6.3! Having mastered proof by contradiction in Modules 6.1 and 6.2, we now turn to one of the most powerful and widely-used proof techniques in mathematics and computer science: Mathematical Induction.

Induction is the primary tool for proving statements about all natural numbers. While you can't check infinitely many cases one by one, induction provides a clever way to prove infinitely many statements with a finite proof.

💡 Why Induction Matters in Computer Science

Mathematical induction is essential for:

  • Algorithm correctness: Proving that algorithms work for all input sizes
  • Recursive function verification: Showing recursive code produces correct results
  • Data structure properties: Proving properties of trees, lists, and graphs
  • Complexity analysis: Establishing running time bounds

  • Identify the base case and inductive step in an induction proof

🎲 The Domino Analogy

The best way to understand mathematical induction is through the domino analogy. Imagine an infinite line of dominoes, numbered 1, 2, 3, 4, and so on forever.

The Setup

Your goal is to knock down all the dominoes. But there are infinitely many of them! How can you possibly knock down infinitely many dominoes?

1n=1
2n=2
3n=3
4n=4
5n=5
6n=6
7n=7
8n=8
...

The Two Key Conditions

To guarantee that all dominoes fall, you need exactly two things:

1. Base Case

Knock down the first domino.

Without this, nothing gets started. Even if dominoes are perfectly arranged, if you never push the first one, none will fall.

2. Inductive Step

Each domino knocks down the next.

The dominoes must be arranged so that whenever one falls, it causes the next one to fall too.

💡 The Key Insight

If both conditions hold:

  • Domino 1 falls (base case)
  • Since 1 falls, 2 falls (inductive step: 1 → 2)
  • Since 2 falls, 3 falls (inductive step: 2 → 3)
  • Since 3 falls, 4 falls (inductive step: 3 → 4)
  • And so on... every domino falls!

Why Both Conditions Are Necessary

What Goes Wrong Without the Base Case?

Suppose the dominoes are perfectly arranged (each will knock down the next), but you never push the first one.

Result: No dominoes fall! The chain reaction never starts.

Mathematical analogy: You might prove "if P(k) then P(k+1)" for all k, but if P(1) is false, the implication tells you nothing useful.

What Goes Wrong Without the Inductive Step?

Suppose you knock down the first domino, but the dominoes are spaced too far apart—each one falls but doesn't reach the next.

Result: Only domino 1 falls!

Mathematical analogy: You prove P(1) is true, but you have no way to extend this to P(2), P(3), etc.

🔄 Interactive: What's Missing?

For each scenario, identify what's missing that prevents us from concluding all dominoes fall.

Scenario 2: "I've pushed domino 1 and it fell, but the dominoes are 10 feet apart."

Scenario 3: "I've verified that each domino, when it falls, will knock down the next one."

📐 The Principle of Mathematical Induction

Now let's translate the domino analogy into precise mathematical language.

The Formal Statement

📐 Principle of Mathematical Induction

Let P(n) be a statement about the positive integer n. If:

  1. Base Case: P(1) is true, and
  2. Inductive Step: For all k ≥ 1, if P(k) is true, then P(k+1) is true

Then P(n) is true for all positive integers n ≥ 1.

Understanding the Components

📖 P(n) - The Predicate

P(n) is a statement that depends on n. For each specific value of n, P(n) is either true or false.

Example: P(n) might be "1 + 2 + 3 + ... + n = n(n+1)/2"

  • P(1) is the statement "1 = 1(2)/2" (which is true)
  • P(2) is the statement "1 + 2 = 2(3)/2" (which is true)
  • P(100) is the statement "1 + 2 + ... + 100 = 100(101)/2"

📖 The Base Case

The base case establishes that P(n₀) is true for some starting value n₀ (usually n₀ = 1 or n₀ = 0).

This is typically the "easy" part—just verify the statement for one specific value.

📖 The Inductive Step

The inductive step proves: For all k ≥ n₀, P(k) → P(k+1)

This is the "if-then" part: IF P(k) is true, THEN P(k+1) must also be true.

📖 The Inductive Hypothesis

When proving the inductive step, we assume P(k) is true for some arbitrary k. This assumption is called the inductive hypothesis.

We then use this assumption to prove P(k+1).

The Logical Form

Symbolically, the Principle of Mathematical Induction states:

[P(1) ∧ (∀k ≥ 1)(P(k) → P(k+1))] → (∀n ≥ 1)P(n)

In words: "If P(1) is true AND for every k, P(k) implies P(k+1), THEN P(n) is true for all n."

Why Is This Valid?

💡 The Chain of Implications

Given the base case and inductive step, here's why P(n) is true for any n:

  • P(1) is true (base case)
  • P(1) → P(2), and P(1) is true, so P(2) is true (modus ponens)
  • P(2) → P(3), and P(2) is true, so P(3) is true (modus ponens)
  • P(3) → P(4), and P(3) is true, so P(4) is true (modus ponens)
  • ...continue for any finite n...

For any specific n, we can reach P(n) in n-1 applications of modus ponens!

Connection to Well-Ordering

📐 Well-Ordering Principle

Every non-empty set of positive integers has a smallest element.

The Principle of Mathematical Induction is actually equivalent to the Well-Ordering Principle. Here's an informal argument:

Why Induction Follows from Well-Ordering ▼

Suppose P(1) is true and P(k) → P(k+1) for all k ≥ 1.

We want to show P(n) is true for all n ≥ 1.

Proof by contradiction:

Suppose not. Then the set S = {n ∈ ℤ⁺ : P(n) is false} is non-empty.

By the Well-Ordering Principle, S has a smallest element m.

Since P(1) is true, we have 1 ∉ S, so m > 1.

Since m is the smallest element of S, we have m-1 ∉ S, meaning P(m-1) is true.

By the inductive step, P(m-1) → P(m), so P(m) is true.

But this contradicts m ∈ S (meaning P(m) is false)!

Therefore, S must be empty, and P(n) is true for all n ≥ 1. ∎

📝 The Induction Proof Template

Every induction proof follows a standard structure. Learning this template will help you write clear, organized proofs.

📋 Standard Induction Proof Template

Step 0: State What You're Proving

Claim: For all integers n ≥ n₀, P(n).

We proceed by induction on n.

Step 1: Base Case

Base Case (n = n₀): We verify that P(n₀) is true.

[Show the explicit calculation or verification]

Thus, P(n₀) holds. ✓

Step 2: Inductive Hypothesis

Inductive Hypothesis: Assume P(k) is true for some arbitrary k ≥ n₀.

[Write out explicitly what P(k) says]

Step 3: Inductive Step

Inductive Step: We must show P(k+1) is true.

[Write out what P(k+1) says—what you need to prove]

[Use the inductive hypothesis to prove P(k+1)]

Thus, P(k+1) holds. ✓

Step 4: Conclusion

Conclusion: By the Principle of Mathematical Induction, P(n) is true for all n ≥ n₀. ∎

A Complete Example

📐 Theorem

For all positive integers n: 1 + 2 + 3 + ... + n = n(n+1)/2

Proof by Induction

Claim: For all integers n ≥ 1, 1 + 2 + 3 + ... + n = n(n+1)/2.

We proceed by induction on n.

Base Case (n = 1):

When n = 1, the left side is: 1

The right side is: 1(1+1)/2 = 1(2)/2 = 1

Since LHS = RHS = 1, the base case holds. ✓

Inductive Hypothesis:

Assume that for some k ≥ 1:

1 + 2 + 3 + ... + k = k(k+1)/2

Inductive Step:

We must show: 1 + 2 + 3 + ... + k + (k+1) = (k+1)(k+2)/2

Starting with the left side:

1 + 2 + 3 + ... + k + (k+1)
= [1 + 2 + 3 + ... + k] + (k+1)
= k(k+1)/2 + (k+1)     [by inductive hypothesis]
= k(k+1)/2 + 2(k+1)/2
= [k(k+1) + 2(k+1)]/2
= [(k+1)(k + 2)]/2
= (k+1)(k+2)/2

This equals the right side, so P(k+1) holds. ✓

Conclusion: By the Principle of Mathematical Induction, for all n ≥ 1:

1 + 2 + 3 + ... + n = n(n+1)/2 ∎

Key Observations About the Proof

💡 Notice These Important Points

  1. The inductive hypothesis is used! In the inductive step, we explicitly used the assumption that 1 + 2 + ... + k = k(k+1)/2.
  2. We proved an implication. We showed: IF P(k) is true, THEN P(k+1) is true. We did NOT prove P(k) is true!
  3. The algebra connects k to k+1. The key step was recognizing that the sum up to k+1 equals (the sum up to k) plus (k+1).

🔄 Interactive: Label the Proof Parts

Read the following induction proof and identify each component.

Proof that n! ≥ 2ⁿ⁻¹ for all n ≥ 1

[A] We proceed by induction on n.

[B] When n = 1: LHS = 1! = 1, RHS = 2⁰ = 1. Since 1 ≥ 1, the base case holds.

[C] Assume k! ≥ 2^(k-1) for some k ≥ 1.

[D] We show (k+1)! ≥ 2^k.

[E] (k+1)! = (k+1) · k! ≥ (k+1) · 2^(k-1) ≥ 2 · 2^(k-1) = 2^k ✓

[F] By induction, n! ≥ 2^(n-1) for all n ≥ 1. ∎

⚠️ Common Misconceptions

Induction is often misunderstood. Let's address the most common misconceptions head-on.

Misconception 1: "Induction is Circular Reasoning"

❌ The Misconception

"You're assuming P(k) is true to prove P(k+1). But how do you know P(k) is true? Aren't you assuming what you're trying to prove?"

✅ The Reality

We are NOT assuming P(k) is true for all k. We are proving a conditional statement:

IF P(k) is true, THEN P(k+1) is true.

This is a valid implication regardless of whether P(k) is actually true!

Remember: P → Q is true whenever Q is true, or whenever P is false. We're proving that the implication holds for all k.

The base case then "activates" the chain: P(1) is true, so by our implication, P(2) is true, so P(3) is true, etc.

Misconception 2: "The Base Case Can Be Any Value"

❌ The Misconception

"I can start my induction wherever I want."

✅ The Reality

The base case determines where your conclusion applies. If you prove:

  • Base case: P(1) is true
  • Inductive step: P(k) → P(k+1) for k ≥ 1

Then you can only conclude P(n) for n ≥ 1.

If you prove:

  • Base case: P(5) is true
  • Inductive step: P(k) → P(k+1) for k ≥ 5

Then you can only conclude P(n) for n ≥ 5. You've said nothing about P(1), P(2), P(3), or P(4)!

Misconception 3: "I Don't Need to Use the Inductive Hypothesis"

❌ The Misconception

"I proved P(k+1) directly without using the assumption that P(k) is true. My proof is still valid!"

✅ The Reality

If you can prove P(k+1) without using P(k), you're doing a direct proof, not induction!

This might actually be fine—but then you should just prove P(n) directly for arbitrary n, without the induction framework.

The power of induction is that we get to USE the inductive hypothesis. If you're not using it, induction is the wrong technique.

Misconception 4: "Checking a Few Cases is Enough"

❌ The Misconception

"I verified P(1), P(2), P(3), P(4), and P(5). That's enough to conclude P(n) is true for all n."

✅ The Reality

Checking finitely many cases NEVER proves an infinite statement!

Consider: "n² + n + 41 is prime for all n ≥ 0"

  • n=0: 41 (prime ✓)
  • n=1: 43 (prime ✓)
  • n=2: 47 (prime ✓)
  • ...
  • n=39: 1601 (prime ✓)
  • n=40: 1681 = 41² (NOT prime ✗)

The pattern held for 40 cases but then failed! You need the inductive step to guarantee it works forever.

Inductive step: Assume k² < k. Then (k+1)² = k² + 2k + 1 < k + 2k + 1 = 3k + 1. Since 3k + 1 > k + 1, we have... hmm, this doesn't work.

Conclusion: The proof fails, so n² < n is false.

💻 Connection to Computer Science

Mathematical induction is deeply connected to recursion in programming. Understanding this connection will help you both write correct recursive code and prove that your algorithms work.

Induction ↔ Recursion

Mathematical Induction

  • Base case: Prove P(1) directly
  • Inductive step: Assume P(k), prove P(k+1)
  • Result: P(n) true for all n

Recursive Function

  • Base case: Return answer for n=1 directly
  • Recursive case: Use answer for k to compute answer for k+1
  • Result: Correct answer for all n

Example: Sum Function

Consider the mathematical statement we proved earlier:

1 + 2 + 3 + ... + n = n(n+1)/2

Here's a recursive implementation and its correctness proof:

/**
 * Computes 1 + 2 + 3 + ... + n recursively.
 * 
 * CORRECTNESS PROOF BY INDUCTION:
 * 
 * Claim: sum(n) returns n(n+1)/2 for all n ≥ 1.
 * 
 * Base Case (n=1): sum(1) returns 1 = 1(2)/2. ✓
 * 
 * Inductive Hypothesis: Assume sum(k) returns k(k+1)/2.
 * 
 * Inductive Step: sum(k+1) returns sum(k) + (k+1)
 *                                = k(k+1)/2 + (k+1)  [by IH]
 *                                = (k+1)(k+2)/2      [algebra]
 * This is correct! ✓
 * 
 * By induction, sum(n) is correct for all n ≥ 1.
 */
public static int sum(int n) {
    // Base case
    if (n == 1) {
        return 1;
    }
    // Recursive case: use sum(n-1) to compute sum(n)
    return sum(n - 1) + n;
}

Example: Factorial Function

/**
 * Computes n! = n × (n-1) × ... × 2 × 1.
 * 
 * CORRECTNESS PROOF BY INDUCTION:
 * 
 * Claim: factorial(n) returns n! for all n ≥ 0.
 * 
 * Base Case (n=0): factorial(0) returns 1 = 0!. ✓
 * 
 * Inductive Hypothesis: Assume factorial(k) returns k!.
 * 
 * Inductive Step: factorial(k+1) returns (k+1) * factorial(k)
 *                                       = (k+1) * k!  [by IH]
 *                                       = (k+1)!      [definition]
 * This is correct! ✓
 * 
 * By induction, factorial(n) is correct for all n ≥ 0.
 */
public static long factorial(int n) {
    // Base case
    if (n == 0) {
        return 1;
    }
    // Recursive case
    return n * factorial(n - 1);
}

The Parallel Structure

💡 Key Insight

Notice the parallel structure:

Induction Proof Recursive Code
Base case: verify P(n₀) Base case: if (n == n₀) return ...
IH: assume P(k) true Recursive call: f(k) gives correct answer
IS: use P(k) to prove P(k+1) Use f(k) to compute f(k+1)

This is why CSLO 4 connects mathematical induction to recursion—they are two sides of the same coin!

Loop Invariants (Preview)

Induction also underlies loop invariants, which we use to prove iterative algorithms correct:

/**
 * Computes 1 + 2 + ... + n iteratively.
 * 
 * LOOP INVARIANT: At the start of iteration i, sum = 1 + 2 + ... + (i-1)
 * 
 * Proof by induction on the iteration number:
 * - Base: Before iteration 1, sum = 0 = (empty sum). ✓
 * - IH: Assume at start of iteration k, sum = 1 + 2 + ... + (k-1).
 * - IS: Iteration k adds k to sum, so after iteration k,
 *       sum = 1 + 2 + ... + (k-1) + k = 1 + 2 + ... + k. ✓
 * 
 * After iteration n, sum = 1 + 2 + ... + n. Correct!
 */
public static int sumIterative(int n) {
    int sum = 0;
    for (int i = 1; i <= n; i++) {
        sum = sum + i;  // Invariant maintained
    }
    return sum;
}

Problem 1: Identify the Components

Problem Statement ▼

For the statement "For all n ≥ 1, 2ⁿ > n", identify:

  1. What is P(n)?
  2. What is P(1)? Is it true?
  3. What is P(k)?
  4. What is P(k+1)?
Solution ▼
  1. P(n): 2ⁿ > n
  2. P(1): 2¹ > 1, i.e., 2 > 1. Yes, this is true!
  3. P(k): 2^k > k (the inductive hypothesis)
  4. P(k+1): 2^(k+1) > k+1 (what we need to prove)

Problem 2: Write the Setup

Problem Statement ▼

For the statement "For all n ≥ 0, 3 divides n³ - n", write:

  1. The base case (what n value and what to verify)
  2. The inductive hypothesis
  3. What you need to prove in the inductive step

(You don't need to complete the proof—just set it up.)

Solution ▼
  1. Base case (n = 0): Verify 3 | (0³ - 0) = 3 | 0. Since 0 = 3·0, this is true. ✓
  2. Inductive hypothesis: Assume 3 | (k³ - k) for some k ≥ 0. That is, k³ - k = 3m for some integer m.
  3. Inductive step goal: Prove 3 | ((k+1)³ - (k+1)).

Problem 3: Complete the Proof

Problem Statement ▼

Prove: For all n ≥ 1, 1 + 3 + 5 + ... + (2n-1) = n².

(The sum of the first n odd numbers equals n².)

Solution ▼

Proof by Induction

Base Case (n = 1):

LHS = 1 (just the first odd number)

RHS = 1² = 1

LHS = RHS ✓

Inductive Hypothesis:

Assume 1 + 3 + 5 + ... + (2k-1) = k² for some k ≥ 1.

Inductive Step:

We must show: 1 + 3 + 5 + ... + (2k-1) + (2(k+1)-1) = (k+1)²

The next odd number after (2k-1) is (2k+1) = 2(k+1)-1.

1 + 3 + 5 + ... + (2k-1) + (2k+1)
= k² + (2k+1)   [by IH]
= k² + 2k + 1
= (k+1)² ✓

Conclusion: By induction, 1 + 3 + 5 + ... + (2n-1) = n² for all n ≥ 1. ∎
