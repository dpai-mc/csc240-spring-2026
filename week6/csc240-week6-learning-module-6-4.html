📚 Introduction

Welcome to Module 6.4! In Module 6.3, you learned the principle of mathematical induction. Now it's time to put that principle to work. In this module, you'll master the art of proving summation formulas—equations that give closed-form expressions for sums like 1 + 2 + 3 + ... + n.

These formulas are incredibly useful in computer science for analyzing algorithm complexity. When you need to count operations in a nested loop or analyze recursive running times, summation formulas are essential tools.

  • Master the "splitting the sum" algebraic technique
  • Verify base cases for summation identities

  • Connect summation formulas to algorithm analysis

Why Summation Formulas Matter

Computer Science Application

Consider this simple nested loop:

for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++) {
        // Some O(1) operation
    }
}

How many times does the inner operation execute?

  • When i = 1: 1 time
  • When i = 2: 2 times
  • When i = 3: 3 times
  • ...
  • When i = n: n times

Total: 1 + 2 + 3 + ... + n = n(n+1)/2 = O(n²)

The summation formula tells us this algorithm is quadratic!

🔧 The Key Technique: Splitting the Sum

Every summation proof by induction uses the same fundamental technique: splitting the sum. This technique is so important that you should internalize it completely.

📋 The Splitting Technique

For any sum from i = 1 to k+1:

∑i=1k+1 f(i) = ∑i=1k f(i) + f(k+1)

In words: The sum up to k+1 equals the sum up to k, plus the (k+1)th term.

Why This Works

This technique is the bridge between P(k) and P(k+1):

∑i=1k+1 f(i) [What we want to evaluate]
= ∑i=1k f(i) + f(k+1) [Split off last term]
= [closed form for sum to k] + f(k+1) [Apply Inductive Hypothesis!]
= [simplify algebraically] [Algebra]
= [closed form for sum to k+1] [This is P(k+1)!]

💡 The Pattern

Every summation induction proof follows this pattern:

  1. Split: Write the sum to k+1 as (sum to k) + (last term)
  2. Apply IH: Replace "sum to k" with its closed form (this is the inductive hypothesis!)
  3. Simplify: Use algebra to combine and simplify
  4. Match: Show the result equals the closed form for k+1

Common Algebraic Manipulations

You'll frequently need these algebraic skills:

Essential Algebra for Summation Proofs

Skill Example
Finding common denominators k(k+1)/2 + (k+1) = k(k+1)/2 + 2(k+1)/2
Factoring k(k+1) + 2(k+1) = (k+1)(k+2)
Expanding (k+1)² = k² + 2k + 1
Recognizing patterns k² + 2k + 1 = (k+1)²

⚠️ Common Mistake

Don't forget to actually use the inductive hypothesis! The whole point of the splitting technique is to create a place where you can substitute the IH.

If you find yourself proving P(k+1) without ever using P(k), you're either doing a direct proof (not induction) or making an error.

➕ Example 1: Sum of First n Integers

Let's prove the most famous summation formula using our induction template.

The Formula

1 + 2 + 3 + ... + n = n(n+1)/2

📐 Theorem

For all positive integers n ≥ 1:

∑i=1n i = n(n+1)/2

The Complete Proof

Proof by Mathematical Induction

Claim: For all n ≥ 1, ∑i=1n i = n(n+1)/2.
Base Case (n = 1):
LHS: ∑i=11 i = 1
RHS: 1(1+1)/2 = 1(2)/2 = 1
Since LHS = RHS = 1, the base case holds. ✓
Inductive Hypothesis: Assume that for some k ≥ 1:
∑i=1k i = k(k+1)/2
Inductive Step: We must show: ∑i=1k+1 i = (k+1)(k+2)/2
Proof of Inductive Step:
∑i=1k+1 i
= ∑i=1k i + (k+1) [Split off last term]
= k(k+1)/2 + (k+1) [By Inductive Hypothesis]
= k(k+1)/2 + 2(k+1)/2 [Common denominator]
= [k(k+1) + 2(k+1)]/2 [Combine fractions]
= (k+1)(k+2)/2 [Factor out (k+1)]
This is exactly what we needed to show! ✓
Conclusion: By the Principle of Mathematical Induction, ∑i=1n i = n(n+1)/2 for all n ≥ 1. ∎

Key Observations

💡 What Made This Work?

  1. We used the IH: The step "= k(k+1)/2 + (k+1)" is where we applied the inductive hypothesis.
  2. Algebra connected k to k+1: Factoring (k+1) from k(k+1) + 2(k+1) gave us (k+1)(k+2).
  3. The formula "updated" correctly: Replacing k with k+1 in k(k+1)/2 gives (k+1)(k+2)/2.

📐 Example 2: Sum of First n Squares

Now let's tackle a more challenging formula: the sum of squares.

The Formula

1² + 2² + 3² + ... + n² = n(n+1)(2n+1)/6

📐 Theorem

For all positive integers n ≥ 1:

∑i=1n i² = n(n+1)(2n+1)/6

Step-by-Step Proof

Base Case (n = 1):

LHS: 1² = 1

RHS: 1(1+1)(2·1+1)/6 = 1(2)(3)/6 = 6/6 = 1

LHS = RHS ✓

Inductive Hypothesis:

Assume for some k ≥ 1:

∑i=1k i² = k(k+1)(2k+1)/6
Goal (What we need to show):
∑i=1k+1 i² = (k+1)(k+2)(2k+3)/6

Note: We get this by replacing k with k+1 in the formula.

Split the Sum:
∑i=1k+1 i² = ∑i=1k i² + (k+1)²
Apply the Inductive Hypothesis:
= k(k+1)(2k+1)/6 + (k+1)²
Find Common Denominator:
= k(k+1)(2k+1)/6 + 6(k+1)²/6
Combine and Factor:
= [k(k+1)(2k+1) + 6(k+1)²]/6
= (k+1)[k(2k+1) + 6(k+1)]/6 [Factor out (k+1)]
= (k+1)[2k² + k + 6k + 6]/6 [Expand]
= (k+1)[2k² + 7k + 6]/6 [Combine like terms]
= (k+1)(k+2)(2k+3)/6 [Factor quadratic]
Conclusion:

This is exactly (k+1)(k+2)(2k+3)/6, which is what we needed to prove!

By induction, ∑i=1n i² = n(n+1)(2n+1)/6 for all n ≥ 1. ∎

The Tricky Part: Factoring

How to factor 2k² + 7k + 6 ▼

To factor 2k² + 7k + 6, we need two numbers that:

  • Multiply to 2 × 6 = 12
  • Add to 7

Those numbers are 3 and 4.

Rewrite: 2k² + 3k + 4k + 6

Group: k(2k + 3) + 2(2k + 3)

Factor: (k + 2)(2k + 3) ✓

📈 Example 3: Geometric Series

The geometric series is crucial for analyzing algorithms with exponential behavior and for understanding recurrence relations.

The Formula (for r ≠ 1)

1 + r + r² + ... + rⁿ = (rn+1 - 1)/(r - 1)

📐 Theorem

For all integers n ≥ 0 and any real number r ≠ 1:

∑i=0n ri = (rn+1 - 1)/(r - 1)

⚠️ Note the Index

This sum starts at i = 0, not i = 1. The base case will be n = 0.

The Proof

Proof by Mathematical Induction

Base Case (n = 0):
LHS: ∑i=00 ri = r⁰ = 1
RHS: (r0+1 - 1)/(r - 1) = (r - 1)/(r - 1) = 1
LHS = RHS ✓
Inductive Hypothesis: Assume for some k ≥ 0:
∑i=0k ri = (rk+1 - 1)/(r - 1)
Inductive Step: We must show: ∑i=0k+1 ri = (rk+2 - 1)/(r - 1)
Proof:
∑i=0k+1 ri
= ∑i=0k ri + rk+1 [Split]
= (rk+1 - 1)/(r - 1) + rk+1 [By IH]
= (rk+1 - 1)/(r - 1) + rk+1(r - 1)/(r - 1) [Common denom]
= [rk+1 - 1 + rk+2 - rk+1]/(r - 1) [Combine]
= (rk+2 - 1)/(r - 1) [Simplify: rk+1 cancels!]
This is exactly what we needed! ✓
Conclusion: By induction, ∑i=0n ri = (rn+1 - 1)/(r - 1) for all n ≥ 0. ∎

Special Cases

Common Applications

Value of r Formula Application
r = 2 1 + 2 + 4 + ... + 2ⁿ = 2n+1 - 1 Binary tree nodes, merge sort analysis
r = 1/2 1 + 1/2 + 1/4 + ... → 2 (as n → ∞) Probability, expected values
r = 10 1 + 10 + 100 + ... + 10ⁿ Decimal place values

💡 CS Application: Total Nodes in a Complete Binary Tree

A complete binary tree of height h has:

  • Level 0: 2⁰ = 1 node (root)
  • Level 1: 2¹ = 2 nodes
  • Level 2: 2² = 4 nodes
  • ...
  • Level h: 2h nodes

Total: 1 + 2 + 4 + ... + 2h = 2h+1 - 1 nodes

This is why a complete binary tree with n nodes has height Θ(log n)!

Problem 1: Sum of Odd Numbers

Prove

1 + 3 + 5 + ... + (2n-1) = n²
Hint ▼

The (k+1)th odd number is 2(k+1) - 1 = 2k + 1.

You'll need to show that k² + (2k+1) = (k+1)².

Complete Solution ▼

Proof

Base Case (n = 1): LHS = 1, RHS = 1² = 1. ✓

IH: Assume 1 + 3 + 5 + ... + (2k-1) = k²

IS: Show 1 + 3 + 5 + ... + (2k-1) + (2k+1) = (k+1)²

1 + 3 + ... + (2k-1) + (2k+1)
= k² + (2k+1) [By IH]
= k² + 2k + 1
= (k+1)² [Perfect square!]

By induction, 1 + 3 + ... + (2n-1) = n² for all n ≥ 1. ∎

Problem 2: Sum of Cubes

Prove

1³ + 2³ + 3³ + ... + n³ = [n(n+1)/2]²
Hint ▼

Notice that the RHS is the square of the sum of first n integers!

For the inductive step, you need to show:

[k(k+1)/2]² + (k+1)³ = [(k+1)(k+2)/2]²

Factor out (k+1)² from the LHS.

Complete Solution ▼

Proof

Base Case (n = 1): LHS = 1³ = 1, RHS = [1(2)/2]² = 1² = 1. ✓

IH: Assume ∑i=1k i³ = [k(k+1)/2]²

IS: Show ∑i=1k+1 i³ = [(k+1)(k+2)/2]²

∑i=1k+1 i³ = [k(k+1)/2]² + (k+1)³ [Split + IH]
= k²(k+1)²/4 + (k+1)³
= (k+1)²[k²/4 + (k+1)] [Factor (k+1)²]
= (k+1)²[k² + 4k + 4]/4
= (k+1)²(k+2)²/4
= [(k+1)(k+2)/2]² ✓

By induction, the formula holds for all n ≥ 1. ∎
