📚 Introduction

Welcome to Module 6.5! In Module 6.4, you mastered induction proofs for summation formulas. Now we expand your induction toolkit to handle two important categories of statements that don't involve sums:

  • Divisibility properties: "For all n ≥ 0, 3 divides 4ⁿ - 1"
  • Inequalities: "For all n ≥ 1, 2ⁿ > n"

These types of proofs appear frequently in computer science when analyzing algorithms, proving correctness of recursive functions, and establishing bounds on computational resources.

🎯 Learning Objectives

  • Apply induction to prove divisibility properties
  • Apply induction to prove inequalities
  • Recognize common patterns in divisibility induction proofs
  • Handle inequalities carefully in inductive steps
  • Choose appropriate base cases for different types of claims
  • Connect these proofs to algorithm analysis

📋 How This Differs from Summation Proofs

In summation proofs, we used the "splitting" technique to separate the (k+1)th term. For divisibility and inequality proofs, we use different strategies:

  • Divisibility: Factor expressions to reveal multiples of the divisor
  • Inequalities: Build on the IH by adding/multiplying both sides

➗ Divisibility Basics

Before diving into proofs, let's review the definition and key properties of divisibility.

📖 Definition: Divisibility

For integers a and b with a ≠ 0, we say a divides b (written a | b) if there exists an integer k such that b = ak.

Equivalently: b is a multiple of a, or b is divisible by a.

Examples

  • 3 | 12 because 12 = 3 × 4 ✓
  • 5 | 0 because 0 = 5 × 0 ✓
  • 7 | 21 because 21 = 7 × 3 ✓
  • 4 ∤ 10 because 10 ≠ 4k for any integer k ✗

Key Properties for Proofs

📋 Properties We'll Use

Property Statement Why It's Useful
Sum Rule If d | a and d | b, then d | (a + b) Combine divisible terms
Difference Rule If d | a and d | b, then d | (a - b) Subtract divisible terms
Multiple Rule If d | a, then d | (ca) for any integer c Scale divisible terms
Transitivity If a | b and b | c, then a | c Chain divisibility

The Key Technique: Factoring for Divisibility

🔑 The Pattern

To prove d | f(k+1) using the fact that d | f(k):

  1. Express f(k+1) in terms of f(k)
  2. Rewrite as: f(k+1) = [something involving f(k)] + [multiple of d]
  3. Apply the Sum Rule: both parts are divisible by d

Preview: Why 4ⁿ⁺¹ - 1 involves 4ⁿ - 1

We can write:

4ⁿ⁺¹ - 1 = 4 · 4ⁿ - 1 = 4 · 4ⁿ - 4 + 3 = 4(4ⁿ - 1) + 3

Now we see: 4ⁿ⁺¹ - 1 = 4(4ⁿ - 1) + 3

If 3 | (4ⁿ - 1), then 3 | 4(4ⁿ - 1), and since 3 | 3, we get 3 | (4ⁿ⁺¹ - 1)!

📐 Divisibility Proofs by Induction

Let's work through complete divisibility proofs, examining the key techniques at each step.

Example 1: 3 | (4ⁿ - 1)

📐 Theorem

For all integers n ≥ 0, 3 divides 4ⁿ - 1.

Proof by Mathematical Induction

Base Case (n = 0):
4⁰ - 1 = 1 - 1 = 0
Since 0 = 3 × 0, we have 3 | 0. ✓
Inductive Hypothesis: Assume for some k ≥ 0 that 3 | (4ᵏ - 1).
This means 4ᵏ - 1 = 3m for some integer m.
Inductive Step: We must show: 3 | (4ᵏ⁺¹ - 1)
Proof:
4ᵏ⁺¹ - 1
= 4 · 4ᵏ - 1 [Exponent rule]
= 4 · 4ᵏ - 4 + 3 [Add and subtract 4]
= 4(4ᵏ - 1) + 3 [Factor out 4]
= 4(3m) + 3 [By IH: 4ᵏ - 1 = 3m]
= 3(4m + 1) [Factor out 3]
Since 4m + 1 is an integer, 4ᵏ⁺¹ - 1 = 3(4m + 1) is a multiple of 3.
Therefore, 3 | (4ᵏ⁺¹ - 1). ✓
Conclusion: By the Principle of Mathematical Induction, 3 | (4ⁿ - 1) for all n ≥ 0. ∎

💡 The Key Move

The crucial step was rewriting 4ᵏ⁺¹ - 1 as 4(4ᵏ - 1) + 3:

  • 4(4ᵏ - 1) is divisible by 3 because 4ᵏ - 1 is (by IH)
  • 3 is obviously divisible by 3
  • By the Sum Rule, their sum is divisible by 3

Example 2: 6 | (n³ - n)

📐 Theorem

For all integers n ≥ 0, 6 divides n³ - n.

0³ - 0 = 0 = 6 × 0, so 6 | 0. ✓

Assume 6 | (k³ - k) for some k ≥ 0.

This means k³ - k = 6m for some integer m.

Show that 6 | ((k+1)³ - (k+1)).

(k+1)³ - (k+1) = (k³ + 3k² + 3k + 1) - (k + 1)
= k³ + 3k² + 3k + 1 - k - 1
= k³ + 3k² + 2k
k³ + 3k² + 2k = (k³ - k) + 3k² + 3k
= (k³ - k) + 3k(k + 1)

By IH, 6 | (k³ - k).

For 3k(k+1): Since k and k+1 are consecutive integers, one is even.

So k(k+1) is even, meaning k(k+1) = 2j for some integer j.

Thus 3k(k+1) = 3(2j) = 6j, so 6 | 3k(k+1).

Since 6 | (k³ - k) and 6 | 3k(k+1), by the Sum Rule:

6 | [(k³ - k) + 3k(k+1)] = 6 | [(k+1)³ - (k+1)]. ✓

By induction, 6 | (n³ - n) for all n ≥ 0. ∎

Common Patterns in Divisibility Proofs

🔑 Pattern 1: aⁿ - 1 is divisible by a - 1

For any integer a ≥ 2 and n ≥ 0: (a - 1) | (aⁿ - 1)

Key technique: Write aⁿ⁺¹ - 1 = a·aⁿ - 1 = a(aⁿ - 1) + (a - 1)

🔑 Pattern 2: Consecutive integers

The product of any k consecutive integers is divisible by k!

Example: n(n+1) is always even (divisible by 2)

Example: n(n+1)(n+2) is always divisible by 6

📊 Inequality Basics

Inequality proofs by induction require careful handling of the inductive step. Let's establish the key techniques.

Key Properties for Inequality Proofs

📋 Properties We'll Use

Property Statement
Addition If a > b, then a + c > b + c
Multiplication (positive) If a > b and c > 0, then ac > bc
Transitivity If a > b and b > c, then a > c
Power rule If a > 1 and n > m ≥ 0, then aⁿ > aᵐ

The Key Technique: Building on the IH

🔑 The Strategy

To prove f(k+1) > g(k+1) using the IH that f(k) > g(k):

  1. Start with what you want: f(k+1)
  2. Relate it to f(k) somehow
  3. Apply the IH to get a lower bound
  4. Show this lower bound ≥ g(k+1)

⚠️ Common Pitfall: Wrong Direction

A common mistake is to start with the conclusion and work backward. Instead:

  • Wrong: "We want to show 2ᵏ⁺¹ > k+1. If this is true, then..."
  • Right: "We have 2ᵏ⁺¹ = 2 · 2ᵏ > 2k [by IH]. Now we show 2k ≥ k+1..."

Always build forward from what you know to what you want to show.

Choosing the Right Base Case

💡 Base Case Selection

For inequalities, the base case value matters more than for other proofs:

  • 2ⁿ > n: True for n ≥ 1 (base case n = 1: 2 > 1 ✓)
  • 2ⁿ > n²: Only true for n ≥ 5 (check: 2⁴ = 16 < 16 = 4²)
  • n! > 2ⁿ: Only true for n ≥ 4 (check: 3! = 6 < 8 = 2³)

Always verify your base case and make sure your conclusion states the correct range!

📈 Inequality Proofs by Induction

Let's work through complete inequality proofs, focusing on the techniques for the inductive step.

Example 1: 2ⁿ > n for all n ≥ 1

📐 Theorem

For all integers n ≥ 1, 2ⁿ > n.

Proof by Mathematical Induction

Base Case (n = 1):
2¹ = 2 > 1 = n ✓
Inductive Hypothesis: Assume 2ᵏ > k for some k ≥ 1.
Inductive Step: We must show: 2ᵏ⁺¹ > k + 1
Proof:
2ᵏ⁺¹ = 2 · 2ᵏ [Exponent rule]
> 2 · k [By IH: 2ᵏ > k, multiply by 2]
= k + k
≥ k + 1 [Since k ≥ 1]
Therefore, 2ᵏ⁺¹ > k + 1. ✓
Conclusion: By induction, 2ⁿ > n for all n ≥ 1. ∎

💡 The Key Insight

We showed 2ᵏ⁺¹ > 2k ≥ k + 1 (when k ≥ 1).

The chain of inequalities: 2ᵏ⁺¹ > 2k ≥ k + 1 gives us 2ᵏ⁺¹ > k + 1 by transitivity.

Example 2: n! > 2ⁿ for all n ≥ 4

📐 Theorem

For all integers n ≥ 4, n! > 2ⁿ.

4! = 24

2⁴ = 16

Since 24 > 16, the base case holds. ✓

Assume k! > 2ᵏ for some k ≥ 4.

Show that (k+1)! > 2ᵏ⁺¹.

(k+1)! = (k+1) · k!
(k+1) · k! > (k+1) · 2ᵏ

[Since k! > 2ᵏ by IH, and k+1 > 0]

We need (k+1) · 2ᵏ ≥ 2 · 2ᵏ = 2ᵏ⁺¹

This requires k + 1 ≥ 2, i.e., k ≥ 1.

Since k ≥ 4, this is satisfied. ✓

(k+1)! = (k+1) · k! > (k+1) · 2ᵏ ≥ 2 · 2ᵏ = 2ᵏ⁺¹

Therefore (k+1)! > 2ᵏ⁺¹. ✓

By induction, n! > 2ⁿ for all n ≥ 4. ∎

Example 3: 2ⁿ ≥ n² for all n ≥ 4

📐 Theorem

For all integers n ≥ 4, 2ⁿ ≥ n².

View Complete Proof ▼

Proof

Base Case (n = 4): 2⁴ = 16 = 4². Since 16 ≥ 16, the base case holds. ✓

Inductive Hypothesis: Assume 2ᵏ ≥ k² for some k ≥ 4.

Inductive Step: We show 2ᵏ⁺¹ ≥ (k+1)².

2ᵏ⁺¹ = 2 · 2ᵏ ≥ 2k² [By IH]

We need to show 2k² ≥ (k+1)² = k² + 2k + 1.

This is equivalent to: k² ≥ 2k + 1, or k² - 2k - 1 ≥ 0.

For k ≥ 4: k² - 2k - 1 = k(k-2) - 1 ≥ 4(2) - 1 = 7 > 0. ✓

Therefore 2ᵏ⁺¹ ≥ 2k² ≥ (k+1)². ✓

By induction, 2ⁿ ≥ n² for all n ≥ 4. ∎

CS Application: Algorithm Complexity

Why These Inequalities Matter

These inequalities help us compare algorithm complexities:

  • 2ⁿ > n: Exponential algorithms are worse than linear
  • n! > 2ⁿ: Factorial is worse than exponential
  • 2ⁿ ≥ n²: Exponential eventually dominates polynomial

When analyzing algorithms, these relationships help us understand which approach is more efficient for large inputs.

Problem 1: Divisibility

Prove

For all n ≥ 0, 5 | (6ⁿ - 1).

Hint ▼

Write 6ⁿ⁺¹ - 1 = 6 · 6ⁿ - 1 = 6 · 6ⁿ - 6 + 5 = 6(6ⁿ - 1) + 5.

Complete Solution ▼

Proof

Base Case (n = 0): 6⁰ - 1 = 0 = 5 × 0. So 5 | 0. ✓

IH: Assume 5 | (6ᵏ - 1), so 6ᵏ - 1 = 5m for some integer m.

IS: Show 5 | (6ᵏ⁺¹ - 1).

6ᵏ⁺¹ - 1 = 6 · 6ᵏ - 1
= 6 · 6ᵏ - 6 + 5
= 6(6ᵏ - 1) + 5
= 6(5m) + 5 = 5(6m + 1)

Since 6m + 1 is an integer, 5 | (6ᵏ⁺¹ - 1). ✓

By induction, 5 | (6ⁿ - 1) for all n ≥ 0. ∎

Problem 2: Inequality

Prove

For all n ≥ 1, 3ⁿ > 2n + 1.

Hint ▼

For the inductive step: 3ᵏ⁺¹ = 3 · 3ᵏ > 3(2k + 1) = 6k + 3.

You need to show 6k + 3 ≥ 2(k+1) + 1 = 2k + 3.

Complete Solution ▼

Proof

Base Case (n = 1): 3¹ = 3 > 2(1) + 1 = 3? No! 3 is not > 3.

Try n = 2: 3² = 9 > 2(2) + 1 = 5. ✓

Actually, let's check n = 1 more carefully: we need 3 > 3, which is false.

So the theorem should be for n ≥ 2, or we use ≥ instead of >.

Revised: Prove 3ⁿ ≥ 2n + 1 for all n ≥ 1.

Base Case (n = 1): 3¹ = 3 ≥ 2(1) + 1 = 3. ✓

IH: Assume 3ᵏ ≥ 2k + 1 for some k ≥ 1.

IS: Show 3ᵏ⁺¹ ≥ 2(k+1) + 1 = 2k + 3.

3ᵏ⁺¹ = 3 · 3ᵏ ≥ 3(2k + 1) [By IH]
= 6k + 3
≥ 2k + 3 [Since 6k ≥ 2k for k ≥ 0]

Therefore 3ᵏ⁺¹ ≥ 2k + 3 = 2(k+1) + 1. ✓

By induction, 3ⁿ ≥ 2n + 1 for all n ≥ 1. ∎

Problem 3: Divisibility with Consecutive Integers

Prove

For all n ≥ 1, 2 | n(n+1).

Complete Solution ▼

Proof

Base Case (n = 1): 1(2) = 2 = 2 × 1. So 2 | 2. ✓

IH: Assume 2 | k(k+1), so k(k+1) = 2m for some integer m.

IS: Show 2 | (k+1)(k+2).

(k+1)(k+2) = (k+1)k + 2(k+1)
= k(k+1) + 2(k+1)
= 2m + 2(k+1) [By IH]
= 2(m + k + 1)

Since m + k + 1 is an integer, 2 | (k+1)(k+2). ✓

By induction, 2 | n(n+1) for all n ≥ 1. ∎

🎓 Summary

Key Techniques Reference

Divisibility Proofs

  • Goal: Show d | f(k+1)
  • Strategy: Write f(k+1) = c·f(k) + d·m
  • Key property: Sum Rule
  • Common pattern: aⁿ - 1 = a(aⁿ⁻¹ - 1) + (a-1)

Inequality Proofs

  • Goal: Show f(k+1) > g(k+1)
  • Strategy: Chain: f(k+1) > ... ≥ g(k+1)
  • Key property: Transitivity
  • Watch for: Correct base case!

5 | (6ⁿ - 1) n = 0 6ⁿ⁺¹ - 1 = 6(6ⁿ - 1) + 5

(a-1) | (aⁿ - 1) n = 0 aⁿ⁺¹ - 1 = a(aⁿ - 1) + (a-1)
