Common Mistakes to Avoid

üí° Learn from Common Errors

Mistake 1: Incorrect Implication Logic

Wrong: Treating p -> q as false when p is false

Correct: Implication is false ONLY when p is true and q is false

Mistake 2: Wrong Row Count

Wrong: Counting variables incorrectly (e.g., counting 'p' twice in "p AND p")

Correct: Count DISTINCT variables only. "p AND p" has 1 variable = 2 rows

Mistake 3: Operator Precedence

Wrong: Evaluating p AND q OR r as p AND (q OR r)

Correct: AND has higher precedence, so it's (p AND q) OR r

Best practice: Always use parentheses for clarity!

Mistake 4: Parentheses Matching

Wrong: Missing closing parenthesis: (p AND q OR r

Correct: Every opening parenthesis needs a closing one

Mistake 5: Case Sensitivity

Issue: Treating 'P' and 'p' as different variables

Recommendation: Convert all input to lowercase or uppercase for consistency

Solution Strategy

Think of binary counting! For 3 variables (p, q, r):

Row 0: 000 ‚Üí F F F
Row 1: 001 ‚Üí F F T
Row 2: 010 ‚Üí F T F
Row 3: 011 ‚Üí F T T
Row 4: 100 ‚Üí T F F
Row 5: 101 ‚Üí T F T
Row 6: 110 ‚Üí T T F
Row 7: 111 ‚Üí T T T

Implementation Hint

Loop from 0 to 2‚Åø - 1, and use bit operations to extract individual truth values:

// Java example
for (int i = 0; i < Math.pow(2, numVars); i++) {
    boolean p = ((i >> 2) & 1) == 1; // bit 2
    boolean q = ((i >> 1) & 1) == 1; // bit 1
    boolean r = (i & 1) == 1;        // bit 0
    // Now evaluate expression with these values
}

The Problem

Evaluating compound expressions like "(p AND q) OR (NOT r)" for given truth values.

Solution Strategy

  1. Parse the expression into a tree or postfix notation
  2. Substitute truth values for variables
  3. Evaluate operators according to their definitions
  4. Work from innermost parentheses outward

Key Functions to Implement

  • not(p): Returns !p
  • and(p, q): Returns p && q
  • or(p, q): Returns p || q
  • implies(p, q): Returns !p || q
  • biconditional(p, q): Returns p == q

The Problem

Creating nicely aligned, readable truth table output in the console.

Solution Strategy

  • Calculate column widths based on header lengths
  • Use printf/cout with formatting specifiers
  • Add separator lines for readability
  • Center or align text appropriately

Java Example

// Format with fixed widths
System.out.printf("%-5s | %-5s | %-10s%n", "p", "q", "p AND q");
System.out.println("------+-------+-----------");
System.out.printf("%-5s | %-5s | %-10s%n", "T", "T", "T");

Debugging Tips

üêõ Common Bugs and Fixes

Bug: Wrong number of rows

Symptom: Your table has 6 rows instead of 8 for a 3-variable expression

Cause: Counting variables incorrectly (maybe counting 'p' twice in "p AND p")

Fix: Count DISTINCT variables only. Use a Set data structure.

Bug: Implication always true

Symptom: p -> q shows all T's in your output

Cause: Incorrect implication logic (maybe treating it like biconditional)

Fix: Implication is: !p || q (false only when p is true and q is false)

Bug: Parentheses not working

Symptom: "(p OR q) AND r" gives same result as "p OR (q AND r)"

Cause: Not respecting parentheses during evaluation

Fix: Parse parentheses first, evaluate innermost expressions first

Bug: Classification incorrect

Symptom: Calling "p AND q" a tautology

Cause: Not checking ALL rows, or logic error in classification

Fix: Tautology = ALL true, Contradiction = ALL false, Contingency = mixed
