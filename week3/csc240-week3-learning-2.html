Welcome to the study of logic‚Äîthe foundation of mathematical reasoning, computer science, and critical thinking! Logic provides the tools to analyze arguments, verify program correctness, and build reliable systems.

üìñ What is Logic?

Logic is the systematic study of reasoning and argumentation. It provides rules for determining when conclusions follow validly from premises.

Propositional Logic (also called sentential logic) deals with propositions and their relationships using logical connectives.

Why Logic Matters in Computer Science

üíª CS Applications of Logic

  • Program Correctness: Prove that programs do what they're supposed to do
  • Boolean Logic: Foundation of digital circuits and conditional statements
  • Database Queries: SQL uses logical operations (AND, OR, NOT)
  • Algorithm Design: Logical reasoning guides algorithm development
  • Artificial Intelligence: Logical inference in expert systems
  • Verification: Formal methods for proving system properties

üìù Propositions and Propositional Variables

Let's start with the fundamental building block of logic: the proposition.

üìñ Definition: Proposition

A proposition is a declarative statement that is either true or false, but not both.

Key characteristics:

  • It's a statement (not a question or command)
  • It has exactly one truth value: True (T) or False (F)
  • The truth value doesn't change arbitrarily

Examples of Propositions

üí° Valid Propositions

  • "2 + 2 = 4" ‚Üí TRUE (a mathematical fact)
  • "The sky is green" ‚Üí FALSE (observably false)
  • "Paris is the capital of France" ‚Üí TRUE (a fact)
  • "10 is an even number" ‚Üí TRUE (by definition)
  • "All dogs are mammals" ‚Üí TRUE (biological fact)
  • "Java is a programming language" ‚Üí TRUE (verifiable fact)

What is NOT a Proposition?

‚ùå NOT Propositions

  • "What time is it?" ‚Üí Question (no truth value)
  • "Close the door" ‚Üí Command (no truth value)
  • "x + 1 = 5" ‚Üí Depends on x (not definitely true or false)
  • "This statement is false" ‚Üí Paradox (can't be consistently true or false)
  • "Chocolate ice cream is the best" ‚Üí Opinion (subjective, no definite truth value)

Why not? These statements either don't make claims, have variable truth values, or create logical paradoxes.

Propositional Variables

To work with propositions abstractly, we use propositional variables (usually lowercase letters like p, q, r, s) to represent propositions.

üî§ Using Variables

We can assign propositions to variables:

  • p: "It is raining"
  • q: "The ground is wet"
  • r: "I will bring an umbrella"

Then we can reason about relationships: "If p, then q" without worrying about the specific content.

Truth Values

üìñ Truth Values

Every proposition has a truth value:

  • True (T): The proposition correctly describes reality
  • False (F): The proposition incorrectly describes reality

Important: A proposition cannot be both true and false, and it cannot be neither.

üîó Logical Connectives

Now that we understand propositions, let's learn how to combine them using logical connectives (also called logical operators).

üìñ Logical Connectives

Logical connectives are operators that combine one or more propositions to form compound propositions.

The truth value of a compound proposition depends on:

  • The truth values of its component propositions
  • The connective(s) used

üìä Constructing Truth Tables

Truth tables are systematic tools for analyzing compound propositions by showing all possible truth value combinations.

üìñ Truth Table

A truth table displays the truth value of a compound proposition for every possible combination of truth values of its component propositions.

How to Build a Truth Table

üìù Step-by-Step Process

  1. Identify variables: List all propositional variables (p, q, r, ...)
  2. Calculate rows: With n variables, you need 2‚Åø rows
  3. List combinations: Write all possible T/F combinations
  4. Add columns: Create columns for sub-expressions
  5. Compute final column: Calculate the complete expression

Two of the most important (and sometimes confusing) logical connectives are implication and biconditional.

Implication (Conditional): p ‚Üí q

üìñ Implication

The implication p ‚Üí q (read "if p then q" or "p implies q") is false ONLY when p is true and q is false.

p is called the hypothesis or antecedent

q is called the conclusion or consequent

T F F
F T T
F F T

ü§î Understanding Implication

Key insight: An implication is only false when the hypothesis is true but the conclusion is false.

When the hypothesis is false, the implication is vacuously true (true by default).

üí° Implication Examples

p: "It is raining"
q: "The ground is wet"
p ‚Üí q: "If it is raining, then the ground is wet"

  • Row 1 (T‚ÜíT=T): It's raining AND ground is wet ‚Üí TRUE (makes sense)
  • Row 2 (T‚ÜíF=F): It's raining BUT ground is NOT wet ‚Üí FALSE (broken promise!)
  • Row 3 (F‚ÜíT=T): NOT raining, but ground IS wet ‚Üí TRUE (maybe sprinkler?)
  • Row 4 (F‚ÜíF=T): NOT raining, ground NOT wet ‚Üí TRUE (consistent)

Why rows 3 & 4 are true: The statement only makes a claim about what happens when it rains. If it's not raining, the statement says nothing, so it can't be false!

Programming Connection

üíª If-Then in Code

    turnOnAirConditioning();
}

This is an implication: "If temperature > 30, then turn on AC"

  • When temp > 30 and AC turns on ‚Üí statement fulfilled (TRUE)
  • When temp > 30 but AC doesn't turn on ‚Üí bug! (FALSE)
  • When temp ‚â§ 30 ‚Üí statement makes no claim (vacuously TRUE)

Key Takeaways
