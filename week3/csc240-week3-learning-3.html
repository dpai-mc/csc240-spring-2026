What is Propositional Logic?

Propositional logic (also called propositional calculus or Boolean logic) is the branch of logic that deals with propositions and their relationships. It forms the foundation for reasoning in mathematics, computer science, and artificial intelligence.

üìñ Definition: Proposition

A proposition is a declarative statement that is either true or false, but not both.

Propositions have a definite truth value: they are either TRUE (T) or FALSE (F).

üí° Examples of Propositions
  • "2 + 2 = 4" - This is a proposition (TRUE)
  • "The sky is green" - This is a proposition (FALSE)
  • "Python is a programming language" - This is a proposition (TRUE)
  • "10 > 15" - This is a proposition (FALSE)
‚ö†Ô∏è NOT Propositions

The following are NOT propositions because they don't have definite truth values:

  • "What time is it?" - This is a question, not a statement
  • "Close the door" - This is a command
  • "x + 5 = 10" - Truth value depends on x (not definite)
  • "This statement is false" - Paradox (cannot be true or false)

Propositional Variables

We use letters like p, q, r, s to represent propositions. These are called propositional variables.

üí° Example: Using Variables

Let:

  • p = "It is raining"
  • q = "I will carry an umbrella"

Each variable can be either TRUE or FALSE depending on the situation.

Why Propositional Logic Matters in Computer Science

üíª Applications in CS
  • Boolean Logic: Foundation of digital circuits and computer hardware
  • Conditional Statements: if-then-else logic in programming
  • Database Queries: WHERE clauses use logical conditions
  • Algorithm Correctness: Proving programs work as intended
  • Artificial Intelligence: Reasoning and decision-making systems

What are Logical Connectives?

Logical connectives (also called logical operators) are symbols used to combine simple propositions into more complex compound propositions.

The Five Basic Logical Connectives

¬¨
Negation (NOT)

Symbol: ¬¨p or ~p

Read as: "not p"

Meaning: The opposite truth value of p

Programming: !p or NOT p

‚àß
Conjunction (AND)

Symbol: p ‚àß q

Read as: "p and q"

Meaning: True only when both p and q are true

Programming: p && q or p AND q

‚à®
Disjunction (OR)

Symbol: p ‚à® q

Read as: "p or q"

Meaning: True when at least one of p or q is true

Programming: p || q or p OR q

‚Üí

Symbol: p ‚Üí q

Read as: "if p then q" or "p implies q"

Meaning: False only when p is true and q is false

Programming: if (p) { q }

‚Üî
Biconditional (IFF)

Symbol: p ‚Üî q

Read as: "p if and only if q"

Meaning: True when p and q have the same truth value

Programming: p == q

‚ö†Ô∏è Important: Inclusive OR

In logic, the OR operator (‚à®) is inclusive. This means p ‚à® q is TRUE when:

  • p is true, OR
  • q is true, OR
  • Both p and q are true

This differs from everyday language where "or" often means "one or the other, but not both" (exclusive or).

Compound Propositions

We can combine multiple connectives to create complex logical expressions.

üí° Example: Compound Propositions

Let:

  • p = "It is raining"
  • q = "I have an umbrella"
  • r = "I will get wet"

Compound statements:

  • p ‚àß ¬¨q = "It is raining and I don't have an umbrella"
  • (p ‚àß ¬¨q) ‚Üí r = "If it is raining and I don't have an umbrella, then I will get wet"
  • p ‚à® q = "It is raining or I have an umbrella (or both)"

What is a Truth Table?

üìñ Definition: Truth Table

A truth table is a mathematical table that shows the truth value of a compound proposition for every possible combination of truth values of its propositional variables.

Truth tables allow us to:

  • Determine the truth value of complex logical expressions
  • Verify logical equivalences
  • Identify tautologies and contradictions
  • Analyze the behavior of logical circuits

Number of Rows in a Truth Table

The number of rows in a truth table depends on the number of propositional variables:

Formula: Number of rows = 2^n

where n = number of distinct propositional variables

üí° Examples: Calculating Rows
  • 1 variable (p): 2^1 = 2 rows
  • 2 variables (p, q): 2^2 = 4 rows
  • 3 variables (p, q, r): 2^3 = 8 rows
  • 4 variables (p, q, r, s): 2^4 = 16 rows

Truth Tables for Basic Connectives

1. Negation (¬¨p)

p ¬¨p
T F
F T

Explanation: Negation flips the truth value. If p is true, ¬¨p is false, and vice versa.

2. Conjunction (p ‚àß q)

T T T

F T F
F F F

Explanation: AND is true only when BOTH p and q are true. If either is false, the result is false.

3. Disjunction (p ‚à® q)

p q p ‚à® q
T T T
T F T
F T T
F F F

Explanation: OR is true when AT LEAST ONE of p or q is true. It's false only when both are false.

4. Implication (p ‚Üí q)

p q p ‚Üí q
T T T
T F F
F T T
F F T
‚ö†Ô∏è Understanding Implication

Implication is the trickiest connective! Key points:

  • p ‚Üí q is FALSE only when p is TRUE and q is FALSE
  • When p is FALSE, p ‚Üí q is always TRUE (vacuously true)
  • Think: "If p, then q" means "p guarantees q"
  • Example: "If it rains, I carry an umbrella" is only broken when it rains and I don't carry an umbrella

5. Biconditional (p ‚Üî q)

p q p ‚Üî q
T T T
T F F
F T F
F F T

Explanation: Biconditional is true when p and q have the SAME truth value (both true or both false).

Order of Operations for Logical Connectives

Just like arithmetic has order of operations (PEMDAS), logical connectives have a precedence order:

‚ö†Ô∏è Operator Precedence (Highest to Lowest)
  1. Negation (¬¨) - Highest precedence, evaluate first
  2. Conjunction (‚àß) - Second
  3. Disjunction (‚à®) - Third
  4. Implication (‚Üí) - Fourth
  5. Biconditional (‚Üî) - Lowest precedence, evaluate last

Use parentheses to override default precedence, just like in arithmetic!

Step-by-Step Method for Complex Truth Tables

üìã The Process
  1. Identify variables: Count distinct propositional variables
  2. Calculate rows: Number of rows = 2^n
  3. List all combinations: Create columns for each variable
  4. Add intermediate columns: Break down the expression into sub-expressions
  5. Evaluate step-by-step: Work from innermost parentheses outward
  6. Final column: The rightmost column is your answer

Example 1: (p ‚àß q) ‚Üí r

Let's construct a truth table for the compound proposition: (p ‚àß q) ‚Üí r

Step 1: Identify variables: p, q, r (3 variables)

Step 2: Calculate rows: 2^3 = 8 rows

üí° Explanation
  • Column 4 (p ‚àß q): Evaluate the conjunction first. It's true only in row 1 and 2 where both p and q are true.
  • Column 5 ((p ‚àß q) ‚Üí r): Now evaluate the implication. Remember: implication is false only when the antecedent is true and consequent is false. This happens only in row 2.

Evaluating ¬¨p ‚à® (q ‚àß r) Row by Row

Column 4 (¬¨p): Negate the value of p in each row.

Column 5 (q ‚àß r): Evaluate conjunction of q and r. True only when both are true (rows 1 and 5).

Column 6 (¬¨p ‚à® (q ‚àß r)): Evaluate disjunction. True when at least one of ¬¨p or (q ‚àß r) is true.

Three Special Types of Propositions

üìñ Tautology

A tautology is a compound proposition that is always TRUE, regardless of the truth values of its propositional variables.

In a truth table, the final column contains all T's.

Example: p ‚à® ¬¨p (Law of Excluded Middle)

p ¬¨p p ‚à® ¬¨p
T F T
F T T

Analysis: The final column is all TRUE, so this is a tautology. This makes intuitive sense: either p is true or p is false‚Äîthere's no third option!

üìñ Contradiction

A contradiction is a compound proposition that is always FALSE, regardless of the truth values of its propositional variables.

In a truth table, the final column contains all F's.

Example: p ‚àß ¬¨p

p ¬¨p p ‚àß ¬¨p
T F F
F T F

Analysis: The final column is all FALSE, so this is a contradiction. This makes sense: p cannot be both true and false simultaneously!

üìñ Contingency

A contingency is a compound proposition that is neither a tautology nor a contradiction.

In a truth table, the final column contains both T's and F's.

Example: p ‚Üí q

p q p ‚Üí q
T T T
T F F
F T T
F F T

Analysis: The final column has both TRUE and FALSE values, so this is a contingency. The truth value depends on the specific values of p and q.

Why These Classifications Matter

üíª Applications in Computer Science
  • Tautologies: Represent logical laws that are always valid (used in theorem proving, program verification)
  • Contradictions: Indicate logical impossibilities (used to detect errors in specifications)
  • Contingencies: Represent conditions that depend on input (most real-world logic)
  • Circuit Design: Tautologies always output 1, contradictions always output 0
  • Software Testing: Tautologies in assertions mean always-true conditions (may indicate bugs)
