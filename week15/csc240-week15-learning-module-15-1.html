1 — Why Trees Matter

Every time you open a file explorer, parse an HTML document, or evaluate an arithmetic expression, you are working with a tree. Trees are the data structures that give hierarchy to otherwise flat information: directories contain subdirectories, programming-language grammars nest expressions inside expressions, and database indexes guide searches through sorted key spaces.

From a graph-theory perspective, a tree is simply a connected, acyclic graph—a concept you first met in Week 12 when we defined graphs, vertices, and edges. In Week 13 you saw that BFS and DFS naturally produce spanning trees of a connected graph, and in Week 14 Kruskal's and Prim's algorithms found minimum spanning trees. This week we turn the spotlight on the tree itself: its vocabulary, its structural properties, and the specialized forms (binary trees, BSTs, expression trees) that dominate computing.

We will also revisit the proof-by-induction technique from Week 6. Two of the most important tree properties—the edge-count theorem and the maximum-node formula for binary trees—have elegant inductive proofs that reinforce why induction is the natural tool for recursive structures.

2 — Formal Definition and Properties

Definition — Tree (Graph-Theoretic)

A tree is an undirected graph T = (V, E) that is connected and acyclic (contains no cycles).

Equivalent Characterizations

The following statements are all equivalent for a graph G with n vertices. Any one can serve as the definition of a tree:

  1. G is connected and acyclic.
  2. G is connected and has exactly n − 1 edges.
  3. G is acyclic and has exactly n − 1 edges.
  4. There is exactly one simple path between every pair of vertices in G.
  5. G is connected, but removing any single edge disconnects it (every edge is a bridge).

Connection to Week 12

Recall that a connected graph is one where a path exists between every pair of vertices. A cycle is a closed walk with no repeated vertices (except start = end). Trees are the “minimally connected” graphs—connected with the fewest possible edges.

The n − 1 Edge Theorem

Theorem: Every tree with n ≥ 1 vertices has exactly n − 1 edges.

Proof by Strong Induction (click to expand)

Base case: n = 1. A single-vertex tree has 0 edges. n − 1 = 0. ✓

Inductive hypothesis: Assume every tree with k vertices (1 ≤ k < n) has exactly k − 1 edges.

Inductive step: Let T be a tree with n vertices (n ≥ 2). Because T is connected and has at least two vertices, it has at least one edge. Pick any edge e = {u, v}. Removing e disconnects T (since every edge in a tree is a bridge), producing exactly two connected components T1 and T2 with n1 and n2 vertices respectively, where n1 + n2 = n and both n1, n2 ≥ 1.

Each component is still acyclic (removing an edge cannot create a cycle) and connected, so each is a tree. By the inductive hypothesis:

  • T1 has n1 − 1 edges
  • T2 has n2 − 1 edges

Total edges in T = edges in T1 + edges in T2 + the removed edge e:

(n1 − 1) + (n2 − 1) + 1 = n1 + n2 − 1 = n − 1. ∎

✓ Quick Self-Check

A tree has 10 vertices. How many edges does it have? Answer: 9. If you’re given a connected graph with 10 vertices and 10 edges, can it be a tree? No—it would contain exactly one cycle.

3 — Rooted Trees and Core Vocabulary

Free Tree vs. Rooted Tree

A free tree (or unrooted tree) has no distinguished vertex. A rooted tree designates one vertex as the root, which imposes a parent–child hierarchy: edges point “downward” from root to leaves.

Definition — Rooted Tree

A rooted tree is a tree in which one vertex r is designated as the root. For every other vertex v, the unique path from r to v determines the parent–child relationships along that path.

Figure 1: A rooted tree with root A, internal nodes B, C, E, and leaves D, F, G, H.

Vocabulary Block

Root
The designated top vertex (A in Figure 1). It has no parent.
Parent
The vertex directly above a given vertex on the path to the root. E.g., parent(E) = B.
Child
A vertex directly below a given vertex. E.g., children(B) = {D, E}.
Sibling
Vertices sharing the same parent. D and E are siblings.
Leaf
A vertex with no children (degree 1 in the rooted tree, except a root that stands alone). Leaves here: D, F, G, H.
Internal node
A non-leaf vertex (has at least one child). Internal nodes: A, B, C, E.
Ancestor
Any vertex on the path from a vertex up to the root. Ancestors of G: E, B, A.
Descendant
Any vertex in the subtree rooted at a given vertex. Descendants of B: D, E, G, H.
Subtree
The tree formed by a vertex and all its descendants. The subtree rooted at E contains {E, G, H}.
Depth
The number of edges on the path from the root to a vertex. depth(A) = 0, depth(E) = 2, depth(G) = 3.
Height
The number of edges on the longest path from a vertex down to a leaf. height(A) = 3, height(E) = 1, height(D) = 0. The height of the tree is the height of the root.
Level
The set of all vertices at the same depth. Level 0 = {A}, Level 1 = {B, C}, Level 2 = {D, E, F}, Level 3 = {G, H}.

Common Confusion: Height vs. Depth

Depth is measured downward from the root to the vertex. Height is measured downward from the vertex to its deepest leaf descendant. They go in opposite directions. The root has depth 0 but may have large height; leaves have height 0 but may have large depth.

4 — Binary Trees

Definition — Binary Tree

A binary tree is a rooted tree in which every vertex has at most two children, distinguished as the left child and the right child.

Types of Binary Trees

Full Binary Tree

Every vertex has 0 or 2 children—no vertex has exactly one child.

Property: If a full binary tree has i internal nodes, it has exactly i + 1 leaves and 2i + 1 total nodes.

Example: An expression tree is always full—every operator has exactly two operands.

Complete Binary Tree

Every level is fully filled except possibly the last, and the last level is filled from left to right with no gaps.

Property: A complete binary tree of height h has between 2h and 2h+1 − 1 nodes (inclusive).

Why it matters: Complete binary trees map perfectly to arrays (see Section 5), which is why heaps use this shape.

Perfect Binary Tree

Every internal node has exactly 2 children and all leaves are at the same depth.

Property: A perfect binary tree of height h has exactly 2h+1 − 1 nodes and 2h leaves.

Note: Every perfect binary tree is both full and complete, but the converse is not true.

Induction Proof: Max-Node Formula

Proof: A binary tree of height h has at most 2h+1 − 1 nodes (click to expand)

Theorem: For h ≥ 0, a binary tree of height h has at most 2h+1 − 1 nodes.

Base case: h = 0. The tree is a single root. Nodes = 1. Formula: 20+1 − 1 = 2 − 1 = 1. ✓

Inductive hypothesis: Assume that for all k < h, a binary tree of height k has at most 2k+1 − 1 nodes.

Inductive step: Consider a binary tree T of height h (h ≥ 1). The root has at most two subtrees, TL and TR, each with height at most h − 1. By the inductive hypothesis, each subtree has at most 2(h−1)+1 − 1 = 2h − 1 nodes.

Total nodes in T:

nodes(T) ≤ 1 + (2h − 1) + (2h − 1) = 1 + 2h+1 − 2 = 2h+1 − 1. ∎

Connection to Week 6: This proof uses the same strong-induction template you practiced with divisibility and summation formulas.

5 — Array Representation of Complete Binary Trees

Because a complete binary tree has no “gaps” except at the trailing end of the last level, we can store it in a contiguous array with no pointers. This is the representation used by binary heaps.

Index Formulas (0-based indexing)

Store the root at index 0, then fill level by level, left to right:

RelationshipFormula
Left child of node at index i2i + 1
Right child of node at index i2i + 2
Parent of node at index i (for i > 0)⌊(i − 1) / 2⌋
Worked Example: Mapping a tree to an array (click to expand)

Consider a complete binary tree with nodes labeled by level order: A, B, C, D, E, F.

    A           Array: [A, B, C, D, E, F]
   / \          Index:  0  1  2  3  4  5
  B   C
 / \ /
D  E F

Verify the formulas:

  • Left child of A (index 0): 2(0) + 1 = 1 → B ✓
  • Right child of A (index 0): 2(0) + 2 = 2 → C ✓
  • Left child of B (index 1): 2(1) + 1 = 3 → D ✓
  • Parent of E (index 4): ⌊(4 − 1) / 2⌋ = ⌊1.5⌋ = 1 → B ✓
  • Parent of F (index 5): ⌊(5 − 1) / 2⌋ = ⌊2⌋ = 2 → C ✓

Why this matters: In Module 15.4 (Binary Search Trees) and in your Final Project, you may choose between pointer-based and array-based representations. Heaps always use this array layout because insert and delete operations need only index arithmetic—no pointer chasing.

1-Based Indexing Variant

Some textbooks store the root at index 1 (leaving index 0 unused). In that case: left child = 2i, right child = 2i + 1, parent = ⌊i / 2⌋. Both conventions are correct—just be consistent.
