1. Why Traversal Matters

A tree traversal is a systematic method for visiting every node in a tree exactly once. Without a traversal strategy, data stored in a tree is inaccessible—we need an algorithm to "walk" the structure.

🔗 Connection to Week 13 – Graph Traversals

In Week 13 you learned two fundamental graph exploration strategies:

  • Depth-First Search (DFS) — explore as deep as possible before backtracking (uses a stack or recursion).
  • Breadth-First Search (BFS) — explore all neighbors at the current depth before going deeper (uses a queue).

Trees are connected acyclic graphs, so the same strategies apply. Preorder, inorder, and postorder are all variants of DFS on a binary tree (they differ only in when the root is processed). Level-order is BFS applied to a tree starting at the root.

Practical Applications

TraversalCommon Use Cases
PreorderCopying a tree; prefix expression generation; serialization
InorderRetrieving BST elements in sorted order

2. The Four Traversals — Definitions & Mnemonics

TraversalVisit OrderMnemonicType
PreorderRoot → Left → RightR-L-R (Root first)DFS
InorderLeft → Root → RightL-R-R (Root in middle)DFS
PostorderLeft → Right → RootL-R-R (Root last)DFS
Level-orderLevel 0 → Level 1 → Level 2 → …Top-down, left-to-rightBFS

💡 Memory Tip

The name tells you when you visit the root relative to its subtrees:

  • Preorder = root is visited before (pre-) the subtrees.
  • Inorder = root is visited in between the left and right subtrees.
  • Postorder = root is visited after (post-) the subtrees.

3. Depth-First Traversals — Pseudocode

All three DFS traversals share the same recursive structure; only the placement of the "visit" step changes.

Preorder Traversal Pseudocode (Root → Left → Right) 
PREORDER(node):
    if node = NULL then return
    VISIT(node)              ← process the current node
    PREORDER(node.left)      ← recurse on left subtree
    PREORDER(node.right)     ← recurse on right subtree
Inorder Traversal Pseudocode (Left → Root → Right) 
INORDER(node):
    if node = NULL then return
    INORDER(node.left)       ← recurse on left subtree
    VISIT(node)              ← process the current node
    INORDER(node.right)      ← recurse on right subtree
Postorder Traversal Pseudocode (Left → Right → Root) 
POSTORDER(node):
    if node = NULL then return
    POSTORDER(node.left)     ← recurse on left subtree
    POSTORDER(node.right)    ← recurse on right subtree
    VISIT(node)              ← process the current node

Key Observation

Each recursive call pushes a frame onto the call stack. The maximum depth of recursion equals the height h of the tree. This is why the space complexity of recursive DFS traversals is O(h).

4. Worked Example — Tracing All Four Traversals

Consider the following labeled binary tree. Try to trace each traversal yourself before revealing the answers below.

            A
          /   \
        B       C
       / \       \
      D   E       F
         / \     /
        G   H   I

Tree structure summary:

  • Root: A
  • A's children: left = B, right = C
  • B's children: left = D, right = E
  • C's children: left = (none), right = F
  • E's children: left = G, right = H
  • F's children: left = I, right = (none)
  • D, G, H, I are leaves

On paper (or mentally), trace each traversal. Then click the buttons below to reveal each answer step by step.

Preorder: Root → Left → Right

Step 1: Visit A (root)
Step 2: Go left → Visit B
Step 3: Go left → Visit D (leaf, backtrack)
Step 4: Go right from B → Visit E
Step 5: Go left → Visit G (leaf, backtrack)
Step 6: Go right from E → Visit H (leaf, backtrack to A)
Step 7: Go right from A → Visit C
Step 8: C has no left child; go right → Visit F
Step 9: Go left → Visit I (leaf, done)
Result: A, B, D, E, G, H, C, F, I

Inorder: Left → Root → Right

Step 1: Go left from A → left from B → Visit D (leaf)
Step 2: Backtrack → Visit B (root of subtree)
Step 3: Go right to E → go left → Visit G (leaf)
Step 4: Backtrack → Visit E
Step 5: Go right from E → Visit H (leaf)
Step 6: Backtrack to root → Visit A
Step 7: Go right from A → C has no left child → Visit C
Step 8: Go right to F → go left → Visit I (leaf)
Step 9: Backtrack → Visit F (done)
Result: D, B, G, E, H, A, C, I, F

Postorder: Left → Right → Root

Step 1: Go left from A → left from B → Visit D (leaf)
Step 2: Go right from B → left from E → Visit G (leaf)
Step 3: Go right from E → Visit H (leaf)
Step 4: Both children of E done → Visit E
Step 5: Both children of B done → Visit B
Step 6: Go right from A → C has no left; go right to F → left → Visit I (leaf)
Step 7: F has no right child → Visit F
Step 8: Both children of C done → Visit C
Step 9: Both children of A done → Visit A (done)
Result: D, G, H, E, B, I, F, C, A

Level-Order: BFS (Top-down, left-to-right)

Step 1: Enqueue A. Dequeue A → Visit A. Enqueue B, C.
Step 2: Dequeue B → Visit B. Enqueue D, E.
Step 3: Dequeue C → Visit C. Enqueue F (no left child).
Step 4: Dequeue D → Visit D (leaf, nothing to enqueue).
Step 5: Dequeue E → Visit E. Enqueue G, H.
Step 6: Dequeue F → Visit F. Enqueue I.
Step 7: Dequeue G → Visit G (leaf).
Step 8: Dequeue H → Visit H (leaf).
Step 9: Dequeue I → Visit I (leaf). Queue empty → done.
Result: A, B, C, D, E, F, G, H, I

5. Level-Order — Queue-Based Algorithm

Unlike the three DFS traversals, level-order does not use recursion. Instead, it uses a queue (FIFO) to process nodes level by level—exactly like BFS from Week 13.

Level-Order Traversal Pseudocode 
LEVEL_ORDER(root):
    if root = NULL then return
    create empty queue Q
    Q.enqueue(root)

    while Q is not empty:
        node ← Q.dequeue()
        VISIT(node)
        if node.left ≠ NULL then Q.enqueue(node.left)
        if node.right ≠ NULL then Q.enqueue(node.right)

Why a queue? A queue ensures that all nodes at depth d are processed before any node at depth d+1. This is the defining property of BFS.

🔗 BFS Connection (Week 13)

In Week 13, BFS on a general graph required a "visited" set to avoid revisiting nodes in cycles. Trees have no cycles, so no visited set is needed—each node is naturally reached exactly once through its unique parent.

6. Complexity Analysis

Recall from Week 11 that Big-O notation describes the upper bound on resource usage as input size grows. Let:

  • n = total number of nodes in the tree
  • h = height of the tree (longest root-to-leaf path)
  • w = maximum width (largest number of nodes at any single level)
TraversalTime ComplexitySpace ComplexitySpace Explanation
PreorderO(n)O(h)Recursion stack depth = tree height
InorderO(n)O(h)Recursion stack depth = tree height
PostorderO(n)O(h)Recursion stack depth = tree height
Level-orderO(n)O(w)Queue holds at most one full level

Why O(n) Time for All?

Every traversal visits each node exactly once and performs a constant amount of work per node. Therefore, total work = c · n = O(n).

💡 Best/Worst Cases for Space

  • Balanced tree: h = O(log n) and w = O(n/2) = O(n). DFS uses less space than BFS.
  • Skewed tree (degenerate): h = O(n) and w = O(1). BFS uses less space than DFS.

Neither strategy is universally better—the tree's shape determines which is more space-efficient.

Level-Order Traversal (Java / C++)

Java

import java.util.LinkedList;
import java.util.Queue;

void levelOrder(TreeNode root) {
    if (root == null) return;
    Queue<TreeNode> q =
        new LinkedList<>();
    q.add(root);

    while (!q.isEmpty()) {
        TreeNode node = q.poll();
        System.out.print(
            node.data + " ");
        if (node.left != null)
            q.add(node.left);
        if (node.right != null)
            q.add(node.right);
    }
}

C++

#include <queue>
using namespace std;

void levelOrder(TreeNode* root) {
    if (root == nullptr) return;
    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        TreeNode* node = q.front();
        q.pop();
        cout << node->data << " ";
        if (node->left != nullptr)
            q.push(node->left);
        if (node->right != nullptr)
            q.push(node->right);
    }
}

Summary

Key Takeaways

  • There are four standard binary tree traversals: three DFS variants (preorder, inorder, postorder) and one BFS variant (level-order).
  • DFS traversals use recursion (implicit stack); level-order uses an explicit queue.
  • All four traversals run in O(n) time. Space depends on tree shape: O(h) for DFS and O(w) for level-order.
  • Inorder traversal of a BST yields elements in sorted order—a critical property you will use in Module 15.3 (Binary Search Trees).
