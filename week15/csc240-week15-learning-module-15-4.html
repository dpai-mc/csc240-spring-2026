📋 Module Overview

A Binary Search Tree (BST) is a binary tree that maintains a sorting invariant: for every node, all keys in the left subtree are smaller and all keys in the right subtree are larger. This property enables efficient search, insertion, and deletion — all in O(h) time where h is the tree's height. This module covers the BST property, core operations (search, insert, delete with all three cases), validation, complexity analysis, and the motivation for balanced trees. BSTs are the most implementation-heavy topic of the week and directly prepare you for AVL trees (Week 16) and the Final Project.

⚠️ Prerequisites

This module assumes you have completed Module 15.1 (tree terminology, binary tree structure, height) and Module 15.2 (inorder traversal). If you are not comfortable performing an inorder traversal by hand, review Module 15.2 before proceeding.

1. The BST Property

Formal Definition

A Binary Search Tree (BST) is a binary tree in which, for every node x:

  • All keys in the left subtree of x are less than x.key.
  • All keys in the right subtree of x are greater than x.key.

This invariant must hold recursively for every node in the tree — not just the immediate children, but the entire subtree.

🔑 Key Point

The BST property is a global constraint on subtrees, not merely a local parent-child relationship. A common mistake is checking only whether left.key < parent.key < right.key without verifying the entire subtree respects the bound. We will revisit this mistake in the Validation section.

Valid BST Example

Consider this binary tree with keys [8, 3, 10, 1, 6, 14, 4, 7, 13]:

8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13

Verification:

  • Node 8: Left subtree keys {1, 3, 4, 6, 7} — all < 8 ✓. Right subtree keys {10, 13, 14} — all > 8 ✓.
  • Node 3: Left subtree keys {1} — all < 3 ✓. Right subtree keys {4, 6, 7} — all > 3 ✓.
  • Node 6: Left subtree keys {4} — all < 6 ✓. Right subtree keys {7} — all > 6 ✓.
  • Node 10: No left subtree. Right subtree keys {13, 14} — all > 10 ✓.
  • Node 14: Left subtree keys {13} — all < 14 ✓. No right subtree ✓.
  • Leaves (1, 4, 7, 13): No subtrees to violate — trivially satisfy the property ✓.

Invalid BST Example (Non-Example)

10 / \ 5 15 / \ / 2 12 13

❌ Violation

Node 12 is in the left subtree of node 10 (it is the right child of 5, which is the left child of 10). However, 12 > 10, which violates the BST property. The BST property requires that all keys in the left subtree of 10 must be less than 10 — including grandchildren and beyond.

Note that locally, node 5 looks correct: left child 2 < 5 and right child 12 > 5. This is why local-only checking is insufficient.

2. Inorder Traversal and Sorted Output

The Key Insight

✨ Fundamental Theorem of BSTs

An inorder traversal (Left → Root → Right) of a BST visits the keys in sorted ascending order.

Why? By the BST property, for any node x:

  1. All keys in the left subtree are less than x.key — and the inorder traversal visits them before x.
  2. The root x is visited next.
  3. All keys in the right subtree are greater than x.key — and the inorder traversal visits them after x.

By induction on the structure of the tree, the entire sequence is sorted.

Example: Inorder on Our Valid BST

Performing inorder traversal on the BST from Section 1:

8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13

Inorder result: 1, 3, 4, 6, 7, 8, 10, 13, 14 — sorted! ✓

🔗 Connection to Module 15.2

In Module 15.2, you learned to perform inorder traversals on general binary trees. Now you see why inorder is special for BSTs: it recovers the sorted order. This connection is also the basis for one of our BST validation methods (Section 6).

Practical Applications

  • Sorting: Insert n keys into a BST, then perform inorder traversal → sorted output. (This is essentially tree sort.)
  • Range queries: Find all keys between a and b by performing a modified inorder traversal.
  • Validation: If inorder traversal does not produce sorted output, the tree is not a valid BST.

3. BST Search

Algorithm

Searching for a key k in a BST exploits the BST property to eliminate half the remaining tree at each step:

ALGORITHM BSTSearch(node, key):
    IF node = NULL THEN
        RETURN NULL              ← key not found
    IF key = node.key THEN
        RETURN node              ← found it
    ELSE IF key < node.key THEN
        RETURN BSTSearch(node.left, key)   ← search left subtree
    ELSE
        RETURN BSTSearch(node.right, key)  ← search right subtree

Time complexity: O(h) where h is the height of the tree. Each recursive call moves one level deeper.

Worked Example

✏️ Try it first!

Using the BST below, trace the search for key 7. Write down which nodes you visit and which direction you go at each step.

8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13
Reveal Search Trace for key = 7
  1. Compare 7 with root 8: 7 < 8 → go left
  2. Compare 7 with node 3: 7 > 3 → go right
  3. Compare 7 with node 6: 7 > 6 → go right
  4. Compare 7 with node 7: 7 = 7 → FOUND!

Nodes visited: 8 → 3 → 6 → 7 (4 comparisons, depth = 3).

Reveal Search Trace for key = 5 (not in tree)
  1. Compare 5 with root 8: 5 < 8 → go left
  2. Compare 5 with node 3: 5 > 3 → go right
  3. Compare 5 with node 6: 5 < 6 → go left
  4. Compare 5 with node 4: 5 > 4 → go right
  5. Node 4 has no right child: NULL → NOT FOUND

Nodes visited: 8 → 3 → 6 → 4 → NULL (4 comparisons + 1 null check).

4. BST Insertion

Algorithm

Insertion follows the same compare-and-branch logic as search. When we reach a NULL position, that is where the new node belongs:

ALGORITHM BSTInsert(node, key):
    IF node = NULL THEN
        RETURN new Node(key)     ← create leaf node here
    IF key < node.key THEN
        node.left ← BSTInsert(node.left, key)
    ELSE IF key > node.key THEN
        node.right ← BSTInsert(node.right, key)
    ELSE
        ← key already exists; handle as needed (ignore/update)
    RETURN node

Key observation: New nodes are always inserted as leaf nodes. The existing tree structure is not modified.

Time complexity: O(h) — same as search.

✅ Result

This insertion order produces a perfectly balanced BST with height 2. With 7 nodes, the minimum possible height is ⌊log₂ 7⌋ = 2. Searching any key requires at most 3 comparisons.

The Degenerate Worst Case

✏️ Try it first!

Now insert the same 7 keys in sorted order: 1, 3, 4, 5, 6, 7, 8. Draw the resulting BST.

Reveal Degenerate BST Construction
1 \ 3 \ 4 \ 5 \ 6 \ 7 \ 8

Each key is larger than all previous keys, so every new node is inserted as the right child of the current rightmost node. The result is a linked list — not a tree in any useful sense!

⚠️ Worst Case

Height = 6 (for 7 nodes). Searching for key 8 requires 7 comparisons — every single node must be visited. This is O(n) performance, no better than a linked list. Compare this to the balanced version above where the same search takes only 3 comparisons.

💡 Critical Insight

The shape of a BST depends entirely on the insertion order. The same set of keys can produce a balanced tree (height ≈ log n) or a degenerate chain (height = n−1). This is the fundamental motivation for self-balancing trees like AVL trees (Week 16).

5. BST Deletion — Three Cases

Deletion is the most complex BST operation because removing a node must preserve the BST property. There are three cases based on the number of children the target node has:

Case (a): Deleting a Leaf Node (0 children)

Strategy: Simply remove the node. No restructuring needed.

CASE A - Leaf Deletion:
    IF node.left = NULL AND node.right = NULL THEN
        Replace node with NULL in its parent

Example: Delete 4 from the balanced BST

Before: Delete 4
5 / \ 3 7 / \ / \ 1 4 6 8
After
5 / \ 3 7 / / \ 1 6 8

Node 4 is a leaf — remove it directly. The BST property is preserved.

Case (b): Deleting a Node with One Child

Strategy: Replace the node with its only child. The child "moves up" to take the deleted node's position.

CASE B - One Child:
    IF node has exactly one child (left OR right) THEN
        Replace node with its child in the parent's link

Example: Delete 3 from the tree (after we already removed 4)

Before: Delete 3
5 / \ 3 7 / / \ 1 6 8

Node 3 has only a left child (1).

After
5 / \ 1 7 / \ 6 8

Node 3 is removed, and its only child (1) takes its place as the left child of 5. BST property preserved: 1 < 5 ✓.

Case (c): Deleting a Node with Two Children

Strategy: Find the node's inorder successor (smallest key in the right subtree) OR inorder predecessor (largest key in the left subtree). Copy that key into the target node, then delete the successor/predecessor (which will be Case a or b).

CASE C - Two Children:
    1. Find the INORDER SUCCESSOR of node:
       successor ← minimum node in node.right subtree
       (go right once, then go left as far as possible)
    2. Copy successor.key into node.key
    3. Delete the successor node from node.right subtree
       (successor has at most one child — its right child —
        so this is Case A or Case B)

💡 Why does this work?

The inorder successor is the smallest key that is still greater than the deleted node's key. Replacing the deleted key with the successor's key maintains the BST property: everything in the left subtree is still smaller, and everything remaining in the right subtree is still larger.

Example: Delete 5 (the root) from the original balanced BST

✏️ Try it first!

Starting with the balanced BST [5, 3, 7, 1, 4, 6, 8], delete the root (5). Find its inorder successor, replace, and delete. Draw the final tree.

Reveal Deletion of Root (Two Children)
5 ← delete this node (has two children) / \ 3 7 / \ / \ 1 4 6 8

Step 1: Find inorder successor of 5. Go to right child (7), then go left as far as possible → node 6. The inorder successor is 6.

Step 2: Copy 6 into the root position (replacing 5).

Step 3: Delete the original node 6 from the right subtree. Node 6 was a leaf (Case a) → simply remove it.

6 / \ 3 7 / \ \ 1 4 8

Verify BST property:

  • Node 6: Left subtree {1,3,4} all < 6 ✓; Right subtree {7,8} all > 6 ✓
  • Node 3: Left {1} < 3 ✓; Right {4} > 3 ✓
  • Node 7: No left child; Right {8} > 7 ✓

Complete Deletion Algorithm

ALGORITHM BSTDelete(node, key):
    IF node = NULL THEN
        RETURN NULL                          ← key not found

    IF key < node.key THEN
        node.left ← BSTDelete(node.left, key)
    ELSE IF key > node.key THEN
        node.right ← BSTDelete(node.right, key)
    ELSE
        ← Found the node to delete
        IF node.left = NULL THEN
            RETURN node.right                ← Case A or B (no left child)
        ELSE IF node.right = NULL THEN
            RETURN node.left                 ← Case B (no right child)
        ELSE
            ← Case C: two children
            successor ← FindMin(node.right)
            node.key ← successor.key
            node.right ← BSTDelete(node.right, successor.key)

    RETURN node

ALGORITHM FindMin(node):
    WHILE node.left ≠ NULL DO
        node ← node.left
    RETURN node

6. BST Validation

Given a binary tree, how do we verify that it satisfies the BST property? There are two standard approaches.

Method 1: Inorder Traversal Check

Since an inorder traversal of a valid BST produces sorted output, we can:

  1. Perform an inorder traversal, collecting keys into a list.
  2. Check that the list is in strictly ascending order.
ALGORITHM ValidateBST_Inorder(root):
    result ← []
    InorderCollect(root, result)
    FOR i FROM 1 TO result.length - 1 DO
        IF result[i] ≤ result[i-1] THEN
            RETURN false
    RETURN true

ALGORITHM InorderCollect(node, result):
    IF node = NULL THEN RETURN
    InorderCollect(node.left, result)
    result.append(node.key)
    InorderCollect(node.right, result)

Time: $$O(n)$$. Space: $$O(n)$$ for the list (can be optimized to $$O(h)$$ by tracking only the previous value).

Method 2: Recursive Min/Max Bound Checking

Pass down valid bounds as you recurse. Each node must fall within the allowed range:

ALGORITHM ValidateBST_Bounds(node, minBound, maxBound):
    IF node = NULL THEN
        RETURN true
    IF node.key ≤ minBound OR node.key ≥ maxBound THEN
        RETURN false
    RETURN ValidateBST_Bounds(node.left, minBound, node.key)
        AND ValidateBST_Bounds(node.right, node.key, maxBound)

― Initial call: ValidateBST_Bounds(root, -∞, +∞)

Time: $$O(n)$$. Space: $$O(h)$$ (recursion stack only — more space-efficient).

The Common Mistake: Local-Only Checking

❌ WRONG Approach

WRONG_Validate(node):
    IF node = NULL THEN RETURN true
    IF node.left ≠ NULL AND node.left.key ≥ node.key THEN
        RETURN false
    IF node.right ≠ NULL AND node.right.key ≤ node.key THEN
        RETURN false
    RETURN WRONG_Validate(node.left) AND WRONG_Validate(node.right)

This only checks immediate children against the parent. It would incorrectly validate our invalid example from Section 1:

10 / \ 5 15 / \ / 2 12 13

The local-only check passes every node:

  • Node 10: left child 5 < 10 ✓, right child 15 > 10 ✓
  • Node 5: left child 2 < 5 ✓, right child 12 > 5 ✓
  • Node 15: left child 13 < 15 ✓

But 12 is in the left subtree of 10 and 12 > 10 — INVALID! The min/max bound method catches this because when we recurse into node 5's right subtree, we pass maxBound = 10. Node 12 fails the check: 12 ≥ 10 → false.

7. Complexity Analysis and Worst-Case Motivation

Time Complexity Summary

All three core BST operations have the same complexity — they all traverse a root-to-leaf (or root-to-node) path:

Operation Best Case (Balanced) Average Case Worst Case (Degenerate)
Search O(log n) O(log n) O(n)
Insert O(log n) O(log n) O(n)
Delete O(log n) O(log n) O(n)

Why O(log n) in the balanced case? A balanced binary tree with n nodes has height h = O(log n). Since each operation follows a single path from root to leaf, it takes at most h + 1 comparisons.

Why O(n) in the worst case? A degenerate BST (linked list shape) has height h = n − 1. Following a path from root to the deepest node visits all n nodes.

Visual Comparison: Balanced vs. Degenerate

Balanced BST (h = 2, n = 7)
5 / \ 3 7 / \ / \ 1 4 6 8

Height = 2 = ⌊log₂ 7⌋
Max comparisons: 3
Search time: O(log n)

Degenerate BST (h = 6, n = 7)
1 \ 3 \ 4 \ 5 \ 6 \ 7 \ 8

Height = 6 = n − 1
Max comparisons: 7
Search time: O(n)

🤔 The Fundamental Problem

A plain BST provides no height guarantee. The shape depends entirely on the insertion order. In the worst case (sorted input), we get O(n) operations — no better than a linked list. We need a mechanism to automatically maintain balance regardless of insertion order. This is exactly what AVL trees provide.

Space Complexity

Component Space
Storage (all nodes) O(n)
Recursive operations (stack depth) O(h)
Iterative operations O(1)

8. Preview: AVL Trees (Week 16)

🔮 Coming Next Week

In Week 16, you will learn about AVL trees — self-balancing BSTs invented by Adelson-Velsky and Landis. An AVL tree maintains a balance factor (the difference in height between left and right subtrees) of at most 1 for every node. When an insertion or deletion violates this constraint, the tree performs rotations (single or double) to restore balance. The result: AVL trees guarantee O(log n) height — and therefore O(log n) search, insert, and delete — regardless of the insertion order. No more degenerate linked-list trees!

9. Code Implementation (Java / C++)

Below are complete implementations of the BST Node class and all operations covered in this module. Both languages use the same recursive approach.

Java Implementation

Node Class

public class BSTNode {
    int key;
    BSTNode left;
    BSTNode right;

    public BSTNode(int key) {
        this.key = key;
        this.left = null;
        this.right = null;
    }
}

Search

public class BST {
    private BSTNode root;

    public BST() {
        this.root = null;
    }

    // Recursive search: returns the node if found, null otherwise
    public BSTNode search(BSTNode node, int key) {
        if (node == null) {
            return null;  // key not found
        }
        if (key == node.key) {
            return node;  // found
        } else if (key < node.key) {
            return search(node.left, key);  // search left
        } else {
            return search(node.right, key); // search right
        }
    }

Insert

    // Recursive insert: returns the (possibly new) root of the subtree
    public BSTNode insert(BSTNode node, int key) {
        if (node == null) {
            return new BSTNode(key);  // create new leaf
        }
        if (key < node.key) {
            node.left = insert(node.left, key);
        } else if (key > node.key) {
            node.right = insert(node.right, key);
        }
        // if key == node.key, duplicate; do nothing
        return node;
    }

    public void insert(int key) {
        root = insert(root, key);
    }

Delete

    // Find the minimum node in a subtree
    private BSTNode findMin(BSTNode node) {
        while (node.left != null) {
            node = node.left;
        }
        return node;
    }

    // Recursive delete: returns the (possibly new) root of the subtree
    public BSTNode delete(BSTNode node, int key) {
        if (node == null) {
            return null;  // key not found
        }

        if (key < node.key) {
            node.left = delete(node.left, key);
        } else if (key > node.key) {
            node.right = delete(node.right, key);
        } else {
            // Found the node to delete
            // Case A: leaf node (both children null)
            // Case B: one child
            if (node.left == null) {
                return node.right;
            } else if (node.right == null) {
                return node.left;
            }
            // Case C: two children
            BSTNode successor = findMin(node.right);
            node.key = successor.key;
            node.right = delete(node.right, successor.key);
        }
        return node;
    }

    public void delete(int key) {
        root = delete(root, key);
    }

Validate BST (Min/Max Bound Method)

    // Recursive validation using bounds
    public boolean isValidBST(BSTNode node, int min, int max) {
        if (node == null) {
            return true;
        }
        if (node.key <= min || node.key >= max) {
            return false;
        }
        return isValidBST(node.left, min, node.key)
            && isValidBST(node.right, node.key, max);
    }

    public boolean isValidBST() {
        return isValidBST(root, Integer.MIN_VALUE, Integer.MAX_VALUE);
    }
}
C++ Implementation

Node Struct

#include <iostream>
#include <climits>
using namespace std;

struct BSTNode {
    int key;
    BSTNode* left;
    BSTNode* right;

    BSTNode(int k) : key(k), left(nullptr), right(nullptr) {}
};

Search

// Recursive search: returns pointer to node if found, nullptr otherwise
BSTNode* search(BSTNode* node, int key) {
    if (node == nullptr) {
        return nullptr;  // key not found
    }
    if (key == node->key) {
        return node;  // found
    } else if (key < node->key) {
        return search(node->left, key);  // search left
    } else {
        return search(node->right, key); // search right
    }
}

Insert

// Recursive insert: returns the (possibly new) root of the subtree
BSTNode* insert(BSTNode* node, int key) {
    if (node == nullptr) {
        return new BSTNode(key);  // create new leaf
    }
    if (key < node->key) {
        node->left = insert(node->left, key);
    } else if (key > node->key) {
        node->right = insert(node->right, key);
    }
    // if key == node->key, duplicate; do nothing
    return node;
}

Delete

// Find the minimum node in a subtree
BSTNode* findMin(BSTNode* node) {
    while (node->left != nullptr) {
        node = node->left;
    }
    return node;
}

// Recursive delete: returns the (possibly new) root of the subtree
BSTNode* deleteNode(BSTNode* node, int key) {
    if (node == nullptr) {
        return nullptr;  // key not found
    }

    if (key < node->key) {
        node->left = deleteNode(node->left, key);
    } else if (key > node->key) {
        node->right = deleteNode(node->right, key);
    } else {
        // Found the node to delete
        // Case A: leaf node OR Case B: one child
        if (node->left == nullptr) {
            BSTNode* temp = node->right;
            delete node;
            return temp;
        } else if (node->right == nullptr) {
            BSTNode* temp = node->left;
            delete node;
            return temp;
        }
        // Case C: two children
        BSTNode* successor = findMin(node->right);
        node->key = successor->key;
        node->right = deleteNode(node->right, successor->key);
    }
    return node;
}

Validate BST (Min/Max Bound Method)

// Recursive validation using bounds
bool isValidBST(BSTNode* node, long minBound, long maxBound) {
    if (node == nullptr) {
        return true;
    }
    if (node->key <= minBound || node->key >= maxBound) {
        return false;
    }
    return isValidBST(node->left, minBound, node->key)
        && isValidBST(node->right, node->key, maxBound);
}

bool isValidBST(BSTNode* root) {
    return isValidBST(root, LONG_MIN, LONG_MAX);
}

📝 C++ Memory Note

Unlike Java, C++ requires manual memory management. In the delete operation, we call delete node; to free memory. In production code, consider using smart pointers (std::unique_ptr) to avoid memory leaks.
