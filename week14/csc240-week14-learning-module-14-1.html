1 · Where We Are

Over the past thirteen weeks you have built a substantial toolkit: propositional and predicate logic, proof techniques, sets, functions, relations, counting, recursion, and—most recently in Weeks 12–13—the rich world of graphs and trees. You learned that a tree is a connected, acyclic graph, and you explored traversals (BFS, DFS) that let you walk through a graph systematically. This week we ask a deceptively simple question: Given a weighted graph, how do we connect every vertex at the lowest total cost?

That question is not academic trivia. Imagine a telecommunications company laying fiber-optic cable between cities. Each potential cable route has a cost (trenching, permits, distance). The company wants every city reachable from every other city, but it also wants to minimize total expenditure. This is the Minimum Spanning Tree (MST) problem, and it arises in network design, circuit wiring, pipeline routing, image segmentation, and even clustering in machine learning.

In this opening module we establish the greedy paradigm—a powerful algorithm-design strategy—and formally define the MST problem. We then prove the Cut Property, the theoretical engine behind every MST algorithm you will study in Modules 14.2–14.3. By the end of this page you should be able to explain why greedy choices work for MSTs, setting the stage for Kruskal's and Prim's algorithms next.

2 · The Greedy Paradigm

2.1 What Does "Greedy" Mean?

A greedy algorithm builds a solution one piece at a time, always choosing the piece that looks best right now, without reconsidering earlier choices. In everyday language: "Take the best available option at each step and never look back."

Greedy Paradigm. An algorithm design strategy in which, at every decision point, the locally optimal choice is made with the expectation that these local optima combine into a globally optimal solution.

Not every problem yields to greediness. For a greedy algorithm to produce an optimal result, the problem must exhibit two structural properties:

Two Key Ingredients

  1. Greedy-Choice Property — A globally optimal solution can be assembled by making locally optimal (greedy) choices. In other words, the choice that looks best right now is safe; it will not prevent us from reaching the overall optimum.
  2. Optimal Substructure — After making a greedy choice, the remaining sub-problem is itself an optimization problem of the same type, and an optimal solution to the whole includes an optimal solution to the sub-problem.

2.2 The Coin-Change Analogy

Suppose you are a cashier in the United States and need to give a customer 87 cents in change using the fewest coins possible. The greedy approach: always pick the largest denomination that does not exceed the remaining amount.

🪙 Greedy Coin-Change Trace (U.S. denominations: 25¢, 10¢, 5¢, 1¢)

Remaining: 87¢ → take 25¢ → 62¢ → take 25¢ → 37¢ → take 25¢ → 12¢ → take 10¢ → 2¢ → take 1¢ → 1¢ → take 1¢ → 0¢.

Result: 6 coins (3 quarters, 1 dime, 2 pennies). This is optimal for U.S. coins.

⚠️ When Greedy Fails — A Cautionary Example

Imagine a fictional currency with denominations 1¢, 3¢, and 4¢. To make 6¢:

  • Greedy: 4¢ + 1¢ + 1¢ = 3 coins.
  • Optimal: 3¢ + 3¢ = 2 coins.

The greedy choice (pick 4¢ first) is not safe here because the denomination set lacks the greedy-choice property. Problems like this require dynamic programming to solve optimally.

2.3 Greedy vs. Brute Force vs. Dynamic Programming

Brute Force examines every possible solution and selects the best. It always finds the optimum but is typically far too slow for large inputs. For an MST on a dense graph with n vertices, the number of spanning trees can grow exponentially (Cayley's formula gives nn−2 labeled trees on Kn).

Dynamic Programming (DP) breaks a problem into overlapping sub-problems, solves each once, and stores results to avoid redundant work. DP is more efficient than brute force and handles problems where greedy fails (e.g., the fictional coin-change example). However, DP typically requires more memory and a more complex formulation than greedy.

Greedy is the simplest and often the fastest of the three, but it only works when the problem structure guarantees that local optima lead to a global optimum. When it works, greedy is elegant and efficient. The MST problem is a classic success story for greedy algorithms.

3 · MST Definitions and Properties

3.1 Recall: Trees (Week 12 Connection)

In Week 12 you learned that a tree is a connected, acyclic, undirected graph. Key facts we will rely on:

  • A tree on n vertices has exactly n − 1 edges.
  • Between any two vertices in a tree there is a unique path.
  • Removing any edge disconnects the tree; adding any edge creates exactly one cycle.

3.2 Spanning Tree

Spanning Tree. Let G = (V, E) be a connected, undirected graph. A spanning tree of G is a subgraph T = (V, ET) that is a tree and includes every vertex of G. Because T is a tree on |V| vertices, it has exactly |V| − 1 edges.

3.3 Minimum Spanning Tree

Minimum Spanning Tree (MST). Let G = (V, E, w) be a connected, undirected, weighted graph where w : E → ℝ assigns a weight (cost) to each edge. A minimum spanning tree of G is a spanning tree T whose total weight

w(T) = Σe ∈ ET w(e)

is as small as possible among all spanning trees of G.

3.4 Existence and Uniqueness

Existence — Why does an MST always exist?

Every connected graph has at least one spanning tree (you can obtain one by repeatedly removing edges from cycles until none remain). Since the number of spanning trees is finite and each has a well-defined total weight, at least one must achieve the minimum. Therefore an MST always exists for a connected weighted graph.

Uniqueness — Is the MST always unique?

Not necessarily. If two or more edges share the same weight, there may be multiple distinct MSTs with the same total cost. However, if all edge weights are distinct, then the MST is unique. (This can be proved using the Cut Property, which we develop next.)

3.5 Example Graph

We will use the following small weighted graph throughout this module and the rest of Week 14.

Work It Out — What is the MST of this graph?

Sort edges by weight: A–C (1), B–E (2), B–C (3), A–B (4), C–D (5), C–E (6), D–E (7).

We need |V| − 1 = 4 edges. Picking the lightest edges that avoid cycles:

  1. A–C (1) — connects A and C. ✓
  2. B–E (2) — connects B and E. ✓
  3. B–C (3) — connects components {A,C} and {B,E}. ✓
  4. C–D (5) — connects D to the rest. ✓

(We skipped A–B (4) because it would create a cycle — A and B were already connected through A–C–B. C–E (6) and D–E (7) were not needed once all vertices were reached.)

4 · The Cut Property

The Cut Property is the theoretical foundation for greedy MST algorithms. It tells us exactly when it is safe to include an edge in the MST—i.e., when a greedy choice is guaranteed to be correct.

4.1 Definitions

Cut. A cut of a connected graph G = (V, E) is a partition of the vertex set into two non-empty subsets (S, V ∖ S).

Crossing Edge. An edge {u, v} ∈ E crosses the cut (S, V ∖ S) if one endpoint is in S and the other is in V ∖ S.

4.2 Statement of the Cut Property

Theorem (Cut Property)

Let G = (V, E, w) be a connected, weighted, undirected graph. For any cut (S, V ∖ S), if edge e is the unique minimum-weight crossing edge, then e belongs to every MST of G.

(If there are ties for the minimum crossing edge, at least one of them is in some MST.)

4.3 Intuitive Explanation

Think of the cut as a wall dividing the vertices into two rooms. To build a spanning tree you must include at least one edge that crosses this wall (otherwise the two rooms are disconnected). Among all edges crossing the wall, the lightest one is the cheapest way to "bridge" the two sides. Choosing any heavier crossing edge instead would only increase the total cost, so the lightest crossing edge is always a safe greedy pick.

4.4 Proof Sketch (by Contradiction)

Expand proof sketch

Setup. Suppose, for contradiction, that e = {u, v} is the unique minimum-weight edge crossing cut (S, V ∖ S), but there exists an MST T* that does not contain e.

  1. Since T* is a spanning tree, it connects u and v via some path P. Because u ∈ S and v ∈ V ∖ S, the path P must cross the cut at least once. Let e′ = {x, y} be a crossing edge on P (with e′ ≠ e).
  2. Because e is the unique minimum-weight crossing edge, we have w(e) < w(e′).
  3. Construct a new tree T′ = T* − e′ + e. Removing e′ from T* splits it into two components; adding e reconnects them (since e also crosses the cut). So T′ is a spanning tree.
  4. w(T′) = w(T*) − w(e′) + w(e) < w(T*).
  5. This contradicts the assumption that T* is a minimum spanning tree. ∎

4.5 Cut Property on Our Example

Worked Example — Applying the Cut Property to Figure 1

Consider the cut S = {A}, V ∖ S = {B, C, D, E}.

Crossing edges: A–B (weight 4) and A–C (weight 1).

The unique minimum-weight crossing edge is A–C (1). By the Cut Property, A–C must be in every MST of G. ✓ (Consistent with our MST found earlier.)

Now consider the cut S = {A, C}, V ∖ S = {B, D, E}.

Crossing edges: A–B (4), B–C (3), C–D (5), C–E (6).

The unique minimum-weight crossing edge is B–C (3). By the Cut Property, B–C must be in every MST. ✓

5 · Preview: Three Greedy Algorithms

In Modules 14.2 and 14.3 you will study two MST algorithms in detail, and in Module 14.4 you will study a shortest-path algorithm. Here is a compact preview so you can see how the greedy paradigm manifests differently in each.

Table 1. Three Greedy Graph Algorithms at a Glance
Algorithm Problem Solved Greedy Choice Module
Kruskal's Minimum Spanning Tree Always add the lightest edge that does not create a cycle (global edge sort). 14.2
Prim's Minimum Spanning Tree Always add the lightest edge connecting a visited vertex to an unvisited vertex (grow a single tree). 14.3
Dijkstra's Single-Source Shortest Paths Always process the unvisited vertex with the smallest tentative distance from the source. 14.4
💡 Key Insight

Both Kruskal's and Prim's solve the same problem (MST) but use different greedy strategies. Their correctness both follow from the Cut Property. Dijkstra's solves a different problem (shortest paths, not MST) and relies on a related but distinct argument.

6 · Guided Visualizer Activity

🖥️ Tool Reference

For this activity, use the VisuAlgo MST visualization tool at visualgo.net/en/mst — open it in a separate browser tab. (Do not worry if the interface has minor updates; the core functionality is stable.)

Follow these steps to build intuition before we formalize the algorithms in Modules 14.2–14.3.

Step-by-Step Instructions

  1. Open the tool and select or draw a small graph with 5–7 vertices and weighted edges. (You can use the "Example Graphs" dropdown or create a custom one.)
  2. Run Kruskal's algorithm using the controls. At each step, observe:
    • Which edge is being considered? Is it the lightest remaining edge?
    • Is the edge accepted or rejected? Why? (Hint: cycles.)
    • How many edges are in the MST so far?
  3. Reset the visualization and run Prim's algorithm on the same graph. Observe:
    • The algorithm grows a single tree from a starting vertex.
    • At each step, which edge is added? It is the lightest edge from the "visited" set to the "unvisited" set.
    • Can you identify the cut being used at each step?
  4. Compare results. Both algorithms should produce an MST with the same total weight (and often the same tree, especially if edge weights are distinct).
  5. Experiment: Try a graph where two edges have the same weight. Do Kruskal's and Prim's produce different MSTs? Do those MSTs have the same total weight?
What to Look For — Key Observations
  • Kruskal's processes edges globally (sorted by weight) and may grow multiple disconnected components that merge over time.
  • Prim's always maintains one connected component that expands outward, like a growing blob.
  • Both algorithms add exactly |V| − 1 edges total.
  • Every edge added by either algorithm can be justified by the Cut Property applied to an appropriate cut.
