1. MST vs. Shortest Path — Two Optimization Goals

Kruskal's, Prim's, and Dijkstra's are all greedy algorithms that operate on weighted graphs, but they solve fundamentally different problems. Confusing them is one of the most common mistakes on exams. This section gives you a clear mental model for telling them apart.

🌲 Minimum Spanning Tree (MST)

Goal: Connect all vertices with the minimum total edge weight.

Output: A tree (connected, acyclic subgraph) containing every vertex and exactly $$|V| - 1$$ edges.

Analogy — City Water-Pipe Network: A city needs to connect every house to the water supply. Pipes are expensive, so the city wants to minimize the total length of pipe laid. The resulting network is a tree — no redundant loops.

Algorithms: Kruskal's, Prim's

Graph requirement: Connected, weighted, undirected.

🗺️ Single-Source Shortest Path (SSSP)

Goal: Find the cheapest path from one source to every other vertex individually.

Output: A distance array $$d[v]$$ and predecessor pointers; the shortest-path tree is generally not an MST.

Analogy — Delivery Driver: A driver at a warehouse needs the fastest route to each customer. The driver does not care about total road length in the city — only about minimizing each individual trip.

Algorithm: Dijkstra's (non-negative weights)

Graph requirement: Weighted (directed or undirected), non-negative edge weights.

DimensionMST (Kruskal's / Prim's)SSSP (Dijkstra's)
Optimization targetMinimize total edge weight of spanning treeMinimize individual path weight from source to each vertex
Output structureA single spanning tree ($$|V|-1$$ edges)Shortest-path tree rooted at source (may differ from MST)
Directed graphs?No — MST is defined for undirected graphsYes — works on directed or undirected
Negative weights?Allowed (MST still well-defined)Not allowed for Dijkstra's
Greedy choiceCheapest edge that doesn't form a cycle (Kruskal's) or cheapest edge crossing the cut (Prim's)Unvisited vertex with smallest tentative distance
Data structureUnion-Find (Kruskal's) or min-heap (Prim's)Min-heap (priority queue)

Imagine a 4-vertex graph with vertices A, B, C, D and the following edges and weights:

  • A–B: 1,   A–C: 4,   B–C: 2,   B–D: 5,   C–D: 3

MST (minimum total weight): select A–B (1), B–C (2), C–D (3). Total weight = 6. This tree connects all four vertices as cheaply as possible.

Shortest paths from A (Dijkstra's): A→B = 1 (via A–B), A→C = 3 (via A–B–C), A→D = 6 (via A–B–C–D). The shortest-path tree uses the same edges in this small example, but in general the two trees differ because MST does not care about distance from a single source — it cares about total cost.

If we change A–C to weight 2, the MST might include A–C instead of B–C, but Dijkstra's from A would still route through B–C if that path is shorter to D. Different optimization goals → different trees.

2. Real-World Applications Gallery

The algorithms you learned this week are not textbook curiosities — they power critical infrastructure. Each card below describes a real application, identifies which algorithm applies, and explains why.

🌐

Fiber-Optic Network Design

Algorithm: MST (Kruskal's or Prim's)
Telecommunications companies must connect cities with fiber-optic cable. Each cable segment has an installation cost proportional to distance and terrain. An MST minimizes total cable cost while ensuring every city is reachable. This is the classic MST application — the problem that originally motivated the algorithm.

📍

GPS / Map Routing

Algorithm: Dijkstra's Shortest Path
When you ask a GPS for directions, the system models the road network as a weighted directed graph (intersections = vertices, road segments = edges, travel time = weights). Dijkstra's algorithm finds the fastest route from your current location to the destination. Production systems use optimized variants (A* search), but the core logic is Dijkstra's.

⚡

Electrical Grid Planning

Algorithm: MST
Power utilities connecting substations to a new development use MST to minimize the total length of high-voltage transmission lines. Redundancy is added later for reliability, but the initial backbone is an MST problem — connect everything at minimum total cost.

📡

OSPF Packet Routing

Algorithm: Dijkstra's Shortest Path
The Open Shortest Path First (OSPF) protocol, used in enterprise and ISP networks, runs Dijkstra's algorithm on each router. Each router builds a map of the network (a weighted directed graph where weights represent link costs) and computes shortest paths to every other router. Routing tables are built directly from Dijkstra's output.

✈️

Airline Route Optimization

Algorithm: Both MST and Dijkstra's
Airlines use MST-like analysis to decide which hub-and-spoke connections to maintain (minimize total operating cost for full network coverage). They use shortest-path algorithms to find minimum-cost itineraries for passengers with connections. The two problems — network design vs. individual routing — mirror the MST vs. SSSP distinction exactly.

🤝

Social Network Analysis

Algorithm: Dijkstra's (weighted) or BFS (unweighted)
In a social network, the "shortest path" between two users represents degrees of separation. If edges are weighted by interaction frequency (stronger ties = lower weight), Dijkstra's finds the strongest-connection path. This is used in friend recommendation, influence propagation modeling, and community detection algorithms.

Knowledge Check 14.5a — MST vs. Shortest Path

A city government wants to lay fiber-optic cable connecting 12 municipal buildings. The goal is to minimize the total cost of cable installation while ensuring every building can communicate with every other building. Which algorithm is most appropriate?

✓ Correct! This is an MST problem. The goal is to connect all buildings with minimum total cable cost. Kruskal's or Prim's algorithm produces a spanning tree that achieves this. Dijkstra's would minimize the path cost from one building to each other building individually — a different optimization goal.
✗ Not quite. The correct answer is (C). The key phrase is "minimize the total cost of cable installation while ensuring every building can communicate." This is the definition of the MST problem — connect all vertices with minimum total edge weight. Dijkstra's (A) minimizes individual path costs, not total network cost. BFS (B) works on unweighted graphs and doesn't optimize cost. DFS (D) is a traversal strategy, not an optimization algorithm.

Knowledge Check 14.5b — Final Project Component Mapping

A student is working on the Final Project and needs to implement a student-built priority queue (min-heap) that is used internally by Dijkstra's algorithm. Which Final Project component does this min-heap implementation satisfy?

✓ Correct! Component 4 (Tree-Based Internal Structure, 50 pts) requires a student-implemented tree-based data structure such as a min-heap, BST, or equivalent. A min-heap used as the priority queue for Dijkstra's or Prim's is a perfect fit. Note that the Dijkstra's algorithm itself belongs to Component 3 — but the priority queue it uses internally satisfies Component 4.
✗ Not quite. The correct answer is (D) Component 4: Tree-Based Internal Structure (50 pts). This component specifically requires a student-implemented tree-based structure (min-heap, BST, etc.) used internally by your algorithms. Component 1 is the graph data structure itself. Component 2 is BFS/DFS. Component 3 is the MST and Dijkstra's algorithms — the min-heap supports Component 3 but satisfies Component 4.

Knowledge Check 14.5c — Greedy Paradigm Identification

Consider the following description: "At each step, the algorithm selects the unvisited vertex with the smallest tentative distance from the source, marks it as visited, and updates the distances of its neighbors." Which greedy choice is being made, and which algorithm is this?

✓ Correct! The description matches Dijkstra's algorithm exactly. The greedy choice is: "always process the unvisited vertex with the smallest tentative distance." This is greedy because the algorithm commits to that vertex's distance being final without considering future edges. The correctness relies on the non-negative weight assumption — if all remaining edges are non-negative, no future path can improve the distance to this vertex. The optimal substructure property is: a shortest path to $$v$$ through $$u$$ consists of a shortest path to $$u$$ plus the edge $$(u, v)$$.
✗ Not quite. The correct answer is (C) Dijkstra's algorithm. The key clues are "unvisited vertex with the smallest tentative distance from the source" and "updates the distances of its neighbors" (the relaxation step). Kruskal's (A) processes edges sorted by weight. Prim's (B) selects the cheapest edge crossing the MST cut — it tracks edge weight to the MST, not distance from a source. BFS (D) uses a FIFO queue, not a priority queue, and does not consider weights.
