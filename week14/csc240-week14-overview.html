Graph Algorithms: Greedy Strategies, MSTs & Shortest Paths

From GPS navigation to network backbone design — this week you learn the algorithms that power them.

Welcome to Week 14

This week marks the algorithmic heart of CSC 240. You will explore the greedy paradigm — a powerful strategy for building optimal solutions one step at a time — and apply it to two classic problems: finding Minimum Spanning Trees (Kruskal's and Prim's algorithms) and computing shortest paths (Dijkstra's algorithm). These are the same ideas behind everyday technologies like GPS routing, internet packet delivery, and cost-efficient network design.

You will also kick off the Final Project this week, giving you a full four weeks to plan, build, and refine it. The project asks you to pull together concepts from across the course, so starting early matters.

⏱️ Honest Workload Note

Week 14 is the most algorithmically intensive week of the semester, with an estimated 9 hours of work. You have two deliverables (PE-09 and PS-09) plus the Final Project assignment to review. Budget your time early, follow the recommended module order below, and don't hesitate to ask questions in the discussion board. You've got this.

Week 14 Learning Goals

By the end of this week you will be able to:

  1. 1. Explain the greedy paradigm and identify problems where a greedy strategy yields an optimal solution.
  2. 2. Trace and implement Kruskal's algorithm using the Union-Find data structure to build a minimum spanning tree.
  3. 3. Trace and implement Prim's algorithm using a priority queue to build a minimum spanning tree.
  4. 4. Trace and implement Dijkstra's algorithm to find shortest paths in a weighted graph with non-negative edge weights.
  5. 5. Compare Kruskal's, Prim's, and Dijkstra's algorithms in terms of strategy, data structures, and time complexity.
  6. 6. Connect these graph algorithms to real-world applications such as network design, GPS routing, and infrastructure optimization.

Key Concepts & Themes

🎯

Greedy Paradigm

Make the locally optimal choice at each step and prove it leads to a globally optimal solution.

🌲

Minimum Spanning Trees

Connect every vertex at minimum total cost — the foundation of efficient network design.

✂️

Kruskal's Algorithm

Sort edges by weight and grow a forest, using Union-Find to avoid cycles.

🔗

Prim's Algorithm

Expand a single tree one vertex at a time, selecting the cheapest crossing edge via a priority queue.

🗺️

Dijkstra's Algorithm

Find shortest paths from a source vertex in graphs with non-negative weights — the engine behind GPS routing.

🧰

Supporting Structures

Union-Find (disjoint sets) and priority queues — essential data structures that make these algorithms efficient.

Time Management Guidance

💡 Recommended Approach

Work through Modules 14.1 – 14.5 in sequence. Each module builds on the previous one. Resist the urge to skip ahead to the assignments before you have the conceptual foundation.

  1. Step 1 — Foundations (Modules 14.1 – 14.2)
    Read the greedy paradigm overview, then study Kruskal's algorithm and Union-Find. Trace the examples by hand.
  2. Step 2 — Prim's Algorithm (Module 14.3)
    Study Prim's algorithm and priority queues. Compare it with Kruskal's approach. After completing this module, you have enough background to start PE-09.
  3. Step 3 — Dijkstra's Algorithm (Module 14.4)
    Work through Dijkstra's shortest-path algorithm. Note similarities with Prim's and differences in what each algorithm optimizes.
  4. Step 4 — Wrap-Up & Project Planning (Module 14.5)
    Review the algorithm comparison, explore real-world applications, and read the Final Project description. Begin your project planning.
  5. Step 5 — Complete & Submit
    Finish PE-09 (programming) and PS-09 (problem set). Review your work before submitting.
