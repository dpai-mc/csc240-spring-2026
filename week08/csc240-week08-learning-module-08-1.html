Binomial Coefficients and the Binomial Theorem

CSC 240 – Discrete Structures  |  Spring 2026  |  Mendocino College

Estimated time: 2.0 hours  |  CSLO 2: Apply combinatorics and set theory to counting problems.

Where We've Been — Where We're Going

In Week 7 you built the foundational toolkit of combinatorics: the sum and product rules, the inclusion-exclusion principle, permutations, combinations, and your first look at Pascal's triangle. You learned that the binomial coefficient C(n,k) counts the number of ways to choose k items from a set of n without regard to order, and that Pascal's identity connects adjacent entries of the triangle.

This module asks a natural follow-up question: what pattern appears in the coefficients when we expand (a+b)n for larger and larger n? The answer — the Binomial Theorem — reveals that the entries of row n in Pascal's triangle are exactly those coefficients. This is not a coincidence; it is a deep combinatorial fact that we will prove and then use to unlock powerful counting tools.

2 · Properties of Binomial Coefficients

Before stating the full theorem, let's establish three fundamental properties of binomial coefficients. Each has both a formal algebraic statement and an intuitive combinatorial explanation.

Property 1 — Symmetry Identity

C(n, k) = C(n, n−k)

Formal statement: For all integers 0 ≤ k ≤ n, C(n,k) = C(n, n−k).

Intuitive explanation: Choosing k items to include in a subset is equivalent to choosing the remaining n−k items to exclude. These two actions describe the same subset, so the counts must be equal.

Numeric example: C(7, 2) = 21 and C(7, 5) = 21. Choosing 2 items from 7 gives the same number of subsets as choosing which 5 items to leave out.

Property 2 — Row Sum Identity

C(n,0) + C(n,1) + C(n,2) + · · · + C(n,n) = 2n

Formal statement: The sum of all binomial coefficients in row n equals 2n.

Intuitive explanation: Every subset of an n-element set has some size k between 0 and n. Summing C(n,k) over all k counts all subsets — and we know an n-element set has exactly 2n subsets (each element is either in or out: 2 choices per element, n elements).

Numeric example: Row 4: 1 + 4 + 6 + 4 + 1 = 16 = 2⁴. ✓

Property 3 — Alternating Sum Identity

C(n,0) − C(n,1) + C(n,2) − · · · + (−1)nC(n,n) = 0

Formal statement: The alternating sum of row n equals 0 for all n ≥ 1.

Intuitive explanation: This says the number of even-sized subsets of an n-element set equals the number of odd-sized subsets. There is a bijection between them: for any subset S, toggle membership of a fixed element (say, element 1) to switch between even and odd size.

Numeric example: Row 4: 1 − 4 + 6 − 4 + 1 = 0. ✓ (Even-size subsets: ∅, {a,b}, {a,c}, {a,d}, {b,c}, {b,d}, {c,d}, {a,b,c,d} = 8. Odd-size subsets: {a},{b},{c},{d},{a,b,c},{a,b,d},{a,c,d},{b,c,d} = 8.)

💡 Connection to the Binomial Theorem
Both the row-sum and alternating-sum identities are special cases of the Binomial Theorem. Setting a = b = 1 gives the row sum; setting a = 1, b = −1 gives the alternating sum. You will see this formally in Section 5.

3 · The Binomial Theorem

Formal Statement

Theorem — The Binomial Theorem

Let a and b be any real numbers and let n be a non-negative integer. Then:

(a + b)n = C(n,0)·anb0 + C(n,1)·an−1b1 + C(n,2)·an−2b2 + · · · + C(n,n)·a0bn

Written in summation notation:

(a + b)n  =  ∑k=0n   C(n, k) · an−k · bk

Anatomy of the General Term

The (k+1)-th term of the expansion (counting from k = 0) is:

Tk+1  =  C(n, k)  ·  an−k  ·  bk
C(n,k) — binomial coefficient; the "weight" of this term
an−k — power of the first base; decreases as k increases
bk — power of the second base; increases as k increases
k — term index; runs from 0 to n

Why Does This Work? (Combinatorial Proof Sketch)

When we multiply out (a+b)(a+b)···(a+b) (n factors), each term in the product is formed by choosing either a or b from each factor:

  • If we choose b from exactly k of the n factors, we get a term an−kbk.
  • The number of ways to choose which k factors contribute the b is exactly C(n, k).
  • Therefore the coefficient of an−kbk in the expansion is C(n, k).

Since k ranges from 0 to n, summing over all choices gives the full expansion.

Worked Examples

Apply the theorem with a = x, b = y, n = 4:

  1. 1 List the terms for k = 0, 1, 2, 3, 4.
  2. 2 Compute each C(4, k): C(4,0)=1, C(4,1)=4, C(4,2)=6, C(4,3)=4, C(4,4)=1.
  3. 3 Write each term: C(4,k)·x4−k·yk.

4 · Finding a Specific Term

One of the most useful applications of the Binomial Theorem is locating a single term in a large expansion without computing the entire thing.

General Term Formula

The (k+1)-th term in the expansion of (a + b)n is:

Tk+1 = C(n, k) · an−k · bk

Important: Terms are indexed starting at k = 0, so the first term has k = 0, the second has k = 1, and so on. The r-th term corresponds to k = r − 1.

Strategy: Identify k by matching the power of b = y. Since we want y², we need k = 2.

  1. 1 n = 5, a = x, b = y, k = 2 (power of y).
  2. 2 Verify: an−k = x5−2 = x³ ✓ and bk = y² ✓
  3. 3 Coefficient: C(5, 2) = 10.
  4. 4 Term: T₃ = 10x³y².
The term containing x³y² in (x+y)⁵ is 10x³y².

4th term ↔ k = 3 (since the first term has k = 0).

  1. 1 n = 6, a = a, b = 2b, k = 3.
  2. 2 T4 = C(6, 3) · a6−3 · (2b)3
  3. 3 C(6, 3) = 20;   (2b)³ = 8b³.
  4. 4 T4 = 20 · a³ · 8b³ = 160a³b³.
The 4th term of (a + 2b)⁶ is 160a³b³.
⚠️ Common Mistake
When b has a coefficient (like 2b), you must raise the entire expression to the power k. (2b)³ = 8b³, not 2b³.

5 · Key Corollaries

Two especially important results follow directly from the Binomial Theorem by substituting specific values for a and b.

Corollary 1 — Subset Counting (a = b = 1)

Set a = 1 and b = 1 in the Binomial Theorem:

(1 + 1)n = C(n,0) + C(n,1) + C(n,2) + · · · + C(n,n)  =  2n

Combinatorial meaning: The total number of subsets of an n-element set is 2n. Each term C(n,k) counts the subsets of size exactly k; summing over all sizes gives all subsets.

Example: A set with 5 elements has C(5,0)+C(5,1)+C(5,2)+C(5,3)+C(5,4)+C(5,5) = 1+5+10+10+5+1 = 32 = 2⁵ subsets.

Corollary 2 — Alternating Sum (a = 1, b = −1)

Set a = 1 and b = −1:

(1 − 1)n = C(n,0) − C(n,1) + C(n,2) − · · · + (−1)nC(n,n)  =  0

Combinatorial meaning: For any set with n ≥ 1 elements, the number of even-sized subsets equals the number of odd-sized subsets.

Practical use: This identity is a quick sanity check when computing with binomial coefficients. If your alternating sum is non-zero, you have made an error.

Corollary 3 — The (1+x)n Form

Setting a = 1 gives a particularly clean form used throughout mathematics and computer science:

(1 + x)n = C(n,0) + C(n,1)x + C(n,2)x² + · · · + C(n,n)xⁿ

Why it matters: This is the generating function for the binomial coefficients in row n — a concept you will explore in Module 8.4. The coefficient of xk is exactly C(n, k).

7 · Java Code Snippet (Bridge to PE-07)

The collapsible panel below contains a Java method that computes C(n,k) using Pascal's triangle row construction — the same approach required in PE-07 Part 1. Study the inline comments carefully; understanding this method will save you significant time on the assignment.

This method builds row n of Pascal's triangle iteratively and returns the k-th entry. It avoids computing large factorials directly and handles integer overflow better than the naive factorial approach for moderate values of n. 

/**
 * Computes C(n, k) using Pascal's triangle row construction.
 * Builds row n iteratively; returns the k-th entry.
 * @param n  total items (n >= 0)
 * @param k  items chosen (0 <= k <= n)
 * @return   C(n, k) as a long to reduce overflow risk
 */
public static long binomialCoefficient(int n, int k) {
    // Edge cases
    if (k < 0 || k > n) return 0;
    if (k == 0 || k == n) return 1;

    // Use symmetry: C(n,k) == C(n, n-k)
    // Choosing the smaller k halves the work
    if (k > n - k) k = n - k;

    // Build only the first k+1 entries of row n
    // using Pascal's identity in place
    long[] row = new long[k + 1];
    row[0] = 1;  // C(n, 0) = 1 always

    for (int i = 1; i <= n; i++) {
        // Update from right to left to avoid overwriting
        // values we still need for this iteration
        for (int j = Math.min(i, k); j >= 1; j--) {
            row[j] += row[j - 1];
        }
    }
    return row[k];
}
⚠️ PE-07 Note — Integer Overflow
For large values of n (say, n > 60), even long can overflow. PE-07 requires you to handle this gracefully — consider printing a warning or using BigInteger for Part 1's overflow-safe version.
