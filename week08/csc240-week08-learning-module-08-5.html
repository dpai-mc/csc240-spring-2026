1 · What Is a Generating Function?

Motivation: One Expression, Infinitely Many Answers

Suppose you are asked a whole family of related counting questions: "How many $$k$$-element subsets does an $$n$$-element set have?" for every possible value of $$k$$. You already know the answer for each individual $$k$$: it is $$\binom{n}{k}$$. But what if you could package all of those answers — for every $$k$$ at once — into a single algebraic expression?

That is exactly what a generating function does. Think of it as a numbered filing cabinet: drawer 0 holds the answer for size 0, drawer 1 holds the answer for size 1, drawer 2 for size 2, and so on. The generating function is the label on the entire cabinet, and you retrieve an individual answer by looking at the coefficient of a particular power of $$x$$.

Formal Definition

Definition — Ordinary Generating Function

Let $$\{a_0, a_1, a_2, \ldots\}$$ be a sequence of numbers (often counting answers). The ordinary generating function of this sequence is the formal power series:

$$G(x) = \sum_{n=0}^{\infty} a_n \, x^n = a_0 + a_1 x + a_2 x^2 + a_3 x^3 + \cdots$$

The variable $$x$$ is a formal placeholder — we are not substituting a number into it. Its only role is to keep track of which coefficient belongs to which position in the sequence.

How to Read a Generating Function

✅ The Reading Rule

The answer to the counting question for size $$n$$ is the coefficient of $$x^n$$ in $$G(x)$$. We write this as $$[x^n]\,G(x) = a_n$$.

For example, if $$G(x) = 3 + 7x + 2x^2 + 5x^3 + \cdots$$, then:

  • $$[x^0]\,G(x) = 3$$ — the answer for size 0 is 3
  • $$[x^1]\,G(x) = 7$$ — the answer for size 1 is 7
  • $$[x^2]\,G(x) = 2$$ — the answer for size 2 is 2
  • $$[x^3]\,G(x) = 5$$ — the answer for size 3 is 5

⚠️ Important: x Is Not a Number
Unlike the $$x$$ in a polynomial equation that you might solve for, the $$x$$ in a generating function is a formal variable — a bookkeeping device. We never "plug in" a value for $$x$$ (at least not in this first-exposure module). The power of $$x$$ simply tells us which position in the sequence we are looking at.

2 · Two Fundamental Examples

Two generating functions appear throughout discrete mathematics and computer science so frequently that you should recognize them on sight. Both are direct consequences of ideas you have already studied this week.

Example 1 — $$G(x) = (1+x)^n$$: Encoding Binomial Coefficients

By the Binomial Theorem (Module 8.1), we can expand $$(1+x)^n$$ as:

$$(1+x)^n = \sum_{k=0}^{n} \binom{n}{k} x^k = \binom{n}{0} + \binom{n}{1}x + \binom{n}{2}x^2 + \cdots + \binom{n}{n}x^n$$

Comparing this with the definition $$G(x) = \sum_{k=0}^{\infty} a_k x^k$$, we can read off the sequence that $$G(x) = (1+x)^n$$ encodes:

Position $$k$$Term in $$(1+x)^n$$Coefficient $$a_k$$Counting Meaning
$$k=0$$$$\binom{n}{0}$$$$1$$Number of 0-element subsets of an $$n$$-set
$$k=1$$$$\binom{n}{1}x$$$$n$$Number of 1-element subsets
$$k=2$$$$\binom{n}{2}x^2$$$$\frac{n(n-1)}{2}$$Number of 2-element subsets
$$\vdots$$$$\vdots$$$$\vdots$$$$\vdots$$
$$k=n$$$$\binom{n}{n}x^n$$$$1$$Number of $$n$$-element subsets
✅ Key Reading Rule for This Example

The coefficient of $$x^k$$ in $$(1+x)^n$$ is $$\binom{n}{k}$$ — the number of $$k$$-element subsets of an $$n$$-element set.

Concrete example with $$n = 5$$:

$$(1+x)^5 = 1 + 5x + 10x^2 + 10x^3 + 5x^4 + x^5$$

Reading the coefficients: there is 1 subset of size 0 (the empty set), 5 subsets of size 1, 10 of size 2, 10 of size 3, 5 of size 4, and 1 of size 5. These are exactly the entries of row 5 in Pascal's triangle.

Notice also: the coefficients sum to $$1+5+10+10+5+1 = 32 = 2^5$$, confirming that a 5-element set has $$2^5$$ total subsets — the row-sum identity from Module 8.1.

Example 2 — $$G(x) = \dfrac{1}{1-x}$$: Encoding the All-Ones Sequence

Recall the geometric series formula: for $$|r| < 1$$, $$\sum_{n=0}^{\infty} r^n = \frac{1}{1-r}$$. In the generating-function setting we treat this as a formal identity — a relationship between power series that holds regardless of convergence concerns:

$$\frac{1}{1-x} = \sum_{n=0}^{\infty} x^n = 1 + x + x^2 + x^3 + x^4 + \cdots$$

The coefficient of every $$x^n$$ is 1. So the sequence encoded by $$G(x) = \frac{1}{1-x}$$ is $$\{1, 1, 1, 1, \ldots\}$$.

💡 Counting Interpretation

"The number of ways to choose exactly $$n$$ identical items from an unlimited supply is always 1." There is only one way to pick 0 items, one way to pick 1, one way to pick 2, and so on — because the items are indistinguishable. The generating function $$\frac{1}{1-x}$$ packages all these trivial answers into one compact expression.

Reading coefficients:

  • $$[x^0]\,\frac{1}{1-x} = 1$$
  • $$[x^5]\,\frac{1}{1-x} = 1$$
  • $$[x^{100}]\,\frac{1}{1-x} = 1$$

The geometric series generating function will become much more powerful in Week 11, where shifting and multiplying such series lets us solve recurrence relations. For now, simply recognize it and know how to read its coefficients.

  2. 2 Every coefficient is 1, so $$[x^4]\,\dfrac{1}{1-x} = 1$$.

3 · Connection to the Binomial Theorem

Now that you have seen both the Binomial Theorem (Module 8.1) and the definition of a generating function (Section 1), we can state precisely how they are related.

The Binomial Theorem Is a Generating-Function Statement

The Binomial Theorem says:

$$(1+x)^n = \sum_{k=0}^{n} \binom{n}{k} x^k$$

Viewed through the lens of generating functions, this equation says: $$(1+x)^n$$ is the generating function for the sequence of binomial coefficients $$\binom{n}{0}, \binom{n}{1}, \ldots, \binom{n}{n}$$.

In other words, the Binomial Theorem is not just an algebraic expansion rule — it is a generating-function identity that packages the entire row $$n$$ of Pascal's triangle into a single expression.

Reading the Binomial Theorem as a Generating Function

The table below translates each part of the Binomial Theorem into generating-function language. Notice that the two perspectives say exactly the same thing — just with different vocabulary.

Binomial Theorem LanguageGenerating Function Language
"Expand $$(1+x)^n$$" "Find the generating function for the row-$$n$$ binomial coefficients"
"The coefficient of $$x^k$$ in the expansion is $$\binom{n}{k}$$" "$$[x^k]\,(1+x)^n = \binom{n}{k}$$"
"Setting $$x=1$$ gives $$\sum_{k=0}^n \binom{n}{k} = 2^n$$" "The sum of all coefficients of $$(1+x)^n$$ equals $$2^n$$"
"The $$k$$-th term is $$\binom{n}{k}x^k$$" "The $$x^k$$ term of the generating function has coefficient $$\binom{n}{k}$$"

The Subset-Counting Identity Revisited

The row-sum identity $$\sum_{k=0}^{n} \binom{n}{k} = 2^n$$ (Corollary 1 from Module 8.1) now has a natural generating-function interpretation:

$$\text{Sum of all coefficients of } (1+x)^n = (1+1)^n = 2^n$$

The total number of subsets of an $$n$$-element set is the sum of all coefficients of the generating function $$(1+x)^n$$, obtained by evaluating at $$x = 1$$. This is a preview of how generating functions connect algebra to combinatorics.

4 · Algorithm Analysis Applications

The counting tools from Weeks 7 and 8 are not just abstract mathematics — they directly determine how long algorithms must run. This section connects the Binomial Theorem, stars and bars, and multinomial coefficients to concrete questions about computational complexity.

4A — Counting Bit Strings: Why $$2^n$$ Is Unavoidable

A bit string of length $$n$$ is a sequence of $$n$$ symbols, each either 0 or 1. By the product rule, there are exactly $$2^n$$ such strings.

The Binomial Theorem gives us a second way to count the same thing. A bit string of length $$n$$ with exactly $$k$$ ones corresponds to choosing which $$k$$ of the $$n$$ positions hold a 1 — that is, $$\binom{n}{k}$$ strings. Summing over all possible values of $$k$$:

$$2^n = \sum_{k=0}^{n} \binom{n}{k}$$

This is the row-sum identity, now interpreted as: the total number of bit strings of length $$n$$ equals the sum of the number of strings of each possible Hamming weight $$k$$.

💡 Algorithm Implication: Exponential Lower Bound

Suppose an algorithm must examine every subset of an $$n$$-element set — for example, a brute-force algorithm testing all possible subsets for some property. Since there are $$2^n$$ subsets, the algorithm must perform at least $$2^n$$ steps. We write this as a lower bound:

$$\text{Any subset-enumeration algorithm requires at least } \Omega(2^n) \text{ time.}$$

This is why brute-force subset search is considered intractable for large $$n$$. No matter how clever the implementation, the exponential count of subsets creates a floor on the running time.

4B — Stars and Bars in Computing: Resource Allocation

The stars and bars method from Module 8.3 models many real computing scenarios directly. Here are two worked examples.

Scenario: An operating system scheduler has $$n = 12$$ identical CPU time slices to distribute among $$k = 4$$ processes. A process may receive zero slices (it will simply wait). How many distinct allocations are possible?

  1. 1 Identify stars and bars: the 12 time slices are the stars; the 4 processes are the bins.
  2. 2 Constraint: bins may be empty (a process can receive 0 slices), so use the unrestricted formula.
  3. 3 Apply: Number of ways $$= \binom{n + k - 1}{k - 1} = \binom{12 + 4 - 1}{4 - 1} = \binom{15}{3}$$.
  4. 4 Compute: $$\binom{15}{3} = \dfrac{15 \cdot 14 \cdot 13}{3 \cdot 2 \cdot 1} = 455$$.
Answer
There are 455 distinct ways to allocate 12 identical CPU time slices across 4 processes when empty allocation is permitted.

Scenario: A system must partition $$n = 16$$ identical memory blocks across $$k = 4$$ segments, and each segment must receive at least 1 block (no empty segments allowed). How many ways can this be done?

  1. 1 Stars and bars: 16 blocks (stars), 4 segments (bins), each bin gets $$\geq 1$$.
  2. 2 Use the restricted formula: $$\binom{n-1}{k-1} = \binom{16-1}{4-1} = \binom{15}{3}$$.
  3. 3 Compute: $$\binom{15}{3} = 455$$.
Answer
There are 455 distinct ways to partition 16 identical memory blocks into 4 non-empty segments.
🔍 Interesting Coincidence?
Both examples gave 455. This is not always the case — it happened here because the unrestricted problem with $$(n, k) = (12, 4)$$ and the restricted problem with $$(n, k) = (16, 4)$$ both reduce to $$\binom{15}{3}$$. In general the two formulas give different answers.

4C — Multinomial Coefficients in Hashing

A hash table with $$k$$ buckets distributes $$n$$ keys across those buckets. If we want to count the number of ways exactly $$k_1$$ keys land in bucket 1, $$k_2$$ in bucket 2, …, $$k_r$$ in bucket $$r$$ (where $$k_1 + k_2 + \cdots + k_r = n$$ and the keys are distinguishable), the answer is the multinomial coefficient:

$$\binom{n}{k_1,\, k_2,\, \ldots,\, k_r} = \frac{n!}{k_1!\, k_2!\, \cdots\, k_r!}$$

Scenario: A hash table has $$r = 3$$ buckets. We insert $$n = 9$$ distinct keys. How many ways can the keys be distributed so that bucket 1 gets 4 keys, bucket 2 gets 3 keys, and bucket 3 gets 2 keys?

  1. 1 Set up: $$n = 9$$, $$k_1 = 4$$, $$k_2 = 3$$, $$k_3 = 2$$. Verify: $$4 + 3 + 2 = 9$$. ✓
  2. 2 Apply the multinomial coefficient: $$\dbinom{9}{4,\,3,\,2} = \dfrac{9!}{4!\,3!\,2!}$$.
  3. 3 Compute: $$\dfrac{362880}{24 \cdot 6 \cdot 2} = \dfrac{362880}{288} = 1260$$.
Answer
There are 1,260 distinct ways to distribute 9 distinct keys into 3 buckets with the specified load distribution.

Why this matters for CS: Hash table performance depends on how evenly keys are distributed. Multinomial coefficients let us count how many input arrangements produce a given load profile — a foundation for analyzing hash function quality probabilistically (a topic you will revisit in Week 9).

4D — Preview: The Probability Connection (Week 9)

🔭 Bridge to Week 9 — Discrete Probability

In Week 9 you will study the Binomial Distribution: the probability of exactly $$k$$ successes in $$n$$ independent binary trials (think coin flips, packet error events, or random bit generation). The formula is:

$$P(X = k) = \binom{n}{k} p^k (1-p)^{n-k}$$

Notice that $$\binom{n}{k}$$ — the binomial coefficient you have been computing all week — is the core of this formula. It counts the number of ways to arrange exactly $$k$$ successes among $$n$$ trials. The counting you practiced this week becomes the numerator in Week 9 probability calculations.

The generating function $$(1+x)^n$$ also connects directly: if you replace $$x$$ with $$\frac{p}{1-p}$$ (formally), the coefficients of the resulting expression encode the binomial probabilities. This is a glimpse of the power of generating functions that you will develop further in Week 11.
