Binomial Coefficient Identities

Where We've Been — Where We're Going

Module 8.1 established the Binomial Theorem and showed that the entries of row n in Pascal's triangle are exactly the coefficients of (a+b)n. You can now expand binomial expressions, find specific terms, and apply the two key corollaries (row-sum = 2n and alternating-sum = 0).

This module goes deeper. Rather than just using binomial coefficients, we will prove relationships among them. The four identities below — symmetry, Pascal's, Vandermonde's, and the Hockey Stick — each express a non-obvious equality between binomial coefficients. Along the way you will learn the combinatorial proof technique: proving an identity by showing that both sides count the same set in two different ways.

1 · Symmetry Identity

Identity 1 Symmetry Identity

Statement. For all integers n ≥ 0 and 0 ≤ k ≤ n:

C(n, k) = C(n, n − k)

Intuitive explanation. Choosing k items to include in a subset is exactly the same decision as choosing the remaining n − k items to exclude. Every k-element subset uniquely determines an (n−k)-element complement, so the two counts must be equal.

Algebraic Proof

We verify the identity directly from the formula C(n,k) = n! / (k!(n−k)!):

C(n, n−k) = n!(n−k)! · (n−(n−k))!  =  n!(n−k)! · k!  =  C(n, k)   ∎

The denominator (n−k)! · k! is the same whether we write C(n,k) or C(n, n−k) — the two factorials simply swap positions.

Worked Example

  1. 1 Compute C(9, 3): 9! / (3! · 6!) = (9·8·7)/(3·2·1) = 504/6 = 84.
  2. 2 Compute C(9, 6): 9! / (6! · 3!) = (9·8·7)/(3·2·1) = 504/6 = 84.
  3. 3 Both equal 84. ✓ The denominators are identical because 3! · 6! = 6! · 3!.
💡 Practical Use
When computing by hand, always choose the smaller of k and n−k. C(20, 17) is much easier to compute as C(20, 3) = 20·19·18 / 6 = 1140 than by expanding 17! in the denominator.
✅ Key Takeaway
C(n, k) = C(n, n−k). Use this to simplify computations: always work with the smaller subscript. This also confirms the left-right mirror symmetry of every row in Pascal's triangle.

2 · Pascal's Identity

Identity 2 Pascal's Identity

Statement. For all integers n ≥ 1 and 1 ≤ k ≤ n − 1:

C(n, k) = C(n−1, k−1) + C(n−1, k)

Reading this identity. Each interior entry of Pascal's triangle equals the sum of the two entries directly above it. This is the construction rule for the triangle — and now we will prove it, not just observe it.

Proof 1 — Algebraic

We expand both terms on the right-hand side using the formula C(n,k) = n! / (k!(n−k)!) and show they sum to C(n,k).

  1. 1 Write out the right-hand side using the factorial formula:
    C(n−1, k−1) + C(n−1, k)
      =  (n−1)!(k−1)!(n−k)!  +  (n−1)!k!(n−1−k)!
  2. 2 Find a common denominator. Note that k! = k · (k−1)! and (n−k)! = (n−k) · (n−k−1)!. The common denominator is k! · (n−k)!:
    =  k · (n−1)!k! · (n−k)!  +  (n−k) · (n−1)!k! · (n−k)!
  3. 3 Factor out (n−1)! from the numerator:
    =  (n−1)! · [k + (n−k)] k! · (n−k)!  =  (n−1)! · n k! · (n−k)!
  4. 4 Since n · (n−1)! = n!:
    =  n!k! · (n−k)!  =  C(n, k)   ∎

Proof 2 — Combinatorial (Double Counting)

This proof avoids algebra entirely. We count the same set in two ways and conclude the two expressions must be equal.

The Set Being Counted

Let S be any set with n elements. Fix one special element — call it x. We count all k-element subsets of S.

Left Side — Direct Count

By definition, the number of k-element subsets of an n-element set is C(n, k).

=
Right Side — Split by x

Every k-subset either contains x or does not:

  • Contains x: choose the remaining k−1 elements from the other n−1 elements → C(n−1, k−1) ways.
  • Does not contain x: choose all k elements from the other n−1 elements → C(n−1, k) ways.

Since these two cases are mutually exclusive and exhaustive, by the Sum Rule: C(n, k) = C(n−1, k−1) + C(n−1, k).  ∎

Connection to Pascal's Triangle

Pascal's identity is exactly the construction rule for Pascal's triangle: each interior entry is the sum of the two entries above it. The diagram below highlights one instance: the entry C(5, 2) = 10 is the sum of C(4, 1) = 4 and C(4, 2) = 6 from the row above.

Pascal's Triangle (rows 3–5)

n=3: 1 3 3 1
n=4: 1 4 6 4 1
n=5: 1 5 10 10 5 1

Pascal's Identity Applied

The gold entry C(5, 2) = 10 is the sum of the two blue entries directly above it:

C(5, 2) = C(4, 1) + C(4, 2)
10  =  4  +  6  ✓

This works for every interior entry in the triangle — Pascal's identity is the triangle's defining recurrence.

Worked Example

  1. 1 Compute the left side: C(8, 3) = 8! / (3! · 5!) = (8·7·6)/(3·2·1) = 336/6 = 56.
  2. 2 Compute C(7, 2): 7! / (2! · 5!) = (7·6)/2 = 21.
  3. 3 Compute C(7, 3): 7! / (3! · 4!) = (7·6·5)/6 = 35.
  4. 4 Check: 21 + 35 = 56 = C(8, 3). ✓
C(8, 3) = C(7, 2) + C(7, 3)  →  56 = 21 + 35 ✓
✅ Key Takeaway
Pascal's identity has two proofs: algebraic (factor and simplify) and combinatorial (fix one element; count subsets that include it vs. exclude it). The combinatorial proof is the model for all double-counting arguments you will see this week.

3 · Vandermonde's Identity

Identity 3 Vandermonde's Identity

Statement. For non-negative integers m, n, and r with r ≤ m + n:

C(m+n, r)  =  C(m,0)·C(n,r) + C(m,1)·C(n,r−1) + · · · + C(m,r)·C(n,0)  =  ∑k=0r C(m, k) · C(n, r−k)

In words: The number of ways to choose r items from a combined group of m + n items equals the sum, over all splits k, of the number of ways to choose k items from the first group and r − k items from the second group.

Combinatorial Proof Sketch

The Set Being Counted

Imagine a committee of r people chosen from a pool of m men and n women (total m + n people).

Left Side — Choose from Everyone

Choose r people from all m + n people at once: C(m+n, r) ways.

=
Right Side — Split by Gender

If exactly k men are chosen (for k = 0, 1, \ldots, r), then exactly r − k women must be chosen:

  • Choose k men from m: C(m, k) ways.
  • Choose r−k women from n: C(n, r−k) ways.
  • By the Product Rule, this split gives C(m,k)·C(n,r−k) committees.

Summing over all valid values of k gives the total: ∑ C(m,k)·C(n,r−k).

Both expressions count the same set of committees, so they are equal.  ∎

Worked Example

Identify the parameters: m = 4, n = 3, r = 3.

Apply Vandermonde's Identity:

∑k=03 C(4, k) · C(3, 3−k) = C(4+3, 3) = C(7, 3)

Compute C(7, 3):

C(7, 3) = 7!3! · 4! = 7 · 6 · 53 · 2 · 1 = 2106 = 35

Verify by expanding the sum directly:

kC(4,k)C(3, 3−k)Product
0C(4,0) = 1C(3,3) = 11
1C(4,1) = 4C(3,2) = 312
2C(4,2) = 6C(3,1) = 318
3C(4,3) = 4C(3,0) = 14
1 + 12 + 18 + 4 = 35 = C(7, 3) ✓
💡 Strategy
When you see a sum of products of two binomial coefficients whose lower indices add to a constant, check whether Vandermonde's identity applies. It reduces the entire sum to a single binomial coefficient.
✅ Key Takeaway
Vandermonde's identity: C(m+n, r) = ∑ C(m,k)·C(n,r−k). The combinatorial proof uses a two-group committee model and the Product Rule. When the lower indices of two C-terms sum to a constant r, Vandermonde collapses the sum to C(m+n, r).

4 · Hockey Stick Identity

Identity 4 Hockey Stick Identity

Statement. For non-negative integers n and r:

C(n,0) + C(n+1,1) + C(n+2,2) + · · · + C(n+r,r)  =  ∑i=0r C(n+i, i)  =  C(n+r+1, r)

In words: Sum the diagonal entries of Pascal's triangle starting at C(n, 0) and going r steps down-right. The sum equals the entry one step further down-right — the "blade" of the hockey stick.

Pascal's Triangle Visualization

The diagram below shows rows 0–7 of Pascal's triangle. The blue cells form the "handle" of the hockey stick (the summed entries) and the gold cell is the "blade" (the result). This example uses n = 2, r = 4:

C(2,0) + C(3,1) + C(4,2) + C(5,3) + C(6,4) = C(7,4)  →  1 + 3 + 6 + 10 + 15 = 35 ✓

Combinatorial Proof Sketch

The set being counted: All (r+1)-element subsets of \{1, 2, 3, \ldots, n+r+1\}. By definition there are C(n+r+1, r+1) = C(n+r+1, r) such subsets (using symmetry, since C(n+r+1, r+1) = C(n+r+1, n) and the identity is stated for C(n+r+1, r)).

The split: Every (r+1)-element subset has a largest element. Let that largest element be n+1+i for some i \in \{0, 1, \ldots, r\} (it must be at least n+1 to leave room for r smaller elements from \{1,\ldots,n+i\}).

  • If the largest element is n+1+i, the remaining r elements must be chosen from \{1, 2, \ldots, n+i\}, giving C(n+i, r) choices.
  • Summing over all valid values of i:
    C(n+r+1, r) = ∑i=0r C(n+i, r)  =  ∑i=0r C(n+i, i)
    (The last equality uses symmetry: C(n+i, r) = C(n+i, i) when (n+i) - r = i, i.e., n = r - r = 0 — more precisely, this form matches the statement when n is the fixed diagonal offset.)

The two counts are equal, proving the Hockey Stick Identity.  ∎

Worked Example

Identify parameters: The sum has the form ∑i=0r C(n+i, i) with n = 1 and r = 3 (four terms, i = 0, 1, 2, 3).

Apply the Hockey Stick:

C(1,0) + C(2,1) + C(3,2) + C(4,3) = C(1+3+1, 3) = C(5, 3)

Compute C(5, 3):

C(5, 3) = 5!3! · 2! = 5 · 42 = 10

Verify directly:

1 + 2 + 3 + 4 = 10 ✓

(Note: C(1,0)=1, C(2,1)=2, C(3,2)=3, C(4,3)=4 — these are just the natural numbers 1 through 4, and their sum is the triangular number T(4) = 10.)

✅ Key Takeaway
The Hockey Stick Identity: ∑i=0r C(n+i, i) = C(n+r+1, r). Visually, it traces a diagonal "handle" in Pascal's triangle; the sum equals the entry at the "blade." The proof classifies subsets by their largest element.

5 · The Combinatorial Proof Technique

Throughout this module you have seen proofs that do not rely on algebra. Instead, they prove an identity by showing that both sides count the same thing. This is the combinatorial proof (also called the double-counting technique). It is one of the most elegant and powerful tools in discrete mathematics.

📐 What Is a Combinatorial Proof?

A combinatorial proof of an identity of the form A = B proceeds in two steps:

  1. Describe a set S whose size can be computed in two different ways.
  2. Show that |S| = A by one counting method and |S| = B by another. Conclude A = B.

The Two-Counting Strategy — A Guide

StepQuestion to AskExample (Pascal's Identity)
1. Name the set What collection of objects does the identity seem to be counting? All k-element subsets of an n-element set S.
2. Count directly What does the left-hand side of the identity equal? C(n, k) — by definition of the binomial coefficient.
3. Find the split What natural partition or case distinction produces the right-hand side? Fix element x: subsets containing x vs. subsets not containing x.
4. Count each part How many objects are in each part of the partition? C(n−1, k−1) containing x; C(n−1, k) not containing x.
5. Apply Sum Rule Do the parts cover the whole set without overlap? Yes — every subset either contains x or it doesn't.
6. Conclude State that both counts equal |S|, so they equal each other. C(n,k) = C(n−1,k−1) + C(n−1,k). ∎
💡 Tip — When to Use Combinatorial Proofs

Ask yourself: "What are we choosing, and how many ways can we do it?" If both sides of the identity can be interpreted as counting the same collection of subsets, committees, or arrangements — a combinatorial proof is likely the cleanest approach.

Algebraic proofs (expanding factorials) always work, but they can be tedious and hide the underlying reason the identity is true. Combinatorial proofs reveal why the identity holds.

Practice Example — Prove C(2n, 2) = 2·C(n, 2) + n²

Try to construct the combinatorial proof yourself before expanding the panel below. Hint: think about choosing 2 items from a set split into two equal halves.

Step 1 — Name the set. Let S be a set of 2n elements, split into two halves: A = \{a_1, \ldots, a_n\} and B = \{b_1, \ldots, b_n\}. Count all 2-element subsets of S.

Step 2 — Count directly (left side). There are C(2n, 2) ways to choose 2 elements from 2n elements.

Step 3 — Find the split (right side). Every 2-element subset of S falls into exactly one of three cases:

  • Both from A: C(n, 2) ways.
  • Both from B: C(n, 2) ways.
  • One from A, one from B: n choices from A times n choices from B = n² ways (Product Rule).

Steps 4–5 — Apply Sum Rule. The three cases are mutually exclusive and exhaustive:

C(2n, 2) = C(n, 2) + C(n, 2) + n² = 2·C(n, 2) + n²   ∎

Verify with n = 3:

C(6, 2) = 15   and   2·C(3,2) + 9 = 2·3 + 9 = 15 ✓
✅ Key Takeaway
A combinatorial proof names a set, counts it two ways, and concludes the two expressions are equal. The strategy: (1) identify what is being chosen, (2) find a natural partition, (3) count each part, (4) apply the Sum Rule. This technique will appear on the Midterm Exam.

6 · Computing C(n, k) in Code

Module 8.1 showed a Pascal's triangle row-construction method for computing C(n, k). Here we compare that approach against the direct factorial formula and discuss integer overflow — a critical concern for PE-07 Part 1.

Method 1 — Direct Factorial Formula

/**
 * C(n,k) via direct factorial formula.
 * WARNING: overflows for moderate n
 * because n! grows extremely fast.
 */
public static long binomDirect(int n, int k) {
    if (k < 0 || k > n) return 0;
    if (k == 0 || k == n) return 1;

    // Use symmetry first
    if (k > n - k) k = n - k;

    long num = 1, den = 1;
    for (int i = 0; i < k; i++) {
        num *= (n - i);  // numerator
        den *= (i + 1);  // denominator k!
    }
    return num / den;
}

Pros: Simple to implement.
Cons: num can overflow long before the division reduces it, even for modest n (e.g., n=66, k=33).

Method 2 — Pascal's Triangle Row

/**
 * C(n,k) via Pascal's triangle row.
 * Builds only the needed portion of
 * row n; avoids large intermediates.
 */
public static long binomPascal(int n, int k) {
    if (k < 0 || k > n) return 0;
    if (k == 0 || k == n) return 1;
    if (k > n - k) k = n - k;

    long[] row = new long[k + 1];
    row[0] = 1;

    for (int i = 1; i <= n; i++) {
        // Right-to-left update using
        // Pascal's identity in-place
        for (int j = Math.min(i,k);
                 j >= 1; j--) {
            row[j] += row[j - 1];
        }
    }
    return row[k];
}

Pros: Only stores k+1 values; intermediate values are smaller; directly implements Pascal's identity.
Cons: Slightly more complex loop logic.

⚠️ PE-07 — Integer Overflow Warning

Both methods use long (64-bit), which overflows for C(n,k) when the result exceeds approximately 9.2 × 1018. This occurs for, e.g., n \approx 66, k = 33.

PE-07 requirement: Your program must detect potential overflow and either print a warning or switch to BigInteger for large inputs. The Pascal's triangle method (Method 2) delays overflow longer because it adds smaller values, but it does not eliminate the problem for very large n.

🔗 Connection to Module 8.2 Identities

The Pascal's triangle row method works because it applies Pascal's Identity (Section 2) at every step: row[j] += row[j-1] computes C(i, j) = C(i−1, j−1) + C(i−1, j) in place. The Symmetry Identity (Section 1) is used at the top of both methods to replace k with n−k when it is smaller, halving the computation.

Module Summary

In this module you have covered:

  • Symmetry Identity: C(n,k) = C(n, n−k). Algebraic proof via factorial formula; combinatorial meaning as include/exclude duality.
  • Pascal's Identity: C(n,k) = C(n−1,k−1) + C(n−1,k). Two proofs: algebraic (common denominator) and combinatorial (fix one element). Direct connection to Pascal's triangle construction.
  • Vandermonde's Identity: ∑ C(m,k)·C(n,r−k) = C(m+n,r). Combinatorial proof via a two-group committee model and the Product Rule.
  • Hockey Stick Identity: ∑i=0r C(n+i,i) = C(n+r+1,r). Visualized as a diagonal path in Pascal's triangle; proved by classifying subsets by their largest element.
  • Combinatorial Proof Technique: The six-step double-counting strategy; applied to the practice identity C(2n,2) = 2·C(n,2) + n².
  • Computing C(n,k) in Code: Comparison of direct formula and Pascal's triangle row method; integer overflow warning relevant to PE-07.
