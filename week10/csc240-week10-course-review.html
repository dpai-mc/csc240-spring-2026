1. Sets and Set Operations (Week 1)

📘 Key Definitions
  • Set — an unordered collection of distinct elements. Written in braces: A = {1, 2, 3}.
  • a ∈ A means element a belongs to set A; a ∉ A means it does not.
  • Empty set ∅ = {} — a set containing no elements.
  • Universal set U — the set of all elements under consideration in a given context.
  • Subset: A ⊆ B means every element of A is also in B. Proper subset: A ⊂ B means A ⊆ B and A ≠ B.
  • Power set 𝒫(A) — the set of all subsets of A. If |A| = n, then |𝒫(A)| = 2ⁿ.
  • Cartesian product: A × B = { (a, b) | a ∈ A, b ∈ B }. Size: |A × B| = |A| · |B|.

Core Set Operations

OperationNotationDefinition
UnionA ∪ BAll elements in A or B (or both)
IntersectionA ∩ BElements that are in both A and B
DifferenceA ∖ BElements in A but not in B
ComplementA' or AᶜElements of U that are not in A
DisjointA ∩ B = ∅A and B share no elements
⚠️ Watch Out — Common Set Mistakes
  • Order does not matter: {1, 2, 3} = {3, 1, 2}.
  • No duplicates: {1, 2, 2, 3} = {1, 2, 3}.
  • ∅ ≠ {∅}: The empty set has 0 elements; {∅} is a set containing the empty set and has 1 element.
  • Membership vs. subset: 1 ∈ {1, 2, 3} but {1} ⊆ {1, 2, 3}. The element 1 and the set {1} are different objects.
▶ Worked Example — Set Operations and Power Set

Let U = {1, 2, 3, 4, 5, 6, 7, 8}, A = {1, 2, 3, 4}, B = {3, 4, 5, 6}.

  1. A ∪ B = {1, 2, 3, 4, 5, 6} — combine all elements; no duplicates.
  2. A ∩ B = {3, 4} — only the elements common to both.
  3. A ∖ B = {1, 2} — in A but not in B.
  4. B ∖ A = {5, 6} — note that A ∖ B ≠ B ∖ A in general.
  5. A' = {5, 6, 7, 8} — elements of U not in A.
  6. |𝒫(A)| = 2⁴ = 16 — the power set of A contains all 16 subsets of A (including ∅ and A itself).

2. Relations and Functions (Week 2)

📘 Key Definitions
  • Binary relation R from A to B: a subset R ⊆ A × B. We write a R b when (a, b) ∈ R.
  • Equivalence relation: a relation that is reflexive, symmetric, and transitive. Equivalence relations partition their set into disjoint equivalence classes.
  • Partial order: a relation that is reflexive, antisymmetric, and transitive (e.g., ≤ on integers, ⊆ on sets).
  • Function f : A → B: a relation where each element of A maps to exactly one element of B. Every function is a relation, but not every relation is a function.

Relation Properties

PropertyFormal Definition
Reflexive∀a ∈ A : (a, a) ∈ R
Symmetric∀a, b : (a, b) ∈ R → (b, a) ∈ R
Transitive∀a, b, c : (a,b) ∈ R ∧ (b,c) ∈ R → (a,c) ∈ R
Antisymmetric∀a, b : (a,b) ∈ R ∧ (b,a) ∈ R → a = b

Function Types

TypeConditionKey point
Injective (one-to-one) f(x₁) = f(x₂) → x₁ = x₂ No two inputs share an output
Surjective (onto) ∀y ∈ B, ∃x ∈ A : f(x) = y Every element of B is hit at least once
Bijective Injective and surjective Perfect pairing; an inverse f⁻¹ exists
📘 Pigeonhole Principle
If n items are distributed into m containers and n > m, then at least one container holds at least ⌈n/m⌉ items.

Classic example: In any group of 13 people, at least two must share a birth month (13 people, 12 months, so at least one month has ≥ 2 people).

CS connection: Any function from a set of size n to a set of size m < n is guaranteed to be non-injective.
⚠️ Watch Out — Relations and Functions
  • Antisymmetric ≠ not symmetric. A relation can be both symmetric and antisymmetric simultaneously (e.g., equality). These are independent properties.
  • Composition order: (g ∘ f)(x) = g(f(x)) — apply f first, then g. In general, g ∘ f ≠ f ∘ g.
  • Inverses require bijection. Only bijective functions have an inverse function.

3. Cardinality and Propositional Logic (Week 3)

Cardinality and Countability

📘 Key Definitions
  • Cardinality |A| — the number of elements in set A.
  • Two sets have the same cardinality if and only if there exists a bijection between them.
  • Countably infinite: same cardinality as ℕ — can be listed in a sequence. Examples: ℤ, ℚ.
  • Uncountably infinite: strictly larger cardinality than |ℕ|. The real numbers ℝ are uncountable (Cantor's diagonal argument).

Propositional Logic

A proposition is a declarative statement with a definite truth value: true (T) or false (F). Propositions are combined using logical connectives:

SymbolNameKey truth condition
¬pNegation (NOT)True when p is false
p ∧ qConjunction (AND)True only when both p and q are true
p ∨ qDisjunction (OR)False only when both p and q are false
p → qImplication (IF…THEN)False only when p is true and q is false
p ↔ qBiconditional (IFF)True when p and q have the same truth value

Tautology: a proposition that is always true, for every assignment (e.g., p ∨ ¬p).
Contradiction: always false (e.g., p ∧ ¬p).
Contingency: true for some assignments and false for others.

4. Logical Equivalences, Normal Forms, and Predicate Logic (Week 4)

Key Logical Equivalences

NameEquivalence
Double negation¬(¬p) ≡ p
De Morgan's (AND)¬(p ∧ q) ≡ ¬p ∨ ¬q
De Morgan's (OR)¬(p ∨ q) ≡ ¬p ∧ ¬q
Contrapositive(p → q) ≡ (¬q → ¬p)
Implication as disjunction(p → q) ≡ (¬p ∨ q)
Absorptionp ∨ (p ∧ q) ≡ p
Distributive (AND over OR)p ∧ (q ∨ r) ≡ (p ∧ q) ∨ (p ∧ r)

CNF (Conjunctive Normal Form): a conjunction of clauses, each a disjunction of literals. Example: (p ∨ ¬q) ∧ (¬p ∨ r).

DNF (Disjunctive Normal Form): a disjunction of clauses, each a conjunction of literals. Example: (p ∧ q) ∨ (¬p ∧ r).

Predicate Logic and Quantifiers

📘 Key Definitions
  • Predicate P(x): a statement whose truth value depends on the value of x from some domain D.
  • Universal quantifier ∀x P(x): "For all x in the domain, P(x) is true." True only when P holds for every element. Disproved by a single counterexample.
  • Existential quantifier ∃x P(x): "There exists at least one x such that P(x)." Proved by a single witness.

Negating quantifiers (De Morgan's laws extended to predicate logic):

¬(∀x P(x)) ≡ ∃x ¬P(x)       ¬(∃x P(x)) ≡ ∀x ¬P(x)
⚠️ Watch Out — Logic Pitfalls
  • p → q is not equivalent to its converse q → p. Only the contrapositive ¬q → ¬p is logically equivalent.
  • Quantifier order matters: ∀x ∃y P(x, y) means "for every x, there exists a (possibly different) y," which is generally stronger than ∃y ∀x P(x, y) ("one fixed y works for all x").
  • A statement of the form ∀x P(x) is true in a vacuously true sense when the domain is empty.

5. Proof Techniques (Weeks 5–6)

TechniqueStructureBest used when
Direct proof Assume P; derive Q The conclusion follows naturally from definitions and assumptions
Proof by contrapositive Assume ¬Q; derive ¬P The negation of the conclusion is easier to work with
Proof by contradiction Assume P ∧ ¬Q; derive a contradiction Existence proofs; irrationality; when the negation leads somewhere useful
Mathematical induction Base case + inductive step (P(k) → P(k+1)) Statements about all natural numbers; sequence formulas
Strong induction Assume all of P(1), …, P(k); prove P(k+1) P(k+1) depends on multiple earlier cases, not just P(k)
📘 Mathematical Induction — Standard Structure
To prove ∀n ≥ n₀, P(n):
  • Base case: Show P(n₀) is true directly.
  • Inductive hypothesis: Assume P(k) holds for some arbitrary k ≥ n₀.
  • Inductive step: Using the hypothesis, prove P(k+1).
  • Conclusion: By the principle of induction, P(n) holds for all n ≥ n₀. □
▶ Worked Example — Induction: Sum of First n Integers

Claim: For all n ≥ 1, 1 + 2 + 3 + ⋯ + n = n(n+1)2.

  1. Base case (n = 1): LHS = 1. RHS = 1 · 22 = 1. ✓
  2. Inductive hypothesis: Assume 1 + 2 + ⋯ + k = k(k+1)/2 for some k ≥ 1.
  3. Inductive step — show the formula holds for k+1: 1 + 2 + ⋯ + k + (k+1) = k(k+1)2 + (k+1) = (k+1) · k + 22 = (k+1)(k+2)2 This matches the formula with n = k+1. ✓
  4. Conclusion: By induction, the formula holds for all n ≥ 1. □

6. Fundamental Counting (Week 7)

📘 Core Counting Rules
  • Sum rule: If tasks A and B cannot both be done simultaneously (mutually exclusive), the total is |A| + |B|.
  • Product rule: If a task consists of m sequential independent steps with n₁, n₂, …, nₘ choices respectively, the total is n₁ · n₂ · ⋯ · nₘ.
  • Inclusion-exclusion (two sets): |A ∪ B| = |A| + |B| − |A ∩ B| For three sets: |A ∪ B ∪ C| = |A| + |B| + |C| − |A ∩ B| − |A ∩ C| − |B ∩ C| + |A ∩ B ∩ C|.

Permutations and Combinations

FormulaWhen to use
Permutation P(n, r) P(n, r) = n!(n − r)! Ordered arrangements of r items from n (order matters)
Combination C(n, r) C(n, r) = n!r!(n − r)! Unordered selections of r items from n (order does not matter)

Pascal's identity: C(n, k) = C(n−1, k−1) + C(n−1, k) — the recurrence that builds Pascal's triangle row by row.

▶ Worked Example — Committees and Passwords

Problem A: A club of 10 people needs to form a 3-person committee. How many distinct committees are possible?

  1. Order does not matter (a committee {Alice, Bob, Carol} is the same as {Carol, Alice, Bob}) → use combinations.
  2. C(10, 3) = 10!3! · 7! = 10 · 9 · 83 · 2 · 1 = 120 distinct committees.

Problem B: How many 3-character passwords use distinct letters from A–Z?

  1. Order matters (password "CAT" ≠ "ACT") and no repetition → use permutations.
  2. P(26, 3) = 26 · 25 · 24 = 15,600 distinct passwords.

7. Advanced Counting — Binomial Theorem and More (Week 8)

📘 The Binomial Theorem
For any real numbers a, b and non-negative integer n: (a + b)ⁿ = ∑k=0n C(n, k) · aⁿ⁻ᵏ · bᵏ The coefficient of aⁿ⁻ᵏ bᵏ is C(n, k) — the entry in row n, position k of Pascal's triangle.

Key corollary (set counting): Setting a = b = 1: ∑k=0n C(n, k) = 2ⁿ This confirms that an n-element set has exactly 2ⁿ subsets (matching the power set formula from Week 1).

Binomial Coefficient Identities

IdentityFormula
SymmetryC(n, k) = C(n, n − k)
Pascal's identityC(n, k) = C(n−1, k−1) + C(n−1, k)
Vandermonde's identity∑k=0r C(m, k) · C(n, r−k) = C(m+n, r)
Hockey stick identity∑i=0r C(n+i, i) = C(n+r+1, r)

Multinomial Coefficients

Count distinct arrangements of n objects when there are r types with k₁, k₂, …, kᵣ indistinguishable items of each type (k₁ + ⋯ + kᵣ = n):

n!k₁! · k₂! · ⋯ · kᵣ!

Example: Distinct arrangements of MISSISSIPPI (11 letters: 1 M, 4 I, 4 S, 2 P): 11! / (1! · 4! · 4! · 2!) = 34,650.

Stars and Bars

Count the ways to distribute n indistinguishable objects into k distinct labeled bins:

  • Bins may be empty: C(n + k − 1, k − 1) ways.
  • Each bin gets at least one object: C(n − 1, k − 1) ways.

Equivalent formulation: count non-negative integer solutions to x₁ + x₂ + ⋯ + xₖ = n.

🔗 Generating Functions — First Look
The ordinary generating function for a counting sequence {a₀, a₁, a₂, …} is the formal power series G(x) = a₀ + a₁x + a₂x² + ⋯. The Binomial Theorem shows that (1+x)ⁿ is the generating function for the binomial coefficients — reading off the coefficient of xᵏ gives C(n, k). Generating functions return in Week 11 to solve recurrence relations.

8. Discrete Probability (Week 9)

📘 Probability Space — Core Definitions
  • Sample space Ω: the set of all possible outcomes of a random experiment.
  • Event E ⊆ Ω: any subset of the sample space (using the set vocabulary from Week 1).
  • Probability measure P: a function assigning a real number to each event, satisfying the three Kolmogorov axioms:
  • Axiom 1: P(E) ≥ 0 for all events E ⊆ Ω.
  • Axiom 2: P(Ω) = 1. (The probability of some outcome occurring is 1.)
  • Axiom 3: If A ∩ B = ∅, then P(A ∪ B) = P(A) + P(B).

Uniform probability spaces — when all outcomes are equally likely: P(E) = |E||Ω| In a uniform space, computing probability reduces to counting — the exact techniques of Weeks 7–8.

Core Rules (derived from the axioms)

RuleFormulaDerivation note
Complement rule P(Eᶜ) = 1 − P(E) From P(E) + P(Eᶜ) = P(Ω) = 1 (Axioms 2 & 3, since E and Eᶜ are disjoint)
Addition rule (general) P(A ∪ B) = P(A) + P(B) − P(A ∩ B) Inclusion-exclusion from Week 7 expressed as probability
Addition rule (mutually exclusive) P(A ∪ B) = P(A) + P(B) when A ∩ B = ∅ Direct from Axiom 3
Conditional probability P(A | B) = P(A ∩ B)P(B), where P(B) > 0 Restricts the effective sample space to B
Independence P(A ∩ B) = P(A) · P(B) Knowing B occurred provides no information about A
📘 Bayes' Theorem
Derived by applying the definition of conditional probability twice: P(A | B) = P(B | A) · P(A)P(B) Interpretation: Bayes' theorem lets you update the prior probability P(A) to a posterior probability P(A | B) after observing evidence B.
📘 Discrete Random Variables and Expected Value
A discrete random variable X is a function that assigns a numerical value to each outcome in Ω. The expected value (long-run average): E[X] = ∑all x x · P(X = x) Example: For a fair six-sided die, E[X] = (1 + 2 + 3 + 4 + 5 + 6)/6 = 3.5.
⚠️ Watch Out — Common Probability Misconceptions
  • P(Ω) = 1, not the count of outcomes. Probabilities are always in [0, 1].
  • Mutually exclusive vs. independent are different concepts. Mutually exclusive: A ∩ B = ∅ (they cannot both happen). Independent: P(A ∩ B) = P(A) · P(B) (one gives no information about the other). Two events with positive probability cannot be both mutually exclusive and independent.
  • P(A | B) ≠ P(B | A) in general. Confusing these is known as the prosecutor's fallacy.
  • P(E) = 0 does not mean E is impossible — this can occur in continuous (non-discrete) settings.
▶ Worked Example — Probability Using Combinations

Problem: A shipment of 20 devices contains 8 defective and 12 good units. A quality tester randomly selects 5 devices. What is the probability that exactly 2 of the selected devices are defective?

  1. Define the sample space: Ω = all ways to choose 5 devices from 20.
    |Ω| = C(20, 5) = 15,504.
  2. Define event E: exactly 2 defective (from 8) and exactly 3 good (from 12).
    Using the product rule (independent choices from each group): |E| = C(8, 2) · C(12, 3) = 28 · 220 = 6,160
  3. Apply uniform probability: P(E) = 6,16015,504 ≈ 0.397

There is approximately a 39.7% chance that the sample contains exactly 2 defective devices. This calculation uses Week 7–8 counting tools directly inside the Week 9 probability framework.

📋 Weeks 1–9 at a Glance
  • Weeks 1–2: Sets and operations are the shared language of the entire course. Relations generalize sets; functions are the special case where each input maps to exactly one output.
  • Weeks 3–4: Cardinality measures and compares set sizes. Propositional and predicate logic give us the formal language for making precise statements about mathematical objects.
  • Weeks 5–6: Direct proof, contrapositive, contradiction, and mathematical induction are the tools for establishing truths rigorously — not just observing patterns.
  • Weeks 7–8: The sum/product rules, permutations, combinations, the Binomial Theorem, multinomial coefficients, and stars and bars all answer the question "how many?"
  • Week 9: Probability formally extends counting. In a uniform space, P(E) = |E|/|Ω|. Conditional probability, independence, Bayes' theorem, and expected value complete the foundational toolkit.
