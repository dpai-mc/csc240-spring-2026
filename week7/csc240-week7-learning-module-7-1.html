Why Counting Matters in Computer Science

Counting — formally known as combinatorics — is one of the most practical areas of discrete mathematics. Every time you analyze an algorithm's complexity, calculate how many passwords are possible, determine the number of test cases needed, or figure out how many ways to assign resources, you are using counting techniques.

📘 Definition: Combinatorics

Combinatorics is the branch of mathematics concerned with counting, arranging, and selecting objects according to specified rules. It answers the question: "How many ways can something happen?"

Real-World Motivations

Consider these everyday computing questions that require counting:

  • Password Security: How many 8-character passwords can be formed using uppercase letters, lowercase letters, and digits? (This determines brute-force attack feasibility.)
  • Algorithm Analysis: How many comparisons does a sorting algorithm make in the worst case?
  • Networking: How many unique IP addresses are possible in IPv4 vs. IPv6?
  • Testing: How many test inputs are needed to cover all combinations of boolean parameters?
  • Database Queries: How many possible join results can a query produce?
  • Bit Strings: How many binary strings of length n exist? (Answer: 2ⁿ)
💡 Connection to Previous Weeks

In Weeks 1–2, you studied sets, relations, and functions. Counting builds directly on set theory. The size of a set (its cardinality) is a counting concept. The union, intersection, and Cartesian product of sets all have counting implications that we will formalize in this module.

The Two Fundamental Questions

Almost every counting problem begins by asking one of two questions:

  1. "OR" question: How many ways can we do Task A or Task B? → Sum Rule
  2. "AND" question: How many ways can we do Task A and then Task B? → Product Rule

These two principles form the foundation of all counting techniques. Every advanced method — permutations, combinations, the binomial theorem — ultimately reduces to applications of these two rules.

Combinatorics has ancient roots. The Indian mathematician Mahāvīra (9th century) studied permutations and combinations. Blaise Pascal (1623–1662) developed Pascal's Triangle, which we'll study in Module 7.3. Gottfried Leibniz (1646–1716) called combinatorics the "art of counting" and considered it foundational to logic and computation — a remarkably prescient view given its central role in modern computer science.

The Sum Rule (Addition Principle)

📘 Definition: The Sum Rule

If a task can be done in one of two ways, where:

  • Way 1 can be done in n₁ ways, and
  • Way 2 can be done in n₂ ways, and
  • The two ways are mutually exclusive (no overlap),

then the task can be done in n₁ + n₂ ways.

📐 Formal Statement (Set Theory Version)

If A and B are disjoint finite sets (i.e., A ∩ B = ∅), then:

|A ∪ B| = |A| + |B|

More generally, for k pairwise disjoint sets A₁, A₂, …, Aₖ:

|A₁ ∪ A₂ ∪ … ∪ Aₖ| = |A₁| + |A₂| + … + |Aₖ|

Key Requirement: Mutually Exclusive

The Sum Rule only applies directly when the categories do not overlap. The keyword to look for is "or" — when a problem says "either this or that," and the options cannot happen simultaneously, use the Sum Rule.

⚠️ Common Mistake

If the categories overlap (are not mutually exclusive), you cannot simply add. You must use the Inclusion-Exclusion Principle (covered in Section 5 of this module) to avoid double-counting.

Examples

📝 Example 2.1: Choosing a Programming Language

A student must choose one programming language for a project. The department offers:

  • 3 compiled languages: Java, C++, Rust
  • 4 interpreted languages: Python, JavaScript, Ruby, PHP

How many choices does the student have?

Solution:

The student picks either a compiled language or an interpreted language (not both). These categories are mutually exclusive — no language appears in both lists.

By the Sum Rule: 3 + 4 = 7 choices.

📝 Example 2.2: Selecting a Data Structure

A programmer needs to choose a data structure for a task. The options are:

  • Linear structures: Array, Linked List, Stack, Queue (4 options)
  • Tree structures: Binary Tree, BST, AVL Tree (3 options)
  • Hash-based structures: HashMap, HashSet (2 options)

How many total data structure options exist?

Solution:

Each category is mutually exclusive (a structure belongs to exactly one category).

By the generalized Sum Rule: 4 + 3 + 2 = 9 options.

📝 Example 2.3: Bit Strings Starting with 1 or Ending with 00

How many bit strings of length 4 either start with 1 or end with 00?

Solution:

Careful! These categories overlap — a string can start with 1 AND end with 00 (e.g., 1100). We cannot use the simple Sum Rule here. We will solve this in Section 5 using Inclusion-Exclusion.

This example illustrates why recognizing mutual exclusivity is critical.

Sum Rule in Code

Here's a simple Java illustration demonstrating the Sum Rule:

Java
/**
 * Sum Rule Example: Counting menu choices
 * A restaurant offers appetizers OR desserts (pick one category)
 */
public class SumRuleDemo {
    public static void main(String[] args) {
        String[] appetizers = {"Soup", "Salad", "Bruschetta", "Wings"};
        String[] desserts = {"Cake", "Pie", "Ice Cream"};

        // Sum Rule: total choices = |appetizers| + |desserts|
        // (assuming you pick from exactly one category)
        int totalChoices = appetizers.length + desserts.length;

        System.out.println("Appetizer options: " + appetizers.length);
        System.out.println("Dessert options:   " + desserts.length);
        System.out.println("Total choices (Sum Rule): " + totalChoices);
        // Output: Total choices (Sum Rule): 7
    }
}

In set theory terms, two sets are disjoint when A ∩ B = ∅. When we add |A| + |B|, we're counting every element exactly once. If A ∩ B ≠ ∅, some elements get counted twice — once in |A| and once in |B|.

Think of it as a Venn diagram: if the circles don't overlap, the total count is the sum of both circles. If they do overlap, you've counted the intersection twice and need to subtract it out.

This is exactly what the Inclusion-Exclusion Principle does: |A ∪ B| = |A| + |B| − |A ∩ B|.

The Product Rule (Multiplication Principle)

📘 Definition: The Product Rule

If a procedure consists of a sequence of steps where:

  • Step 1 can be done in n₁ ways, and
  • Step 2 can be done in n₂ ways (regardless of how Step 1 was done),

then the entire procedure can be done in n₁ × n₂ ways.

📐 Formal Statement (Set Theory Version)

If A and B are finite sets, then the number of elements in their Cartesian product is:

|A × B| = |A| · |B|

More generally, for k sets:

|A₁ × A₂ × … × Aₖ| = |A₁| · |A₂| · … · |Aₖ|

Key Requirement: Independent Sequential Steps

The Product Rule applies when you are performing a sequence of choices, and the number of options at each step does not depend on previous choices. The keyword is "and" or "then" — when you do one thing and then another thing.

💡 Connection to Cartesian Products (Week 1)

Recall from Week 1 that the Cartesian product A × B is the set of all ordered pairs (a, b) where a ∈ A and b ∈ B. The Product Rule is simply counting the size of a Cartesian product! If |A| = 3 and |B| = 4, then |A × B| = 12.

Visualizing the Product Rule: Tree Diagrams

A tree diagram is the best way to visualize the Product Rule. Each level of the tree represents one step, and each branch represents a choice at that step.

📝 Example 3.1: Outfit Selection

You have 3 shirts (Red, Green, Blue) and 2 pants (Jeans, Khakis). How many outfits can you make?

Solution (with Tree Diagram):

Step 1: Choose a shirt (3 ways). Step 2: Choose pants (2 ways).

By the Product Rule: 3 × 2 = 6 outfits.

📝 Example 3.2: License Plates

A license plate consists of 3 letters followed by 3 digits. How many distinct plates are possible?

Solution:

This is a 6-step sequential procedure:

  • Position 1 (letter): 26 choices
  • Position 2 (letter): 26 choices
  • Position 3 (letter): 26 choices
  • Position 4 (digit): 10 choices
  • Position 5 (digit): 10 choices
  • Position 6 (digit): 10 choices

By the Product Rule: 26 × 26 × 26 × 10 × 10 × 10 = 26³ × 10³ = 17,576 × 1,000 = 17,576,000 plates.

📝 Example 3.3: Binary Strings

How many bit strings of length n exist?

Solution:

Each position has 2 choices (0 or 1), and there are n positions.

By the Product Rule: 2 × 2 × … × 2 (n times) = 2ⁿ

  • n = 3: 2³ = 8 strings (000, 001, 010, 011, 100, 101, 110, 111)
  • n = 8: 2⁸ = 256 (one byte's worth of values)
  • n = 32: 2³² = 4,294,967,296 (IPv4 addresses!)
📝 Example 3.4: Password Strength Analysis

A password must be exactly 8 characters long. Characters can be uppercase letters (26), lowercase letters (26), or digits (10). How many possible passwords exist?

Solution:

Character set size: 26 + 26 + 10 = 62 (Sum Rule for the character set)

Each of the 8 positions can be any of 62 characters (Product Rule for the sequence):

62⁸ = 218,340,105,584,896 ≈ 2.18 × 10¹⁴

That's over 218 trillion possible passwords! Notice how both rules work together: the Sum Rule determines the size of the character set, and the Product Rule counts the total combinations.

Product Rule in Code

Java
/**
 * Product Rule Example: Counting all possible binary strings of length n
 */
public class ProductRuleDemo {
    public static void main(String[] args) {
        int n = 8; // bit string length

        // Product Rule: 2 choices per position, n positions
        long totalStrings = (long) Math.pow(2, n);

        System.out.println("Bit string length: " + n);
        System.out.println("Total bit strings: " + totalStrings);
        // Output: Total bit strings: 256

        // Password example: 62 chars, 8 positions
        int charSetSize = 26 + 26 + 10; // Sum Rule for char set
        long totalPasswords = (long) Math.pow(charSetSize, 8); // Product Rule

        System.out.println("\nCharacter set size: " + charSetSize);
        System.out.println("Password length: 8");
        System.out.println("Total passwords: " + totalPasswords);
        // Output: Total passwords: 218340105584896
    }
}

When the Product Rule Does NOT Apply

⚠️ Dependent Choices

The standard Product Rule assumes the number of choices at each step is independent of previous choices. If choosing one option at Step 1 changes the number of options at Step 2, you need to be more careful.

Example: Choosing 2 different letters from {A, B, C} without repetition:

  • Step 1: 3 choices
  • Step 2: 2 choices (one letter already used)
  • Total: 3 × 2 = 6 (not 3 × 3 = 9)

The Product Rule still works here — the key is that Step 2 always has 2 choices regardless of which letter was chosen in Step 1. We'll explore this more in Module 7.2 (Permutations).

Sum Rule vs. Product Rule — How to Decide

One of the most common challenges in counting problems is deciding which rule to apply. Here's a systematic approach:

Feature Sum Rule (Addition) Product Rule (Multiplication)
Keyword "or", "either…or" "and", "then", "followed by"
Structure Choose one from multiple categories Make a sequence of choices
Set operation Union: A ∪ B Cartesian Product: A × B
Requirement Categories must be disjoint Steps are independent
Formula n₁ + n₂ n₁ × n₂
Result tends to be Smaller Larger

Decision Framework

Ask yourself these questions when approaching a counting problem:

  1. Am I choosing ONE item from multiple categories? → Sum Rule
  2. Am I making a SEQUENCE of choices to build a compound outcome? → Product Rule
  3. Do the categories overlap? → Inclusion-Exclusion (Section 5)
  4. Are both rules needed? → Often yes! Many problems use both.
📝 Example 4.1: Combined Rules — Variable Names

In a programming language, a valid variable name:

  • Must start with a letter (uppercase or lowercase) or underscore
  • The second character can be any letter, digit, or underscore

How many 2-character variable names are possible?

Solution:

Step 1 (first character): Letters + underscore = 26 + 26 + 1 = 53 choices (Sum Rule to build the character set)

Step 2 (second character): Letters + digits + underscore = 26 + 26 + 10 + 1 = 63 choices (Sum Rule again)

Total: 53 × 63 = 3,339 valid 2-character variable names (Product Rule for the sequence)

📝 Example 4.2: Combined Rules — Network Routing

A data packet can travel from Server A to Server B via:

  • Route Type 1: Through 2 intermediate routers, with 3 choices for the first router and 4 choices for the second
  • Route Type 2: Through a direct fiber link (5 different fibers available)

How many total routes exist?

Solution:

Route Type 1: 3 × 4 = 12 routes (Product Rule — sequential routers)

Route Type 2: 5 routes

Total: 12 + 5 = 17 routes (Sum Rule — choose one route type)

The Inclusion-Exclusion Principle

What happens when our categories overlap? We can't simply add because elements in the overlap would be counted twice. The Inclusion-Exclusion Principle corrects for this double-counting.

📘 Definition: Inclusion-Exclusion Principle (Two Sets)

For any two finite sets A and B:

|A ∪ B| = |A| + |B| − |A ∩ B|

We include both sets, then exclude the overlap to avoid counting it twice.

💡 Special Case: Disjoint Sets

When A ∩ B = ∅, we have |A ∩ B| = 0, and the formula reduces to the Sum Rule: |A ∪ B| = |A| + |B|. So the Inclusion-Exclusion Principle is a generalization of the Sum Rule.

Visualizing Inclusion-Exclusion

Venn Diagram Intuition:

|A| counts everything in the left circle.
|B| counts everything in the right circle.
The overlap (A ∩ B) gets counted twice — once in |A| and once in |B|.
So we subtract it once: |A| + |B| − |A ∩ B|

📝 Example 5.1: Bit Strings (Revisited from Example 2.3)

How many bit strings of length 4 either start with 1 or end with 00?

Solution:

Let A = set of 4-bit strings starting with 1

Let B = set of 4-bit strings ending with 00

|A|: First bit is fixed as 1, remaining 3 bits are free: 1 × 2 × 2 × 2 = 8

|B|: Last two bits are fixed as 00, first 2 bits are free: 2 × 2 × 1 × 1 = 4

|A ∩ B|: Start with 1 AND end with 00: 1 × 2 × 1 × 1 = 2 (strings: 1000, 1100)

By Inclusion-Exclusion: |A ∪ B| = 8 + 4 − 2 = 10

📝 Example 5.2: CS Students with Skills

In a class of 40 CS students: 25 know Java, 18 know Python, and 10 know both Java and Python. How many students know at least one of these languages?

Solution:

Let J = students who know Java, P = students who know Python.

|J ∪ P| = |J| + |P| − |J ∩ P| = 25 + 18 − 10 = 33 students know at least one language.

This means 40 − 33 = 7 students know neither Java nor Python.

📝 Example 5.3: Integers Divisible by 2 or 3

How many positive integers from 1 to 100 are divisible by 2 or 3?

Solution:

Let A = multiples of 2 in {1, …, 100}: |A| = ⌊100/2⌋ = 50

Let B = multiples of 3 in {1, …, 100}: |B| = ⌊100/3⌋ = 33

A ∩ B = multiples of 6 (both 2 and 3): |A ∩ B| = ⌊100/6⌋ = 16

|A ∪ B| = 50 + 33 − 16 = 67

Three-Set Inclusion-Exclusion (Preview)

📐 Inclusion-Exclusion for Three Sets

For three finite sets A, B, and C:

|A ∪ B ∪ C| = |A| + |B| + |C| − |A ∩ B| − |A ∩ C| − |B ∩ C| + |A ∩ B ∩ C|

Pattern: Add singles, subtract pairs, add triple. This alternating pattern continues for any number of sets.

Inclusion-Exclusion in Code

Java
/**
 * Inclusion-Exclusion Example:
 * Count integers from 1 to n divisible by a OR b
 */
public class InclusionExclusionDemo {
    public static void main(String[] args) {
        int n = 100;
        int a = 2, b = 3;

        int divByA = n / a;                    // multiples of a
        int divByB = n / b;                    // multiples of b
        int divByBoth = n / lcm(a, b);         // multiples of lcm(a,b)

        // Inclusion-Exclusion: |A ∪ B| = |A| + |B| - |A ∩ B|
        int divByAorB = divByA + divByB - divByBoth;

        System.out.println("Integers 1-" + n + " divisible by " + a + ": " + divByA);
        System.out.println("Integers 1-" + n + " divisible by " + b + ": " + divByB);
        System.out.println("Integers 1-" + n + " divisible by both: " + divByBoth);
        System.out.println("Divisible by " + a + " or " + b + ": " + divByAorB);
        // Output: Divisible by 2 or 3: 67
    }

    static int gcd(int a, int b) {
        while (b != 0) {
            int temp = b;
            b = a % b;
            a = temp;
        }
        return a;
    }

    static int lcm(int a, int b) {
        return a / gcd(a, b) * b;
    }
}

📋 Module 7.1 Summary

  • Combinatorics is the mathematics of counting, essential for algorithm analysis, security, and data structure design.
  • Sum Rule: If task can be done way 1 (n₁ ways) OR way 2 (n₂ ways), and ways are mutually exclusive, total = n₁ + n₂. Corresponds to |A ∪ B| for disjoint sets.
  • Product Rule: If a procedure has Step 1 (n₁ ways) AND Step 2 (n₂ ways), total = n₁ × n₂. Corresponds to |A × B| (Cartesian product).
  • Deciding which rule: "OR" → Sum Rule. "AND/THEN" → Product Rule. Many problems use both.
  • Inclusion-Exclusion: When categories overlap, |A ∪ B| = |A| + |B| − |A ∩ B|. Generalizes the Sum Rule.
  • CS Applications: Password strength (Product Rule), character set construction (Sum Rule), survey overlap (Inclusion-Exclusion).
