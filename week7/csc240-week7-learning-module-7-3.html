🔗 Connection to Module 7.2

In Module 7.2 you learned that P(n, r) = n!/(n − r)! counts ordered arrangements of r items from n. But what if order doesn't matter? Choosing {A, B, C} is the same as choosing {C, A, B} when forming a committee. Combinations remove the ordering — every unordered group of r items corresponds to exactly r! ordered arrangements. So we divide: C(n, r) = P(n, r) / r!.

1

Combinations — Unordered Selections

A combination answers the question: "How many ways can I choose r items from n items when the order doesn't matter?"

📘 Definition: r-Combination (Binomial Coefficient)

An r-combination of a set of n distinct elements is an unordered selection of r elements. The count is denoted C(n, r), also written (nr) or ⁿCᵣ:

C(n, r) = n! / (r! × (n − r)!)

where 0 ≤ r ≤ n. If r > n or r < 0, then C(n, r) = 0.

Derivation from Permutations

Observation: Every unordered group of r items can be arranged in r! different orders.
Therefore: P(n, r) counts each unordered group r! times (once per internal arrangement).
To get unordered count: Divide out the redundant orderings.
Formula: C(n, r) = P(n, r) / r! = [n!/(n − r)!] / r! = n! / (r! × (n − r)!)
📝 Example 1.1: Seeing the Division in Action

From {A, B, C, D}, how many ways can we choose 2 elements?

Solution:

Permutations (ordered): P(4, 2) = 4 × 3 = 12

All 12: AB, AC, AD, BA, BC, BD, CA, CB, CD, DA, DB, DC

Each pair appears twice (e.g., AB and BA). Since 2! = 2:

Combinations: C(4, 2) = 12 / 2 = 6

The 6 combinations: {A,B}, {A,C}, {A,D}, {B,C}, {B,D}, {C,D}

📝 Example 1.2: Team Selection

How many ways can a 4-person team be selected from 20 students?

Solution:

Order doesn't matter — {Alice, Bob, Carol, Dave} = {Dave, Carol, Alice, Bob}.

C(20, 4) = (20 × 19 × 18 × 17) / (4 × 3 × 2 × 1) = 116,280 / 24 = 4,845

📝 Example 1.3: Bit Strings with Exactly k Ones

How many bit strings of length 10 have exactly 4 ones?

Solution:

Choose which 4 of 10 positions get a 1. Order of selection doesn't matter.

C(10, 4) = 10! / (4! × 6!) = 5,040 / 24 = 210

📝 Example 1.4: Poker Hands

How many 5-card hands from a standard 52-card deck?

Solution:

C(52, 5) = (52 × 51 × 50 × 49 × 48) / (5!) = 311,875,200 / 120 = 2,598,960

📝 Example 1.5: Complementary Counting

How many 5-card hands contain at least one ace?

Solution:

At least one ace = Total − No aces = C(52, 5) − C(48, 5) = 2,598,960 − 1,712,304 = 886,656

Combination Formula in Code

Java
public class Combinations {
    public static long comb(int n, int r) {
        if (r < 0 || r > n) return 0;
        if (r == 0 || r == n) return 1;
        if (r > n - r) r = n - r;  // Symmetry optimization
        long result = 1;
        for (int i = 0; i < r; i++) {
            result = result * (n - i) / (i + 1);
        }
        return result;
    }

    public static void main(String[] args) {
        System.out.println("C(20, 4)  = " + comb(20, 4));   // 4845
        System.out.println("C(10, 4)  = " + comb(10, 4));   // 210
        System.out.println("C(52, 5)  = " + comb(52, 5));   // 2598960
        System.out.println("C(10, 0)  = " + comb(10, 0));   // 1
        System.out.println("C(10, 10) = " + comb(10, 10));  // 1
    }
}

⚠️ The "Swap Test"

Combinations with Repetition — Stars and Bars

What if we make an unordered selection where we can choose the same item more than once?

📘 Definition: Combination with Repetition

A combination with repetition selects r items from n types where each type may be chosen multiple times and order does not matter:

C(n + r − 1, r) = (n + r − 1)! / (r! × (n − 1)!)

The Stars and Bars Method

  • Stars (★) represent the r items being chosen
  • Bars (|) separate items into n categories — we need n − 1 bars
  • Total symbols: r + (n − 1) = n + r − 1
  • Choose which positions are stars (or bars): C(n + r − 1, r)
📐 Stars and Bars Theorem

The number of ways to place r identical items into n distinct categories is:

C(n + r − 1, r) = C(n + r − 1, n − 1)
📝 Example 3.1: Ice Cream Scoops

5 flavors, buying 3 scoops, can repeat. How many selections?

Solution:

C(5 + 3 − 1, 3) = C(7, 3) = 35

★ ★ | | ★ | | means: 2 of flavor 1, 0 of flavor 2, 1 of flavor 3, 0 of flavors 4–5.

📝 Example 3.2: Distributing Identical Cookies

12 identical cookies distributed among 4 children?

Solution:

Stars = cookies (r = 12), categories = children (n = 4).

C(4 + 12 − 1, 12) = C(15, 3) = (15 × 14 × 13) / 6 = 455

📝 Example 3.3: Solutions to an Equation

How many non-negative integer solutions to x₁ + x₂ + x₃ = 10?

Solution:

Distributing 10 units into 3 variables: C(3 + 10 − 1, 10) = C(12, 2) = 66

Pascal's Triangle

Pascal's Triangle is a triangular array where each entry is a binomial coefficient C(n, r). It provides a visual, computation-free way to find combinations and reveals deep patterns.

📘 Definition: Pascal's Triangle
  • Row n contains C(n, 0), C(n, 1), …, C(n, n)
  • Boundary entries are 1: C(n, 0) = C(n, n) = 1
  • Each interior entry = sum of the two entries directly above it

The First 8 Rows

Patterns in Pascal's Triangle

  1. Boundary values: Edges are always 1
  2. Symmetry: Each row reads the same forwards and backwards
  3. Row sums: Sum of row n = 2ⁿ (e.g., row 4: 1+4+6+4+1 = 16 = 2⁴)
  4. Diagonals: Column 1 = natural numbers; Column 2 = triangular numbers
  5. Powers of 11: First rows give 1, 11, 121, 1331, 14641

Although named after Blaise Pascal (1623–1662), this triangle was known centuries earlier. Persian mathematician Omar Khayyám (1048–1131) described it, Chinese mathematician Yang Hui published it in 1261, and Indian mathematician Pingala (2nd century BCE) had related ideas. Pascal's contribution was a systematic study of its properties and application to probability.

Building Pascal's Triangle in Code

Java
public class PascalTriangle {
    public static int[][] buildTriangle(int numRows) {
        int[][] triangle = new int[numRows][];
        for (int n = 0; n < numRows; n++) {
            triangle[n] = new int[n + 1];
            triangle[n][0] = 1;          // C(n, 0) = 1
            triangle[n][n] = 1;          // C(n, n) = 1
            for (int r = 1; r < n; r++) {
                // Pascal's Identity
                triangle[n][r] = triangle[n-1][r-1] + triangle[n-1][r];
            }
        }
        return triangle;
    }

    public static void main(String[] args) {
        int[][] tri = buildTriangle(8);
        for (int n = 0; n < tri.length; n++) {
            System.out.print(" ".repeat((tri.length - n - 1) * 3));
            for (int r = 0; r <= n; r++)
                System.out.printf("%5d ", tri[n][r]);
            System.out.println();
        }
    }
}

Pascal's Identity

The addition rule that builds Pascal's Triangle is not just a computational trick — it is a deep identity with a beautiful combinatorial proof. Pascal's Identity is the single most important property of binomial coefficients.

📐 Theorem: Pascal's Identity

For integers n and r with 1 ≤ r ≤ n − 1:

C(n, r) = C(n − 1, r − 1) + C(n − 1, r)

In words: each interior entry of Pascal's Triangle equals the sum of the two entries directly above it.

Algebraic Proof

We can verify the identity by expanding both sides using the factorial formula:

📜 Proof (Algebraic)

Starting with the right-hand side:

C(n−1, r−1) + C(n−1, r)

= (n−1)! / ((r−1)!(n−r)!) + (n−1)! / (r!(n−1−r)!)

Finding a common denominator of r!(n−r)!:

= (n−1)! × r / (r!(n−r)!) + (n−1)! × (n−r) / (r!(n−r)!)

= (n−1)! × [r + (n − r)] / (r!(n−r)!)

= (n−1)! × n / (r!(n−r)!)

= n! / (r!(n−r)!)

= C(n, r) ✓

■

Combinatorial Proof

The combinatorial proof is more elegant and gives intuitive understanding of why the identity is true. This is the style of proof emphasized in the Saylor CS202 textbook.

📜 Proof (Combinatorial — Double Counting)

Goal: Show that C(n, r) = C(n−1, r−1) + C(n−1, r).

Setup: Consider a set S with n elements. We want to count the number of r-element subsets of S. By definition, this is C(n, r). We will count the same quantity a different way.

Key idea: Fix one specific element x ∈ S. Every r-element subset of S either contains x or does not contain x. These two cases are mutually exclusive and exhaustive.

Case 1 — x is included: If x is in the subset, we need to choose the remaining r − 1 elements from the other n − 1 elements (everything in S except x). The number of such subsets is C(n − 1, r − 1).

Case 2 — x is excluded: If x is not in the subset, we need to choose all r elements from the other n − 1 elements. The number of such subsets is C(n − 1, r).

Conclusion: By the Sum Rule (Module 7.1), since every r-element subset falls into exactly one case:

C(n, r) = C(n − 1, r − 1) + C(n − 1, r)

■

💡 Why Combinatorial Proofs Matter

A combinatorial proof (also called a "proof by counting" or "bijective proof") counts the same set in two different ways. If both methods count the same set, the results must be equal. This technique is powerful because it provides understanding, not just verification. You'll encounter it frequently in discrete mathematics and theoretical computer science.

📝 Example 5.1: Verifying Pascal's Identity

Verify Pascal's Identity for C(6, 2).

Solution:

Left side: C(6, 2) = 6! / (2! × 4!) = 720 / (2 × 24) = 15

Right side: C(5, 1) + C(5, 2) = 5 + 10 = 15 ✓

Combinatorial interpretation: From {a, b, c, d, e, f}, choose 2. Fix element f:

  • Subsets containing f: {f, a}, {f, b}, {f, c}, {f, d}, {f, e} → C(5, 1) = 5
  • Subsets not containing f: {a,b}, {a,c}, {a,d}, {a,e}, {b,c}, {b,d}, {b,e}, {c,d}, {c,e}, {d,e} → C(5, 2) = 10

Total: 5 + 10 = 15 = C(6, 2) ✓

📝 Example 5.2: Using Pascal's Identity for Computation

Compute C(8, 3) using Pascal's Identity (without the factorial formula).

Solution:

C(8, 3) = C(7, 2) + C(7, 3)

C(7, 2) = C(6, 1) + C(6, 2) = 6 + 15 = 21

C(7, 3) = C(6, 2) + C(6, 3) = 15 + 20 = 35

C(8, 3) = 21 + 35 = 56

Check: 8!/(3! × 5!) = 40320/(6 × 120) = 40320/720 = 56 ✓

Properties of Binomial Coefficients

Binomial coefficients satisfy many elegant properties. Understanding these helps you simplify calculations and verify results.

Property 1: Boundary Values

📐 Boundary Values

For any non-negative integer n:

C(n, 0) = 1     and     C(n, n) = 1

Interpretation: There is exactly one way to choose nothing (the empty set), and exactly one way to choose everything (the entire set).

Property 2: Symmetry

📐 Symmetry Property

For 0 ≤ r ≤ n:

C(n, r) = C(n, n − r)

Interpretation: Choosing r items to include is the same as choosing n − r items to exclude. Every selection of items determines a complementary non-selection.

📜 Proof (Algebraic)

C(n, n − r) = n! / ((n − r)! × (n − (n − r))!) = n! / ((n − r)! × r!) = C(n, r) ✓

■

📝 Example 6.1: Symmetry in Practice

C(100, 97) looks intimidating, but by symmetry: C(100, 97) = C(100, 3) = (100 × 99 × 98) / 6 = 161,700

This is why the Java code in Section 1 includes the optimization if (r > n - r) r = n - r;

Property 3: Row Sum

📐 Row Sum Property

For any non-negative integer n:

C(n, 0) + C(n, 1) + C(n, 2) + … + C(n, n) = 2ⁿ

The sum of all entries in row n of Pascal's Triangle equals 2ⁿ.

📜 Proof (Combinatorial)

Left side: Σ C(n, r) for r = 0 to n counts all subsets of an n-element set, grouped by size. C(n, 0) counts 0-element subsets, C(n, 1) counts 1-element subsets, …, C(n, n) counts n-element subsets.

Right side: 2ⁿ is the total number of subsets of an n-element set (from Week 1 — each element is either in or out, giving 2 choices per element, so 2ⁿ total).

Since both sides count the same thing (all subsets of an n-element set), they are equal.

■

📝 Example 6.2: Verifying the Row Sum

Row 5 of Pascal's Triangle: 1, 5, 10, 10, 5, 1

Verification:

1 + 5 + 10 + 10 + 5 + 1 = 32 = 2⁵ ✓

This means a 5-element set has 32 total subsets: 1 empty set + 5 singletons + 10 pairs + 10 triples + 5 four-element subsets + 1 full set.

Property 4: Alternating Sum

📐 Alternating Row Sum

For any positive integer n:

C(n, 0) − C(n, 1) + C(n, 2) − C(n, 3) + … + (−1)ⁿ C(n, n) = 0

Proof sketch: Substitute x = −1 into the Binomial Theorem (Week 8): (1 + (−1))ⁿ = 0ⁿ = 0.

Property 5: Vandermonde's Identity (Preview)

📐 Vandermonde's Identity

For non-negative integers m, n, r with r ≤ m + n:

C(m + n, r) = Σ C(m, k) × C(n, r − k) for k = 0 to r

Interpretation: Choosing r items from two groups (m items and n items) can be done by choosing k from the first group and r − k from the second, summed over all valid k.

📋 Module 7.3 Summary

  • Combination C(n, r) = n!/(r!(n−r)!): Counts unordered selections of r items from n. Derived from permutations: C(n, r) = P(n, r)/r!. Also counts r-element subsets.
  • Permutation vs. Combination: Does order matter? Yes → permutation. No → combination. Use the "swap test" to decide. The flowchart covers all four formula types.
  • Stars and Bars: Combinations with repetition = C(n+r−1, r). Distributing r identical items into n categories. Equivalent to non-negative integer solutions of x₁ + x₂ + … + xₙ = r.

  • Pascal's Identity: C(n, r) = C(n−1, r−1) + C(n−1, r). Combinatorial proof: fix one element, split into "include" and "exclude" cases.
  • Key Properties: Boundary values (C(n,0)=C(n,n)=1), Symmetry (C(n,r)=C(n,n−r)), Row Sum (Σ C(n,r)=2ⁿ), Alternating Sum (= 0).
