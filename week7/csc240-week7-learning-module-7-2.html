The Factorial Function

Before we can count permutations, we need the factorial function — one of the most important functions in combinatorics. You may have encountered it in prior math courses, but here we define it rigorously and connect it to counting.

📘 Definition: Factorial

For any non-negative integer n, the factorial of n, written n! (read "n factorial"), is defined as:

n! = n × (n − 1) × (n − 2) × … × 2 × 1

By convention: 0! = 1

Recursive Definition

The factorial can also be defined recursively, which connects directly to the recursion topics you'll study in Week 10:

📐 Recursive Definition of Factorial

Base case: 0! = 1

Recursive case: n! = n × (n − 1)! for n ≥ 1

This means: to compute 5!, compute 5 × 4!, which requires 4 × 3!, which requires 3 × 2!, which requires 2 × 1!, which requires 1 × 0! = 1.

⚠️ Factorial Growth is Explosive

Notice that 20! ≈ 2.4 × 10¹⁸ — that's over 2 quintillion. For comparison, 2⁶⁴ ≈ 1.8 × 10¹⁹. Factorials grow faster than exponentials: for large n, n! ≫ 2ⁿ. This is why brute-force algorithms that check all permutations (like the naive traveling salesman solution) become infeasible very quickly.

Why 0! = 1

This convention may seem arbitrary, but it has strong justifications:

  1. Combinatorial argument: 0! counts the number of ways to arrange 0 objects. There is exactly one way to arrange nothing — do nothing. So 0! = 1.
  2. Recursive consistency: Since n! = n × (n − 1)!, we need 1! = 1 × 0!. Since 1! = 1, we get 0! = 1.
  3. Formula consistency: The permutation formula P(n, n) = n!/(n − n)! = n!/0! should equal n!, which requires 0! = 1.

Factorial in Code

Java
/**
 * Factorial implementations: iterative and recursive
 */
public class Factorial {

    // Iterative version
    public static long factorialIterative(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be non-negative");
        long result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }

    // Recursive version
    public static long factorialRecursive(int n) {
        if (n < 0) throw new IllegalArgumentException("n must be non-negative");
        if (n == 0) return 1;           // Base case: 0! = 1
        return n * factorialRecursive(n - 1);  // Recursive case
    }

    public static void main(String[] args) {
        for (int i = 0; i <= 12; i++) {
            System.out.printf("%2d! = %,d%n", i, factorialIterative(i));
        }
        // Note: long overflows at 21! (exceeds Long.MAX_VALUE)
    }
}
💡 Overflow Warning

In Java, long can hold values up to 9,223,372,036,854,775,807 (≈ 9.2 × 10¹⁸). Since 20! ≈ 2.4 × 10¹⁸ fits, but 21! ≈ 5.1 × 10¹⁹ does not. For larger factorials, use java.math.BigInteger. This is a real-world consideration when implementing combinatorial algorithms!

Permutations of n Distinct Objects

📘 Definition: Permutation

A permutation of a set of distinct objects is an ordered arrangement of all those objects. The key word is ordered — the sequence in which objects appear matters.

The number of permutations of n distinct objects is n!

Why n! — The Product Rule Derivation

Arranging n distinct objects in a line is a sequential decision process:

Position 1: Choose any of the n objects → n choices
Position 2: Choose from the remaining n − 1 objects → n − 1 choices
Position 3: Choose from the remaining n − 2 objects → n − 2 choices
⋮
Position n: Only 1 object remains → 1 choice

By the Product Rule (Module 7.1):

Total arrangements = n × (n − 1) × (n − 2) × … × 2 × 1 = n!
🔗 Connection to Week 2: Bijections

A permutation of a set S = {s₁, s₂, …, sₙ} is equivalent to a bijection from S to itself. The function maps each position (1, 2, …, n) to the element placed there. Since the mapping is one-to-one and onto, it's a bijection. The count n! equals the number of bijections from an n-element set to itself.

📝 Example 2.1: Arranging Books on a Shelf

How many ways can 8 students line up for a photo?

Solution:

We need to arrange all 8 students in a line (order matters — who stands where).

Number of arrangements = 8! = 8 × 7 × 6 × 5 × 4 × 3 × 2 × 1 = 40,320

📝 Example 2.2: Scheduling Processes

An operating system has 5 processes in the ready queue. How many different execution orders are possible?

Solution:

Each execution order is a permutation of the 5 processes.

Number of orders = 5! = 120

This is relevant to CPU scheduling algorithms — a brute-force scheduler that checks all orderings would need to evaluate 120 possibilities for just 5 processes. For 20 processes, that's 20! ≈ 2.4 × 10¹⁸ — completely infeasible.

📝 Example 2.3: Seating at a Round Table

How many ways can 6 people sit around a circular table?

Solution:

In a circular arrangement, rotations of the same arrangement are considered identical (rotating everyone one seat clockwise gives the same relative seating).

Fix one person's position to break the rotational symmetry. The remaining 5 people can be arranged in 5! ways.

Circular permutations of n objects = (n − 1)! = 5! = 120

r-Permutations: Partial Arrangements P(n, r)

Often we don't arrange all n objects — we select and arrange only r of them. This is called an r-permutation.

📘 Definition: r-Permutation

An r-permutation of a set of n distinct elements is an ordered arrangement of r elements chosen from the set (without repetition).

The number of r-permutations of n objects is denoted P(n, r) (also written ⁿPᵣ or P(n, r)):

P(n, r) = n! / (n − r)! = n × (n − 1) × (n − 2) × … × (n − r + 1)

This is the product of r consecutive descending integers starting from n.

Derivation Using the Product Rule

Selecting and arranging r objects from n distinct objects:

Position 1: n choices
Position 2: n − 1 choices
Position 3: n − 2 choices
⋮
Position r: n − r + 1 choices

By the Product Rule: n × (n − 1) × … × (n − r + 1)

This product can be written compactly as n!/(n − r)! because:

n!/(n − r)! = [n × (n−1) × … × (n−r+1) × (n−r) × … × 1] / [(n−r) × … × 1]

The (n − r)! in the denominator cancels the "tail" of the factorial, leaving exactly the first r factors.

💡 Special Case: P(n, n) = n!

When r = n (arrange all objects), P(n, n) = n!/(n − n)! = n!/0! = n!/1 = n!. This confirms that full permutations are a special case of r-permutations.

Examples

📝 Example 3.1: Club Officers

A club of 12 members elects a president, vice president, and treasurer (all different people). How many outcomes are possible?

Solution:

We are selecting 3 people from 12 and the order matters (president ≠ vice president ≠ treasurer).

P(12, 3) = 12!/(12 − 3)! = 12!/9! = 12 × 11 × 10 = 1,320

📝 Example 3.2: Podium Finishes

In a race with 20 runners, how many ways can the gold, silver, and bronze medals be awarded?

Solution:

P(20, 3) = 20 × 19 × 18 = 6,840

📝 Example 3.3: Injective Functions

How many injective (one-to-one) functions exist from A = {a, b, c, d} to B = {1, 2, 3, 4, 5, 6}?

Solution:

An injective function from A to B assigns each element of A a distinct element of B. This is equivalent to choosing 4 elements from B and arranging them (assigning one to each element of A).

P(6, 4) = 6!/2! = 6 × 5 × 4 × 3 = 360

Connection to Week 2: The number of injective functions from an m-element set to an n-element set (where m ≤ n) is P(n, m).

P(n, r) in Code

Java
/**
 * Compute P(n, r) = n! / (n-r)!
 * Uses the product form to avoid computing large factorials
 */
public class Permutations {

    public static long perm(int n, int r) {
        if (r < 0 || r > n) {
            throw new IllegalArgumentException("Requires 0 <= r <= n");
        }
        long result = 1;
        for (int i = 0; i < r; i++) {
            result *= (n - i);  // n, n-1, n-2, ..., n-r+1
        }
        return result;
    }

    public static void main(String[] args) {
        System.out.println("P(12, 3) = " + perm(12, 3));  // 1320
        System.out.println("P(20, 3) = " + perm(20, 3));  // 6840
        System.out.println("P(6, 4)  = " + perm(6, 4));   // 360
        System.out.println("P(5, 5)  = " + perm(5, 5));   // 120 = 5!
        System.out.println("P(5, 0)  = " + perm(5, 0));   // 1
    }
}

Permutations with Repetition Allowed

In the permutations we've studied so far, each object can be used at most once. But what if repetition is allowed — what if we can reuse objects?

📘 Definition: Permutations with Repetition

An r-permutation with repetition from a set of n elements is an ordered arrangement of r elements where each element may be used any number of times.

The number of such arrangements is:

nʳ (n to the power r)

Why nʳ — The Product Rule Again

When repetition is allowed, each of the r positions has n choices independently:

Position 1: n choices
Position 2: n choices (repetition allowed!)
Position 3: n choices
⋮
Position r: n choices

By the Product Rule: n × n × n × … × n (r times) = nʳ

💡 This Is the Product Rule from Module 7.1!

Permutations with repetition are exactly the Product Rule with constant choices. The password and bit string examples from Module 7.1 were actually permutations with repetition! A bit string of length n is an n-permutation with repetition from {0, 1}: 2ⁿ possibilities.

Comparison: With vs. Without Repetition

Type Formula Example (n=5, r=3) Count
Without repetition P(n, r) = n!/(n−r)! P(5, 3) = 5 × 4 × 3 60
With repetition nʳ 5³ = 5 × 5 × 5 125

Repetition always gives more outcomes because the pool of choices never shrinks.

📝 Example 4.1: Strings from an Alphabet

How many 5-character strings can be formed from {a, b, c} if repetition is allowed?

Solution:

Each of the 5 positions can be a, b, or c (3 choices per position, repetition allowed).

Total = 3⁵ = 243 strings

Examples: aaaaa, aabbc, cbacb, ccccc, …

📝 Example 4.2: PIN Codes

A PIN consists of 4 digits (0–9). How many PINs are possible?

Solution:

10 choices per position, 4 positions, repetition allowed.

Total = 10⁴ = 10,000 PINs (from 0000 to 9999)

📝 Example 4.3: Function Counting

How many functions exist from a set A with 4 elements to a set B with 3 elements?

Solution:

A function assigns each element of A to some element of B. Each of the 4 elements of A has 3 choices (and different elements can map to the same target — repetition is allowed).

Total = 3⁴ = 81 functions

Connection to Week 2: The number of functions from an m-element set to an n-element set is nᵐ. Compare this to injective functions: P(n, m). Since P(3, 4) is undefined (can't have an injection from 4 elements to 3), there are 0 injective functions here, but 81 total functions.

Permutations of Multisets (Identical Items)

What if some of the objects we're arranging are identical? If we have repeated elements, swapping two identical items doesn't create a new arrangement. We need to divide out the over-counting.

📘 Definition: Multiset Permutation

Given n objects where there are n₁ identical objects of type 1, n₂ identical objects of type 2, …, and nₖ identical objects of type k (with n₁ + n₂ + … + nₖ = n), the number of distinct arrangements is:

n! / (n₁! × n₂! × … × nₖ!)

This is also called a multinomial coefficient.

Why We Divide

Consider arranging the letters in "BANANA":

The word has 6 letters: B A N A N A

  • B appears 1 time
  • A appears 3 times
  • N appears 2 times

If all 6 letters were distinct, there would be 6! = 720 arrangements. But since the 3 A's are identical, any rearrangement of just the A's among their positions doesn't create a new word. Similarly for the 2 N's.

We divide by the number of ways to rearrange each group of identical items:

6! / (1! × 3! × 2!) = 720 / (1 × 6 × 2) = 720 / 12 = 60
📝 Example 5.1: Arrangements of "BANANA"

How many distinct arrangements of the letters in "BANANA" are there?

Solution:

n = 6 total letters. B: 1, A: 3, N: 2.

Distinct arrangements = 6! / (1! × 3! × 2!) = 720 / 12 = 60

📝 Example 5.2: Arrangements of "MISSISSIPPI"

How many distinct arrangements of the letters in "MISSISSIPPI" are there?

Solution:

n = 11 total letters. M: 1, I: 4, S: 4, P: 2.

Distinct arrangements = 11! / (1! × 4! × 4! × 2!)

= 39,916,800 / (1 × 24 × 24 × 2)

= 39,916,800 / 1,152 = 34,650

📝 Example 5.3: Binary String Arrangements

How many bit strings of length 10 have exactly 4 ones (and 6 zeros)?

Solution:

We're arranging 10 bits: 4 identical 1's and 6 identical 0's.

Distinct arrangements = 10! / (4! × 6!) = 3,628,800 / (24 × 720) = 3,628,800 / 17,280 = 210

Preview: This is also C(10, 4) — a combination! We're choosing which 4 of the 10 positions get a 1. You'll study this formula in Module 7.3.

Multiset Permutations in Code

Java
/**
 * Count distinct permutations of a string with repeated characters
 */
public class MultisetPermutations {

    public static long factorial(int n) {
        long result = 1;
        for (int i = 2; i <= n; i++) result *= i;
        return result;
    }

    public static long distinctPermutations(String s) {
        // Count frequency of each character
        java.util.Map<Character, Integer> freq = new java.util.HashMap<>();
        for (char c : s.toCharArray()) {
            freq.put(c, freq.getOrDefault(c, 0) + 1);
        }

        // Compute n! / (n1! * n2! * ... * nk!)
        long numerator = factorial(s.length());
        long denominator = 1;
        for (int count : freq.values()) {
            denominator *= factorial(count);
        }

        return numerator / denominator;
    }

    public static void main(String[] args) {
        System.out.println("BANANA:      " + distinctPermutations("BANANA"));
        // Output: 60
        System.out.println("MISSISSIPPI: " + distinctPermutations("MISSISSIPPI"));
        // Output: 34650
        System.out.println("ABCDE:       " + distinctPermutations("ABCDE"));
        // Output: 120 (all distinct, so 5!)
    }
}

📋 Module 7.2 Summary

  • Factorial: n! = n × (n−1) × … × 1, with 0! = 1. Grows explosively — faster than exponentials.
  • Full permutation (n!): The number of ways to arrange all n distinct objects. Equivalent to counting bijections from a set to itself.
  • r-Permutation P(n, r) = n!/(n−r)!: The number of ways to select and arrange r objects from n distinct objects without repetition. Counts injective functions.
  • Permutations with repetition (nʳ): r positions, each with n independent choices. Equivalent to the Product Rule with constant step sizes. Counts all functions from an r-set to an n-set.
  • Multiset permutations n!/(n₁!n₂!…nₖ!): Arrangements when some objects are identical. Divide out the over-counting from swapping identical items.
  • Key decision: Does order matter? → Yes: permutation. Can items repeat? → Determines which formula.
📐 Quick Formula Reference
Type Formula When to Use
Full permutation n! Arrange ALL n distinct objects
r-permutation P(n, r) = n!/(n−r)! Choose and arrange r from n, no repetition
With repetition nʳ r positions, n choices each, repetition OK
Multiset n!/(n₁!n₂!…nₖ!) Arrange n objects with identical groups
Circular (n−1)! Arrange n objects in a circle
