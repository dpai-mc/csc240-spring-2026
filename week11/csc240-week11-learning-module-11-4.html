Recurrence Relations: Defining, Modeling, and Solving

This module is the mathematical core of PE-08.

1Definition and Notation

📐 Formal Definition

A recurrence relation for a sequence {an} is an equation that expresses an in terms of one or more earlier terms an−1, an−2, …, an−k. The relation alone defines a family of sequences; a unique sequence is selected by specifying initial conditions — explicit values for a0, a1, …, ak−1.

The standard form of a degree-k linear recurrence relation with constant coefficients is:

an = c1an−1 + c2an−2 + ⋯ + ckan−k initial conditions: a0 = v0,   a1 = v1,   …   ak−1 = vk−1

where c1, c2, …, ck are real constants with ck ≠ 0. The degree (or order) k is the distance between subscript n and the earliest subscript referenced on the right-hand side.

Recurrence Relation vs. Recursive Definition

These two constructs have the same algebraic form but serve different purposes:

Feature Recursive Definition Recurrence Relation
Purpose Defines what a sequence is — enumerates its values Describes the cost of a computation as a function of input size
Context Pure mathematics; data structures Algorithm analysis and complexity theory
Goal Enumerate values or prove sequence properties Derive a closed-form Big-O bound for T(n)
Example F(n) = F(n−1) + F(n−2) defines the Fibonacci numbers T(n) = 2T(n/2) + n describes the runtime cost of merge sort
⚠ Common Confusion

F(n) = F(n−1) + F(n−2) and T(n) = T(n−1) + 1 look structurally identical — and both are recurrence relations. The distinction lies in interpretation: F(n) is a number sequence; T(n) is the count of operations an algorithm performs on an input of size n. Always clarify what the variable represents before setting up a recurrence.

Why Initial Conditions Are Required

💭 Concept Question Ungraded

Consider an = 2an−1. The same rule produces entirely different sequences depending on the starting value:

  • a0 = 1  →  1, 2, 4, 8, 16, …
  • a0 = 3  →  3, 6, 12, 24, 48, …
  • a0 = −5  →  −5, −10, −20, −40, …

Why does the recurrence alone not uniquely determine the sequence?

A recurrence relation is a rule for extending a sequence — it specifies how each new term relates to earlier ones, but it says nothing about where the sequence starts. Without initial conditions, infinitely many distinct sequences all satisfy the same rule. Uniqueness is guaranteed only when you supply exactly k initial values for a degree-k recurrence. Those values anchor the sequence so that every subsequent term is determined by one and only one application of the rule. This is directly analogous to an ordinary differential equation: the general solution is a family of curves, and initial conditions select the unique curve that passes through a given point.

2Modeling Algorithms as Recurrence Relations

Translating pseudocode into a recurrence relation is a core algorithm analysis skill. Apply these five steps consistently:

  1. 1
    Identify the size parameter n Determine the quantity governing problem size: array length, number of elements, tree height, etc.
  2. 2
    Count work done at the current call level Count only operations performed directly at this level — comparisons, swaps, arithmetic. Do not count work inside recursive calls; that will be handled by the T(·) terms.
  3. 3
    Identify recursive calls How many recursive calls are made, and what is the subproblem size passed to each call?
  4. 4
    Write the recurrence equation Combine: T(n) = [sum of costs of all recursive calls] + [work done at the current level].
  5. 5
    State the base case Identify when recursion stops and record T(base case) = [cost at that point].

💡 Anatomy of the Merge Sort Recurrence

T(n) = 2T(n/2) + n
The leading 2 counts the number of recursive calls. The divisor 2 in n/2 is the subproblem-size reduction factor. The trailing n is the cost of the merge step executed after both recursive calls return. This form — aT(n/b) + f(n) — is the signature of divide-and-conquer recurrences; Module 11.5 handles them with the Master Theorem.

🔍 Mini-Case: Insertion Sort Ungraded

Apply the five-step method to derive the recurrence for worst-case insertion sort. In the worst case (input in reverse order), inserting A[n] into the already-sorted subarray A[1..n−1] requires exactly n−1 comparisons.

InsertionSort(A, n):
  if n == 1: return                     // base case: one element is already sorted
  InsertionSort(A, n − 1)               // sort the first n − 1 elements
  insert A[n] into its correct position // worst case: n − 1 comparisons needed

Work through the five steps and write T(n).

  1. Size parameter: n = number of elements to sort.
  2. Work at current level: n−1 comparisons (worst case) to insert A[n].
  3. Recursive calls: One call, on n−1 elements → contributes T(n−1).
  4. Recurrence:
T(n) = T(n−1) + (n−1),    T(1) = 0
  1. Base case: T(1) = 0 — a single-element array is already sorted; zero comparisons needed.

Iterating to closed form: T(n) = (n−1) + (n−2) + ⋯ + 1 + 0 = n(n−1)2  →  Θ(n2).

3Generating Terms by Iteration

Before attempting an algebraic closed form, generate the first several terms by direct substitution — also called unrolling or forward iteration. This technique applies the recurrence rule starting from the initial conditions and stepping forward one index at a time.

Method

  1. Write down all initial conditions.
  2. Substitute the smallest undefined index n into the recurrence; compute the resulting value using already-known terms.
  3. Repeat, incrementing the index, until you have the desired number of terms.

Worked Example: First 8 Fibonacci Terms

Recurrence: F(n) = F(n−1) + F(n−2),   F(0) = 0,   F(1) = 1.

n 012 345 67
Computation given given 0 + 11 + 11 + 2 2 + 33 + 55 + 8
F(n) 01 12 35 813

Each F(n) entry is computed by summing the two entries immediately to its left. The first two entries come directly from the initial conditions; no computation is needed for them.

🔗 PE-08 Connection — "Generate First n Terms"

PE-08 requires your program to generate the first n terms of a user-supplied recurrence relation. The iteration method above is exactly what your code must implement:

  1. Read the recurrence coefficients and initial conditions as input; store the initial values in an array.
  2. Run a loop from index k (= degree of the recurrence) up to the requested term count, computing each new value by applying the recurrence to previously stored entries.
  3. Collect and output the complete sequence.

Verification target: For F(n) = F(n−1) + F(n−2), a0 = 0, a1 = 1, requesting 8 terms, your program should produce: 0, 1, 1, 2, 3, 5, 8, 13.

💡 Two Reasons to Generate Terms First
  1. Model validation: If the numbers look wrong (unexpected sign, growth rate, or pattern), revisit your recurrence setup before investing time in the algebra.
  2. Closed-form verification: After deriving a closed form, plug in small values of n and compare against the iteration table. A mismatch reveals an algebra error immediately — for example, Binet's formula (Section 4) should give F(6) = 8, exactly matching the table above.

4Solving Linear Homogeneous Recurrences — Characteristic Equation Method

📐 Definition: Linear Homogeneous Recurrence with Constant Coefficients

A recurrence is linear homogeneous with constant coefficients when it has the form

an = c1an−1 + c2an−2 + ⋯ + ckan−k

Linear: every an−j appears to the first power only.  Homogeneous: the right-hand side contains only previous a terms — no additional function of n.  Constant coefficients: the cj values do not depend on n.

Five-Step Visual Walkthrough

  1. 1
    Write the recurrence in standard form Move all a terms to the left so the right side equals zero:
    an − c1an−1 − c2an−2 − ⋯ − ckan−k = 0
  2. 2
    Substitute the trial solution an = rn Replacing an−j = rn−j and dividing every term by rn−k yields the characteristic equation — a polynomial in r:
    rk − c1rk−1 − c2rk−2 − ⋯ − ck = 0
  3. 3
    Find the characteristic roots r1, r2, …, rk Factor or apply the quadratic formula. Each distinct root will contribute an independent term to the general solution.
  4. 4
    Write the general solution (distinct roots) When all k roots are distinct, the general solution is:
    an = α1r1n + α2r2n + ⋯ + αkrkn
    The constants α1, …, αk are as yet undetermined.
  5. 5
    Apply initial conditions to determine the constants Substitute a0 = v0, a1 = v1, …, ak−1 = vk−1 into the general solution. This produces a linear system of k equations in k unknowns. Solve the system to obtain the closed form.

Fully Worked Example: Fibonacci Closed Form (Binet's Formula)

We apply all five steps to F(n) = F(n−1) + F(n−2),   F(0) = 0,   F(1) = 1.

Step 1 — Standard Form
F(n) − F(n−1) − F(n−2) = 0
Step 2 — Characteristic Equation

Substitute F(n) = rn; divide through by rn−2:

rn − rn−1 − rn−2 = 0 ÷ rn−2 r2 − r − 1 = 0
Step 3 — Characteristic Roots (Quadratic Formula)
r = 1 ± √52

Name the two distinct roots:

φ = 1 + √52 ≈ 1.6180    (the golden ratio) ψ = 1 − √52 ≈ −0.6180
Step 4 — General Solution (two distinct roots)
F(n) = A · φn + B · ψn
Step 5 — Apply Initial Conditions

From F(0) = 0:

A · φ0 + B · ψ0 = 0  ⟹  A + B = 0  ⟹  B = −A

From F(1) = 1:

A · φ + (−A) · ψ = 1  ⟹  A(φ − ψ) = 1

Compute φ − ψ directly:

φ − ψ = 1+√52 − 1−√52 = 2√52 = √5

Therefore:

A · √5 = 1  ⟹  A = 1√5     B = −A = −1√5
Result — Binet's Formula
F(n) = φn − ψn√5 where   φ = 1+√52,    ψ = 1−√52

Verification (n = 6): φ6 ≈ 17.944, ψ6 ≈ 0.056;   (17.944 − 0.056) ÷ 2.236 ≈ 8.00 ✓ — matches F(6) = 8 from the iteration table in Section 3.

Although φ and ψ are irrational, the formula produces a positive integer for every n ≥ 0 because the irrational √5 parts cancel exactly.

Extension: Repeated Characteristic Roots

📐 Repeated Root Rule

If the characteristic equation has a root r with multiplicity m — meaning (x − r)m divides the characteristic polynomial — then that root contributes m linearly independent terms to the general solution:

(α1 + α2n + α3n2 + ⋯ + αmnm−1) · rn

The ordinary distinct-root formula rn is the special case m = 1. Each additional copy of the same root introduces a factor of n multiplying rn.

Repeated Root — Worked Example

Solve: an = 4an−1 − 4an−2,   a0 = 1,   a1 = 2.

Characteristic Equation
r2 − 4r + 4 = 0  ⟹  (r − 2)2 = 0  ⟹  r = 2   (double root, m = 2)
General Solution (Repeated Root, m = 2)
an = (α1 + α2n) · 2n
Apply Initial Conditions
a0 = 1:   (α1 + 0) · 20 = 1  ⟹  α1 = 1 a1 = 2:   (1 + α2) · 21 = 2  ⟹  1 + α2 = 1  ⟹  α2 = 0
Closed Form
an = 2n

Verification: a0=1 ✓, a1=2 ✓, a2=4(2)−4(1)=4=22 ✓, a3=4(4)−4(2)=8=23 ✓.

The α2 = 0 result collapses the polynomial factor to a constant: (1 + 0·n)·2n = 2n. The solution is purely geometric even though the characteristic root had multiplicity 2.

5Linear Non-Homogeneous Recurrences

📐 Definition: Linear Non-Homogeneous Recurrence

A recurrence of the form

an = c1an−1 + c2an−2 + ⋯ + ckan−k + f(n)

where f(n) ≠ 0 is a function of n alone (called the driving function or forcing function) is a linear non-homogeneous recurrence relation. The associated homogeneous recurrence is obtained by setting f(n) = 0.

Solution Strategy: Homogeneous + Particular

The general solution combines two independent components:

an  =  an(h)  +  an(p)
  • an(h) — the homogeneous solution: solve the associated homogeneous recurrence (f(n) = 0) using the characteristic equation method from Section 4. This part contains the free constants αi.
  • an(p) — a particular solution: guess a trial form for an(p) whose shape is guided by f(n), substitute it into the full recurrence, and equate coefficients to solve for the constants in the guess.

After forming an = an(h) + an(p), apply the initial conditions to the combined general solution (not to the homogeneous part alone) to determine the remaining constants.

Choosing the Particular Solution Form

f(n) Type Standard Particular Solution Guess Resonance Exception
Polynomial of degree d
e.g., 3n + 2  (d = 1)		 pdnd + pd−1nd−1 + ⋯ + p0 If 1 is a characteristic root with multiplicity j, multiply guess by nj.
Exponential
e.g., 3n		 C · 3n If 3 is a characteristic root with multiplicity j, use C · nj · 3n.
Polynomial × Exponential
e.g., n · 2n		 (pdnd + ⋯ + p0) · 2n Apply both rules above as needed.
⚠ Resonance Pitfall

If the base of f(n)'s exponential matches a characteristic root, the naive particular guess is already part of the homogeneous solution and yields 0 = f(n) when substituted — making the constants unsolvable. Fix: multiply the particular guess by n (or nj for multiplicity j) to make it linearly independent from the homogeneous solution. This situation is called resonance, in analogy with driven oscillators in differential equations.

Worked Example: Polynomial Driving Function

Solve: an = 2an−1 + n,   a0 = 1.

Step 1 — Homogeneous Solution

Associated homogeneous recurrence: an = 2an−1.   Characteristic equation: r − 2 = 0  ⟹  r = 2.

an(h) = A · 2n
Step 2 — Particular Solution

f(n) = n is degree-1; characteristic root r = 2 ≠ 1, so no resonance. Guess: an(p) = cn + d.

Substitute into the full recurrence:

cn + d = 2[c(n−1) + d] + n cn + d = 2cn − 2c + 2d + n cn + d = (2c + 1)n + (−2c + 2d)

Equate coefficients by power of n:

n1:   c = 2c + 1  ⟹  −c = 1  ⟹  c = −1 n0:   d = −2c + 2d  ⟹  −d = −2c = 2  ⟹  d = −2
an(p) = −n − 2
Step 3 — General Solution
an = A · 2n + (−n − 2) = A · 2n − n − 2
Step 4 — Apply Initial Condition
a0 = 1:   A · 20 − 0 − 2 = 1  ⟹  A − 2 = 1  ⟹  A = 3
Closed Form
an = 3 · 2n − n − 2

Verification:

  • a0 = 3(1) − 0 − 2 = 1 ✓
  • a1 = 3(2) − 1 − 2 = 3;   check via recurrence: 2(1) + 1 = 3 ✓
  • a2 = 3(4) − 2 − 2 = 8;   check: 2(3) + 2 = 8 ✓
  • a3 = 3(8) − 3 − 2 = 19;   check: 2(8) + 3 = 19 ✓

💡 Module 11.5 Connection

The Binary Search and Merge Sort rows use divide-and-conquer recurrences of the form T(n) = aT(n/b) + f(n). Although their closed forms are listed above, deriving them via the characteristic equation method is awkward — the n/b subproblem size breaks the constant-coefficient assumption. The Master Theorem (Module 11.5) handles these in three pattern-matching cases without any algebra, and is the tool PE-08 requires for its divide-and-conquer analysis component.

▶ Up Next: Module 11.5 — The Master Theorem

The characteristic equation method works cleanly for recurrences like T(n) = T(n−1) + 1 where each call reduces the problem by a fixed amount. Divide-and-conquer recurrences — T(n) = 2T(n/2) + n, T(n) = T(n/2) + 1 — involve subproblem sizes of the form n/b, which breaks the constant-coefficient assumption and makes the characteristic equation method awkward to apply. In Module 11.5 you will learn the Master Theorem, a three-case pattern-matching tool that resolves the complexity of any T(n) = aT(n/b) + f(n) recurrence in seconds — completing the merge sort and binary search analyses and satisfying PE-08's divide-and-conquer requirement.
